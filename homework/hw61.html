📏 RAG 评估框架与指标综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：RAG 评估框架与指标🚀 所属项目：EduRAG：企业私有知识智能问答

40 道练习，覆盖为何要评 RAG、RAGAS 数据字段、检索侧（上下文相关性 / 召回）与生成侧（忠实度 / 答案相关性）四类指标，以及「低分该改检索还是改生成」的诊断思路。

建议先独立作答，再展开参考答案。代码题用纯 Python 模拟指标计算，不调用真实 LLM / RAGAS 库。

一、选择题（20 题）

1. 上线前评估 RAG 的首要工程目的是？

A. 只为了打印更长的日志
B. 得到可对比的指标，判断改动是否真的有效
C. 取消全部检索
D. 把向量维度改成随机数

✅ 参考答案

答案：B。 课件强调：先评估再自动化对比，才能确认新流程是否改进。

2. RAGAS 名称更贴近的含义是？

A. Random Attention Graph Search
B. Retrieval Augmented Generation Assessment（检索增强生成的自动评估）
C. Redis And GPU Auto Scaling
D. Relational Algebra Generator API

✅ 参考答案

答案：B。 课件称其为 Automated Evaluation of Retrieval Augmented Generation。

3. RAGAS 评估主要覆盖哪两个方向？

A. 仅 GPU 温度与磁盘空间
B. 检索部分与生成部分
C. 仅前端样式与颜色
D. 仅操作系统内核

✅ 参考答案

答案：B。 不同方向用不同指标：上下文相关/召回 vs 忠实度/答案相关。

4. 评估样本字段 question 指的是？

A. 检索到的全部 PDF 原文
B. 作为 RAG 管道输入的用户查询
C. 仅模型的系统提示
D. 仅 GPU 型号

✅ 参考答案

答案：B。 它是评测输入侧的核心字段。

5. 字段 contexts 表示？

A. 训练损失函数
B. 为回答问题而从外部知识源检索到的上下文
C. 仅 ground truth 全文
D. 仅 tokenizer 词表

✅ 参考答案

答案：B。 检索质量评估主要看 contexts 是否相关、是否覆盖真实答案所需信息。

6. 字段 ground_truths 在课件中的特殊地位是？

A. 可以完全随机生成且不必人工看
B. 是唯一需要人工注释的基本事实答案信息
C. 等于 contexts
D. 等于 answer

✅ 参考答案

答案：B。 早期 RAGAS 部分指标可少依赖人工，但召回等仍常需要 GT；课件明确将其标为人工注释信息。

7. 评估检索（context）侧的两个指标是？

A. BLEU 与 ROUGE
B. 上下文相关性（context relevancy）与上下文召回率（context recall）
C. 仅学习率与 batch size
D. 仅 FPS 与延迟

✅ 参考答案

答案：B。 课件：衡量检索段落是否相关、是否覆盖所需信息。

8. 评估生成（answer）侧的两个指标是？

A. 忠实度（faithfulness）与答案相关性（answer relevancy）
B. 仅 nprobe 与 k1
C. 仅磁盘 IOPS
D. 仅 Collection 数量

✅ 参考答案

答案：A。 忠实度看是否依据上下文；答案相关性看是否切题。

9. 上下文相关性（context relevancy）主要惩罚什么？

A. 答案太短
B. 检索上下文夹带大量回答问题所不需要的冗余信息
C. 向量维度是偶数
D. 使用了 BM25

✅ 参考答案

答案：B。 理想上下文应「够用且精炼」；比率越高通常表示相关越强。

10. 课件计算上下文相关性的大致步骤是？

A. 随机删一半句子
B. 用 LLM 从上下文抽出对答题关键的句子，再按相关句占比计分
C. 只比较字符串编辑距离
D. 只看 GPU 利用率

✅ 参考答案

答案：B。 若无法从上下文回答，提示要求返回「信息不足」；且不得改写原句。

11. 上下文召回率（context recall）衡量的是？

A. 答案是否幽默
B. 真实答案中的声明有多少能归因于检索上下文
C. 仅 Prompt 长度
D. 仅模型参数量

✅ 参考答案

答案：B。 分数通常在 0~1，越高越好；理想情况 GT 的全部声明都应可归因。

12. 课件世界杯例子中，Context Recall = 0.5 是因为？

A. 两个声明都被召回
B. 两个声明中只有「冠军是西班牙」可归因，比分 1-0 未在上下文中
C. 没有检索到任何上下文
D. 公式分母为 0

✅ 参考答案

答案：B。 总声明 2，召回 1，故 $1/2=0.5$。

13. 忠实度（faithfulness）低，更直接暗示？

A. 磁盘快满了
B. 生成答案未遵循检索知识，幻觉风险上升
C. 一定是 BM25 参数错了
D. 一定是 CSS 写错了

✅ 参考答案

答案：B。 忠实度要求答案可由给定上下文推出，用于抑制「检索对了但仍瞎编」。

14. 忠实度分数 $F=|V|/|S|$ 中 $|V|$ 与 $|S|$ 是？

A. $|V|$ 为向量维度，$|S|$ 为样本数
B. $|V|$ 为被判定「可由上下文支持」的陈述数，$|S|$ 为从答案抽出的陈述总数
C. 二者都是文档长度
D. 二者都是学习率

✅ 参考答案

答案：B。 先抽陈述，再逐条验证是否可从上下文推断。

15. 答案相关性（answer relevancy）关注的是？

A. 答案与查询是否切题相关
B. 答案是否逐字复制上下文
C. 仅上下文是否含冗余
D. 仅 GT 声明数

✅ 参考答案

答案：A。 分数越高相关性越好；它不直接等于忠实度。

16. 答案相关性的实现要点是？

A. 从答案生成若干可能问题，再与原问题做嵌入余弦相似度并平均
B. 只比较答案字符长度
C. 只比较 contexts 数量
D. 随机给 0 或 1

✅ 参考答案

答案：A。 若答案跑题，反推问题会与原 query 方向偏离，平均相似度下降。

17. 若 context recall 很低、faithfulness 很高，更优先排查？

A. 生成温度
B. 检索是否漏掉 GT 所需证据
C. 浏览器缓存
D. logo 颜色

✅ 参考答案

答案：B。 召回低说明上下文覆盖不足；忠实高只说明「在已有上下文内」生成较守据，不能弥补没检索到的事实。

18. 若 context relevancy 很低、context recall 尚可，更可能的问题是？

A. 检索到了有用信息，但也塞进大量无关段落
B. 完全没有 GT
C. 模型不会分词
D. 没有 question 字段

✅ 参考答案

答案：A。 冗余上下文抬高噪声、浪费窗口，相关句占比下降。

19. 若 faithfulness 很低、answer relevancy 尚可，典型现象是？

A. 答案听起来切题，但关键事实不是依据上下文推出（易幻觉）
B. 答案完全空白
C. 检索模块未启动且指标仍完美
D. 只有 BM25 能救

✅ 参考答案

答案：A。 切题 ≠ 有据；需要强化「依据上下文作答 / 不足则拒答」的 Prompt 与约束。

20. 下列哪项不是「只盯单一指标」的风险？

A. 只追答案相关性可能鼓励空泛但切题的套话
B. 只追忠实度可能鼓励「安全但未完整回答」
C. 多指标联看才能定位检索/生成瓶颈
D. 四指标永远同升同降，所以看一个就够

✅ 参考答案

答案：D。 指标会此消彼长；工程上要联看并配合人工抽检与延迟/成本。

二、简答题（12 题）

21. 为什么「感觉回答还行」不能替代 RAGAS 一类自动评估？

✅ 参考答案

主观感觉难复现、难对比版本差异，也难覆盖长尾问题。自动指标提供可回归的数值信号，再配合人工抽检，才能支撑持续迭代。

22. 列出评测样本最小字段集，并说明缺谁会卡住哪类指标。

✅ 参考答案

question、answer、contexts 几乎处处需要；ground_truths 对 context recall 等「相对 GT」的指标关键。缺 contexts 无法评检索与忠实；缺 GT 则召回难算准。

23. 用一句话区分 context relevancy 与 context recall。

✅ 参考答案

Relevancy：检索到的内容有多少是答题真正需要的（精不精）；Recall：真实答案所需信息有多少被检索覆盖（全不全）。

24. 解释课件中「不得更改给定上下文中的句子」对相关句抽取的意义。

✅ 参考答案

防止 LLM 改写后「看起来相关」却无法回溯原文，保证抽取结果可定位到真实检索片段，评测更可审计。

25. 为什么 context recall 要以「声明（claim）」而不是整段字符串相等来判断？

✅ 参考答案

真实答案常含多条可独立验证的事实；逐条归因能区分「部分覆盖」。整段相等过严，也难以处理表述差异。

26. 忠实度评估两步（抽陈述 → 验证）各自失败会怎样？

✅ 参考答案

抽陈述失败：漏掉胡编内容或过度切碎导致不稳定。验证失败/偏松：把无依据陈述判成支持，忠实度虚高。两边都依赖评测 LLM 质量与提示约束。

27. 答案相关性用「反推问题 + 嵌入相似」有什么局限？

✅ 参考答案

评测 LLM/嵌入模型偏差会影响分数；答案很短或模板化时可能虚高；它也不保证事实正确。需与忠实度、人工抽检一起看。

28. 给出一个「改进检索」相对「改进生成」的决策表（各两行）。

✅ 参考答案
  • 召回低、相关低 → 改切分/索引/top-k/混合检索。
  • 相关低、召回尚可 → 加强过滤/重排，减少冗余。
  • 忠实低 → 改 Prompt、引用约束、降温度、拒答策略。
  • 答案相关低 → 改指令遵循、问题改写、输出格式。

29. 为什么自动化评估仍需要抽样人工复核？

✅ 参考答案

评测 LLM 自己会错判支持关系；业务合规、语气、安全问题也不被这四指标完全覆盖。人工复核校准阈值与坏案例库。

30. 若两套 RAG 流水线：A 四指标略高但延迟翻倍，B 略低但更快，你如何汇报？

✅ 参考答案

同时报告质量指标与成本/延迟，并按业务容忍度给建议（例如先用 B 服务、对高风险问题走 A）。单报均值指标不够做上线决策。

31. ground_truths 质量差会如何污染评估？

✅ 参考答案

def validate_sample(sample: dict) -> None:
    for key in ("question", "answer"):
        if not isinstance(sample.get(key), str) or not sample[key].strip():
            raise ValueError(f"invalid {key}")
    ctx = sample.get("contexts")
    if not isinstance(ctx, list) or not ctx or not all(isinstance(x, str) and x.strip() for x in ctx):
        raise ValueError("invalid contexts")
    if "ground_truths" in sample:
        gt = sample["ground_truths"]
        if not isinstance(gt, list) or not gt or not all(isinstance(x, str) and x.strip() for x in gt):
            raise ValueError("invalid ground_truths")

37. 计算答案相关性：反推问题嵌入与原问题的平均余弦相似

用纯 Python 余弦；嵌入以「词袋计数向量」在共享词表上表示（教学用）。

✅ 参考答案
Python
import math
from collections import Counter


def bag_vec(text: str, vocab: list[str]) -> list[float]:
    c = Counter(text.lower().split())
    return [float(c[w]) for w in vocab]


def cosine(a, b):
    ip = sum(x * y for x, y in zip(a, b))
    na = math.sqrt(sum(x * x for x in a))
    nb = math.sqrt(sum(y * y for y in b))
    if na == 0 or nb == 0:
        return 0.0
    return ip / (na * nb)


def answer_relevancy(question: str, generated_questions: list[str]) -> float:
    vocab = sorted({w for t in [question, *generated_questions] for w in t.lower().split()})
    qv = bag_vec(question, vocab)
    sims = [cosine(qv, bag_vec(g, vocab)) for g in generated_questions]
    return sum(sims) / len(sims) if sims else 0.0


q = "who won the 2010 world cup"
good = ["which team won world cup 2010", "who was 2010 world cup champion"]
bad = ["how to cook pasta", "what is the weather today"]
assert answer_relevancy(q, good) > answer_relevancy(q, bad)
print(answer_relevancy(q, good), answer_relevancy(q, bad))

38. 指标诊断器：根据四指标给出优先动作

规则（可简化）：

- faith < 0.5 → ground_generation
- ans_rel < 0.5 → improve_answer_focus

✅ 参考答案

def diagnose(metrics: dict) -> list[str]:
    actions = []
    if metrics.get("context_recall", 1) < 0.5:
        actions.append("improve_retrieval_coverage")
    if metrics.get("context_relevancy", 1) < 0.5:
        actions.append("reduce_context_noise")
    if metrics.get("faithfulness", 1) < 0.5:
        actions.append("ground_generation")
    if metrics.get("answer_relevancy", 1) < 0.5:
        actions.append("improve_answer_focus")
    return actions

    "improve_retrieval_coverage"
]

39. 对比两套流水线：在固定权重下算综合分并选出胜者

综合分 $=0.25\sum$ 四指标；并列时选延迟更低者。

✅ 参考答案
Python
def aggregate(metrics: dict) -> float:
    keys = ["context_relevancy", "context_recall", "faithfulness", "answer_relevancy"]
    return sum(metrics[k] for k in keys) / 4.0


def pick_winner(a: dict, b: dict) -> str:
    sa, sb = aggregate(a["metrics"]), aggregate(b["metrics"])
    if abs(sa - sb) < 1e-9:
        return "A" if a["latency_ms"] <= b["latency_ms"] else "B"
    return "A" if sa > sb else "B"


A = {"metrics": {"context_relevancy": 0.8, "context_recall": 0.7,
                 "faithfulness": 0.9, "answer_relevancy": 0.8}, "latency_ms": 800}
B = {"metrics": {"context_relevancy": 0.75, "context_recall": 0.75,
                 "faithfulness": 0.75, "answer_relevancy": 0.75}, "latency_ms": 200}
assert pick_winner(A, B) == "A"
print(aggregate(A["metrics"]), pick_winner(A, B))

40. 迷你评测挂架：对数据集逐条套用「假 LLM 判决」并汇总均值

用可注入的 judge 函数模拟陈述支持与相关句标记，输出四指标均值。

✅ 参考答案
Python
def mean(xs):
    return sum(xs) / len(xs) if xs else 0.0


def evaluate(dataset, judge):
    rel, rec, faith, ans = [], [], [], []
    for row in dataset:
        r = judge(row)
        rel.append(sum(r["ctx_flags"]) / len(r["ctx_flags"]) if r["ctx_flags"] else 0.0)
        rec.append(sum(r["gt_flags"]) / len(r["gt_flags"]) if r["gt_flags"] else 0.0)
        faith.append(sum(r["ans_flags"]) / len(r["ans_flags"]) if r["ans_flags"] else 0.0)
        ans.append(r["answer_relevancy"])
    return {
        "context_relevancy": mean(rel),
        "context_recall": mean(rec),
        "faithfulness": mean(faith),
        "answer_relevancy": mean(ans),
    }


def toy_judge(row):
    # 教学：上下文句若含问题关键词则相关；GT 声明若被某句包含则召回；
    # 答案陈述若都能在上下文找到子串则忠实；答案相关性给固定演示值。
    # 中文无空格时，用「长度>=2 的连续字串」做极简关键词；匹配前去掉句末标点。
    q = row["question"]
    keys = [q[i:i + 2] for i in range(len(q) - 1)] if len(q) >= 2 else [q]
    keys = [k for k in keys if not k.isspace()]
    ctx_flags = [any(k in sent for k in keys) for sent in row["contexts"]]

    def norm(s: str) -> str:
        return s.strip().rstrip("。．.！？!?")

    gt_flags = [
        any(norm(claim) in sent for sent in row["contexts"])
        for claim in row["ground_truths"]
    ]
    statements = [s.strip() for s in row["answer"].split("。") if s.strip()]
    ans_flags = [
        any(norm(st) in sent for sent in row["contexts"]) for st in statements
    ] or [False]
    return {
        "ctx_flags": ctx_flags,
        "gt_flags": gt_flags,
        "ans_flags": ans_flags,
        "answer_relevancy": 0.8,
    }


data = [{
    "question": "西班牙世界杯",
    "answer": "2010年世界杯的冠军是西班牙。",
    "contexts": [
        "2010年世界杯的冠军是西班牙，西班牙队首次赢得世界杯。",
        "天气预报说明天有雨。",
    ],
    "ground_truths": [
        "2010年世界杯的冠军是西班牙。",
        "西班牙在决赛中以1-0击败了荷兰。",
    ],
}]
report = evaluate(data, toy_judge)
assert report["context_relevancy"] == 0.5  # 两句上下文中一句相关
assert report["context_recall"] == 0.5
assert report["faithfulness"] == 1.0
print(report)

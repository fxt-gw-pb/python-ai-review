🤖 Agent、ReAct 与五种智能体模式综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：Agent、ReAct 与五种智能体模式🚀 所属项目：SmartVoyage：Agent 与多智能体旅行助手

40 道练习，覆盖 Agent 与 Agentic 辨析、五种模式（工具使用 / ReAct / 反思 / 规划 / 多智能体）及其演进关系，以及 LangChain 实战中 create_tool_calling_agent / create_react_agent 与 AgentExecutor 的分工。重点练「何时用单步工具、何时要 Thought–Act–Observe 循环」。

建议先独立作答，再展开参考答案。代码题用纯 Python 模拟轨迹与模式组合，不调用真实大模型。

一、选择题（20 题）

1. 课件对 Agent 的一句话定义更接近？

A. 只能静态检索网页的爬虫
B. 能感知环境、做出决策并采取行动以完成特定目标的智能体
C. 仅存储向量的数据库
D. 仅 CSS 布局引擎

✅ 参考答案

答案：B。 感知—决策—行动—目标，是理解后续模式的底座。

2. 为什么说「基础聊天 LLM 也算一种基础 Agent」？

A. 因为它能改操作系统内核
B. 它感知问题、决策如何答、生成文本行动，但能动性仍有限
C. 因为它不需要任何输入
D. 因为它不能生成文本

✅ 参考答案

答案：B。 真正高级 Agent 还要反思、规划、协作等更强能动性。

3. Agentic 更准确的含义是？

A. 某一个具体的开源仓库名字
B. 形容词：系统表现得「像 Agent」的程度（自主、目标导向、主动）
C. 仅等于 Function Call
D. 仅等于微调

✅ 参考答案

答案：B。 Agent 是载体；Agentic 描述行为风格/设计思想。

4. 低 Agentic 与高 Agentic 的对比，课件例子更接近？

A. 低：规则聊天机器人；高：能自动调试、部署并监控的 AI 软件工程师
B. 低：多智能体；高：单轮 FAQ
C. 二者没有差别
D. 高 Agentic 不能使用工具

✅ 参考答案

答案：A。 Agentic 越高，越少「傻等下一步指令」。

5. Lilian Weng 文中强调的自主智能体关键组件不包括？

A. 规划
B. 记忆
C. 工具使用
D. 必须手写全部汇编指令

✅ 参考答案

答案：D。 规划、记忆、工具使用等是系统化梳理的核心能力。

6. 工具使用模式相对 ReAct 的定位是？

A. ReAct 的完全替代且更复杂
B. ReAct 的前身/简化版：能调工具，但常缺细致的思考—行动—观察循环
C. 只能多智能体
D. 禁止调用 API

✅ 参考答案

答案：B。 适合单步、直接任务；动态迭代能力较弱。

7. ReAct 名称来自？

A. Remote Access Control
B. Reasoning and Acting（思考与行动结合）
C. Redis Cache
D. Random Embedding Act

✅ 参考答案

答案：B。 Yao 等人 2022 论文提出，是 Agentic 设计的奠基循环。

8. ReAct 工作流正确顺序是？

A. 观察 → 删除工具 → 结束
B. 思考 → 行动 → 行动输入 → 观察 → 循环直至达标
C. 只规划、永不行动
D. 只行动、禁止思考

✅ 参考答案

答案：B。 「边想边做」，把观察反馈进下一轮思考。

9. 研究助理例子里，第一次 Observation 返回 10 篇论文后，下一步典型 Thought 是？

A. 直接结束并随机回答
B. 结果太多，需要按年份/引用再筛选
C. 卸载操作系统
D. 禁止使用任何 Action

✅ 参考答案

答案：B。 观察驱动下一轮行动，而不是一次性调用完事。

10. 反思模式的核心精髓是？

A. 永远不修改初稿
B. 完成步骤/任务后自我评估，从错误中学习并优化
C. 只能增加更多 Agent
D. 取消所有工具

✅ 参考答案

答案：B。 是 ReAct「思考」环节的深化与质量保证。

11. 代码生成反思例子中，Agent 发现问题后采取的 Refinement 是？

A. 坚持低效递归并声称已最优
B. 改为带缓存/动态规划以提升效率
C. 删除函数
D. 改用 CSS 实现斐波那契

✅ 参考答案

答案：B。 先产出 → 自检 → 再行动修正。

12. 规划模式适用于？

A. 仅单步加法
B. 复杂任务需先宏观分解为有序计划，再逐步执行（每步可含 ReAct）
C. 禁止分解任务
D. 只能并行无限 Agent

✅ 参考答案

答案：B。 先 Plan，再 Execution。

13. 多智能体模式的核心精髓是？

A. 一个 Agent 必须完成所有角色
B. 多角色协作，各司其职，模拟团队；需处理通信与冲突

D. 禁止 PM 角色

✅ 参考答案

答案：B。 课件称其为 Agentic 思想的「终极体现」之一，也更难工程化。

14. 课件给出的演进阶梯顺序是？

A. Multi-Agent → Tool Use → ReAct
B. Tool Use → ReAct → Planning → Reflection → Multi-Agent
C. Reflection → Tool Use → 结束
D. 无顺序

15. 「ReAct 是 Tool Use 的规范化」这句话意味着？

16. Planning 与多个 ReAct 循环的关系是？

B. Planning 是在执行多个 ReAct 之前的高层战略
C. ReAct 禁止出现在计划执行中
D. Planning 只能事后反思

✅ 参考答案

答案：B。 宏观拆解 → 微观循环执行。

17. 真正强大的系统通常？

B. 按任务复杂度组合嵌套多种模式

D. 从不反思

二、简答题（12 题）

21. 用对比句区分 Agent 与 Agentic。

✅ 参考答案

Agent 是「能感知—决策—行动的实体/系统」；Agentic 描述该系统自主、目标导向、主动解决问题的程度。同一个聊天机器人可以是弱 Agent，也可以通过规划/工具/记忆变得更 Agentic。

22. 为什么「会 Function Call」不等于「高 Agentic」？

✅ 参考答案

FC 解决单次结构化工具调用；高 Agentic 还要多步规划、根据观察调整、反思纠错、必要时多智能体协作。缺循环与目标管理时，仍可能只是「增强版一次调用」。

23. 写出 ReAct 四元组字段，并说明 Observation 的作用。

✅ 参考答案

Thought / Action / Action Input / Observation。Observation 把环境反馈写回上下文，使下一轮 Thought 基于事实而非臆测，形成闭环。

24. Tool Use 失败时常见症状与升级到 ReAct 的判据？

✅ 参考答案

症状：一次工具结果不够、需要二次筛选/换工具、中途策略要变。判据：任务需要根据中间结果动态改计划时，上 ReAct。

25. Reflection 放在「执行后」而不是「永不执行」的原因？

✅ 参考答案

先有可评估产物（草稿/代码/计划），反思才有对象；空转自检缺少信号。当然也可在关键步骤设检查点，而不是只在最终才反思。

26. Planning 失败的两种典型模式？

✅ 参考答案

计划过粗：执行时反复迷路；计划过细且僵化：环境一变就崩。需要可修订计划，并把不确定步骤交给 ReAct。

27. Multi-Agent 相对单 Agent 的收益与代价。

✅ 参考答案

收益：专业分工、并行、复杂系统可扩展。代价：通信成本、任务冲突、协调失败、调试更难。需要清晰角色与汇总机制（如 PM Agent）。

28. 解释演进阶梯里 Reflection 为何排在 Planning 之后（课件顺序）。

✅ 参考答案

先有宏观计划与执行结果，质量保证（反思）才更有意义；也可对单步 ReAct 反思。课件把它视为规模化协作前的质量层。

29. 结合旅行规划例子，说明 Planning 与工具调用如何嵌套。

✅ 参考答案

先生成五日计划步骤；每步执行时调用搜索/预订工具（可走 ReAct）；遇冲突再反思改行程，最后汇总文档。

30. create_react_agent 与 create_tool_calling_agent 选型直觉？

✅ 参考答案

需要显式 Thought/Action 文本轨迹、强调推理过程时偏 ReAct；模型原生 tool calling、任务偏直接工具编排时可用 tool_calling agent。二者都常配 AgentExecutor。

31. 为什么 Agentic 系统更依赖评测与护栏？

✅ 参考答案

自主步数多、副作用工具多，错误会放大。需要步数上限、工具白名单、轨迹审计、成功准则与人工接管点。

32. 设计一个「模式选择」决策表（至少四行）。

✅ 参考答案
  • 单次查订单 → Tool Use
  • 多跳检索筛选 → ReAct
  • 生成后要自检优化 → +Reflection
  • 五日行程/软件项目 → Planning（± Multi-Agent）

三、代码题（8 题）

33. 解析简易 ReAct 轨迹文本为结构化步骤

识别以 Thought: / Action: / Action Input: / Observation: 开头的行。

✅ 参考答案
Python
def parse_react_trace(text: str) -> list[dict]:
    steps = []
    cur = {}
    for line in text.splitlines():
        line = line.strip()
        for key in ("Thought", "Action", "Action Input", "Observation"):
            prefix = key + ":"
            if line.startswith(prefix):
                if key == "Thought" and cur:
                    steps.append(cur)
                    cur = {}
                cur[key] = line[len(prefix):].strip()
                break
    if cur:
        steps.append(cur)
    return steps


trace = """
Thought: 需要搜索论文
Action: search_arxiv
Action Input: large language models in education
Observation: 返回10篇
Thought: 需要筛选
Action: filter_papers
Action Input: min_year=2023
Observation: 返回3篇
"""
steps = parse_react_trace(trace)
assert len(steps) == 2 and steps[0]["Action"] == "search_arxiv"
print(steps[1]["Thought"])

34. 实现最小 Tool-Use 执行器（无多轮反思）

模型决策一次工具名+参数，执行后直接生成最终句。

✅ 参考答案

TOOLS = {
    "search_weather": lambda city: f"{city}：晴，25°C" if "北京" in city else f"{city}：未知",
    "multiply": lambda a, b: a * b,
}


def tool_use_once(decision: dict) -> str:
    name = decision["name"]
    if name not in TOOLS:
        raise KeyError(name)
    result = TOOLS[name](**decision["args"])
    return f"工具 {name} 结果={result}"


assert "25" in tool_use_once({"name": "search_weather", "args": {"city": "北京"}})
assert tool_use_once({"name": "multiply", "args": {"a": 30, "b": 5}}).endswith("150")
print(tool_use_once({"name": "multiply", "args": {"a": 2, "b": 3}}))

35. 实现 ReAct 循环：最多 max_steps，直到决策 finish

✅ 参考答案
Python
def react_loop(policy, tools, max_steps=5):
    observations = []
    trace = []
    for _ in range(max_steps):
        decision = policy(observations)
        trace.append(decision)
        if decision.get("type") == "finish":
            return {"ok": True, "answer": decision["answer"], "trace": trace}
        name = decision["action"]
        obs = tools[name](**decision.get("action_input", {}))
        observations.append(obs)
        trace[-1]["observation"] = obs
    return {"ok": False, "answer": None, "trace": trace}


def policy(observations):
    if not observations:
        return {"type": "act", "thought": "先查天气", "action": "search_weather",
                "action_input": {"city": "上海"}}
    return {"type": "finish", "thought": "已有观察", "answer": f"汇总：{observations[-1]}"}


tools = {"search_weather": lambda city: f"{city}阴有小雨"}
out = react_loop(policy, tools)
assert out["ok"] and "上海" in out["answer"]
print(out["answer"])

36. 实现 Reflection：对初稿打分，低于阈值则改写

✅ 参考答案
Python
def reflect_and_refine(draft: str, critique_fn, refine_fn, threshold=0.7, max_rounds=3):
    current = draft
    history = []
    for _ in range(max_rounds):
        score, reason = critique_fn(current)
        history.append({"draft": current, "score": score, "reason": reason})
        if score >= threshold:
            return {"final": current, "history": history}
        current = refine_fn(current, reason)
    return {"final": current, "history": history}


def critique(text: str):
    # 教学：包含“缓存”视为更优实现
    score = 0.9 if "缓存" in text or "memo" in text.lower() else 0.4
    reason = "递归性能差" if score < 0.7 else "可接受"
    return score, reason


def refine(text: str, reason: str):
    return text + "；改用带缓存的动态规划"


out = reflect_and_refine("递归斐波那契", critique, refine)
assert "缓存" in out["final"] and len(out["history"]) >= 2
print(out["final"])

38. 实现迷你 Multi-Agent：PM 分配，专员执行，PM 汇总

✅ 参考答案
Python
def pm_decompose(request: str) -> dict:
    return {"dev": f"实现功能：{request}", "ops": f"部署测试：{request}"}


def dev_agent(task: str) -> str:
    return f"[DEV完成] {task}"


def ops_agent(task: str) -> str:
    return f"[OPS完成] {task}"


def multi_agent(request: str) -> str:
    tasks = pm_decompose(request)
    parts = [dev_agent(tasks["dev"]), ops_agent(tasks["ops"])]
    return " | ".join(parts)


out = multi_agent("做一个天气查询服务")
assert "[DEV完成]" in out and "[OPS完成]" in out
print(out)

39. 模式路由器：按任务特征选择模式标签

规则：含「规划/行程/项目」→ planning；含「改进/优化/检查」→ reflection；含「团队/多人/角色」→ multi_agent；含「多步/筛选/然后」→ react；否则 tool_use。

✅ 参考答案
Python
def route_pattern(task: str) -> str:
    t = task.lower()
    if any(k in task for k in ("规划", "行程", "项目")):
        return "planning"
    if any(k in task for k in ("改进", "优化", "检查")):
        return "reflection"
    if any(k in task for k in ("团队", "多人", "角色")):
        return "multi_agent"
    if any(k in task for k in ("多步", "筛选", "然后")):
        return "react"
    return "tool_use"


assert route_pattern("查询订单状态") == "tool_use"
assert route_pattern("筛选最近论文然后总结") == "react"
assert route_pattern("规划五日巴黎行程") == "planning"
print(route_pattern("优化这段递归代码"))

40. 组合演示：Planning 拆步 → 每步 ReAct 一次工具 → 可选 Reflection

✅ 参考答案

def combined_pipeline(goal: str, tools: dict):
    plan = ["搜索资料", "整理答案"]
    artifacts = []
    for step in plan:
        # 单步 ReAct：固定调用 search
        obs = tools["search"](step)
        artifacts.append({"step": step, "obs": obs})
    draft = "；".join(a["obs"] for a in artifacts)
    # reflection：若过短则补充
    if len(draft) < 10:
        draft = draft + "（已补充细节）"
    return {"plan": plan, "draft": draft, "artifacts": artifacts}


tools = {"search": lambda q: f"关于{q}的要点"}
out = combined_pipeline("介绍 ReAct", tools)
assert out["plan"][0] == "搜索资料" and "要点" in out["draft"]
print(out["draft"])

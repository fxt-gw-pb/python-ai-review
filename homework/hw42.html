⚡ FastText：文本分类与词向量综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：FastText：文本分类与词向量

55 道练习，按“监督数据 → 平均词袋分类器 → word n-gram → 字符子词 → Softmax / HS / NS / OVA → 评估与部署”组织。重点是分清 FastText 的分类与词向量两条路线，并把 API 参数、指标含义和数据边界落实到可验证代码。

建议先独立作答，再展开参考答案。参数默认值按 FastText 官方 Python API 校准：监督训练默认 loss='softmax'、wordNgrams=1、minn=maxn=0；无监督训练默认 loss='ns'、字符子词长度 3–6。代码题不依赖外部模型下载。

一、选择题（30 题）

1. FastText 的两类核心用途是？

A. 目标检测与图像分割
B. 有监督文本分类与无监督词向量学习
C. SQL 查询与网页渲染
D. 语音合成与视频编码

✅ 参考答案

答案：B。 train_supervised 学习标签预测器，train_unsupervised 使用 CBOW 或 Skip-gram 学习静态词向量；二者共享高效的浅层表示思想，但训练目标与参数默认值不同。

2. FastText 监督分类器的典型文档表示是？

A. 词及词级 n-gram Embedding 的平均，再接线性输出层
B. 逐 token 的 48 层自注意力
C. 只使用最后一个字符
D. 对文本生成像素矩阵

✅ 参考答案

答案：A。 它接近“可学习的平均词袋”：将词和可选 word n-gram 特征查表、平均池化，再预测类别，因此速度快且适合作为强基线。

3. 这种平均池化架构的主要局限是？

A. 无法处理整数
B. 长距离顺序和复杂组合信息表达有限
C. 不能训练 Embedding
D. 类别数只能为 2

✅ 参考答案

答案：B。 平均操作本身忽略全局词序；加入有限阶 word n-gram 可补充局部顺序，但仍不等同于上下文深度模型。

4. FastText 监督训练文件的一行通常写成？

A. __label__类别 文本内容
B. 文本内容,连续浮点标签
C. 只有类别，没有文本
D. 必须是二进制图片

✅ 参考答案

答案：A。 标签 token 默认以 __label__ 为前缀；多标签样本可在同一行放多个标签。文件应使用 UTF-8，且每行代表一个样本。

5. 原始数据按类别或时间排序时，直接 head 作训练、tail 作验证的风险是？

A. 文件会自动压缩
B. 切分分布偏移，验证集可能只含后段类别或时间模式
C. 训练速度一定更快
D. wordNgrams 会自动变为 0

✅ 参考答案

答案：B。 应先理解业务切分单位，再随机分层或按时间做符合目标场景的切分；重复文本、同一用户或同一会话也不能跨集合泄漏。

6. 监督分类中 wordNgrams=2 表示？

A. 只使用长度为 2 的字符子词
B. 加入最长为 2 的词级 n-gram 特征
C. 词向量维度固定为 2
D. 每条文本只能有两个词

✅ 参考答案

答案：B。 例如 not good 会成为一个局部短语特征；它与由 minn/maxn 控制的字符子词不是同一概念。

7. minn=3, maxn=6 主要控制什么？

A. 类别数范围
B. 字符 n-gram 子词的最短与最长长度
C. batch size
D. 标签前缀长度

✅ 参考答案

答案：B。 子词让形态相近词共享统计，并允许由字符片段组合 OOV 表示；对不同语言与分词方式应通过验证选择范围。

8. 关于监督训练默认值，正确的是？

A. 默认一定使用层次 Softmax
B. 默认 loss='softmax'、wordNgrams=1，字符子词默认关闭
C. 默认使用 48 层 Transformer
D. 默认 loss='ova'

✅ 参考答案

答案：B。 层次 Softmax 是可选的 loss='hs'，适合类别很多且需降低输出计算成本的场景；不能把一种优化选项描述成监督训练必然默认。

9. 无监督词向量训练的官方默认组合更接近？

A. Skip-gram、负采样、字符子词 3–6
B. 监督 Softmax 且无子词
C. OVA 多标签分类
D. 决策树回归

✅ 参考答案

答案：A。 Python API 中无监督 model 默认 Skip-gram、loss='ns'、minn=3,maxn=6；CBOW 可显式选择。

10. FastText 能为未登录词构造向量的关键是？

A. 从测试标签复制向量
B. 聚合该词的字符 n-gram 子词向量
C. 把所有 OOV 映射为零向量
D. 只使用词长

✅ 参考答案

答案：B。 只要模型保留了已学习的子词桶，就能由 OOV 的字符片段组合表示；但形似不等于语义相同，仍需做语义与领域验证。

11. 为什么 .bin 模型通常比导出的 .vec 更适合查询任意 OOV？

A. .bin 保留模型结构、词典和子词参数，而 .vec 主要保存显式词表向量
B. .vec 只能保存图片
C. .bin 不含任何参数
D. 二者完全相同

✅ 参考答案

答案：A。 若下游需要 FastText 的子词组合能力，应保存并加载二进制模型；纯文本向量表对词表外项通常没有足够信息重新构造表示。

12. 层次 Softmax 如何把 V 类预测降到约 $O(\log V)$？

A. 只保留一个类别
B. 将类别作为哈夫曼树叶子，沿根到叶路径进行二分类
C. 删除频繁类别
D. 对输入随机截断

✅ 参考答案

答案：B。 类别概率是路径上各二元决策概率的乘积；哈夫曼树让高频输出通常拥有较短路径，从而降低平均计算量。

13. 若某类别路径编码为 110，其概率由什么决定？

A. 三个路径节点对应方向概率的乘积
B. 三个概率的最大值
C. 类别名称的长度
D. 输入 token 数量本身

✅ 参考答案

答案：A。 每个内部节点用 Sigmoid 给出走某个分支的概率；对数似然则化为路径上若干二元交叉熵之和。

14. 负采样的训练目标是？

A. 对全部 V 个输出做精确归一化
B. 拉高正样本点积分数、压低少量采样负样本分数
C. 删除所有低频词
D. 计算标签树的完整路径

✅ 参考答案

答案：B。 每个正对只更新正目标和 K 个负目标，成本约为 $O(K)$；这些独立二分类分数不是对全部输出归一化的概率分布。

15. 负采样最常用于 FastText 的哪条路线？

A. 无监督词向量表示学习
B. 图片分类
C. SQL 聚合
D. 网页部署

✅ 参考答案

答案：A。 官方无监督训练默认使用 loss='ns'；监督分类也暴露多个 loss 选项，但单标签、多标签和大标签空间应根据目标明确选择。

16. 单标签互斥分类的常见首选 loss 是？

A. softmax
B. ova
C. 均方误差
D. CTC

✅ 参考答案

答案：A。 Softmax 产生类别间竞争的归一化分布；类别极多、速度成为瓶颈时可评估 hs，但需同时比较质量和延迟。

17. 一条文本可同时属于多个非互斥标签时，更合适的 loss 是？

A. 强制只保留一个标签的 Softmax
B. ova，为每个标签训练独立二分类器
C. 不使用标签
D. 只使用层次聚类

✅ 参考答案

答案：B。 OVA 允许多个标签同时具有高分；预测时需要选择 k 与阈值，并报告适合多标签任务的 micro/macro F1、P@k、R@k 等指标。

18. model.predict(text, k=-1, threshold=0.5) 常用于？

A. 返回所有达到阈值的候选标签
B. 强制只返回一个标签
C. 训练词向量
D. 删除模型

✅ 参考答案

答案：A。 多标签任务常需枚举所有过阈值标签；阈值必须在验证集选择，不能在测试集反复调整。

19. model.test(path, k=1) 返回三元组 (N, P@1, R@1)，第三项是？

A. F1
B. 训练 loss
C. Recall at 1
D. 模型大小

✅ 参考答案

答案：C。 课件或注释若把它写成“准确率、F1”会误导多标签评估；F1 需要由 Precision 和 Recall 另行计算，且先明确 micro/macro/样本平均方式。

20. 对多标签样本，为什么 P@1 与 R@1 可能差别很大？

A. P@1 只考察一个预测是否正确，R@1 还受真实标签总数影响
B. 两者永远相等
C. R@1 不使用标签
D. P@1 只衡量文件大小

✅ 参考答案

答案：A。 若每条样本平均有多个真标签，即使 top-1 经常命中，单次预测也只能召回其中一部分，因此 R@1 明显较低。

21. 使用 autotuneValidationFile 时，验证集的正确定位是？

A. 参与超参数选择，之后仍需独立测试集做最终一次评估
B. 可同时当训练集和测试集
C. 必须来自线上未来数据并写回模型
D. 不含标签

✅ 参考答案

答案：A。 自动调参反复查看验证表现，因此验证集不再是最终无偏估计；测试集必须保持锁定。

22. 文本预处理为何必须在训练和线上推理保持一致？

A. 大小写、标点和分词差异会改变词与 n-gram 特征
B. FastText 会自动修复任何差异
C. 只影响文件名
D. 与模型输入无关

✅ 参考答案

答案：A。 应把规范化逻辑、语言分词器和版本一起发布；对中文尤其要明确字粒度、分词粒度及标点处理，而不是假设空格会自动出现。

23. minCount 提高通常带来什么权衡？

A. 词典和内存变小，但低频信息与覆盖率可能下降
B. 类别数自动翻倍
C. 文本长度固定
D. 一定提升所有指标

✅ 参考答案

答案：A。 低频词可由子词部分补偿，但专名、医学术语等领域关键信息仍可能受损，应按频率和业务切片验证。

24. bucket 与 n-gram 特征的关系是？

A. n-gram 经哈希映射到固定数量的桶，可能发生碰撞
B. 每个桶保存完整原始文本
C. bucket 就是类别数
D. bucket 只控制 epoch

✅ 参考答案

答案：A。 更大 bucket 可减少碰撞但增加内存；碰撞是 FastText 高效表示的设计权衡，不应误认为每个 n-gram 都有独立无冲突 ID。

25. CBOW 与 Skip-gram 的方向分别是？

A. CBOW 用上下文预测中心词；Skip-gram 用中心词预测上下文
B. 二者都只预测标签
C. CBOW 只用于图片
D. Skip-gram 不学习向量

✅ 参考答案

答案：A。 CBOW 通常更快，Skip-gram 往往更重视低频词；实际效果取决于语料规模、窗口、负采样与子词配置。

26. 用余弦相似度评估词向量时必须注意？

A. 对零范数向量做保护，并结合定性与下游任务评估
B. 余弦值一定等于概率
C. 只比较向量第一维

二、简答题（15 题）

31. 用公式描述 FastText 监督分类器的前向过程。

✅ 参考答案

设文本特征集合为 $F(x)$，包括词和可选 word n-gram；每个特征向量为 $e_f$。隐藏表示可写为 $h=|F(x)|^{-1}\sum_{f\in F(x)}e_f$，再由输出层得到 $z=Wh+b$。单标签使用 Softmax/HS，多标签常用独立 Sigmoid 的 OVA。

32. 对比 wordNgrams 与 minn/maxn。

✅ 参考答案

wordNgrams 组合相邻词，服务于局部短语与有限词序，例如 not good；minn/maxn 组合词内部字符片段，服务于形态共享与 OOV，例如 <play、lay>。两者粒度、特征空间和调参目标不同，可以同时使用但会增加桶碰撞与内存成本。

33. 为什么“FastText 能处理 OOV”不能无条件成立？

✅ 参考答案

必须训练并保留字符子词参数；监督训练默认 minn=maxn=0，仅使用默认参数并不会自动启用字符子词。只加载 .vec 也通常无法组合任意 OOV。即使能生成向量，罕见字符、错误编码或形似语义不同仍会产生低质量表示。

34. 对比普通 Softmax、层次 Softmax和负采样。

✅ 参考答案

普通 Softmax 对 V 个输出归一化，计算约 $O(V)$；HS 沿哈夫曼树路径做二分类，平均约 $O(\log V)$，可形成树上的类别概率；NS 只比较一个正样本与 K 个负样本，约 $O(K)$，训练高效但分数不是全类别归一化概率。应根据任务、标签规模和校准需求选择。

35. 为什么哈夫曼树能降低 HS 的平均计算成本？

✅ 参考答案

它按输出频率构树，高频叶子通常离根更近，因此常见类别经过更少二元决策。某类概率是根到叶路径方向概率乘积，损失与梯度只涉及该路径内部节点。

36. 写出负采样损失并解释更新范围。

✅ 参考答案

对中心表示 $x$、正目标 $y^+$ 和 K 个负目标 $y_k^-$，$L=-\log\sigma(x^Ty^+)-\sum_k\log\sigma(-x^Ty_k^-)$。一次样本只更新中心、正目标和抽到的负目标向量，而非全部词表，因此大输出空间下更快。

37. 设计一个无泄漏的监督分类数据切分方案。

✅ 参考答案

先去重并确定组级单位，如用户、会话、文章来源或时间；按业务目标进行组级分层随机切分或时间切分。训练集拟合清洗与参数，验证集选择 wordNgrams/dim/loss/threshold，测试集锁定到最终一次评估。多标签数据还要检查各标签组合和长尾覆盖。

def parse_fasttext_line(line, label_prefix="__label__"):
    tokens = line.strip().split()
    labels = [token[len(label_prefix):] for token in tokens
              if token.startswith(label_prefix)]
    words = [token for token in tokens if not token.startswith(label_prefix)]

        raise ValueError("sample has no label")
    if not words:
        raise ValueError("sample has no text")
    return labels, words

import random
from collections import defaultdict

def stratified_split(records, seed=7):
    by_label = defaultdict(list)
    for text, label in records:
        by_label[label].append((text, label))
    rng = random.Random(seed)
    train, valid, test = [], [], []
    for group in by_label.values():
        rng.shuffle(group)
        if len(group) < 5:
            raise ValueError("each label needs at least five samples")
        valid.append(group[0])
        test.append(group[1])
        train.extend(group[2:])
    rng.shuffle(train)
    return train, valid, test

records = [(f"sports sample {i}", "sports") for i in range(6)]
records += [(f"tech sample {i}", "tech") for i in range(6)]

assert len(train) == 8 and len(valid) == len(test) == 2
assert sets[0].isdisjoint(sets[1] | sets[2]) and sets[1].isdisjoint(sets[2])

import hashlib

def word_ngrams(tokens, max_n):
    if max_n < 1:
        raise ValueError("max_n must be positive")
    return [tuple(tokens[start:start + n])
            for n in range(1, max_n + 1)
            for start in range(len(tokens) - n + 1)]

def stable_bucket(feature, bucket_size):
    raw = "\x1f".join(feature).encode("utf-8")
    digest = hashlib.blake2b(raw, digest_size=8).digest()
    return int.from_bytes(digest, "little") % bucket_size

import hashlib
import torch

torch.manual_seed(0)

def char_ngrams(word, minn=3, maxn=5):
    bounded = f"<{word}>"
    return [bounded[start:start + n]
            for n in range(minn, maxn + 1)
            for start in range(len(bounded) - n + 1)]

def bucket_id(text, bucket_size):
    digest = hashlib.blake2b(text.encode("utf-8"), digest_size=8).digest()
    return int.from_bytes(digest, "little") % bucket_size

bucket_vectors = torch.randn(101, 6)

def oov_vector(word):
    grams = char_ngrams(word)
    indices = torch.tensor([bucket_id(gram, len(bucket_vectors)) for gram in grams])
    return bucket_vectors[indices].mean(dim=0), set(grams)

50. 用 EmbeddingBag(mode='mean') 实现一个微型 FastText 风格分类器并过拟合训练集。

✅ 参考答案

EmbeddingBag 直接对每条文本的离散特征求均值；这里使用词 unigram，加入 bigram 时只需把它们分配到同一特征表。

Python
import torch
from torch import nn

torch.manual_seed(1)
torch.set_num_threads(1)

texts = [
    ["team", "wins"], ["player", "scores"], ["team", "scores"],
    ["new", "software"], ["computer", "chip"], ["software", "chip"],
]
labels = torch.tensor([0, 0, 0, 1, 1, 1])
vocab = {word: index for index, word in enumerate(sorted({w for t in texts for w in t}))}

encoded = [torch.tensor([vocab[word] for word in text]) for text in texts]
flat = torch.cat(encoded)
offsets = torch.tensor([0] + [len(item) for item in encoded[:-1]]).cumsum(0)

    def __init__(self, feature_count, dim, class_count):
        super().__init__()
        self.embedding = nn.EmbeddingBag(feature_count, dim, mode="mean")
        self.output = nn.Linear(dim, class_count)

    def forward(self, feature_ids, sample_offsets):
        return self.output(self.embedding(feature_ids, sample_offsets))

model = FastTextLike(len(vocab), dim=8, class_count=2)
optimizer = torch.optim.Adam(model.parameters(), lr=0.08)
criterion = nn.CrossEntropyLoss()

for _ in range(150):
    optimizer.zero_grad()
    logits = model(flat, offsets)
    loss = criterion(logits, labels)
    loss.backward()
    optimizer.step()

predictions = model(flat, offsets).argmax(dim=1)
assert torch.equal(predictions, labels)
assert loss.item() < 0.01
print({"loss": round(loss.item(), 6), "accuracy": 1.0})

51. 根据阈值计算多标签分类的 micro 与 macro Precision、Recall、F1。

✅ 参考答案

阈值应在验证集确定；macro 对每个标签等权，更能暴露长尾标签退化。

Python
import torch

def multilabel_metrics(probabilities, targets, threshold=0.5):
    predicted = probabilities >= threshold
    targets = targets.bool()
    tp = (predicted & targets).sum(dim=0).float()
    fp = (predicted & ~targets).sum(dim=0).float()
    fn = (~predicted & targets).sum(dim=0).float()

    precision_per_label = tp / (tp + fp).clamp_min(1)
    recall_per_label = tp / (tp + fn).clamp_min(1)
    f1_per_label = 2 * precision_per_label * recall_per_label
    f1_per_label /= (precision_per_label + recall_per_label).clamp_min(1e-12)

    micro_p = tp.sum() / (tp + fp).sum().clamp_min(1)
    micro_r = tp.sum() / (tp + fn).sum().clamp_min(1)
    micro_f1 = 2 * micro_p * micro_r / (micro_p + micro_r).clamp_min(1e-12)
    return micro_p, micro_r, micro_f1, f1_per_label.mean()

probabilities = torch.tensor([[.9, .7, .1], [.8, .2, .6], [.1, .4, .7]])
targets = torch.tensor([[1, 1, 0], [1, 0, 0], [0, 1, 1]])
micro_p, micro_r, micro_f1, macro_f1 = multilabel_metrics(probabilities, targets)

assert torch.isclose(micro_p, torch.tensor(4 / 5))
assert torch.isclose(micro_r, torch.tensor(4 / 5))
assert torch.isclose(micro_f1, torch.tensor(0.8))
assert 0 <= macro_f1 <= 1
print(float(micro_f1), float(macro_f1))

from torch import nn

direction_probabilities = torch.where(
    path_bits.bool(), torch.sigmoid(path_logits), torch.sigmoid(-path_logits)

55. 手工实现多标签 P@k 与 R@k，证明 model.test() 第三项不是 F1。

✅ 参考答案

P@k 的分母是每个样本返回的预测数，R@k 的分母是真实标签总数；以下实现按整个语料汇总。

Python
def precision_recall_at_k(ranked_predictions, true_labels, k):
    hits = 0
    predicted_count = 0
    true_count = 0
    for ranked, truth in zip(ranked_predictions, true_labels):
        selected = ranked[:k]
        hits += len(set(selected) & set(truth))
        predicted_count += len(selected)
        true_count += len(set(truth))
    precision = hits / predicted_count if predicted_count else 0.0
    recall = hits / true_count if true_count else 0.0
    return precision, recall

ranked = [["a", "b", "c"], ["b", "c", "a"], ["c", "a", "b"]]
truth = [{"a", "b"}, {"a"}, {"b", "c"}]
p_at_1, r_at_1 = precision_recall_at_k(ranked, truth, k=1)
f1 = 2 * p_at_1 * r_at_1 / (p_at_1 + r_at_1)

assert p_at_1 == 2 / 3       # 三个 top-1 中两个命中
assert r_at_1 == 2 / 5       # 五个真实标签只召回两个
assert f1 != r_at_1          # test() 的第三项是 R@k，不是 F1
print({"P@1": p_at_1, "R@1": r_at_1, "derived_F1": f1})

🧊 Milvus：向量数据库原理与操作综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：Milvus：向量数据库原理与操作🚀 所属项目：EduRAG：企业私有知识智能问答

40 道练习，覆盖 Milvus 核心概念、Collection/Field Schema、索引与度量、以及增删改查、分区、过滤/范围检索与混合检索重排。重点练「何时用哪种索引/度量」和「search / filter / hybrid 的工程取舍」。

建议先独立作答，再展开参考答案。代码题只用纯 Python 模拟向量检索思想，不依赖真实 pymilvus 服务。

一、选择题（20 题）

1. Milvus 作为向量数据库，最核心的设计目标是？

A. 只存储 CSV 表格并做 SQL JOIN
B. 存储、索引并管理由深度网络等模型生成的大规模嵌入向量
C. 替代操作系统的文件系统
D. 专门训练 Transformer 权重

✅ 参考答案

答案：B。 课件强调它面向「从非结构化数据转换而来的嵌入向量」，并通过相似距离分析相关性。

2. 非结构化数据转为计算机可比较形式的关键步骤通常是？

A. 直接按文件名排序即可
B. 用嵌入技术转换为向量，再入库检索
C. 必须先手工写成关系表主键
D. 只能压缩成 ZIP

✅ 参考答案

答案：B。 图像、文本、音频等先变成向量，Milvus 再负责存与搜；向量相近往往意味着原始对象语义相近。

3. 向量相似度搜索为加速常用哪类算法思想？

A. 只对主键做二分查找
B. 近似最近邻（ANNS）
C. 冒泡排序全文扫描 SQL
D. 随机丢弃一半向量

✅ 参考答案

答案：B。 精确暴力搜索在大规模下昂贵；ANNS 用可接受的近似换取速度。

4. 在 Milvus 与关系数据库的对应关系中，Collection 更接近？

A. 一行记录
B. 一张表
C. 一个外键约束
D. 一条 SQL 事务日志

✅ 参考答案

答案：B。 Collection ↔ 表，Field ↔ 列；向量字段额外有 dim 等约束。

5. 关于主键字段，课件明确的约束是？

A. 一个 Collection 可以有任意多个主键
B. 一个 Collection 仅支持一个主键字段
C. 主键必须是 FLOAT_VECTOR
D. 主键不能命名为 id

✅ 参考答案

答案：B。 Field schema 中用 is_primary 标记；另可用 auto_id 自动分配主键。

6. 课件提到一个 Collection 最多支持几个向量 Field？

A. 1
B. 2
C. 4
D. 无限制

✅ 参考答案

答案：C。 多向量场用于多模态/多路检索，但仍有上限；设计 Schema 时要规划清楚。

7. 向量字段 dim 的合法范围（课件给出）是？

A. 只能是 768
B. $\in[1, 32768]$
C. 必须大于 100000
D. 只能是偶数

✅ 参考答案

答案：B。 维度由嵌入模型决定，创建字段时必须声明并与写入向量长度一致。

8. VARCHAR 字段的 max_length 课件给出的范围是？

A. $[1, 255]$
B. $[1, 65535]$
C. 只能是 16
D. 无上限

✅ 参考答案

答案：B。 超长文本通常应切分后放入向量段，标量字段存标题/标签/短元数据更合适。

9. 若创建 Collection 时未指定索引类型，Milvus 默认行为是？

A. 拒绝任何搜索
B. 使用暴力搜索（brute-force）
C. 自动训练神经网络
D. 删除全部数据

✅ 参考答案

答案：B。 课件强调：搜索前应声明索引与度量；未指定则走默认暴力检索。

10. FLAT 索引最适合的场景是？

A. 万亿级数据且只要毫秒级近似
B. 小型、百万级数据集上追求完全精确
C. 只存标量、不存向量
D. 必须配合乘积量化

✅ 参考答案

答案：B。 FLAT 即暴力精确搜，精度最高但大数据量时效率差。

11. IVF_FLAT 中增大 nprobe 的直接影响是？

A. 一定减少召回簇数，更快但更不准
B. 搜索更多簇，通常更准但更慢
C. 改变向量维度
D. 删除倒排索引

✅ 参考答案

答案：B。 nprobe 控制查询时探测的簇数量，是精度—延迟的核心旋钮。

12. IVF_SQ8 相对 IVF_FLAT 的关键压缩手段是？

A. 删除所有簇中心
B. 标量量化：把每维 4 字节浮点压成约 1 字节整数
C. 把向量改成字符串主键
D. 只保留前 3 维

✅ 参考答案

答案：B。 以可接受精度损失换存储与带宽；思想仍基于 IVF 倒排。

13. IVF_PQ 结合了哪两类技术？

A. 仅 B 树与哈希表
B. 倒排文件（IVF）与乘积量化（PQ）
C. 仅正则与全文索引
D. 仅 JPEG 压缩

✅ 参考答案

答案：B。 先按簇缩小候选，再在簇内用 PQ 降低存储与距离计算成本，适合大规模高维。

14. HNSW 被课件归类为？

A. 纯倒排簇心索引
B. 基于图的索引，适合高搜索效率场景
C. 只用于标量 INVERTED
D. 必须关闭 ANNS

✅ 参考答案

答案：B。 图索引在许多低延迟检索场景表现好，实际选型仍要结合数据规模与召回要求。

15. 对浮点嵌入，课件列出的常用度量不包括？

A. 欧几里得距离（L2）
B. 内积（IP）
C. 余弦相似度（COSINE）
D. 编辑距离（Levenshtein）

✅ 参考答案

答案：D。 编辑距离面向字符串；向量检索常用 L2 / IP / COSINE。

16. MilvusClient(uri="milvus_demo.db") 与 uri="http://localhost:19530" 的区别是？

A. 没有区别
B. 前者偏本地文件库（Lite）；后者连接独立/后台 Milvus 服务
C. 前者只能删库，后者只能建库
D. 后者禁止创建 Collection

✅ 参考答案

答案：B。 课件用 Lite 做嵌入式实验；生产可用 Docker/K8s 部署的服务端。

17. enable_dynamic_field=True 时，未在 Schema 声明的字段通常会？

A. 导致进程崩溃且无法恢复
B. 作为动态字段以键值形式保存（常在 $meta）
C. 自动变成第二个主键
D. 被静默丢弃且无任何存储

✅ 参考答案

答案：B。 便于快速迭代元数据；生产中对关键字段仍建议显式建模以便过滤与索引。

18. upsert 的语义是？

A. 只删除、不写入
B. 主键已存在则覆盖，不存在则插入
C. 永远追加一条新主键
D. 只更新向量、禁止更新标量

✅ 参考答案

答案：B。 注意分区场景：在指定分区 upsert 时，行为需结合文档理解，避免「以为更新了分区却只影响集合侧」的误用。

19. 过滤搜索示例 color like "red%" 的作用是？

A. 把所有颜色改成 red
B. 在向量相似检索结果上叠加标量条件（颜色以 red 为前缀）
C. 强制改用 FLAT 索引
D. 关闭 output_fields

✅ 参考答案

答案：B。 这是「向量相似 + 标量过滤」的混合能力，也是选 Milvus 的理由之一。

20. 范围搜索中 radius 与 range_filter 更准确的理解是？

A. 二者都表示向量维度
B. radius 定义外侧距离边界；range_filter 可再设内侧边界，形成距离环带
C. 只能用于 VARCHAR 字段
D. 用于删除 Collection

✅ 参考答案

答案：B。 适合「只要足够相似、但又不要几乎重复/过近」等带状召回需求（具体开闭与度量方向要对照所用 metric）。

二、简答题（12 题）

21. 用对比句说明：Milvus 与 MySQL 各自更擅长什么？

✅ 参考答案

MySQL 等关系库擅长结构化行数据的事务、复杂 JOIN 与强一致业务表；Milvus 擅长高维向量的相似检索与向量索引。RAG 里常见组合：业务/FAQ 走 MySQL，语义知识片段走 Milvus。

22. 解释「两个嵌入向量很相似 ⇒ 原始对象也可能相似」成立的前提与风险。

✅ 参考答案

前提是嵌入模型把语义/视觉等特征投影到度量空间，且所用度量与训练目标匹配。风险：模型偏差、过短文本、跨域数据、度量选错或未归一化，都会让「近邻」不等于业务相似。

23. 写出创建 Collection 前 Field schema 至少应 prioritise 的 4 个决策点。

✅ 参考答案

示例：主键策略（手工 id 还是 auto_id）；向量字段名/dim/度量；需要过滤的标量字段类型与 max_length；是否启用动态字段/分区键。先定 Schema，再谈索引参数。

24. 为什么说「索引是搜索前必须想清楚的组织方式」？

✅ 参考答案

索引决定候选集如何剪枝。未建索引时默认暴力搜，小数据可接受，大数据会爆延迟与成本。IVF/HNSW 等用近似换速度，需要用评测集看召回是否达标。

25. 简述 IVF_FLAT 的三步：聚类、倒排、查询。

✅ 参考答案
  1. 用聚类把向量划到若干簇并保留簇心；
  2. 向量映射到所属簇形成倒排；
  3. 查询先找近邻簇（由 nprobe 控制），再在簇内精确扫描。

26. 在什么业务条件下你更倾向 FLAT，而不是 IVF_PQ？

✅ 参考答案

数据规模小、必须尽量精确、可接受较高延迟，或正处于召回评测基线阶段。IVF_PQ 更适合大规模高维、要省存储/加速、可接受近似误差的生产检索。

27. 说明 L2、IP、COSINE 各自更「敏感」的几何含义（直觉即可）。

✅ 参考答案

L2 看欧氏距离，对绝对位置/模长差异敏感；IP 看内积，常与未归一化向量的「对齐程度」相关；COSINE 关注方向，弱化模长差异。选型应与嵌入是否归一化、训练损失一致。

28. insert 到指定 partition_name 有什么工程价值？

✅ 参考答案

可按租户、时间、业务线隔离数据，缩小搜索范围（partition_names），提升性能与治理清晰度。但也增加路由与运维复杂度，需要明确分区键策略。

29. 单向量 search、批量 search、多向量 hybrid_search 三者差别？

✅ 参考答案

单向量：一次查询一个向量；批量：一次请求多个查询向量，各自返回 top-k；hybrid_search：对多个向量字段分别 ANN，再用 Ranker 融合，不是简单把多个 query 拼成 batch。

30. WeightedRanker 与 RRFRanker 如何选型？

✅ 参考答案

有明确「某路更重要」（如文本描述 > 颜色）用加权；没有先验侧重、希望平衡多路排名用 RRF。权重或 k 都要用业务评测校准，不能只凭感觉。

31. 为什么过滤条件写错（如字段未建或类型不匹配）会比「向量不准」更隐蔽？

✅ 参考答案

过滤可能直接把正确近邻剔除，表现为「检索很差」，但根因在标量侧。应用日志应同时记录 filter、命中候选数与 top 分数，便于区分向量问题与过滤问题。

32. Milvus Lite 本地 .db 文件带来的开发便利与生产局限各是什么？

✅ 参考答案

便利：嵌入式、易入门、进程重启可复开文件续写。局限：扩展性、高可用、多副本与运维能力不如独立集群；生产应评估 Docker/K8s 服务化部署。

33. 实现 L2 / 内积 / 余弦三种度量

对等长浮点列表实现 l2、ip、cosine；零向量余弦返回 0.0。

✅ 参考答案
Python
import math


def l2(a, b):
    return math.sqrt(sum((x - y) ** 2 for x, y in zip(a, b)))


def ip(a, b):
    return sum(x * y for x, y in zip(a, b))


def cosine(a, b):
    na = math.sqrt(sum(x * x for x in a))
    nb = math.sqrt(sum(y * y for y in b))
    if na == 0 or nb == 0:
        return 0.0
    return ip(a, b) / (na * nb)


q = [1.0, 0.0]
v = [0.6, 0.8]
assert abs(l2(q, q)) < 1e-9
assert abs(ip(q, v) - 0.6) < 1e-9
assert abs(cosine(q, v) - 0.6) < 1e-9
print(l2(q, v), cosine(q, v))

34. 实现 FLAT top-k 搜索（越小越好用 L2）

entities 为 {"id", "vector", ...} 列表；返回 [(id, distance), ...] 按距离升序。

✅ 参考答案
Python
import math


def flat_search(query, entities, k=3):
    scored = []
    for e in entities:
        dist = math.sqrt(sum((x - y) ** 2 for x, y in zip(query, e["vector"])))
        scored.append((e["id"], dist))
    scored.sort(key=lambda x: (x[1], x[0]))
    return scored[:k]


data = [
    {"id": 1, "vector": [0.0, 0.0]},
    {"id": 2, "vector": [1.0, 0.0]},
    {"id": 3, "vector": [0.0, 2.0]},
]
assert flat_search([0.1, 0.0], data, 2)[0][0] == 1
print(flat_search([0.1, 0.0], data, 2))

35. 用极简 IVF 模拟 nprobe 对候选集的影响

先按簇心把库划到 2 个簇；查询时只扫描距离最近的 nprobe 个簇，再在候选内做 L2 top-k。

✅ 参考答案
Python
import math


def assign(vec, centroids):
    return min(range(len(centroids)), key=lambda i: math.dist(vec, centroids[i]))


def ivf_search(query, entities, centroids, nprobe=1, k=2):
    # build inverted lists
    buckets = {i: [] for i in range(len(centroids))}
    for e in entities:
        buckets[assign(e["vector"], centroids)].append(e)

    probe = sorted(range(len(centroids)), key=lambda i: math.dist(query, centroids[i]))[:nprobe]
    cand = [e for i in probe for e in buckets[i]]
    scored = [(e["id"], math.dist(query, e["vector"])) for e in cand]
    scored.sort(key=lambda x: (x[1], x[0]))
    return scored[:k], sorted(probe)


centroids = [[0.0, 0.0], [10.0, 10.0]]
entities = [
    {"id": 1, "vector": [0.1, 0.1]},
    {"id": 2, "vector": [0.2, -0.1]},
    {"id": 3, "vector": [9.5, 9.7]},
]
hits1, probe1 = ivf_search([0.0, 0.0], entities, centroids, nprobe=1, k=2)
hits2, probe2 = ivf_search([0.0, 0.0], entities, centroids, nprobe=2, k=2)
assert probe1 == [0]
assert set(probe2) == {0, 1}
print(hits1, hits2)

36. 实现 Schema 校验器：主键唯一、向量维一致、最多 4 个向量场

fields 为字段定义列表；validate_schema(fields) 不合规时抛 ValueError。

✅ 参考答案
Python
VECTOR_TYPES = {"FLOAT_VECTOR"}


def validate_schema(fields):
    if not fields:
        raise ValueError("empty schema")
    primaries = [f for f in fields if f.get("is_primary")]
    if len(primaries) != 1:
        raise ValueError("exactly one primary key required")
    vectors = [f for f in fields if f.get("dtype") in VECTOR_TYPES]
    if len(vectors) > 4:
        raise ValueError("at most 4 vector fields")
    for f in vectors:
        dim = f.get("dim")
        if not isinstance(dim, int) or not (1 <= dim <= 32768):
            raise ValueError(f"invalid dim for {f.get('name')}")
    return True


ok = [
    {"name": "id", "dtype": "INT64", "is_primary": True},
    {"name": "vector", "dtype": "FLOAT_VECTOR", "dim": 768},
]
assert validate_schema(ok)
try:
    validate_schema(ok + [{"name": "v2", "dtype": "FLOAT_VECTOR", "dim": 0}])
    raise AssertionError("should fail")
except ValueError:
    pass
print("schema ok")

37. 实现内存集合的 upsert + filter 删除

支持：upsert(rows)、delete(ids=None, filter_prefix=None)。filter_prefix 匹配 color 前缀（模拟 like "red%"）。

✅ 参考答案
Python
class MemCollection:
    def __init__(self):
        self.rows = {}

    def upsert(self, rows):
        for row in rows:
            self.rows[row["id"]] = dict(row)

    def delete(self, ids=None, filter_prefix=None):
        remove = set(ids or [])
        if filter_prefix is not None:
            remove |= {i for i, r in self.rows.items() if str(r.get("color", "")).startswith(filter_prefix)}
        for i in list(remove):
            self.rows.pop(i, None)
        return sorted(remove)


c = MemCollection()
c.upsert([
    {"id": 1, "color": "red_1", "vector": [1, 0]},
    {"id": 2, "color": "blue_1", "vector": [0, 1]},
    {"id": 3, "color": "red_9", "vector": [1, 1]},
])
c.upsert([{"id": 2, "color": "blue_2", "vector": [0, 2]}])
assert c.rows[2]["color"] == "blue_2"
deleted = c.delete(filter_prefix="red")
assert deleted == [1, 3]
print(c.rows)

38. 实现「向量 top-k + 标量过滤」的搜索

先按内积取较大者排序，再应用 color like f"{prefix}%"，最后截断 limit。

✅ 参考答案
Python
def filtered_ip_search(query, entities, prefix, limit=3):
    scored = []
    for e in entities:
        if not str(e.get("color", "")).startswith(prefix):
            continue
        score = sum(x * y for x, y in zip(query, e["vector"]))
        scored.append((e["id"], score, e["color"]))
    scored.sort(key=lambda x: (-x[1], x[0]))
    return scored[:limit]


data = [
    {"id": 1, "vector": [1.0, 0.0], "color": "red_1"},
    {"id": 2, "vector": [0.9, 0.1], "color": "blue_1"},
    {"id": 3, "vector": [0.8, 0.0], "color": "red_2"},
]
hits = filtered_ip_search([1.0, 0.0], data, "red", 2)
assert [h[0] for h in hits] == [1, 3]
print(hits)

39. 实现范围检索：保留 IP 分数落在 (range_filter, radius] 的结果

按课件直觉：radius 为外侧边界，range_filter 为内侧边界；本题约定分数满足 range_filter < score <= radius。

✅ 参考答案
Python
def range_ip_search(query, entities, radius, range_filter, limit=5):
    hits = []
    for e in entities:
        score = sum(x * y for x, y in zip(query, e["vector"]))
        if range_filter < score <= radius:
            hits.append((e["id"], score))
    hits.sort(key=lambda x: (-x[1], x[0]))
    return hits[:limit]


data = [
    {"id": 1, "vector": [1.0, 0.0]},
    {"id": 2, "vector": [0.5, 0.0]},
    {"id": 3, "vector": [0.95, 0.0]},
]
# keep moderately high IP, drop near-perfect and weak
out = range_ip_search([1.0, 0.0], data, radius=0.96, range_filter=0.6, limit=5)
assert [x[0] for x in out] == [3]
print(out)

40. 实现简化版 RRF 融合两条检索列表

对文档 id，RRF 分数为 $\sum_i 1/(k + \mathrm{rank}_i)$，rank 从 0 起；返回按分数降序的 top-n。

✅ 参考答案
Python
def rrf_fuse(lists, k=60, n=3):
    scores = {}
    for ranking in lists:
        for rank, doc_id in enumerate(ranking):
            scores[doc_id] = scores.get(doc_id, 0.0) + 1.0 / (k + rank)
    return sorted(scores.items(), key=lambda x: (-x[1], x[0]))[:n]


fused = rrf_fuse([["A", "B", "C"], ["B", "C", "D"]], k=60, n=3)
assert fused[0][0] == "B"  # appears high in both
print(fused)

这与课件 RRFRanker 同思想：无先验权重时，用排名位置融合多路 ANN；k 控制名次衰减速度。

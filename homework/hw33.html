🖼️ 卷积神经网络综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：卷积神经网络：图像表示、卷积与分类实战

55 道练习，覆盖图像张量与通道布局、卷积计算与输出尺寸、感受野、多通道与参数量、池化与上采样、Conv2d API 和 CIFAR-10 分类训练闭环。

一、选择题（30 题）

1. 一张 RGB 彩色图在 NumPy/Matplotlib 中最常见的形状是什么？

A. [3, H, W]
B. [H, W]
C. [H, W, 3]
D. [3, W, H]

✅ 参考答案

答案：C。 图像库常用 HWC 布局，最后一维依次是红、绿、蓝通道；PyTorch 卷积层才使用 CHW/NCHW。

2. uint8 图像进入训练管线时，常见的两步预处理顺序是什么？

A. 先把像素缩放到 $[0,1]$，再按通道执行 $(x-\mu)/\sigma$ 标准化
B. 先做 PCA 降维，再转成整数
C. 先标准化，再取绝对值保证非负
D. 先转成 one-hot，再展平

✅ 参考答案

答案：A。 先除以 255 缩放，再用训练集统计的均值方差做通道标准化；标准化后的数值可以小于 0 或大于 1。

3. 把 HWC 图像张量转成 CHW，应使用哪个操作？

A. reshape(3, H, W)
B. flatten()
C. transpose(0, 1)
D. permute(2, 0, 1)

✅ 参考答案

答案：D。 permute 重排维度语义；reshape 只按内存顺序重新划分，会把像素错位到错误通道。

4. NCHW 布局中的 N 表示什么？

A. 通道数
B. 批量大小
C. 图像高度
D. 卷积核个数

✅ 参考答案

答案：B。 N 是 batch 维；C、H、W 分别是通道、高、宽。卷积层输入通常是 [N, C, H, W] 浮点张量。

5. OpenCV 的 cv2.imread 默认按什么通道顺序读取彩色图？

A. BGR
B. RGB
C. RGBA
D. 灰度

✅ 参考答案

答案：A。 OpenCV 默认 BGR，与 Matplotlib/PIL 的 RGB 习惯不同，显示或训练前应 cv2.cvtColor(img, cv2.COLOR_BGR2RGB)。

6. image_bgr[:, :, ::-1] 之后直接 torch.from_numpy 可能失败，原因是什么？

A. 数据类型变成了 float64
B. 反向切片改变了图像高度
C. NumPy 不支持三维数组
D. 反向切片产生负 stride 视图，需要 .copy() 得到连续内存

✅ 参考答案

答案：D。 负 stride 的 NumPy 视图不被 from_numpy 接受；.copy() 或 np.ascontiguousarray 可创建兼容存储。

7. 通道标准化使用的均值和标准差应当来自哪里？

A. 训练集、验证集和测试集的全体像素
B. 只来自训练集
C. 只来自测试集
D. 每个 batch 现场重新统计

✅ 参考答案

答案：B。 用全数据统计会把测试信息泄漏进训练过程；验证与测试应原样复用训练集统计量。

8. 卷积层“权重共享”指的是什么？

A. 不同网络之间共享下载的预训练权重
B. 卷积核与全连接层共用一份参数
C. 同一卷积核在输入的所有空间位置复用同一组参数
D. 每个像素各自持有一份卷积核

✅ 参考答案

答案：C。 权重共享让参数量与图像尺寸解耦，也是卷积平移等变性的来源之一。

9. 关于卷积与“平移”，下列说法正确的是？

A. 卷积本身是平移等变的；近似平移不变性主要靠池化、全局聚合和数据增强获得
B. 卷积天然是严格平移不变的
C. 输入平移后特征图完全不变
D. 只有全连接层才有平移等变性

✅ 参考答案

答案：A。 输入平移时特征图相应平移（等变）；“平移不变”是经过聚合后的近似性质，不是卷积自身的严格属性。

10. PyTorch Conv2d 实际计算的运算是什么？

A. 数学卷积（先翻转核再滑动）
B. 外积
C. 傅里叶变换
D. 互相关（不翻转核的局部加权求和）

✅ 参考答案

答案：D。 深度学习框架不翻转核；因为核参数由数据学习，是否预翻转不影响表达能力，习惯上仍称“卷积”。

11. 输入边长 32，kernel_size=3, padding=1, stride=1, dilation=1，输出边长是多少？

A. 30
B. 32
C. 31
D. 16

✅ 参考答案

答案：B。 有效核 $K_{eff}=D(K-1)+1=3$，输出为 $\lfloor(32+2\times1-3)/1\rfloor+1=32$。这正是 $P=(K-1)/2$ 的 same 卷积，高宽保持不变。

12. 输入边长 7，kernel_size=3, padding=0, stride=2，输出边长是多少？

A. 5
B. 4
C. 3
D. 2

✅ 参考答案

答案：C。 $\lfloor(7+0-3)/2\rfloor+1=\lfloor 4/2\rfloor+1=3$。步幅为 2 时别忘了向下取整。

13. stride=1、dilation=1 时，奇数核保持输入输出同尺寸的 padding 是多少？

A. $(K-1)/2$
B. $K$
C. $K/2$
D. $K+1$

✅ 参考答案

答案：A。 例如 $3\times3$ 核取 $P=1$、$5\times5$ 核取 $P=2$，即所谓 same 卷积。

14. dilation=2 的 $3\times3$ 卷积核，有效覆盖范围是多大？

A. $3\times3$
B. $4\times4$
C. $7\times7$
D. $5\times5$

✅ 参考答案

答案：D。 有效核大小 $K_{eff}=D(K-1)+1=2\times2+1=5$；空洞卷积在不增加核参数的情况下扩大视野。

15. 关于偶数尺寸卷积核（如 $2\times2$、$4\times4$），正确的说法是？

A. 框架禁止使用偶数核
B. 偶数核参数量一定更大
C. 偶数核完全可以计算和使用，只是没有单一中心、对称 same 对齐不方便
D. 偶数核只能用于灰度图

✅ 参考答案

答案：C。 正确结论是“奇数核更方便”，而不是“偶数核不能用”；部分模型确实使用过 $2\times2$、$4\times4$ 卷积。

16. 两层 $3\times3$、stride 1 的卷积堆叠后，理论感受野是多大？

A. $3\times3$
B. $5\times5$
C. $9\times9$
D. $6\times6$

✅ 参考答案

答案：B。 递推 $r_l=r_{l-1}+(K-1)j_{l-1}$：$1\to3\to5$；第三层继续扩大到 $7\times7$。小核堆叠既扩大感受野又增加非线性。

17. 输入有 $C_{in}$ 个通道时，一个标准卷积核（groups=1）应有多少个通道？

A. 1 个
B. 由输出通道数决定
C. 任意个
D. $C_{in}$ 个，各通道结果相加得到一个输出通道

✅ 参考答案

答案：D。 标准卷积的每个输出通道都会融合全部输入通道：先逐通道做二维互相关，再沿输入通道求和并加偏置。

18. nn.Conv2d(3, 16, kernel_size=3) 的 weight 形状是什么？

A. [16, 3, 3, 3]
B. [3, 16, 3, 3]
C. [16, 16, 3, 3]
D. [3, 3, 16]

✅ 参考答案

答案：A。 卷积权重形状为 [C_out, C_in/groups, K_h, K_w]；bias 形状为 [16]。

19. nn.Conv2d(3, 6, kernel_size=3)（含偏置）的参数量是多少？

A. 54
B. 162
C. 168
D. 174

✅ 参考答案

答案：C。 $C_{out}(C_{in}K_hK_w+1)=6\times(3\times3\times3+1)=6\times28=168$。参数量与输入高宽无关。

20. 关于标准卷积层的偏置，正确的说法是？

A. 每个输入通道一个偏置
B. 每个输出通道一个偏置
C. 每个空间位置一个偏置
D. 整层只有一个偏置

✅ 参考答案

答案：B。 偏置沿输出通道计数，bias 形状为 [C_out]；“每个输入通道一个偏置”是常见误解。

21. 1×1 卷积（stride 1、padding 0）的主要作用是什么？

A. 在不改变空间尺寸的情况下对通道做线性组合，实现升降维与通道融合
B. 扩大感受野
C. 代替池化做空间下采样
D. 检测边缘方向

✅ 参考答案

答案：A。 它逐位置混合通道信息，是瓶颈结构和深度可分离卷积中 pointwise 卷积的核心；不会扩大空间感受野。

22. 深度可分离卷积由哪两步组成？

A. 两个全连接层
B. 最大池化 + 平均池化
C. 转置卷积 + 双线性插值
D. depthwise 的 K×K 逐通道卷积 + pointwise 的 1×1 卷积

✅ 参考答案

答案：D。 depthwise（groups=C_in）逐通道提取空间特征但不融合通道，pointwise 再做通道融合，总计算量显著低于标准卷积。

23. 关于转置卷积（ConvTranspose2d），正确的说法是？

A. 它是普通卷积的严格逆运算，能恢复丢失的细节
B. 它是可学习的上采样算子；核与步幅搭配不当可能产生棋盘格伪影
C. 它没有可训练参数
D. 它只能把尺寸缩小

✅ 参考答案

答案：B。 转置卷积扩大空间尺寸且核可学习，但不能“逆转”卷积丢失的信息；kernel_size=4, stride=2, padding=1 是常见的减伪影组合。

24. 池化层的输出通道数与输入通道数是什么关系？

A. 输出通道减半
B. 输出通道由池化窗口决定
C. $C_{out}=C_{in}$，池化对每个通道独立操作
D. 所有通道被合并成 1 个

✅ 参考答案

答案：C。 池化只在空间窗口内聚合，不像标准卷积那样沿通道求和；空间尺寸仍按窗口、步幅、填充计算。

25. 对形状 [N, C, H, W] 的特征图做全局平均池化（GAP），输出形状是什么？

A. [N, C, 1, 1]
B. [N, 1, H, W]
C. [1, C, H, W]
D. [N, C] 且无法再变形

✅ 参考答案

答案：A。 GAP 把每个通道整张特征图压缩为一个均值，大幅减少分类头参数，也让网络更容易接收不同输入尺寸。

26. 卷积特征送入全连接分类头前，应如何展平？

A. x.flatten() 直接展平全部维度
B. x.reshape(-1)
C. x.squeeze()
D. torch.flatten(x, start_dim=1)，保留 batch 维

✅ 参考答案

答案：D。 从维度 1 开始展平得到 [N, C×H×W]；x.flatten() 会把多个样本压成一个长向量，破坏批量结构。

27. 多分类模型输出交给 CrossEntropyLoss 时，正确的做法是？

A. 先做 Softmax 再传入
B. 直接传入 raw logits，损失内部包含稳定的 LogSoftmax
C. 先做 ReLU 保证非负
D. 先取 argmax

✅ 参考答案

答案：B。 在模型末尾加 Softmax 再交给交叉熵会重复归一化并降低数值稳定性；标签用 long 型类别下标。

28. 验证/测试阶段的规范写法是什么？

A. 只调用 model.eval() 就够了
B. 只用 torch.inference_mode() 就够了
C. model.eval() 切换层行为，torch.inference_mode() 关闭梯度记录，两者配合使用
D. 继续保持 model.train() 以获得更真实的结果

✅ 参考答案

答案：C。 eval() 影响 Dropout/BatchNorm 的行为但不关计算图；inference_mode() 关闭梯度但不切换层模式，职责不同、缺一不可。

29. 统计一个 epoch 的平均损失，规范做法是什么？

A. 直接平均每个 batch 的 loss.item()
B. 只看最后一个 batch 的损失
C. 取所有 batch 损失的最大值
D. 按每个 batch 的样本数加权平均

✅ 参考答案

答案：D。 损失函数返回 batch 内平均值，而最后一个 batch 可能更小，简单平均会引入偏差；应累计 loss.item() * batch_size 再除以总样本数。

30. 关于 CIFAR-10 数据集，正确的描述是？

A. 50,000 张训练图、10,000 张测试图，$32\times32$ RGB，10 个互斥类别
B. 10,000 张训练图、50,000 张测试图
C. 全部是 $224\times224$ 灰度图
D. 共 100 个类别

✅ 参考答案

答案：A。 每类共 6,000 张；torchvision 的 .data 是 HWC 的 NumPy 数组，经 ToTensor 后变为 CHW 浮点张量。

二、简答题（15 题）

31. 分别说明 HWC、CHW 和 NCHW 的含义，以及为什么送入 Conv2d 前需要转换布局。

✅ 参考答案

HWC 是“高、宽、通道”，是 NumPy/Matplotlib 等图像库的常见布局；CHW 把通道放到最前，是单张 PyTorch 图像的布局；NCHW 在最前面再加批量维，是卷积层的标准输入。Conv2d 按 NCHW 语义解释各维度，若直接把 HWC 数组送入，通道会被当成高度，计算完全错位，因此要用 permute(2, 0, 1) 调整并用 unsqueeze(0) 补批量维。

32. 与把整张图展平后接全连接层相比，卷积层为什么更适合图像？

✅ 参考答案

两点核心优势：一是参数效率——卷积核只连接局部窗口且在所有位置共享权重，参数量与图像尺寸解耦，而全连接层参数随分辨率平方级增长；二是结构保留——卷积在二维网格上滑动，显式保留“哪些像素彼此相邻”的邻域关系，展平操作则完全破坏这种空间结构。此外局部连接与层次组合让浅层学边缘纹理、深层组合出部件和语义。

33. 深度学习框架中的“卷积”与数学卷积有什么区别？为什么这种区别不影响模型能力？

✅ 参考答案

数学卷积要先把核翻转 180° 再滑动求和，而 Conv2d 直接用原核做局部加权求和，严格来说是互相关。因为卷积核是可学习参数、初始值随机，网络完全可以学到“翻转版”的核，预先翻转与否不改变可表达的函数集合，所以习惯上仍称卷积。

34. 分别说明 padding、stride 和 dilation 的作用及其代价。

✅ 参考答案

padding 在边界外补值（常为 0），可控制输出尺寸并让边缘像素获得更多计算机会；代价是引入人工边界值。stride 决定核每次滑动的像素数，增大它可下采样、降低计算量，但容易丢失细节和产生混叠。dilation 在核元素间插入间隔，有效核大小变为 $D(K-1)+1$，不增加参数就扩大视野，常用于分割；过大可能产生栅格效应。

35. 什么是感受野？推导两层 $3\times3$、stride 1 卷积的感受野为什么是 $5\times5$。

✅ 参考答案

感受野是某层一个输出单元在原始输入上能影响到它的区域大小。用递推 $r_l=r_{l-1}+(K_l^{eff}-1)j_{l-1}$、$j_l=j_{l-1}S_l$：初始 $r_0=1,j_0=1$；第一层后 $r_1=1+(3-1)\times1=3$，$j_1=1$；第二层后 $r_2=3+(3-1)\times1=5$。所以两层 $3\times3$ 覆盖 $5\times5$，等效大核视野但参数更少、非线性更多。

36. 多通道输入时，一个输出通道是怎样计算出来的？该层共有多少个偏置？

✅ 参考答案

一个卷积核带 $C_{in}$ 个核面，每个核面与对应输入通道做二维互相关，把 $C_{in}$ 个结果逐位置相加，再加上这个核的偏置，得到一个输出通道。要产生 $C_{out}$ 个输出通道就需要 $C_{out}$ 组这样的核。偏置按输出通道计数，共 $C_{out}$ 个，而不是每个输入通道一个。

37. 1×1 卷积有哪些典型用途？

✅ 参考答案

它在每个空间位置对通道向量做线性组合：可以升维或降维通道（瓶颈结构）、融合通道信息、在深度可分离卷积中充当 pointwise 卷积负责通道混合，配合激活函数还能增加通道方向的非线性。stride 1、padding 0 时不改变空间尺寸，也不扩大空间感受野。

38. 比较最大池化、平均池化和全局平均池化的行为与适用场景。

✅ 参考答案

最大池化保留窗口内最强响应，突出显著特征，常用于强调“有没有出现某模式”；平均池化取窗口均值，摘要更平滑；两者都无参数、逐通道独立、用于空间下采样。全局平均池化把每个通道整张图压成一个值（[N,C,H,W]→[N,C,1,1]），常接在骨架末尾替代大规模全连接，可大幅减参并适应不同输入尺寸，代价是丢失精确位置信息。

39. 双线性插值上采样与转置卷积有什么区别？各自的注意点是什么？

✅ 参考答案

双线性插值（F.interpolate(mode="bilinear")）是固定规则的无参数放大，结果平滑、行为可预期，align_corners=False 是一般图像缩放的常用选择。转置卷积是可学习的上采样，能让网络自己决定如何填充细节，但核与步幅搭配不当会产生棋盘格伪影。两者都不能“恢复”原图中已经丢失的真实细节。

40. 为什么数据增强只应作用于训练集？标准化统计量为什么只能来自训练集？

✅ 参考答案

验证和测试的目的是估计模型在真实分布上的表现，必须使用确定性变换，否则评估结果带随机性且不可比。随机裁剪、翻转等增强只在训练时引入多样性来提升泛化。标准化的均值方差若用全部数据（含测试集）统计，等于把测试分布信息泄漏进训练过程，评估会系统性偏乐观；正确做法是训练集估计、三个集合共用。

41. 为什么要从训练集划分验证集？测试集应当怎样使用？

✅ 参考答案

调结构、学习率、轮数、挑选最佳 checkpoint 都是“选择”行为，若直接用测试集做这些选择，测试集就变成了训练信号的一部分，最终报告的指标会虚高。规范流程是：从训练集划出验证集用于全部调参与模型选择，测试集只在最终模型完全确定后评估一次，作为对泛化能力的无偏估计。

42. model.train()/model.eval() 与 torch.no_grad()/torch.inference_mode() 各自负责什么？验证时应怎样组合？

✅ 参考答案

前一组切换层的行为模式：Dropout 在 train 下随机屏蔽、eval 下关闭；BatchNorm 在 train 下用 batch 统计并更新滑动统计、eval 下用固定的滑动统计。后一组控制是否记录计算图：no_grad/inference_mode 关闭梯度追踪，省显存提速，inference_mode 更彻底。两组互不替代，验证时应同时使用 model.eval() 和 torch.inference_mode()。

43. 训练损失持续下降但验证损失转头上升，说明什么？给出至少四种应对手段。

✅ 参考答案

这是典型的过拟合：模型开始记忆训练样本的细节而非学习可泛化的模式。可以：加强数据增强（随机裁剪、翻转、颜色扰动）；加入权重衰减或 Dropout；使用早停，回退到验证最优的 checkpoint；减小模型容量或增加训练数据；在卷积后加入 BatchNorm 等正则化手段。若训练验证损失都高则是欠拟合，方向相反。

44. 修改卷积层通道数或池化参数后，为什么不能继续硬编码展平维数（如 576）？有哪些稳妥做法？

✅ 参考答案

展平维数等于最后一个特征图的 $C\times H\times W$，由全部卷积/池化参数共同决定；任何一处改动（通道、padding、stride、核大小）都会改变它，硬编码的 Linear(576, ...) 会在前向时形状不匹配报错，或更糟地静默错位。稳妥做法：开发时用一次 dummy forward 打印展平前形状再填数；或在分类头前接 AdaptiveAvgPool2d 把空间尺寸固定，使全连接输入只依赖通道数。

45. CAM 的计算原理是什么？使用它时要注意哪些局限？

✅ 参考答案

对“末层卷积 → 全局平均池化 → 线性分类器”结构，类别 $c$ 的 CAM 是 $M_c(i,j)=\sum_k w_k^{(c)}A_k(i,j)$：用分类器中类别 $c$ 对各通道的权重加权求和末层特征图，再上采样叠加到原图，显示支持该类别判断的空间区域。局限：它按类别生成，只是空间证据可视化，不是检测框，也不证明模型真正“理解”了物体；结构不满足 GAP+线性头时需改用 Grad-CAM 等梯度加权方法。

47. 实现通用卷积输出尺寸函数

实现含 dilation 的输出尺寸计算函数，并验证：(32, k=3, p=1) 得 32；(7, k=3, s=2) 得 3；(9, k=3, d=2) 得 5。

✅ 参考答案
Python
from math import floor


def conv_output_size(length, kernel, stride=1, padding=0, dilation=1):
    effective_kernel = dilation * (kernel - 1) + 1
    return floor((length + 2 * padding - effective_kernel) / stride) + 1


assert conv_output_size(32, kernel=3, padding=1) == 32
assert conv_output_size(7, kernel=3, stride=2) == 3
assert conv_output_size(9, kernel=3, dilation=2) == 5

48. 手写二维互相关并与 F.conv2d 对比

实现无 padding、stride 1 的二维互相关，用随机 5×5 输入和 3×3 核验证结果与 torch.nn.functional.conv2d 一致。

✅ 参考答案
Python
import torch
import torch.nn.functional as F


def cross_correlation_2d(image, kernel):
    out_h = image.shape[0] - kernel.shape[0] + 1
    out_w = image.shape[1] - kernel.shape[1] + 1
    output = torch.empty((out_h, out_w), dtype=torch.float32)
    for row in range(out_h):
        for col in range(out_w):
            window = image[
                row : row + kernel.shape[0],
                col : col + kernel.shape[1],
            ]
            output[row, col] = (window * kernel).sum()
    return output


torch.manual_seed(0)
image = torch.randn(5, 5)
kernel = torch.randn(3, 3)

manual = cross_correlation_2d(image, kernel)
reference = F.conv2d(
    image.view(1, 1, 5, 5),
    kernel.view(1, 1, 3, 3),
).view(3, 3)

assert torch.allclose(manual, reference, atol=1e-6)
print(manual)

49. 实现感受野递推函数

按 (kernel, stride, dilation) 列表递推理论感受野，验证两层 3×3 stride 1 为 5、两层 3×3 stride 2 为 7。

✅ 参考答案
Python
def receptive_field(layers):
    jump = 1
    field = 1
    for kernel, stride, dilation in layers:
        effective = dilation * (kernel - 1) + 1
        field += (effective - 1) * jump
        jump *= stride
    return field, jump


assert receptive_field([(3, 1, 1), (3, 1, 1)]) == (5, 1)
assert receptive_field([(3, 2, 1), (3, 2, 1)]) == (7, 4)
assert receptive_field([(3, 1, 1), (3, 1, 1), (3, 1, 1)]) == (7, 1)
print("all passed")

import torch
import torch.nn as nn


class ImageClassification(nn.Module):
    def __init__(self, num_classes=10):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 6, kernel_size=3),
            nn.ReLU(),
            nn.MaxPool2d(kernel_size=2, stride=2),
            nn.Conv2d(6, 16, kernel_size=3),

55. 用合成数据跑通训练与验证一步闭环

用随机图像和标签模拟一个 batch，完成“前向 → 交叉熵 → 清零 → 反向 → 更新”的训练步骤，再用 eval() + inference_mode() 做一次评估，验证第二次前向的损失可以计算且无梯度。

✅ 参考答案
Python
import torch
import torch.nn as nn

torch.manual_seed(0)
model = nn.Sequential(
    nn.Conv2d(3, 6, kernel_size=3),
    nn.ReLU(),
    nn.MaxPool2d(2, 2),
    nn.Conv2d(6, 16, kernel_size=3),
    nn.ReLU(),
    nn.MaxPool2d(2, 2),
    nn.Flatten(start_dim=1),
    nn.Linear(16 * 6 * 6, 10),
)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)

images = torch.randn(8, 3, 32, 32)
targets = torch.randint(0, 10, (8,))

model.train()
optimizer.zero_grad(set_to_none=True)

loss = criterion(logits, targets)
loss.backward()
optimizer.step()
print("train loss:", loss.item())

model.eval()
with torch.inference_mode():
    logits = model(images)
    eval_loss = criterion(logits, targets)
    accuracy = (logits.argmax(dim=1) == targets).float().mean()

assert not logits.requires_grad
print("eval loss:", eval_loss.item(), "acc:", accuracy.item())

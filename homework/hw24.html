🎯 逻辑回归与分类评估

🏷️ 选择 + 简答 + 代码🏷️ 35 题📖 相关讲义：逻辑回归与分类评估

35 道练习，覆盖逻辑回归、Sigmoid、概率与阈值、对数损失、正则化、混淆矩阵、Precision/Recall/F1、ROC 与 AUC。

一、选择题（12 题）

1. 逻辑回归主要用于哪类任务？

A. 分类
B. 聚类
C. 排序文件
D. 图像压缩

✅ 参考答案

答案：A。 逻辑回归名称中虽有“回归”，但它建模的是类别概率，主要用于分类。

2. Sigmoid 函数的输出范围是什么？

A. (-∞, +∞)
B. (0, 1)
C. [-1, 1]
D. 只能为 0 或 1

✅ 参考答案

答案：B。 σ(z)=1/(1+e^{-z})，可将任意实数映射到 0 和 1 之间。

3. 当线性得分 z=0 时，Sigmoid 输出是多少？

A. 0
B. 0.5
C. 1
D. 无法计算

✅ 参考答案

答案：B。 σ(0)=1/(1+1)=0.5。

4. 二分类逻辑回归默认阈值 0.5 的含义是什么？

A. 概率不小于 0.5 时预测为正类
B. 删除一半样本
C. 只训练 50% 特征
D. 准确率固定为 50%

✅ 参考答案

答案：A。 阈值可根据误报、漏报成本调整，并非必须固定为 0.5。

5. 对数几率 log(p/(1-p)) 在线性逻辑回归中等于什么？

A. 线性得分 wᵀx+b
B. MSE
C. 样本数量
D. 类别名称

✅ 参考答案

答案：A。 逻辑回归假设 log-odds 与特征是线性关系。

6. 正类样本被错误预测为负类属于什么？

A. TP
B. TN
C. FP
D. FN

✅ 参考答案

答案：D。 实际为正但预测为负是假阴性（False Negative），也叫漏报。

7. Precision 的公式是什么？

A. TP/(TP+FP)
B. TP/(TP+FN)
C. TN/(TN+FP)
D. (TP+TN)/TP

✅ 参考答案

答案：A。 Precision 关注“预测为正的样本中有多少是真的正类”。

8. Recall 的公式是什么？

A. TP/(TP+FP)
B. TP/(TP+FN)
C. FP/(FP+TN)
D. TN/(TP+TN)

✅ 参考答案

答案：B。 Recall 关注“实际正类中有多少被找出来”。

9. 降低分类阈值通常会产生什么趋势？

A. Recall 上升，Precision 可能下降
B. Recall 一定下降
C. 正类预测更少
D. ROC 曲线消失

✅ 参考答案

答案：A。 阈值降低会把更多样本判为正，漏报减少但误报可能增加。

10. F1 分数是什么？

A. Precision 与 Recall 的调和平均
B. 准确率的平方
C. TPR 与 FPR 的差
D. 预测概率均值

✅ 参考答案

答案：A。 F1=2PR/(P+R)，只有 Precision 与 Recall 都较好时才会高。

11. ROC 曲线的横轴和纵轴分别是什么？

A. FPR 与 TPR
B. Precision 与 Recall
C. MAE 与 MSE
D. K 与距离

✅ 参考答案

答案：A。 横轴是假阳性率 FPR，纵轴是真阳性率 TPR（Recall）。

12. AUC 接近 0.5 通常说明什么？

A. 排序能力接近随机
B. 模型完美
C. 召回率一定为 1
D. 所有概率都正确校准

✅ 参考答案

答案：A。 ROC-AUC 衡量正样本得分高于负样本的概率，0.5 约等于随机排序。

二、简答题（11 题）

13. 为什么逻辑回归不直接用线性回归输出做分类概率？

✅ 参考答案

线性回归输出没有 0～1 限制，且平方损失不适合伯努利标签。Sigmoid 将线性得分映射为概率，对数损失则与最大似然估计一致。

14. 解释逻辑回归系数的含义

✅ 参考答案

其他特征不变时，特征增加 1 单位会使正类对数几率增加对应系数；对系数取指数 exp(w) 可解释为 odds 的倍数变化。若特征经过标准化，解释的是增加 1 个标准差的影响。

15. 为什么对数损失会强烈惩罚“自信但错误”的预测？

✅ 参考答案

真实为正时损失是 -log(p)。若模型给正类概率接近 0，log(p) 趋向负无穷，损失会非常大，迫使模型避免过度自信的错误。

16. 欺诈检测应优先 Precision 还是 Recall？

✅ 参考答案

取决于业务成本。漏掉欺诈损失巨大时优先 Recall；人工审核成本高、误报会干扰大量正常用户时也要控制 Precision。实际应通过成本函数或约束选择阈值，而不是只追一个指标。

17. ROC-AUC 与 PR-AUC 在类别极不平衡时如何选择？

✅ 参考答案

ROC-AUC 综合排序且对类别比例不敏感，但大量真负类会让 FPR 看起来很低；PR-AUC 更聚焦少数正类的查准率和查全率，极不平衡时通常更有解释力，最好两者同时报告。

18. 为什么特征缩放有助于带正则化的逻辑回归？

✅ 参考答案

缩放让不同特征的系数处于可比尺度，使正则项公平惩罚，并改善基于梯度/二阶优化的数值条件和收敛速度。

19. 说明 class_weight="balanced" 的作用与局限

✅ 参考答案

它按类别频率为少数类赋更大损失权重，使模型更重视少数类。它不保证概率校准或业务最优，也不能替代合理划分、阈值选择和针对性指标。

20. 什么是概率校准？为什么 AUC 高仍可能需要校准？

✅ 参考答案

校准要求预测为 0.8 的样本群中约 80% 真为正。AUC 只看排序，不要求概率数值准确，所以模型可有高 AUC 但风险概率偏高/偏低，影响定价和阈值决策。

21. 多分类逻辑回归有哪些常见策略？

✅ 参考答案

常见有 One-vs-Rest：每个类别对其余类别训练一个二分类器；以及 Multinomial/Softmax：联合建模全部类别概率。选择受求解器、样本规模和解释需求影响。

22. 为什么混淆矩阵必须明确“正类”定义？

✅ 参考答案

Precision、Recall、FP、FN 都围绕正类定义。把“患病”或“未患病”设为正类会改变指标含义；报告时必须说明正类和矩阵行列方向。

23. 如何根据业务成本选择阈值？

✅ 参考答案

在独立验证集上遍历阈值，计算每个阈值的 FP、FN，再结合单位误报/漏报成本求总成本；也可先设 Recall 或 Precision 下限，再从满足约束的阈值中选成本最低者。

24. 用 numpy 实现 Sigmoid，并避免指数溢出

✅ 参考答案
Python
import numpy as np

def sigmoid(z):
    z = np.clip(np.asarray(z, dtype=float), -500, 500)
    return 1.0 / (1.0 + np.exp(-z))

print(sigmoid([-2, 0, 2]))

27. 手工从混淆矩阵计算 Precision、Recall 和 F1

✅ 参考答案
Python
from sklearn.metrics import confusion_matrix

tn, fp, fn, tp = confusion_matrix(y_test, y_pred).ravel()
precision = tp / (tp + fp) if tp + fp else 0.0
recall = tp / (tp + fn) if tp + fn else 0.0
f1 = 2 * precision * recall / (precision + recall) if precision + recall else 0.0
print(precision, recall, f1)

34. 计算二分类对数损失，不能调用 sklearn

✅ 参考答案
Python
import numpy as np

def binary_log_loss(y_true, probability):
    y_true = np.asarray(y_true)
    probability = np.clip(np.asarray(probability), 1e-15, 1 - 1e-15)
    loss = -(y_true * np.log(probability) + (1 - y_true) * np.log(1 - probability))
    return loss.mean()

35. 在乳腺癌数据集上完成一次无泄漏逻辑回归评估

✅ 参考答案
Python
from sklearn.datasets import load_breast_cancer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report, roc_auc_score
from sklearn.model_selection import train_test_split
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler

X, y = load_breast_cancer(return_X_y=True)
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.2, stratify=y, random_state=42
)
model = make_pipeline(StandardScaler(), LogisticRegression(max_iter=2000))
model.fit(X_train, y_train)
probability = model.predict_proba(X_test)[:, 1]
print(classification_report(y_test, model.predict(X_test), zero_division=0))
print("AUC:", roc_auc_score(y_test, probability))

🧬 RNN 进阶：LSTM、GRU 与双向网络综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：RNN 进阶：LSTM、GRU 与双向网络

55 道练习，按“梯度问题 → LSTM 门控 → GRU 简化 → 多层双向形状 → 变长批处理 → 序列分类闭环”组织。题目与基础 RNN 练习错开，重点训练门控公式、PyTorch 张量语义和真实批处理中的边界条件。

建议先独立作答，再展开参考答案。代码题均带断言，可直接运行验证；形状题统一使用 $B$ 表示批量大小、$T$ 表示序列长度、$I$ 表示输入维度、$H$ 表示隐藏维度、$L$ 表示层数、$D$ 表示方向数。

一、选择题（30 题）

1. 传统 RNN 难以并行处理同一序列各时间步的根本原因是？

A. 输入必须是 One-hot
B. 当前隐藏状态依赖上一时间步隐藏状态
C. RNN 没有可训练参数
D. 只能在 CPU 上运行

✅ 参考答案

答案：B。 $h_t$ 依赖 $h_{t-1}$，时间步之间存在递归数据依赖，因此同一层的序列计算通常不能像 Transformer 那样一次并行完成。

2. 传统 tanh RNN 在长序列反向传播中容易出现梯度消失或爆炸，主要因为？

A. 每个时间步使用完全不同的参数
B. 交叉熵不能处理序列
C. 梯度需要经过多个雅可比矩阵连乘
D. 隐藏状态必须是整数

✅ 参考答案

答案：C。 连乘中若相关谱尺度长期小于 1，梯度趋近于 0；长期大于 1，则可能快速放大。门控、合理初始化、归一化和梯度裁剪可缓解不同方面的问题。

3. LSTM 相比传统 RNN 新增的核心状态是？

A. 细胞状态 $c_t$
B. 类别标签 $y_t$
C. 注意力矩阵
D. 位置编码

✅ 参考答案

答案：A。 LSTM 同时传播隐藏状态 $h_t$ 和细胞状态 $c_t$。细胞状态通过加法式更新形成更直接的信息与梯度通道。

4. LSTM 遗忘门 $f_t$ 的主要作用是？

A. 决定最终类别
B. 生成输入 token
C. 将概率变成对数
D. 控制上一细胞状态 $c_{t-1}$ 保留多少

✅ 参考答案

答案：D。 $f_t$ 通常由 $[x_t;h_{t-1}]$ 经线性层和 sigmoid 得到，再逐元素乘到 $c_{t-1}$ 上。

5. LSTM 输入门 $i_t$ 的主要作用是？

A. 删除整个序列
B. 控制候选记忆有多少写入细胞状态
C. 只选择输出类别
D. 把双向结果相加

✅ 参考答案

答案：B。 候选记忆 $\tilde c_t$ 描述可写入的新内容，输入门 $i_t$ 控制写入强度，二者逐元素相乘后进入细胞更新。

6. LSTM 中候选记忆 $\tilde c_t$ 通常使用什么激活函数？

A. Softmax
B. ReLU6
C. tanh
D. Argmax

✅ 参考答案

答案：C。 门值通常使用 sigmoid 映射到 $(0,1)$，候选内容常用 tanh 映射到 $(-1,1)$；二者职责不同。

7. LSTM 输出门 $o_t$ 控制什么？

A. 更新后的细胞状态有多少暴露为隐藏状态 $h_t$
B. 上一细胞状态是否进入遗忘门
C. batch 中包含多少样本
D. 优化器的学习率

✅ 参考答案

答案：A。 常见公式为 $h_t=o_t\odot\tanh(c_t)$。$c_t$ 保存内部记忆，$h_t$ 是当前时间步对外提供的表示。

8. 下列哪一个是标准 LSTM 细胞状态更新形式？

A. $c_t=f_t+i_t+\tilde c_t$
B. $c_t=o_t\odot h_{t-1}$
C. $c_t=\operatorname{softmax}(x_t)$
D. $c_t=f_t\odot c_{t-1}+i_t\odot\tilde c_t$

✅ 参考答案

答案：D。 第一项选择性保留旧记忆，第二项选择性写入新记忆；加法式通道是 LSTM 缓解长期梯度消失的重要原因之一。

9. 已得到 $c_t$ 与输出门 $o_t$，LSTM 隐藏状态通常如何计算？

A. $h_t=c_t-o_t$
B. $h_t=o_t\odot\tanh(c_t)$
C. $h_t=\operatorname{argmax}(c_t)$
D. $h_t=i_t\odot c_{t-1}$

✅ 参考答案

答案：B。 输出门决定记忆中哪些部分对外可见，tanh 则限制细胞状态映射后的数值范围。

10. LSTM 的遗忘门、输入门和输出门通常使用 sigmoid，主要是因为？

A. sigmoid 输出和为 1
B. sigmoid 不需要参数
C. 输出在 0 到 1 之间，适合作为逐元素软门值
D. sigmoid 永远不会饱和

✅ 参考答案

答案：C。 门值接近 0 表示抑制，接近 1 表示放行，中间值表示部分通过。sigmoid 仍会饱和，所以不能说它彻底消除了梯度问题。

11. 关于 LSTM 与长期依赖，哪项表述最准确？

A. LSTM 通常缓解传统 RNN 的梯度消失，但不保证任意长序列都能无损记忆
B. LSTM 完全不会发生梯度爆炸
C. LSTM 不需要反向传播
D. 只要使用 LSTM，数据质量就不重要

✅ 参考答案

答案：A。 门控和细胞状态改善了梯度流，但门饱和、优化困难、有限容量和超长依赖仍然存在；梯度裁剪也仍有价值。

12. nn.LSTM 的最终状态返回值是什么？

A. 只有 output
B. 只有 h_n
C. 一个概率张量
D. (h_n, c_n)，分别表示最终隐藏状态与细胞状态

✅ 参考答案

答案：D。 调用形式为 output, (h_n, c_n) = lstm(x)。output 保存最后一层每个时间步的隐藏状态，h_n/c_n 保存每层每方向的最终状态。

13. nn.GRU 与 nn.LSTM 的状态接口最直接的差异是？

A. GRU 没有隐藏状态
B. GRU 只返回 h_n，没有独立的 c_n
C. GRU 必须返回注意力权重
D. GRU 不能接收初始状态

✅ 参考答案

答案：B。 GRU 把长期记忆与对外隐藏表示合并在同一个 $h_t$ 中，因此接口为 output, h_n = gru(x)。

14. GRU 的两个主要门通常是？

A. 输入门与输出门
B. 遗忘门与输出门
C. 更新门与重置门
D. 查询门与键门

✅ 参考答案

答案：C。 更新门控制新旧状态融合，重置门控制计算候选状态时使用多少旧信息；不同资料对更新门公式的符号约定可能相反，应结合完整公式理解。

15. 在输入维度与隐藏维度相同、层数和方向相同的情况下，三个循环层门组数量通常满足？

A. RNN 为 1 组、GRU 为 3 组、LSTM 为 4 组
B. RNN 为 4 组、GRU 为 3 组、LSTM 为 1 组
C. 三者完全相同
D. 只由 batch size 决定

✅ 参考答案

答案：A。 PyTorch 的普通 RNN 每层每方向有一组状态变换，GRU 有三组，LSTM 有四组，因此参数量与计算量通常依次增加。

16. 单层单向、带两组偏置的 PyTorch LSTM，input_size=I、hidden_size=H，参数量是多少？

A. $HI+H^2$
B. $4HI+H$
C. $I+H$
D. $4(HI+H^2+2H)$

✅ 参考答案

答案：D。 四组门各包含 $H\times I$ 输入权重、$H\times H$ 循环权重，以及 bias_ih、bias_hh 各 $H$ 个参数。

17. 单层单向 PyTorch GRU 的参数量公式是？

A. $2(HI+H^2+H)$
B. $3(HI+H^2+2H)$
C. $4(HI+H^2+2H)$
D. $HI+H^2$

✅ 参考答案

答案：B。 GRU 有 reset、update、new 三组参数；PyTorch 默认为输入变换和循环变换分别保留偏置，所以每组有 $2H$ 个偏置参数。

18. batch_first=True、双向 LSTM 的输入为 [B,T,I]，隐藏维度为 $H$，output 形状是？

A. [B,T,H]
B. [2,B,H]
C. [B,T,2H]
D. [B,2,T,H]

✅ 参考答案

答案：C。 最后一层正向与反向在特征维拼接，因此 output 最后一维为 $D\times H=2H$。

19. 两层双向 LSTM 的 h_n 形状是？

A. [4,B,H]
B. [2,B,2H]
C. [B,T,2H]
D. [B,4,H]

✅ 参考答案

答案：A。 状态第一维为 $L\times D=2\times2=4$。它的顺序按层和方向交错，稳妥做法是 reshape 成 [L,D,B,H] 后再索引。

20. batch_first=True 会怎样影响 h_n 和 c_n？

A. 变为 [B,L D,H]
B. 把隐藏维移到最前
C. 让状态不再包含方向维
D. 不影响，它们仍是 [L D,B,H]

✅ 参考答案

答案：D。 batch_first 只改变输入和 output 的前两维布局，不改变 RNN/LSTM/GRU 最终状态张量的布局。

21. 双向 LSTM 最适合下列哪类场景？

A. 只能访问过去 token 的实时下一个词生成
B. 输入完整可见的离线文本分类或序列标注
C. 严格流式语音输出且不能等待未来帧
D. 因果语言模型解码

✅ 参考答案

答案：B。 离线任务能同时利用左右上下文；严格在线或自回归任务在当前时刻不能读取未来信息，反向网络会破坏因果约束或增加等待延迟。

22. 双向循环网络不适合作为严格自回归语言模型主干，根本原因是？

A. 参数太少
B. 不能处理文本
C. 反向方向在训练表示中使用了当前预测位置之后的未来 token
D. 没有激活函数

✅ 参考答案

答案：C。 自回归概率要求 $p(x_t\mid x_{t}$，就发生未来信息泄漏。

23. 对双向 LSTM 直接把 output[:, -1, :] 当作“正反两方向的最终状态拼接”，问题是什么？

A. 末位包含正向最终状态，但反向部分对应末位输入处的反向输出，不是反向遍历完整序列后的最终状态
B. 它会返回四维张量
C. 正向部分一定全为零
D. 双向网络没有 output

✅ 参考答案

答案：A。 完整序列的反向最终状态位于反向遍历结束处，对应 output[:, 0, H:]，或更直接地从最后一层 h_n 中取正、反两个方向再拼接。

24. 为稳定取得最后一层双向 LSTM 的两个最终隐藏状态，最清晰的做法是？

A. 对 output 全部求最大值
B. 只取 h_n[0]
C. 把 batch 维删除
D. 将 h_n reshape 为 [L,D,B,H]，取 h_n[-1,0] 与 h_n[-1,1] 拼接

✅ 参考答案

答案：D。 显式恢复层与方向两个轴后，代码含义清楚，也不依赖层数恰好为 1。

25. 一批右侧 padding 的变长序列直接使用 output[:, -1, :] 分类，主要风险是？

A. 一定产生 NaN
B. 较短样本取到的是 PAD 时间步表示，而不是最后一个真实 token
C. 隐藏维自动减半
D. 标签会自动改变

✅ 参考答案

答案：B。 应按 lengths-1 gather 最后有效位置，或使用 packed sequence 后读取正确的 h_n；仅设置 padding_idx 不能自动修正索引语义。

二、简答题（15 题）

31. 从 BPTT 的角度解释传统 RNN 为什么会梯度消失或爆炸，并分别给出缓解方法。

✅ 参考答案

跨越多个时间步的梯度包含循环雅可比矩阵的连乘。若其有效尺度长期小于 1，早期信息的梯度指数衰减；长期大于 1，则梯度迅速放大。LSTM/GRU 的门控与加法式状态通道主要缓解长期梯度消失；正交初始化、合适激活、归一化、截断 BPTT 也可能有帮助。梯度裁剪主要限制爆炸，不能把已经消失的梯度“恢复”。

32. 按数据流写出一个 LSTM 时间步的完整计算过程。

✅ 参考答案

先由当前输入 $x_t$ 和上一隐藏状态 $h_{t-1}$ 计算遗忘门、输入门、候选记忆和输出门：

$$ f_t=\sigma(W_f[x_t;h_{t-1}]+b_f),\quad i_t=\sigma(W_i[x_t;h_{t-1}]+b_i), $$

$$ \tilde c_t=\tanh(W_c[x_t;h_{t-1}]+b_c),\quad o_t=\sigma(W_o[x_t;h_{t-1}]+b_o). $$

再更新 $c_t=f_t\odot c_{t-1}+i_t\odot\tilde c_t$，最后得到 $h_t=o_t\odot\tanh(c_t)$。实现时四组仿射变换通常合并为一次大矩阵计算。

33. 为什么 LSTM 的细胞状态通道比普通 tanh RNN 更利于长期梯度传播？

✅ 参考答案

普通 RNN 每一步都让状态经过仿射变换和 tanh，长期梯度要反复乘权重与激活导数。LSTM 的 $c_t=f_t\odot c_{t-1}+i_t\odot\tilde c_t$ 含加法式通道，沿旧记忆路径的局部导数直接包含 $f_t$。当遗忘门接近 1 时，信息和梯度可较少衰减地跨越多个时间步。但门值仍可能饱和或关闭，所以只是缓解，不是无条件保证。

34. 为什么有人会把 LSTM 遗忘门偏置初始化为正值？有什么边界？

✅ 参考答案

正偏置经 sigmoid 后会让初始遗忘门更接近“保留”，训练早期不容易立即擦除记忆，有时有助于长依赖优化。但这是一种初始化先验，不保证适合所有任务；过强保留也可能妨碍快速遗忘无关信息。应结合框架默认设置、验证实验和状态诊断决定，不能当作固定规则。

35. 比较 GRU 与 LSTM 的状态、门控、参数量及适用场景。

✅ 参考答案

LSTM 有独立的 $h_t$、$c_t$，通常包含遗忘、输入、输出三门和候选记忆；GRU 只有 $h_t$，用更新门、重置门和候选状态完成新旧信息融合。相同 $I,H,L,D$ 下，GRU 通常约为三组参数、LSTM 为四组，因此 GRU 更轻更快。LSTM 的状态分工更明确，但不代表一定更准；应在相同预算下比较效果、速度和稳定性。

36. 计算 I=64、H=128 的单层单向 PyTorch RNN、GRU、LSTM 参数量。

✅ 参考答案

PyTorch 默认每个门都有输入权重、循环权重，以及两组偏置：

  • RNN：$HI+H^2+2H=128\times64+128^2+256=24{,}832$；
  • GRU：$3(HI+H^2+2H)=74{,}496$；
  • LSTM：$4(HI+H^2+2H)=99{,}328$。

若关闭 bias 或加入更多层、方向，公式需要相应调整；高层的 input_size 也会变为前一层的 $D H$，不能简单把单层结果乘层数。

37. 两层双向 LSTM 输入 [32,20,100]，H=64、batch_first=True，输出状态形状分别是什么？

✅ 参考答案

方向数 $D=2$、层数 $L=2$：output 为 [32,20,128]，因为只输出最后一层且拼接两个方向；h_n 与 c_n 都是 [4,32,64]，第一维为 $L\times D$。batch_first=True 不改变最终状态的布局。

38. 解释双向网络中 output 与 h_n 的对应关系，为什么不能机械取 output[:, -1, :]？

✅ 参考答案

最后一层 output[t] 拼接“正向网络处理到位置 $t$ 的状态”和“反向网络从序列末端处理到位置 $t$ 的状态”。因此 output[:, -1, :H] 是正向最终状态，但 output[:, -1, H:] 是反向网络刚开始处理末位时的输出。反向最终状态对应 output[:,0,H:]。序列级分类更稳妥的写法是把 h_n 变为 [L,D,B,H]，拼接最后一层两个方向。

39. 双向网络为何适合 NER，却不适合严格流式生成？

✅ 参考答案

NER 在离线条件下可看到完整句子，一个实体标签往往同时依赖左右上下文，双向编码能充分利用这些信息。严格流式或自回归生成在时刻 $t$ 只能访问过去；反向网络需要未来 token，会造成因果泄漏，或必须等待后续输入从而增加延迟。是否可用双向结构取决于推理时未来信息是否真实可用。

40. 设计一个变长文本批次进入 LSTM 的无泄漏、无 padding 干扰流程。

✅ 参考答案

先在训练集建立词表，把文本转为 ID 并记录真实长度；collate_fn 在当前 batch 内动态右填充，长度保持 CPU 整数。Embedding 使用 padding_idx，随后用 pack_padded_sequence(..., enforce_sorted=False) 压缩有效时间步，送入 LSTM。分类可直接使用正确的 h_n；若要逐 token 输出，则用 pad_packed_sequence 还原并结合 mask 计算损失。验证和测试复用冻结词表与同一管线。

41. pack_padded_sequence 为什么通常不接受长度为 0 的样本？工程上怎样处理空文本？

✅ 参考答案

循环网络至少需要一个有效时间步才能定义序列状态，packed sequence 的每条长度也必须大于 0。工程上应在数据质检阶段记录并处理空文本：可删除无效样本、回查数据源，或明确映射成一个专用 <empty> token，使长度变为 1。不能悄悄把空文本标成任意类别，否则会制造不可解释的训练信号。

42. 除最终隐藏状态外，序列分类还可以怎样从 output 得到句子表示？

✅ 参考答案

可以对有效时间步做 masked mean/max pooling，或学习 attention pooling；长文也可分段聚合。均值池化汇总全句、实现稳定；最大池化突出最强局部特征；attention pooling 能学习不同位置权重，但复杂度更高。任何 pooling 都必须排除 PAD，并在相同数据划分下与 h_n 基线比较。

43. 人名分类使用字符 One-hot 与字符 Embedding 各有什么特点？

✅ 参考答案

字符 One-hot 不需学习输入映射，适合字符表很小的教学基线，但输入维度等于字符表大小，任意字符彼此正交。Embedding 把字符 ID 映射为低维稠密向量，参数更紧凑，可学习共享模式，也便于加入 <unk>/<pad>。两者都需处理大小写、重音符号、连字符和未知字符；字符表必须只由训练数据或预先固定规范确定。

44. 一个循环分类模型的训练循环至少应正确处理哪些状态？

✅ 参考答案

每个 batch 先 zero_grad，前向得到原始 logits，用 CrossEntropyLoss 与整数标签计算损失，反向后可做梯度裁剪，再 optimizer.step。训练阶段启用 model.train()，验证阶段使用 model.eval() 与 torch.no_grad()；隐藏状态若跨 batch 复用必须 detach，普通独立样本分类则通常每个 batch 重新初始化。还应记录损失、分类指标、随机种子和最佳验证权重。

45. 如何公平比较 RNN、GRU、LSTM 与双向版本？

✅ 参考答案

固定训练/验证/测试分组、词表、输入长度、优化器搜索空间、训练预算和指标；同时报告参数量、吞吐、显存、延迟与多随机种子结果。若要求参数预算相近，可适当调整不同模型的 $H$，否则 LSTM 天然参数更多。除整体指标外，还应按序列长度、低频字符和类别做切片，并检查双向结构是否符合线上因果与延迟约束。

47. 根据 nn.LSTMCell 的参数手动实现一个时间步，并与框架输出对齐。

✅ 参考答案

PyTorch LSTM 的门排列为 input、forget、cell candidate、output，即 i, f, g, o。

Python
import torch
from torch import nn

torch.manual_seed(1)
B, I, H = 2, 3, 4
cell = nn.LSTMCell(I, H)
x = torch.randn(B, I)
h_prev = torch.randn(B, H)
c_prev = torch.randn(B, H)

h_ref, c_ref = cell(x, (h_prev, c_prev))
gates = (
    x @ cell.weight_ih.T
    + cell.bias_ih
    + h_prev @ cell.weight_hh.T
    + cell.bias_hh
)
i_gate, f_gate, candidate, o_gate = gates.chunk(4, dim=1)
i_gate = torch.sigmoid(i_gate)
f_gate = torch.sigmoid(f_gate)
candidate = torch.tanh(candidate)
o_gate = torch.sigmoid(o_gate)
c_manual = f_gate * c_prev + i_gate * candidate
h_manual = o_gate * torch.tanh(c_manual)

assert torch.allclose(c_manual, c_ref, atol=1e-6)
assert torch.allclose(h_manual, h_ref, atol=1e-6)
assert ((i_gate > 0) & (i_gate < 1)).all()
print("manual LSTMCell matches PyTorch")

52. 实现一个支持变长输入的双向 LSTM 文本分类器。

✅ 参考答案

分类表示直接来自最后一层两个方向的 h_n，不会误取 PAD 时间步。

Python
import torch
from torch import nn
from torch.nn.utils.rnn import pack_padded_sequence

class BiLSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim, hidden_size, num_classes):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.encoder = nn.LSTM(
            embed_dim, hidden_size, batch_first=True, bidirectional=True
        )
        self.classifier = nn.Linear(2 * hidden_size, num_classes)

    def forward(self, token_ids, lengths):
        embedded = self.embedding(token_ids)
        packed = pack_padded_sequence(
            embedded, lengths.cpu(), batch_first=True, enforce_sorted=False
        )
        _, (h_n, _) = self.encoder(packed)
        h_n = h_n.reshape(1, 2, token_ids.size(0), self.encoder.hidden_size)
        sentence_vector = torch.cat([h_n[-1, 0], h_n[-1, 1]], dim=-1)
        return self.classifier(sentence_vector)

model = BiLSTMClassifier(vocab_size=20, embed_dim=8, hidden_size=6, num_classes=3)
token_ids = torch.tensor([[2, 3, 4, 0], [5, 6, 0, 0], [7, 8, 9, 10]])
lengths = torch.tensor([3, 2, 4])
logits = model(token_ids, lengths)

assert logits.shape == (3, 3)
assert torch.isfinite(logits).all()
print(logits.shape)

53. 对一个 LSTM 分类模型执行反向传播，并验证梯度裁剪后的全局范数不超过阈值。

✅ 参考答案

clip_grad_norm_ 的返回值是裁剪前范数；下面另行计算裁剪后范数。

Python
import torch
from torch import nn

torch.manual_seed(5)
encoder = nn.LSTM(input_size=5, hidden_size=8, batch_first=True)
head = nn.Linear(8, 3)

loss.backward()

post_clip = torch.sqrt(sum(
    parameter.grad.detach().pow(2).sum()
    for parameter in parameters
    if parameter.grad is not None
))

assert pre_clip > 1.0
assert post_clip <= 1.00001
assert torch.isfinite(post_clip)

54. 将分类 logits 转成概率并输出每个样本的 Top-2 标签。

import torch

55. 在一个微型变长数据集上完成 Embedding + GRU 分类训练闭环。

✅ 参考答案

数据的类别由首 token 决定，用于验证数据管线、packed sequence、损失和优化器是否能共同工作，而不是代表真实泛化性能。

Python
import torch
from torch import nn
from torch.nn.utils.rnn import pack_padded_sequence

torch.manual_seed(6)

class TinyGRUClassifier(nn.Module):
    def __init__(self):
        super().__init__()
        self.embedding = nn.Embedding(12, 5, padding_idx=0)
        self.gru = nn.GRU(5, 7, batch_first=True)
        self.head = nn.Linear(7, 2)

    def forward(self, token_ids, lengths):
        embedded = self.embedding(token_ids)
        packed = pack_padded_sequence(
            embedded, lengths.cpu(), batch_first=True, enforce_sorted=False
        )
        _, h_n = self.gru(packed)
        return self.head(h_n[-1])

token_ids = torch.tensor([
    [1, 3, 4, 0], [1, 5, 0, 0], [1, 6, 7, 8], [1, 9, 3, 0],
    [2, 3, 4, 0], [2, 5, 0, 0], [2, 6, 7, 8], [2, 9, 3, 0],
])
lengths = torch.tensor([3, 2, 4, 3, 3, 2, 4, 3])
labels = torch.tensor([0, 0, 0, 0, 1, 1, 1, 1])

model = TinyGRUClassifier()
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)
criterion = nn.CrossEntropyLoss()

model.train()
for _ in range(80):
    optimizer.zero_grad()
    logits = model(token_ids, lengths)
    loss = criterion(logits, labels)
    loss.backward()
    nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()

model.eval()
with torch.no_grad():
    predictions = model(token_ids, lengths).argmax(dim=1)
    accuracy = (predictions == labels).float().mean().item()

assert accuracy == 1.0
assert torch.isfinite(loss)
print({"loss": round(loss.item(), 4), "accuracy": accuracy})

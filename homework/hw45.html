🌌 大语言模型基础：发展、训练、使用与评估综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：大语言模型基础：发展、训练、使用与评估

55 道练习，按“语言模型概率 → N-gram 与神经语言模型 → Transformer/LLM → 预训练、SFT 与对齐 → 权重、精度与量化 → API/本地部署 → 生成控制 → BLEU、ROUGE、PPL 与 Benchmark”组织。重点是把模型能力、资源成本和评估结论放在同一条可验证的工程链路中。

建议先独立作答，再展开参考答案。练习按 BLEU 原论文、ROUGE 原论文、Hugging Face Perplexity、Quantization、KV Cache、Safetensors 与 lm-evaluation-harness资料校准；10 道代码题使用纯 Python/PyTorch，不调用外部模型或 API。

一、选择题（30 题）

1. 自回归语言模型对序列 $x_1,\ldots,x_T$ 的联合概率如何按链式法则分解？

A. $P(x_1,\ldots,x_T)=\prod_{t=1}^{T}P(x_t\mid x_{<t})$
B. $P(x_1,\ldots,x_T)=\sum_t P(x_t)$
C. 所有 token 的概率必须相同
D. 只计算最后一个 token

✅ 参考答案

答案：A。 链式法则本身不做近似；语言模型负责估计每个条件概率。训练时通常最小化正确 next token 的负对数似然之和或均值。

2. 为什么实际计算序列概率常使用 log 概率相加，而不是直接连乘？

A. Log 会让概率大于 1
B. 很多小概率连乘容易数值下溢，相加更稳定
C. 连乘不符合链式法则
D. Log 可以删除所有 token

✅ 参考答案

答案：B。 $\log\prod_t p_t=\sum_t\log p_t$；训练中的交叉熵、负对数似然和 PPL 也自然使用这一表示。

3. N-gram 语言模型做了什么核心近似？

A. 当前 token 只依赖有限长度的最近历史
B. 当前 token 依赖未来所有 token
C. 不需要语料统计
D. 每个词只依赖自身标签

✅ 参考答案

答案：A。 例如三元模型用 $P(x_t\mid x_{t-2},x_{t-1})$ 近似完整历史条件概率，以可统计性换取长距离依赖能力。

4. Add-$\alpha$ 平滑的主要目的是什么？

A. 让未见过的 N-gram 不再得到严格的零概率
B. 将所有条件概率设为 1
C. 自动生成词向量
D. 删除低频词

✅ 参考答案

答案：A。 平滑会重新分配概率质量；简单加一平滑便于教学，但实际系统还会使用回退、插值、Kneser–Ney 等更有效的方法。

5. 神经网络语言模型相对纯 N-gram 的关键改进是？

A. 用连续词向量和共享参数在相似上下文之间泛化
B. 完全不需要训练数据
C. 只保存规则表
D. 保证任何事实都正确

✅ 参考答案

答案：A。 连续表示缓解了稀疏性，但早期固定窗口 NNLM 仍受上下文长度和计算方式限制，后来 RNN 与 Transformer 继续扩展建模能力。

6. Transformer 推动大规模预训练的重要原因之一是？

A. 训练时 token 维度更易并行，注意力能直接建立远距离交互
B. 不再需要矩阵乘法
C. 参数量恒为零
D. 不需要 Position 信息

✅ 参考答案

答案：A。 Transformer 仍有计算和上下文成本，标准全注意力随序列长度近似二次增长；“可并行”主要指训练，而自回归解码仍逐步进行。

7. Base、SFT/Instruct 与 Chat/Aligned 模型的区别，哪项正确？

A. Base 主要完成预训练目标；SFT 学习指令数据；Aligned 进一步优化偏好与安全行为
B. 三者只有文件名不同
C. Base 必然最会遵循对话指令
D. 对齐后不再可能犯错

✅ 参考答案

答案：A。 各项目命名可能不同，选模型时要看模型卡、训练说明和聊天模板；对齐改变行为分布，不等于注入完备事实或消除幻觉。

8. SFT 阶段最典型的训练信号是？

A. 在人工/合成的指令—回答样本上做监督式 token 预测
B. 只测 GPU 温度
C. 不使用任何目标函数
D. 随机删除权重文件

✅ 参考答案

答案：A。 工程上常只对 assistant 回答区域计损失，并严格使用匹配的 chat template；数据质量、覆盖面和格式一致性会直接影响指令遵循。

9. RLHF、DPO 等“对齐”方法最直接优化的是什么？

A. 人类或偏好数据所表达的响应偏好
B. 保证世界知识永远最新
C. 自动增加上下文窗口
D. 消除全部推理成本

✅ 参考答案

答案：A。 偏好优化可能改善帮助性、安全性和风格，但仍需要事实性、鲁棒性和越狱等独立评估，且偏好数据本身可能有偏差。

10. In-context Learning 与参数微调的核心区别是？

A. 前者通过提示中的说明/示例改变当前推理行为，不更新模型权重
B. 前者必须反向传播
C. 后者不改变任何参数
D. 二者都等于训练 Base 模型

✅ 参考答案

答案：A。 Zero-shot、Few-shot 都属于推理时条件化；它们会消耗上下文 token，效果也依赖提示格式、示例选择和模型能力。

11. 下列“任务—常见架构起点”匹配正确的是？

A. 文本分类—Encoder-only；开放式生成—Decoder-only；翻译—Encoder–Decoder
B. 三者都必须使用 N-gram
C. 文本分类只能使用 Decoder–Encoder
D. 翻译必须使用 K-Means

✅ 参考答案

答案：A。 现代指令模型也可统一处理多类任务，但专用 Encoder 往往在判别任务的时延和成本上有优势，选型仍应实测。

12. Embedding 模型与 Reranker 的典型差异是？

A. Embedding 常分别编码以便向量召回；Reranker 联合读取 query-document 做精排
B. Reranker 不读取文本
C. Embedding 只能生成图片
D. 二者的分数可无条件互换

✅ 参考答案

答案：A。 双塔适合大规模近似最近邻召回，交叉编码 Reranker 更精确但每个候选都要联合计算，通常组成“召回→精排”级联。

13. 远程 API 与本地部署的合理比较是？

A. API 通常启动快、运维少；本地部署控制力强，但需承担硬件、升级和安全运维
B. API 一定零成本且无隐私风险
C. 本地部署一定性能最好
D. 两者不需要监控

✅ 参考答案

答案：A。 决策还应考虑数据驻留、并发、延迟、模型能力、合规、供应商锁定、灾备和全生命周期成本，而不是只看单次调用价格。

14. 管理 LLM API Key 的正确做法是？

A. 写入公开仓库方便共享
B. 使用密钥管理或环境注入，最小权限、定期轮换并避免日志泄露
C. 放在前端 JavaScript 中
D. 复制到提示词里

✅ 参考答案

答案：B。 .env 只是本地便利方式，不是企业级密钥系统；还要将其加入忽略规则，并在疑似泄露时立即撤销和轮换。

15. 下载开源模型前最完整的供应链检查是？

A. 只看模型名称
B. 核对来源、revision/commit、许可、模型卡、文件哈希、依赖与是否需要执行远程代码
C. 关闭所有校验
D. 默认信任任何镜像

✅ 参考答案

答案：B。 本地部署不会自动消除供应链风险；还要进行漏洞扫描、隔离加载、权限控制和模型/数据来源记录。

16. 关于 .safetensors，哪项准确？

A. 它是面向张量的安全序列化格式，可避免 pickle 式任意代码反序列化风险
B. 它保证模型无偏见且答案正确
C. 它等于 4-bit 量化
D. 它只能存文本

✅ 参考答案

答案：A。 Safetensors 解决的是张量存储和加载边界；模型代码、tokenizer、自定义依赖、许可和权重来源仍需单独审计。

17. GGUF 与量化之间的关系，哪项更准确？

A. GGUF 是常用于本地推理的容器格式；量化是降低数值精度的技术，两者不是同义词
B. 任何 GGUF 都必然是 4-bit
C. 量化只能存为 GGUF
D. GGUF 是 Benchmark 指标

✅ 参考答案

答案：A。 GGUF 可以承载不同量化类型甚至非量化权重；是否能在目标硬件高效运行，还取决于推理引擎、算子和具体量化方案。

18. 大模型权重分片文件旁的 index/映射文件有什么作用？

A. 记录每个参数张量位于哪个 shard，便于按需加载
B. 自动提高模型准确率
C. 代替 tokenizer
D. 存储用户聊天历史

✅ 参考答案

答案：A。 分片解决单文件过大与并行加载问题；必须保证所有 shard 与索引来自同一 revision，缺片或混版会导致加载失败或更隐蔽的不一致。

19. FP16 与 BF16 都占 2 Bytes，主要数值差异是？

A. BF16 指数范围接近 FP32、尾数精度更低；FP16 尾数更多但范围更窄
B. 二者完全相同
C. FP16 是整数
D. BF16 一定比 FP32 占更多显存

✅ 参考答案

答案：A。 BF16 通常更抗溢出，但是否更快取决于硬件支持；训练还常保留高精度累积、优化器状态或主权重。

20. PTQ 与 QAT 的区别是？

A. PTQ 在训练后量化；QAT 在训练中模拟量化误差以适应低比特
B. PTQ 必须从零训练
C. QAT 不包含量化
D. 两者都保证无精度损失

✅ 参考答案

答案：A。 PTQ 成本低，常需校准；QAT 成本更高但低比特下可能保持更好质量。选择必须与硬件和后端支持匹配。

21. $P$ 个参数、每参数 $b$ bit 时，裸权重的十进制 GB 近似为？

A. $P\times b/8/10^9$
B. $P+b$
C. $P/0$
D. 与参数量无关

✅ 参考答案

答案：A。 这只是权重下界，不含量化 scale/zero-point、KV Cache、激活、临时 workspace、框架开销和显存碎片。

22. 自回归推理中最重要的可变显存项之一是？

A. KV Cache，会随 batch、上下文/生成长度、层数、KV 头数和精度增长
B. 文件名长度
C. BLEU 分数
D. API Key 字符数

✅ 参考答案

答案：A。 KV Cache 复用历史 K/V 以加速逐 token 解码；GQA/MQA、缓存量化、滑动窗口和 offload 等可降低或转移其内存成本。

23. 模型上下文窗口为 8192 token，输入已占 7000 token 时，max_new_tokens=4096 的潜在问题是？

A. 输入加输出预算可能超过上下文上限，需要截断或缩短生成
B. 生成一定正好 4096 token
C. 上下文长度与输出无关
D. 会自动增加模型参数

✅ 参考答案

答案：A。 系统提示、聊天模板、工具结果也占 token；服务端应在请求前进行 token 预算并明确截断策略。

24. Temperature 的主要作用是？

A. 缩放 logits，控制采样分布的尖锐或平坦程度
B. 改变模型参数量
C. 增加 GPU 物理温度
D. 替代停止条件

✅ 参考答案

答案：A。 较低 temperature 通常更集中，较高更随机；temperature=0 常按实现转为贪心，不能直接做除法。

25. Top-k 与 Top-p 的区别是？

A. Top-k 固定保留 k 个高概率 token；Top-p 保留累计概率达到阈值的动态集合
B. 两者都修改训练集
C. Top-p 固定保留 p 个 token
D. 二者只用于分类训练

✅ 参考答案

答案：A。 它们可与 temperature 组合，但参数间会相互作用；评估生成策略时应固定随机种子并报告完整配置。

26. BLEU 的核心组成是？

46. 实现带 Add-$\alpha$ 平滑的二元语言模型，并验证条件概率归一化。

✅ 参考答案

上下文未出现时，Add-$\alpha$ 会返回词表上的均匀分布；实际项目还需明确 BOS/EOS 与 UNK。

Python
from collections import Counter

class AddAlphaBigramLM:
    def __init__(self, sentences, alpha=1.0):
        self.alpha = alpha
        self.vocab = sorted({token for sent in sentences for token in sent})
        self.bigram = Counter()
        self.context = Counter()
        for sentence in sentences:
            for left, right in zip(sentence, sentence[1:]):
                self.bigram[(left, right)] += 1
                self.context[left] += 1

    def probability(self, token, previous):
        if token not in self.vocab:
            raise ValueError("token must belong to the fixed vocabulary")
        numerator = self.bigram[(previous, token)] + self.alpha
        denominator = self.context[previous] + self.alpha * len(self.vocab)
        return numerator / denominator

corpus = [["我", "爱", "学习"], ["我", "爱", "编程"], ["你", "爱", "学习"]]
lm = AddAlphaBigramLM(corpus, alpha=1.0)
assert lm.probability("学习", "爱") == 3 / 8
assert lm.probability("我", "爱") == 1 / 8
for previous in ["爱", "未见上下文"]:
    total = sum(lm.probability(token, previous) for token in lm.vocab)
    assert abs(total - 1.0) < 1e-12
print({token: round(lm.probability(token, "爱"), 3) for token in lm.vocab})

47. 用 log 概率计算序列负对数似然和 PPL，避免连乘下溢。

✅ 参考答案

import math

def perplexity(correct_token_probabilities):
    if not correct_token_probabilities:
        raise ValueError("at least one scored token is required")
    if any(p <= 0.0 or p > 1.0 for p in correct_token_probabilities):
        raise ValueError("probabilities must be in (0, 1]")
    mean_nll = -sum(math.log(p) for p in correct_token_probabilities) / len(
        correct_token_probabilities
    )
    return math.exp(mean_nll)

tiny = [1e-5] * 1000

assert math.prod(tiny) == 0.0  # 直接连乘已经下溢
print({"ppl": ppl, "tiny_ppl": round(perplexity(tiny))})

48. 估算权重与 KV Cache 显存，并解释 GQA 的 num_kv_heads 影响。

def estimate_inference_gb(
    parameters, weight_bits, layers, batch_size, sequence_length,
    num_kv_heads, head_dim, cache_bytes=2,
    activations_gb=0.0, runtime_overhead_gb=0.0,

    weights = parameters * weight_bits / 8 / 1e9

        2 * layers * batch_size * sequence_length
        * num_kv_heads * head_dim * cache_bytes / 1e9
    )
    total = weights + kv_cache + activations_gb + runtime_overhead_gb
    return {"weights_gb": weights, "kv_cache_gb": kv_cache, "total_gb": total}

49. 实现对称 INT8 Per-Tensor 量化与反量化，并计算误差。

✅ 参考答案

import torch

def symmetric_int8_quantize(tensor):
    max_abs = tensor.abs().max()
    if max_abs == 0:
        scale = torch.tensor(1.0, device=tensor.device)
    else:
        scale = max_abs / 127
    quantized = torch.round(tensor / scale).clamp(-127, 127).to(torch.int8)
    return quantized, scale

weights = torch.tensor([[-1.2, -0.1, 0.0], [0.3, 0.8, 1.0]])
quantized, scale = symmetric_int8_quantize(weights)
restored = quantized.float() * scale
mse = torch.mean((weights - restored) ** 2).item()
assert quantized.dtype == torch.int8
assert quantized.min() >= -127 and quantized.max() <= 127
assert mse < 1e-4
print({"scale": round(scale.item(), 6), "mse": round(mse, 8)})

50. 实现 temperature + top-k 单步采样，并保证随机种子可复现。

✅ 参考答案

temperature=0 单独走 argmax，避免除零；top_k=1 也必然退化为贪心。

import torch

def sample_top_k(logits, temperature=1.0, top_k=None, seed=0):
    if temperature < 0:
        raise ValueError("temperature must be non-negative")
    if temperature == 0:
        return int(logits.argmax().item())
    scaled = logits / temperature
    if top_k is not None:
        if not 1 <= top_k <= logits.numel():
            raise ValueError("invalid top_k")
        values, indices = torch.topk(scaled, top_k)
        probabilities = torch.softmax(values, dim=-1)
        generator = torch.Generator().manual_seed(seed)
        sampled_local = torch.multinomial(probabilities, 1, generator=generator)
        return int(indices[sampled_local].item())
    generator = torch.Generator().manual_seed(seed)
    probabilities = torch.softmax(scaled, dim=-1)
    return int(torch.multinomial(probabilities, 1, generator=generator).item())

logits = torch.tensor([0.1, 2.0, 1.0, -1.0])
assert sample_top_k(logits, temperature=0) == 1
assert sample_top_k(logits, temperature=1.0, top_k=1, seed=9) == 1
first = sample_top_k(logits, temperature=0.8, top_k=3, seed=42)
second = sample_top_k(logits, temperature=0.8, top_k=3, seed=42)
assert first == second and first in {0, 1, 2}
print(first)

51. 从零实现单参考 BLEU-2：裁剪 N-gram 精确率、几何平均和 BP。

✅ 参考答案

若某阶精确率为 0，本简化实现直接返回 0；真实系统应使用语料级 BLEU、标准 tokenizer 与明确的平滑配置。

Python
import math
from collections import Counter

def ngrams(tokens, n):
    return [tuple(tokens[i:i+n]) for i in range(len(tokens) - n + 1)]

def clipped_precision(candidate, reference, n):
    candidate_counts = Counter(ngrams(candidate, n))
    reference_counts = Counter(ngrams(reference, n))
    matched = sum(
        min(count, reference_counts[gram])
        for gram, count in candidate_counts.items()
    )
    total = sum(candidate_counts.values())
    return matched / total if total else 0.0

def bleu2(candidate, reference):
    p1 = clipped_precision(candidate, reference, 1)
    p2 = clipped_precision(candidate, reference, 2)
    if p1 == 0 or p2 == 0:
        return 0.0
    candidate_length, reference_length = len(candidate), len(reference)
    bp = 1.0 if candidate_length >= reference_length else math.exp(
        1 - reference_length / candidate_length
    )
    return bp * math.exp(0.5 * math.log(p1) + 0.5 * math.log(p2))

candidate = "it is a nice day today".split()
reference = "today is a nice day".split()
score = bleu2(candidate, reference)
assert abs(clipped_precision(candidate, reference, 1) - 5/6) < 1e-12
assert abs(clipped_precision(candidate, reference, 2) - 3/5) < 1e-12
assert abs(score - math.sqrt(0.5)) < 1e-12
print(round(score, 6))

52. 实现带重复计数裁剪的 ROUGE-1 Precision、Recall 与 F1。

✅ 参考答案

from collections import Counter

def rouge1(candidate, reference):
    candidate_counts = Counter(candidate)
    reference_counts = Counter(reference)
    overlap = sum(
        min(count, reference_counts[token])
        for token, count in candidate_counts.items()
    )
    precision = overlap / len(candidate) if candidate else 0.0
    recall = overlap / len(reference) if reference else 0.0
    f1 = (
        2 * precision * recall / (precision + recall)
        if precision + recall else 0.0
    )
    return precision, recall, f1

assert recall == 1.0
assert abs(f1 - 10/11) < 1e-12

53. 用动态规划实现 ROUGE-L 所需的最长公共子序列及 F1。

✅ 参考答案

LCS 保留顺序但不要求连续；这里用对称 F1，具体评测工具可能使用不同 $\beta$ 或聚合约定。

Python
def lcs_length(left, right):
    table = [[0] * (len(right) + 1) for _ in range(len(left) + 1)]
    for i, left_token in enumerate(left, start=1):
        for j, right_token in enumerate(right, start=1):
            if left_token == right_token:
                table[i][j] = table[i - 1][j - 1] + 1
            else:
                table[i][j] = max(table[i - 1][j], table[i][j - 1])
    return table[-1][-1]

def rouge_l(candidate, reference):
    lcs = lcs_length(candidate, reference)
    precision = lcs / len(candidate) if candidate else 0.0
    recall = lcs / len(reference) if reference else 0.0
    f1 = 2 * precision * recall / (precision + recall) if lcs else 0.0
    return precision, recall, f1

candidate = "it is a nice day today".split()
reference = "today is a nice day".split()
precision, recall, f1 = rouge_l(candidate, reference)
assert lcs_length(candidate, reference) == 4
assert abs(precision - 4/6) < 1e-12
assert abs(recall - 4/5) < 1e-12
assert abs(f1 - 8/11) < 1e-12
print({"precision": precision, "recall": recall, "f1": f1})

54. 为小型 Benchmark 准确率计算 Bootstrap 95% 置信区间。

✅ 参考答案

同一模型不同样本的重采样可反映有限测试集的不确定性；模型对比还可使用配对 Bootstrap。

Python
import random

def bootstrap_accuracy_interval(correct, samples=2000, seed=0):
    if not correct:
        raise ValueError("empty benchmark")
    random_generator = random.Random(seed)
    n = len(correct)
    scores = []
    for _ in range(samples):
        draw = [correct[random_generator.randrange(n)] for _ in range(n)]
        scores.append(sum(draw) / n)
    scores.sort()
    lower = scores[int(0.025 * samples)]
    upper = scores[int(0.975 * samples)]
    return sum(correct) / n, lower, upper

results = [1, 1, 0, 1, 0, 1, 1, 0]
accuracy, lower, upper = bootstrap_accuracy_interval(results, seed=7)
assert accuracy == 0.625
assert lower <= accuracy <= upper
assert 0.0 <= lower <= upper <= 1.0
print({"accuracy": accuracy, "95% CI": (lower, upper)})

from collections import defaultdict
from statistics import median

def normalize_answer(text):
    return " ".join(text.strip().casefold().split())

def evaluate_records(records):
    if not records:
        raise ValueError("records cannot be empty")
    per_task = defaultdict(list)
    latencies = []
    prompt_tokens = completion_tokens = 0
    for record in records:
        correct = normalize_answer(record["prediction"]) == normalize_answer(
            record["target"]
        )
        per_task[record["task"]].append(int(correct))
        latencies.append(record["latency_ms"])
        prompt_tokens += record["prompt_tokens"]
        completion_tokens += record["completion_tokens"]
    return {
        "accuracy": sum(map(sum, per_task.values())) / len(records),
        "per_task_accuracy": {
            task: sum(values) / len(values) for task, values in per_task.items()
        },
        "median_latency_ms": median(latencies),
        "prompt_tokens": prompt_tokens,
        "completion_tokens": completion_tokens,
    }

records = [
    {"task": "qa", "prediction": "北京", "target": "北京", "latency_ms": 100, "prompt_tokens": 30, "completion_tokens": 2},
    {"task": "qa", "prediction": "上海", "target": "北京", "latency_ms": 140, "prompt_tokens": 35, "completion_tokens": 2},
    {"task": "format", "prediction": " YES ", "target": "yes", "latency_ms": 80, "prompt_tokens": 20, "completion_tokens": 1},
    {"task": "format", "prediction": "no", "target": "no", "latency_ms": 120, "prompt_tokens": 25, "completion_tokens": 1},
]
report = evaluate_records(records)
assert report["accuracy"] == 0.75
assert report["per_task_accuracy"] == {"qa": 0.5, "format": 1.0}
assert report["median_latency_ms"] == 110.0
assert report["prompt_tokens"] == 110 and report["completion_tokens"] == 6
print(report)

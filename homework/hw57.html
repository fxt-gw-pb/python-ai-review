💬 医疗 KBQA：NLU、Cypher 编排与可追溯回答综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：医疗 KBQA：NLU、Cypher 编排与可追溯回答

55 道练习，按“输入与安全路由 → 意图/槽位 → 实体链接 → DST 与对话策略 → 查询模板与 Neo4j → 证据化回答 → 端到端评测与上线治理”组织。做题时始终记住：KBQA 的目标不是生成一段听起来合理的话，而是在明确能力边界内，把问题可靠地映射到可审核查询，并返回带状态和证据的结果。

建议先独立作答，再展开参考答案。本练习依据原课件第七章和站内重组讲义设计，并参考 复杂多跳 KBQA 的查询图生成研究、PubMedQA 原始论文以及 Neo4j 官方的 Cypher 参数说明、Python Driver 查询接口和事务语义校准。10 道代码题只用 Python 标准库模拟 NLU、策略、模板编排、答案合同与评测，不连接真实数据库，不处理真实患者数据。

一、选择题（30 题）

1. 模板式医疗 KBQA 与纯生成式问答最核心的区别是？

A. KBQA 的事实答案主要来自结构化图查询，生成只负责受控表达 \ B. KBQA 不需要理解用户问题 \ C. 纯生成式系统一定有可靠来源 \ D. KBQA 只能返回“是”或“否”

✅ 参考答案

答案：A。 模板式 KBQA 将语义帧映射为审核过的查询，答案可追溯到图中实体、关系和来源。生成式模型可用于改写，但不能把模型记忆当成图查询证据。

2. 在“慢性支气管炎有哪些症状？”中，意图识别主要输出什么？

A. disease_symptoms 及其置信度 \ B. 疾病在句中的字符偏移 \ C. Neo4j 密码 \ D. 最终处方

✅ 参考答案

答案：A。 意图回答“用户想做什么”；槽位和 NER 负责提取“涉及什么对象”，实体链接再确定知识库中的稳定实体。

3. 槽位填充的直接目标是什么？

A. 为当前意图所需的参数填入 mention、类型和候选值 \ B. 为所有用户生成相同回复 \ C. 自动扩展数据库权限 \ D. 绕过实体链接

✅ 参考答案

答案：A。 槽位 Schema 应由业务意图定义。缺失、歧义或不合法的槽位必须进入追问或兜底，而不是强行执行。

4. NER 已识别 mention“感冒”，为什么仍不能直接用于图查询？

A. mention 可能有别名、歧义或知识库未收录，需要链接到稳定实体 ID \ B. Neo4j 不支持中文 \ C. NER 已经返回完整答案 \ D. 所有 mention 都是疾病

✅ 参考答案

答案：A。 NER 输出文本片段和类型，实体链接输出 entity_id 或 NIL。用原始名称查询会受到别名、同名和拼写变化影响。

5. 一个良好的语义帧至少应包含什么？

A. 意图、意图置信度、槽位 mention、已链接实体 ID、链接置信度 \ B. 只有原始问题 \ C. 只有最终自然语言答案 \ D. 数据库全部节点

✅ 参考答案

答案：A。 还可包含候选列表、偏移、否定、时态、会话信息和模型版本。显式合同让 NLU、策略和查询层能分别测试。

6. 用户问“慢支有哪些症状，又该吃什么药？”，最合理的处理是？

A. 识别多意图并按策略拆分执行，或明确询问先回答哪一项 \ B. 随机保留一个意图 \ C. 把整句直接作为 Cypher \ D. 默认只回答药物剂量

✅ 参考答案

答案：A。 多意图需要显式表示和排序。能安全并行的知识检索可拆分后合并证据；涉及高风险动作时应缩小范围或澄清。

7. 规则模板做意图识别的典型特点是？

A. 规则清晰、易审计，但长尾召回和迁移成本可能较差 \ B. 不需要定义意图 \ C. 能自动覆盖所有口语表达 \ D. 输出天然校准的概率

✅ 参考答案

答案：A。 小型封闭域可先用规则建立可解释基线，再用分类模型扩展覆盖。无论方法如何，都要维护未支持类和回归样例。

8. 意图分类模型在离线测试准确率高，但线上总是过度自信，最需要补充什么？

A. 置信度校准、拒识阈值和分桶可靠性评估 \ B. 把阈值固定为 0 \ C. 删除 unsupported 状态 \ D. 只增加回复长度

✅ 参考答案

答案：A。 分类正确率不等于置信度可信。策略依赖分数时，要评估校准误差，并在不同意图、实体和风险等级上分别设阈值。

9. 意图明确但必需 disease 槽位缺失，系统应优先？

A. 返回 clarification_needed 并追问疾病名称 \ B. 随机选择热门疾病 \ C. 查询所有疾病 \ D. 把空字符串绑定到参数

✅ 参考答案

答案：A。 缺槽是正常对话状态。追问应只询问最关键缺失信息，并保留已确认的意图和其他槽位。

10. 实体链接得到两个分数接近的“感冒”候选时，最佳策略是？

A. 展示简短区分信息，请用户确认候选 \ B. 永远取第一个 \ C. 合并两个实体 ID \ D. 使用名称拼接查询

✅ 参考答案

答案：A。 澄清选项应包含足够区分的规范名称、类型或定义，避免泄露内部调试分数。确认结果写入当前会话状态。

11. 实体链接输出 NIL 表示什么？

A. 当前知识库没有足够可信的匹配实体 \ B. 实体一定不存在于现实世界 \ C. 数据库连接失败 \ D. 意图一定错误

✅ 参考答案

答案：A。 NIL 是覆盖边界，不是事实否定。系统可请求更多信息、转文档检索或返回未收录，而不能强制链接到最相似实体。

12. DST 在多轮 KBQA 中的主要职责是？

A. 跟踪已确认意图、槽位、候选、澄清进度和状态转移 \ B. 替代图数据库 \ C. 训练词向量 \ D. 保存所有用户的同一个全局变量

✅ 参考答案

答案：A。 DST 为对话策略提供当前状态。状态必须按会话隔离，并明确来源、更新时间和有效期。

13. 为什么澄清状态需要 TTL？

A. 防止过期上下文在很久后的新问题中被错误复用，并减少敏感数据留存 \ B. TTL 可以提高模型参数量 \ C. TTL 会自动纠正实体链接 \ D. 没有 TTL 就不能查询 Neo4j

✅ 参考答案

答案：A。 还要防止并发覆盖、跨用户串话和重放攻击。过期后应回到新会话流程，而不是猜测用户仍在回答旧问题。

14. PL 或查询编排层的主要输入输出是什么？

A. 输入已验证语义帧和策略动作，输出模板 ID、参数与执行计划 \ B. 输入数据库密码，输出训练标签 \ C. 输入任意文本，输出任意写 Cypher \ D. 输入答案，输出原始问题

✅ 参考答案

答案：A。 编排层应与 NLU 和数据库驱动解耦，查询模板有版本、权限、超时和返回 Schema。

15. 为什么不能直接执行大模型根据用户问题生成的任意 Cypher？

A. 可能产生注入、越权、昂贵查询、写操作或 Schema 幻觉 \ B. 大模型不能输出文本 \ C. Cypher 不能读取图 \ D. 任意 Cypher 一定比模板慢

✅ 参考答案

答案：A。 封闭域优先使用白名单模板；若探索受限的语义解析，也要做 AST/语法验证、只读权限、成本限制和沙箱评测，不能只靠提示词声明安全。

16. 查询疾病症状时，最适合绑定到 Cypher 参数的是？

A. 已链接且类型验证通过的 disease_id \ B. 用户整句问题 \ C. 未审核的关系类型字符串 \ D. 数据库管理员密码

✅ 参考答案

答案：A。 参数化避免把数据值拼进查询结构，并利于查询计划复用。输入 ID 仍需前缀、长度、所属类型和访问范围校验。

17. 用户输入中的“症状”应如何决定 HAS_SYMPTOM 关系？

A. 由审核过的 intent→template 白名单映射决定 \ B. 直接把用户词拼到关系类型位置 \ C. 对任何词都创建新关系类型 \ D. 由数据库随机选择

✅ 参考答案

答案：A。 自然语言表达可以多样，但底层 Schema 是受控的。意图注册表负责把多种问法统一到固定模板和返回合同。

18. 图查询返回空结果时，为什么要先区分“实体不存在”和“实体存在但暂无该关系”？

A. 两种状态的用户含义、排障方向和评测标签不同 \ B. 两者一定表示数据库宕机 \ C. 两者都应回答“没有这种疾病” \ D. 空结果必须转成随机答案

✅ 参考答案

答案：A。 前者可能是链接或覆盖问题，后者表示已知实体暂无该类知识。还要区分过滤后为空、权限不足、数据过期和系统错误。

19. 可追溯答案合同为何需要 evidence？

A. 让用户和开发者知道答案来自哪些来源、版本和更新时间 \ B. 用来隐藏查询错误 \ C. 自动证明来源绝对正确 \ D. 替代回答状态

✅ 参考答案

答案：A。 evidence 支持核验、冲突展示和问题定位。来源存在不代表结论适用于所有人，仍需医学边界与适用条件。

20. status 字段最重要的价值是？

A. 区分 answered、clarification_needed、not_found、unsupported、unsafe 和 system_error \ B. 让所有失败看起来像成功 \ C. 保存用户密码 \ D. 替代 trace_id

✅ 参考答案

答案：A。 客户端和监控可以按状态采取不同动作。不能把未命中、拒答和系统异常都包装成一段自信回答。

21. 查询返回多个症状时，为保证接口稳定，通常还应做什么？

A. DISTINCT 去重、确定性排序、数量上限和截断说明 \ B. 依赖数据库未声明的默认顺序 \ C. 重复每个结果三次 \ D. 删除来源

✅ 参考答案

答案：A。 集合型答案的排序和分页属于返回合同。超大结果应限制成本并告诉用户如何缩小问题。

22. “哪些疾病既有咳嗽症状又推荐某药物？”比单跳问题更难，主要因为？

A. 需要构造含多个约束和关系的查询图，并控制候选组合 \ B. 它没有任何实体 \ C. Neo4j 只能做单跳查询 \ D. 只需把答案模板加长

✅ 参考答案

答案：A。 多跳和多约束 KBQA 要同时处理实体、关系、连接结构和约束作用域。查询图研究表明，尽早引入约束有助于缩小搜索空间。

import unicodedata


def validate_input(text, max_chars=200):
    if not isinstance(text, str):
        raise TypeError("text must be a string")
    normalized = unicodedata.normalize("NFKC", text)
    for char in normalized:
        category = unicodedata.category(char)
        if category.startswith("C") and char not in "\t\n\r":
            raise ValueError("control character is not allowed")
    normalized = " ".join(normalized.split())
    if not normalized:
        raise ValueError("question is empty")
    if len(normalized) > max_chars:
        raise ValueError("question is too long")
    return normalized

INTENT_KEYWORDS = {
    "disease_symptoms": ("症状", "表现"),
    "disease_drugs": ("药", "药物"),
    "disease_foods": ("吃什么", "饮食"),
}


def rule_parse(question, lexicon):
    intent_hits = [
        intent
        for intent, words in INTENT_KEYWORDS.items()
        if any(word in question for word in words)
    ]
    mentions = []
    for surface, candidates in lexicon.items():
        start = question.find(surface)
        if start >= 0:
            mentions.append({
                "mention": surface,
                "start": start,
                "end": start + len(surface),
                "candidates": list(candidates),
            })
    status = "ok"
    if len(intent_hits) != 1:
        status = "intent_ambiguous"
    elif len(mentions) != 1:
        status = "entity_ambiguous"
    return {
        "status": status,
        "intent_candidates": sorted(intent_hits),
        "mentions": sorted(mentions, key=lambda item: (item["start"], item["end"])),
    }

48. 实现 validate_frame(frame)，按意图注册表检查置信度、必需槽位、实体类型、NIL 与候选逻辑。

✅ 参考答案

REGISTRY = {
    "disease_symptoms": {"slot": "disease", "prefix": "disease:"},
    "disease_drugs": {"slot": "disease", "prefix": "disease:"},
}


def probability(value):
    return (
        isinstance(value, (int, float))
        and not isinstance(value, bool)
        and 0.0 <= float(value) <= 1.0
    )


def validate_frame(frame):
    errors = []
    intent = frame.get("intent", {})
    name = intent.get("name")
    if name not in REGISTRY:
        errors.append("unsupported_intent")
    if not probability(intent.get("confidence")):
        errors.append("invalid_intent_confidence")

    slots = frame.get("slots")
    if not isinstance(slots, dict):
        return sorted(set(errors + ["invalid_slots"]))
    if name in REGISTRY:
        slot_name = REGISTRY[name]["slot"]
        slot = slots.get(slot_name)
        if not isinstance(slot, dict):
            errors.append("missing_required_slot")
        else:
            entity_id = slot.get("entity_id")
            is_nil = slot.get("is_nil")
            if is_nil is not True and (
                not isinstance(entity_id, str)
                or not entity_id.startswith(REGISTRY[name]["prefix"])
            ):
                errors.append("invalid_entity_id")
            if not probability(slot.get("confidence")):
                errors.append("invalid_link_confidence")
            candidates = slot.get("candidates", [])
            if is_nil is True and entity_id is not None:
                errors.append("nil_has_entity_id")
            if not isinstance(candidates, list):
                errors.append("invalid_candidates")
    return sorted(set(errors))

SUPPORTED = {"disease_symptoms", "disease_drugs", "disease_foods"}


def decide_action(frame, safety):
    if safety.get("emergency") or safety.get("personal_diagnosis"):
        return {"action": "unsafe", "reason": "medical_safety"}

    intent = frame.get("intent", {})
    if intent.get("name") not in SUPPORTED:
        return {"action": "fallback", "reason": "unsupported_intent"}

    slot = frame.get("slots", {}).get("disease")
    if not slot:
        return {"action": "clarify", "reason": "missing_disease"}
    if slot.get("is_nil"):
        return {"action": "clarify", "reason": "entity_nil"}

    candidates = sorted(
        slot.get("candidates", []),
        key=lambda item: item["score"],
        reverse=True,
    )
    if len(candidates) >= 2 and candidates[0]["score"] - candidates[1]["score"] < 0.08:
        return {"action": "clarify", "reason": "close_candidates"}

    confidence = min(
        float(intent.get("confidence", 0.0)),
        float(slot.get("confidence", 0.0)),
    )
    if confidence >= 0.80:
        return {"action": "execute", "reason": "trusted_frame"}
    if confidence >= 0.45:
        return {"action": "clarify", "reason": "medium_confidence"}
    return {"action": "fallback", "reason": "low_confidence"}

50. 实现带 TTL 的 SessionStore，按 session_id 隔离澄清状态，并防止调用方修改内部对象。

✅ 参考答案

from copy import deepcopy


class SessionStore:
    def __init__(self):
        self._items = {}

    def put(self, session_id, state, now, ttl_seconds):
        if not session_id or ttl_seconds <= 0:
            raise ValueError("invalid session or ttl")
        self._items[session_id] = {
            "expires_at": now + ttl_seconds,
            "state": deepcopy(state),
        }

    def get(self, session_id, now):
        item = self._items.get(session_id)
        if item is None:
            return None
        if now >= item["expires_at"]:
            del self._items[session_id]
            return None
        return deepcopy(item["state"])

    def pop(self, session_id, now):
        state = self.get(session_id, now)
        self._items.pop(session_id, None)
        return state


store = SessionStore()
state = {"pending_intent": "disease_symptoms", "candidates": ["disease:1"]}

loaded["candidates"].append("also-tampered")
assert store.get("session-a", now=120.0)["candidates"] == ["disease:1"]
assert store.get("session-a", now=130.0) is None
assert store.get("session-b", now=110.0) is None
print("isolated")

}


def compile_query(intent, slots, limit=50):
    if intent not in QUERIES:
        raise ValueError("intent is not allowlisted")
    disease_id = slots.get("disease_id")
    if (
        not isinstance(disease_id, str)
        or not disease_id.startswith("disease:")
        or len(disease_id) > 80
    ):
        raise ValueError("invalid disease_id")
    if not isinstance(limit, int) or isinstance(limit, bool) or not 1 <= limit <= 100:
        raise ValueError("invalid limit")
    return QUERIES[intent], {"disease_id": disease_id, "limit": limit}

52. 实现 build_response()，区分实体不存在、关系为空、成功和系统错误，并构造稳定答案合同。

✅ 参考答案

def build_response(
    trace_id,
    intent,
    entity,
    entity_exists,
    rows=None,
    error_code=None,
):
    base = {
        "trace_id": trace_id,
        "intent": intent,
        "entities": [entity] if entity else [],
        "values": [],
        "evidence": [],
        "disclaimer": "仅用于知识检索，不替代专业医疗建议。",
    }
    if error_code is not None:
        return dict(base, status="system_error", reason=error_code, answer="")
    if not entity_exists:
        return dict(base, status="not_found", reason="entity_missing", answer="")

    rows = rows or []
    by_id = {}
    evidence = set()
    for row in rows:
        by_id[row["id"]] = row["name"]
        if row.get("source"):
            evidence.add(row["source"])
    if not by_id:
        return dict(base, status="not_found", reason="relation_missing", answer="")

    values = [
        {"id": item_id, "name": by_id[item_id]}
        for item_id in sorted(by_id, key=lambda key: (by_id[key], key))
    ]
    return dict(
        base,
        status="answered",
        reason=None,
        answer="图谱中收录了 " + str(len(values)) + " 项相关知识。",
        values=values,
        evidence=[{"source": source} for source in sorted(evidence)],
    )


entity = {"id": "disease:001", "name": "某疾病"}
missing = build_response("t1", "disease_symptoms", entity, False)
assert missing["reason"] == "entity_missing"
empty = build_response("t2", "disease_symptoms", entity, True, [])
assert empty["reason"] == "relation_missing"
answered = build_response("t3", "disease_symptoms", entity, True, [
    {"id": "s:2", "name": "咳嗽", "source": "doc:2"},
    {"id": "s:2", "name": "咳嗽", "source": "doc:1"},
])
assert answered["status"] == "answered"
assert len(answered["values"]) == 1 and len(answered["evidence"]) == 2
print(answered)

from collections import defaultdict


def aggregate_food_claims(rows):
    grouped = defaultdict(lambda: {
        "name": None,
        "do_eat_sources": set(),
        "not_eat_sources": set(),
    })
    for row in rows:
        item = grouped[row["food_id"]]
        item["name"] = row["name"]
        if row["relation"] == "DO_EAT":
            item["do_eat_sources"].add(row["source"])
        elif row["relation"] == "NOT_EAT":
            item["not_eat_sources"].add(row["source"])
        else:
            raise ValueError("unsupported food relation")

    result = []
    for food_id in sorted(grouped):
        item = grouped[food_id]
        do_sources = sorted(item["do_eat_sources"])
        not_sources = sorted(item["not_eat_sources"])
        result.append({
            "food_id": food_id,
            "name": item["name"],
            "do_eat_sources": do_sources,
            "not_eat_sources": not_sources,
            "conflict": bool(do_sources and not_sources),
        })
    return result


rows = [
    {"food_id": "food:1", "name": "食物甲",
     "relation": "DO_EAT", "source": "doc:a"},
    {"food_id": "food:1", "name": "食物甲",
     "relation": "NOT_EAT", "source": "doc:b"},
    {"food_id": "food:2", "name": "食物乙",
     "relation": "DO_EAT", "source": "doc:a"},
]

54. 实现 evaluate(cases)，计算意图 Macro F1、答案集合 F1、证据覆盖率和 unsafe 召回率。

✅ 参考答案
Python
from collections import defaultdict


def set_f1(gold, predicted):
    gold, predicted = set(gold), set(predicted)
    if not gold and not predicted:
        return 1.0
    if not gold or not predicted:
        return 0.0
    overlap = len(gold & predicted)
    precision = overlap / len(predicted)
    recall = overlap / len(gold)
    return 2 * precision * recall / (precision + recall) if overlap else 0.0


def evaluate(cases):
    labels = sorted({
        case["gold_intent"] for case in cases
    } | {
        case["pred_intent"] for case in cases
    })
    per_label = defaultdict(lambda: {"tp": 0, "fp": 0, "fn": 0})
    for case in cases:
        gold, pred = case["gold_intent"], case["pred_intent"]
        for label in labels:
            per_label[label]["tp"] += int(gold == label and pred == label)
            per_label[label]["fp"] += int(gold != label and pred == label)
            per_label[label]["fn"] += int(gold == label and pred != label)

    f1_values = []
    for label in labels:
        counts = per_label[label]
        denominator = 2 * counts["tp"] + counts["fp"] + counts["fn"]
        f1_values.append(2 * counts["tp"] / denominator if denominator else 0.0)

    answer_f1 = sum(
        set_f1(case["gold_values"], case["pred_values"]) for case in cases
    ) / len(cases)
    answered = [case for case in cases if case["pred_values"]]
    provenance = sum(
        bool(case["pred_evidence"]) for case in answered
    ) / len(answered) if answered else 1.0
    unsafe_gold = [case for case in cases if case["gold_status"] == "unsafe"]
    unsafe_recall = sum(
        case["pred_status"] == "unsafe" for case in unsafe_gold
    ) / len(unsafe_gold) if unsafe_gold else 1.0
    return {
        "intent_macro_f1": sum(f1_values) / len(f1_values),
        "answer_set_f1": answer_f1,
        "provenance_coverage": provenance,
        "unsafe_recall": unsafe_recall,
    }


cases = [
    {"gold_intent": "symptom", "pred_intent": "symptom",
     "gold_values": ["s1", "s2"], "pred_values": ["s1"],
     "pred_evidence": ["doc:1"], "gold_status": "answered",
     "pred_status": "answered"},
    {"gold_intent": "unsafe", "pred_intent": "unsafe",
     "gold_values": [], "pred_values": [], "pred_evidence": [],
     "gold_status": "unsafe", "pred_status": "unsafe"},
]
metrics = evaluate(cases)
assert metrics["intent_macro_f1"] == 1.0
assert metrics["unsafe_recall"] == 1.0
assert 0.0 < metrics["answer_set_f1"] < 1.0
print(metrics)

55. 实现 regression_gate(results, limits)，检查状态准确率、安全召回、证据覆盖、trace 覆盖和敏感日志泄露。

✅ 参考答案

def ratio(numerator, denominator):
    return numerator / denominator if denominator else 1.0


def regression_gate(results, limits):
    total = len(results)
    status_accuracy = ratio(
        sum(row["pred_status"] == row["gold_status"] for row in results),
        total,
    )
    unsafe = [row for row in results if row["gold_status"] == "unsafe"]
    unsafe_recall = ratio(
        sum(row["pred_status"] == "unsafe" for row in unsafe),
        len(unsafe),
    )
    answered = [row for row in results if row["pred_status"] == "answered"]
    evidence_coverage = ratio(
        sum(bool(row.get("evidence")) for row in answered),
        len(answered),
    )
    trace_coverage = ratio(
        sum(bool(row.get("trace_id")) for row in results),
        total,
    )
    leakage_count = sum(bool(row.get("sensitive_log")) for row in results)

    metrics = {
        "status_accuracy": status_accuracy,
        "unsafe_recall": unsafe_recall,
        "evidence_coverage": evidence_coverage,
        "trace_coverage": trace_coverage,
        "leakage_count": leakage_count,
    }
    failures = []
    for name in (
        "status_accuracy", "unsafe_recall",
        "evidence_coverage", "trace_coverage",
    ):
        if metrics[name] < limits["min_" + name]:
            failures.append(name + "_below_min")
    if leakage_count > limits["max_leakage_count"]:
        failures.append("sensitive_log_leakage")
    return {"passed": not failures, "metrics": metrics, "failures": failures}


results = [
    {"gold_status": "answered", "pred_status": "answered",
     "evidence": ["doc:1"], "trace_id": "t1", "sensitive_log": False},
    {"gold_status": "unsafe", "pred_status": "unsafe",
     "evidence": [], "trace_id": "t2", "sensitive_log": False},
]
limits = {
    "min_status_accuracy": 0.99,
    "min_unsafe_recall": 1.0,
    "min_evidence_coverage": 1.0,
    "min_trace_coverage": 1.0,
    "max_leakage_count": 0,
}
report = regression_gate(results, limits)
assert report["passed"] is True
bad = [dict(row) for row in results]
bad[1]["pred_status"] = "answered"
bad[1]["sensitive_log"] = True
failed = regression_gate(bad, limits)
assert failed["passed"] is False
assert "sensitive_log_leakage" in failed["failures"]
print(report)

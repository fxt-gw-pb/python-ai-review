🐳 Docker：部署 Milvus 与 Redis 综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：Docker 部署 Milvus 与 Redis🚀 所属项目：EduRAG：企业私有知识智能问答

40 道练习，覆盖用 Docker Compose 同时拉起单机 Milvus 与 Redis 的前置条件、端口与版本、up -d / ps / down、Windows（WSL2）与 Apple Silicon 差异，以及 pymilvus / redis 客户端验证与排障。重点练「依赖服务（etcd/minio）+ 端口映射 + 数据卷持久化」。

建议先独立作答，再展开参考答案。代码题用纯 Python 模拟连接参数、compose 服务清单与端口冲突改写，不启动真实 Docker。

二、简答题（12 题）

21. 为什么用 Compose 而不是手工逐个 docker run？

✅ 参考答案

Milvus 单机常依赖 etcd、minio 等多容器，Compose 用一份 YAML 声明网络、端口、卷与启动顺序，便于一键 up/down 与复现环境。

22. 区分宿主机端口与容器端口：19531:19530 含义？

✅ 参考答案

左侧 19531 是宿主机端口，右侧 19530 是容器内服务端口。客户端连 localhost:19531，流量转发到容器 19530。

23. 列出启动成功后你应用 docker ps 核对的要点。

✅ 参考答案

相关容器状态为 Up；端口映射正确；容器名符合预期（如 milvus、redis、etcd、minio）；无疯狂重启（Restarting）。

24. Windows 路径为何建议先规划独立项目目录？

✅ 参考答案

放置 docker-compose.yml 与 ./volumes 数据，避免散落系统盘；也方便整目录备份/清理，权限与路径空格问题更可控。

25. 验证 Milvus「部署成功」的最小证据是什么？

✅ 参考答案

connections.connect 成功且 utility.get_server_version() 返回合理版本字符串（课件示例约 2.4.x），无连接超时/拒绝。

26. Redis decode_responses=True 的作用？

✅ 参考答案

让客户端自动把返回值解码为 str，而不是 bytes，便于打印与断言（如 "Hello, Redis!"）。

27. 何时需要 docker compose down 后又删 volumes？

✅ 参考答案

要彻底清空持久化数据（坏数据、想重来、释放磁盘）时。仅 down 默认不一定删除命名卷/绑定目录里的文件，需按课件删 ./volumes/...。

28. M1 上启用 Rosetta 的适用场景？

✅ 参考答案

个别镜像/工具仍按 x86_64 构建或出现兼容错误时，用 Rosetta 模拟跑 AMD64 镜像。若官方已提供 ARM64 且运行正常，可不开启。

29. 同时部署 Milvus 与 Redis 时，如何避免「连错端口」？

✅ 参考答案

牢记职责：向量库走 19530（及管理/REST 9091），缓存/KV 走 6379；客户端库分别用 pymilvus 与 redis，不要混用端口。

30. 数据未持久化时，优先检查 compose 哪两类配置？

✅ 参考答案

volumes 是否挂到宿主机路径；Redis 是否启用 AOF（--appendonly yes）且目录可写。Milvus 侧同样核对数据目录挂载。

31. 手动 docker pull 仍失败，可能原因？

✅ 参考答案

网络/代理/DNS、镜像源限流、tag 写错、磁盘满、Docker 未登录私有仓库等。应核对镜像名与 Desktop 网络设置。

32. 从「容器 Up」到「业务可用」还缺什么？

✅ 参考答案

应用侧连通性验证（SDK 读写）、正确的认证/密码（若 compose 设了 Redis 密码）、以及业务配置里的 host/port 与映射一致。

三、代码题（8 题）

33. 解析端口映射字符串 host:container

✅ 参考答案
Python
def parse_port_map(spec: str) -> tuple[int, int]:
    host, container = spec.split(":", 1)
    return int(host), int(container)


assert parse_port_map("19530:19530") == (19530, 19530)
assert parse_port_map("19531:19530") == (19531, 19530)
print(parse_port_map("6380:6379"))

34. 端口冲突时生成新的映射建议

✅ 参考答案
Python
BUSY = {19530, 9091, 6379}


def remap_if_busy(host_port: int, container_port: int) -> str:
    p = host_port
    while p in BUSY:
        p += 1
    # 若原 host 已被占，换一个空闲宿主机端口
    if host_port in BUSY and host_port == container_port:
        # 演示：19530 被占 -> 19531
        p = host_port + 1
        while p in BUSY:
            p += 1
    return f"{p}:{container_port}"


assert remap_if_busy(19530, 19530) == "19531:19530"
print(remap_if_busy(6379, 6379))

35. 校验 Compose 必备服务名是否齐全

✅ 参考答案

REQUIRED = {"etcd", "minio", "milvus", "redis"}


def missing_services(names: set[str]) -> list[str]:
    # 允许实际 key 带前缀，做包含匹配
    have = set()
    for req in REQUIRED:
        if any(req in n.lower() for n in names):
            have.add(req)
    return sorted(REQUIRED - have)


assert missing_services({"etcd", "minio", "standalone-milvus"}) == ["redis"]
print(missing_services({"etcd", "minio", "milvus_standalone", "redis"}))

40. 部署状态机：从 pull 到 verify 的合法步骤

✅ 参考答案

STEPS = ["prepare", "compose_up", "docker_ps", "verify_milvus", "verify_redis"]


def advance(done: list[str], action: str) -> list[str]:
    expect = STEPS[len(done)]
    if action != expect:
        raise ValueError(f"expect {expect}, got {action}")
    return done + [action]


state = []
for a in STEPS:
    state = advance(state, a)
assert state[-1] == "verify_redis"
print(" -> ".join(state))

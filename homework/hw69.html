🗜️ 模型压缩：量化、蒸馏与剪枝综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：模型压缩：量化、蒸馏与剪枝🚀 所属项目：投满分：智能文本分类全流程

40 道练习，覆盖模型压缩工程目标与四条技术路线、量化（对称 / min-max、QAT / PTQ / DQ）、知识蒸馏（软标签、温度 $T$、$\alpha$、$T^2$）与剪枝（结构/非结构、L1、流水线顺序）。重点练「先剪再蒸后量化」和「动态量化为何常在 CPU」。

建议先独立作答，再展开参考答案。代码题用纯 Python / 少量公式演算模拟 scale、KL 与剪枝掩码，不依赖 GPU 训练。

一、选择题（20 题）

1. 模型压缩的工程目标更接近？

A. 无条件增大参数量
B. 在目标硬件上满足精度前提下，最小化推理成本（速度 / 算力 / 显存 / 功耗）
C. 只美化训练曲线颜色
D. 取消全部推理

✅ 参考答案

答案：B。 让模型更小、更快、更省。

2. 课件列出的四种主流技术不包括？

A. 剪枝 Pruning
B. 量化 Quantization、知识蒸馏、低秩因式分解
C. 必须手写操作系统调度器
D. A 与 B 均属四种技术覆盖范围

✅ 参考答案

答案：C。 四条：剪枝、量化、蒸馏、低秩分解。

3. 量化主要做什么？

A. 删除全部网络层
B. 降低参数（及常含激活）存储/计算精度，如 float32→int8
C. 只增加词表大小
D. 禁止部署到边缘设备

✅ 参考答案

答案：B。 同时可作用于权重与激活。

4. 剪枝主要作用于？

A. 仅数据集标签字符串
B. 模型权重：删除不重要参数
C. 仅 CSS 样式表
D. 仅日志级别

✅ 参考答案

答案：B。 冗余参数可删且性能影响可很小。

5. 课件给出的主流压缩流水线顺序更接近？

A. 先量化 → 再剪枝 → 禁止蒸馏
B. 剪枝/低秩 → 蒸馏恢复精度 → 量化 PTQ/QAT → 推理引擎部署
C. 只蒸馏一次即结束，永不量化
D. 随机打乱且无评估

✅ 参考答案

答案：B。 剪完用蒸馏补精度，再量化加速。

6. int8 常见值域是？

A. $-128$ 到 $127$
B. 仅 $0$ 到 $1$ 的浮点
C. 无限整数
D. 仅两个比特

✅ 参考答案

答案：A。 最常用低精度之一。

7. 对称量化（课件权重量化示意）中 zero-point 通常？

A. 固定为 $0$
B. 必须等于 $255$
C. 随机每步重采样且无公式
D. 禁止计算 scale

✅ 参考答案

答案：A。 $\mathrm{scale}=\max(|w|)/127$，$q=\mathrm{round}(w/\mathrm{scale})$。

8. QAT / PTQ / 动态量化（DQ）中，训练后无需校准数据、权重先量化、激活推理时再算参数的是？

A. QAT
B. PTQ
C. Dynamic Quantization（DQ）
D. 低秩分解

✅ 参考答案

答案：C。 文本类 CPU 场景常用；最简单、改动小。

9. 量化感知训练（QAT）的特点是？

A. 训练中模拟量化误差，精度通常最好但成本最高
B. 禁止任何训练
C. 只能量化 bias 不能量化权重
D. 必须在浏览器完成

✅ 参考答案

答案：A。 适合精度极敏感任务。

10. PyTorch quantize_dynamic 课件强调？

A. 必须在 GPU 上完成，CPU 会报错
B. 需在 CPU 上完成；动态量化当前主要支持 QuantizedCPU
C. 只能量化数据集文件名
D. 模型体积必然增大 4 倍

✅ 参考答案

答案：B。 体积常约缩到 ~1/4；GPU 直接动态量化会报错。

11. 知识蒸馏的核心思想是？

A. 学生只背硬标签 one-hot，禁止看教师
B. 大教师指导小学生，学生还学习教师的预测分布（软标签）
C. 删除教师全部参数后随机初始化学生
D. 蒸馏等于量化的别名

✅ 参考答案

答案：B。 Hinton 等提出的经典路线。

12. 软标签相对硬标签的优势是？

A. 丢失全部类别关系
B. 保留不确定性与类间相似度信息
C. 只能是整数 0/1
D. 禁止使用温度 $T$

✅ 参考答案

答案：B。 如 $[0.2,0.7,0.1]$ vs one-hot。

13. 软标签蒸馏总损失课件形式更接近？

A. 仅 MSE，无 KL/CE
B. $L=\alpha\cdot T^{2}\cdot \mathrm{KL}(p_t,p_s)+(1-\alpha)\cdot\mathrm{CE}(y_{\mathrm{true}},z_s)$
C. 仅 L1 正则
D. 仅准确率 Acc

✅ 参考答案

答案：B。 $p=\mathrm{softmax}(z/T)$。

14. 损失里乘 $T^{2}$ 的主要理由（课件）？

A. 纯属排版装饰
B. 平衡温度缩放后的梯度尺度；试验表明有助于训练效果
C. 强制 $T=0$
D. 取消软标签

✅ 参考答案

答案：B。 $\partial\mathrm{KL}/\partial z_s=(p_s-p_t)/T$，乘 $T^{2}$ 后梯度含因子 $T$。

15. 工业常用 $\alpha$、$T$ 起步建议更接近？

A. $\alpha=0.7$，$T=4$，再细调
B. $\alpha=0$，$T=0$ 且禁止验证
C. $\alpha=100$，$T=-1$
D. 只调学习率，从不碰 $\alpha/T$

✅ 参考答案

答案：A。 $\alpha$ 常 $0.5$–$0.9$，$T$ 常 $2$–$10$。

16. 非结构化 vs 结构化剪枝，工程上更能直接「降低规则结构体积/便于加速」的是？

A. 非结构化（稀疏矩阵，细粒度）
B. 结构化（整行/列/通道/层，粗粒度）
C. 二者完全等同
D. 剪枝不能动权重

✅ 参考答案

答案：B。 非结构化更细但常需稀疏库才真正加速。

17. 工程最常用的基于规则剪枝是？

A. 随机剪枝（课件称无价值）
B. L1 范数剪枝：去 $|w|$ 最小的权重
C. 只剪 bias 名称字符串
D. 禁止敏感度分析

✅ 参考答案

答案：B。 常配合敏感度分析决定每层剪多少。

18. 模块级剪枝相对全局剪枝？

A. 对指定模块（如 Linear/Conv）单独剪，工程更主流
B. 必须一次剪掉整个数据集
C. 禁止用于 BERT
D. 只能随机剪

✅ 参考答案

答案：A。 全局剪追求统一稀疏率，小模型可试。

19. 「剪枝 → 蒸馏 → 量化」顺序的直观原因？

A. 先制造冗余再增大模型
B. 先去冗余，再用蒸馏补精度，最后量化加速部署
C. 量化必须永远第一步
D. 蒸馏与剪枝互斥不能组合

✅ 参考答案

答案：B。 与课件常用压缩顺序一致。

20. 累积数据类型（Accumulation）指？

A. 低精度运算的中间结果用更高精度保存，减轻误差累积
B. 删除全部中间结果
C. 只把 int8 存成字符串
D. 与量化无关

✅ 参考答案

答案：A。 如 int8 计算常用 int32 累加。

二、简答题（12 题）

21. 用一句话区分量化、蒸馏、剪枝。

✅ 参考答案

量化降精度；蒸馏让小模型学大模型的行为/分布；剪枝删不重要参数。三者都可组合服务部署。

22. 为什么说量化「用精度换性能」？

✅ 参考答案

低比特表示与计算更快、更省带宽/存储，但存在舍入误差，任务指标可能略降；需用校准、QAT 或评估闭环把损失控在可接受范围。

23. 写出对称量化中 scale 与量化/反量化公式（int8，$q\in[-128,127]$）。

✅ 参考答案

$\mathrm{scale}=\dfrac{\max(|w|)}{127}$，$q=\mathrm{round}(w/\mathrm{scale})$，反量化 $w\approx q\cdot\mathrm{scale}$（zero-point 为 0）。

24. 对比 PTQ 与 DQ：校准数据与激活处理有何不同？

✅ 参考答案

PTQ：训练后用校准集统计激活的 scale/zero-point，推理可全程低精度。DQ：权重先量化，激活的量化参数在推理时动态计算，通常无需校准集，实现更简单。

25. 解释温度 $T$ 变大时软标签会发生什么。

✅ 参考答案

$\mathrm{softmax}(z/T)$ 更平滑，类别概率差变小，学生更容易学到类间相对关系；但过大可能过糊，需与 $\alpha$ 一起调。

26. KL 散度在蒸馏里衡量什么？写出课件形式 $KL(p_t,p_s)$。

✅ 参考答案

衡量教师分布 $p_t$ 与学生分布 $p_s$ 的差异：$KL(p_t,p_s)=\sum p_t\log\frac{p_t}{p_s}$。越小越接近。

27. 为何要先做「学生单独训练」基线再比蒸馏？

✅ 参考答案

否则无法判断提升来自蒸馏还是数据/训练技巧。应用同一验证集对比 Acc/F1（及延迟、体积）。

28. 非结构化剪枝为何「不一定降低部署体积」？

✅ 参考答案

权重变稀疏后若仍按稠密格式存储/计算，文件与算子未必变小变快；需稀疏格式与内核支持。结构化剪枝更易直接受益。

29. 敏感度分析如何指导剪枝比例？

✅ 参考答案

逐层（或逐模块）试剪并测精度，找出不敏感层可多剪、敏感层少剪，再配合 L1 等规则执行，避免一刀切破坏关键层。

30. 低秩因式分解压缩参数的基本想法？

✅ 参考答案

大矩阵 $W\approx AB$（或多个小矩阵乘积），用更少参数近似原变换，降低存储与计算，主要作用于权重。

31. 压缩评估闭环应对比哪些指标？

✅ 参考答案

至少：任务精度（Acc/F1 等）、模型体积、延迟/吞吐、目标设备能否跑起来；压缩前保存基线，压缩后同协议复测。

32. 「蒸馏 + 量化」为何常被称作更优组合？

✅ 参考答案

蒸馏先把能力塞进小模型，量化再进一步降精度加速；课件称压缩效果可更显著（量级上常见数倍），但仍以实测为准。

三、代码题（8 题）

33. 对称量化：算 scale、量化与反量化误差

✅ 参考答案
Python
import math


def symmetric_quantize(w: list[float]):
    max_abs = max(abs(x) for x in w) or 1.0
    scale = max_abs / 127.0
    q = [int(round(x / scale)) for x in w]
    q = [max(-128, min(127, v)) for v in q]
    recon = [v * scale for v in q]
    return scale, q, recon


scale, q, recon = symmetric_quantize([0.0, 1.2, -0.5, 2.0])
assert scale == 2.0 / 127.0
err = max(abs(a - b) for a, b in zip([0.0, 1.2, -0.5, 2.0], recon))
assert err < 0.02
print(round(scale, 6), q[:2], round(err, 6))

34. min-max 量化：计算 scale 与 zero-point（示意 $[0,255]$）

✅ 参考答案

def minmax_params(w: list[float], qmin=0, qmax=255):
    wmin, wmax = min(w), max(w)
    scale = (wmax - wmin) / (qmax - qmin) if wmax != wmin else 1.0
    qzero = int(round(-wmin / scale)) + qmin
    qzero = max(qmin, min(qmax, qzero))
    return scale, qzero


s, z = minmax_params([1.0, 3.0, 5.0])
assert abs(s - (5 - 1) / 255) < 1e-12
print(round(s, 6), z)

✅ 参考答案

import math


def softmax(logits, T=1.0):
    scaled = [x / T for x in logits]
    m = max(scaled)
    exps = [math.exp(x - m) for x in scaled]
    s = sum(exps)
    return [e / s for e in exps]


p1 = softmax([2.0, 1.0, 0.1], T=1)
p4 = softmax([2.0, 1.0, 0.1], T=4)
# T 更大应更平滑：最大概率下降
assert max(p4) < max(p1)
print([round(x, 3) for x in p1], [round(x, 3) for x in p4])

import math


def kl(pt, ps):
    return sum(p * math.log(p / q) for p, q in zip(pt, ps))

def l1_prune(weights: list[float], amount: float = 0.3) -> list[float]:
    n = len(weights)
    k = int(n * amount)
    order = sorted(range(n), key=lambda i: abs(weights[i]))
    out = weights[:]
    for i in order[:k]:
        out[i] = 0.0
    return out


w = [0.01, -2.0, 0.02, 3.0, -0.005]
pruned = l1_prune(w, 0.4)  # 剪 2 个最小
assert pruned.count(0.0) == 2
assert abs(pruned[1]) == 2.0  # 大切幅保留
print(pruned)

PIPELINE = ["prune", "distill", "quantize", "deploy"]


def next_ok(done: list[str], nxt: str) -> bool:
    if nxt not in PIPELINE:
        return False
    expect = PIPELINE[len(done)]
    return nxt == expect and done == PIPELINE[: len(done)]


assert next_ok([], "prune")
assert next_ok(["prune"], "distill")
assert not next_ok(["prune"], "quantize")  # 跳过蒸馏

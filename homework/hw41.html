🔁 RNN Seq2Seq 机器翻译实战综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：RNN Seq2Seq 机器翻译实战

55 道练习，按“平行语料 → 变长 batch → GRU Encoder → Attention → GRU Decoder → Teacher Forcing → 自回归评估”组织。重点是理解 RNN Seq2Seq 的逐步数据流，并构建无泄漏、可训练、可正确停止与评估的翻译闭环。

建议先独立作答，再展开参考答案。代码题统一使用 <PAD>=0、<SOS>=1、<EOS>=2、<UNK>=3，并通过断言覆盖 OOV、序列长度、Attention Mask、隐藏状态形状、Teacher Forcing、梯度裁剪和自回归解码。

一、选择题（30 题）

1. 经典 RNN Seq2Seq 机器翻译系统的核心结构是？

A. 只使用一个分类器预测整句类别
B. Encoder 将源序列编码，Decoder 根据编码结果逐步生成目标序列
C. 将源句与目标句分别聚类
D. 只计算词频，不学习参数

✅ 参考答案

答案：B。 Encoder 依次读取源 token 并形成隐藏表示；Decoder 从 <SOS> 开始，以隐藏状态和上下文为条件预测下一个目标 token，直到 <EOS>。

2. 构建词表、词频阈值和最大长度策略的正确顺序是？

A. 先切分 train/valid/test，只在训练集拟合这些规则
B. 使用测试集拟合后再切分
C. 用全部数据建词表，因为词表不是模型参数
D. 每个 epoch 用验证集更新词表

✅ 参考答案

答案：A。 词表与长度分位数同样携带数据分布信息；先看验证或测试语料会产生预处理泄漏。验证、测试阶段只能复用训练集冻结的规则。

3. 验证句中的新词不在训练词表时，应优先映射到？

A. <PAD>
B. <EOS>
C. <UNK>，或由已冻结的子词模型切分为已知单元
D. 当前 batch 中最常见的词

✅ 参考答案

答案：C。 <PAD> 表示不存在的补齐位置，会被 Embedding、Attention、损失和指标忽略；把真实 OOV 映射成 PAD 会悄悄删除输入信息。

4. 目标序列 [SOS, je, suis, EOS, PAD] 的 Teacher Forcing 输入与监督标签应是？

A. 输入 [SOS,je,suis,EOS]，标签 [je,suis,EOS,PAD]
B. 输入与标签都保留完整序列
C. 输入 [je,suis,EOS]，标签 [SOS,je,suis]
D. 输入只有 [PAD]

✅ 参考答案

答案：A。 Decoder 在位置 $t$ 接收前一个目标 token，并监督预测下一个 token；标签中的 PAD 再由 ignore_index 排除。

5. 单层单向 GRU Encoder 输入为 [B,S,E]，隐藏维度为 H，其两个主要输出形状是？

A. 所有时间步 [B,S,H]，最终隐藏状态 [1,B,H]
B. 所有时间步 [S,H]，最终隐藏状态 [B,S]
C. 都是 [B,V]
D. 所有时间步 [B,H,S]，最终隐藏状态 [B,1]

✅ 参考答案

答案：A。 batch_first=True 只改变序列输出的前两轴；GRU 的隐藏状态仍按 [num_layers * num_directions, B, H] 排列。

6. 不带 Attention 的早期 Seq2Seq 在长句上容易退化，主要原因是？

A. Decoder 不能使用 Embedding
B. 整个源句被压入一个固定长度向量，形成信息瓶颈
C. GRU 不支持反向传播
D. 目标词表一定比源词表小

✅ 参考答案

答案：B。 单一最终隐藏状态要承载整句细节，输入变长后容易遗忘；Attention 让 Decoder 每一步重新读取不同源位置。

7. 在 GRU + Attention Decoder 的第 $t$ 步，常见的 Query、Key、Value 分别来自？

A. Query 来自当前 Decoder 隐藏状态，Key/Value 来自 Encoder 各时间步输出
B. 三者都来自目标标签
C. Query 来自 PAD，Key/Value 来自损失
D. Query 来自词表大小，Key/Value 来自 batch size

✅ 参考答案

答案：A。 Decoder 状态表达“当前需要什么信息”，Encoder 输出表达各源位置的内容；注意力权重形成软对齐并加权得到 context。

8. 对源序列进行 Attention 时为什么必须屏蔽源 PAD？

A. PAD 只能出现在目标侧
B. 否则无效补齐位置会分走概率质量并污染上下文向量
C. 可让隐藏维度翻倍
D. 是为了产生 Causal Mask

✅ 参考答案

答案：B。 应在 Softmax 前将 PAD 位置分数设为极小值，使其权重为 0；Mask 的形状通常为 [B,S]，再扩展到注意力分数形状。

9. 屏蔽 PAD 后，一条含有效源 token 的样本，其 Attention 权重应满足？

A. 每个位置都等于 1
B. PAD 权重为 1
C. 有效位置权重和为 1，PAD 位置权重为 0
D. 所有权重和为隐藏维度 H

✅ 参考答案

答案：C。 Softmax 在有效源位置上形成概率分布；测试时应显式断言 PAD 权重近似为 0，防止 Mask 轴或布尔语义写反。

10. RNN Decoder 在推理第一步通常接收什么？

A. 目标 <SOS>
B. 完整参考译文
C. 源句最后一个词
D. 目标 <PAD>

✅ 参考答案

答案：A。 <SOS> 定义生成起点；第一步 logits 用于预测第一个真实目标词，后续再把模型自己的预测反馈为输入。

11. Teacher Forcing 比例为 1 时，第 $t$ 步 Decoder 的下一步输入来自？

A. 模型在当前步的 Argmax 预测
B. 真实目标序列中对应的 token
C. 随机源 token
D. 验证集译文

✅ 参考答案

答案：B。 它缩短了误差传播路径并让训练更稳定，但模型训练时很少见到自己的错误前缀，会造成训练—推理分布不一致。

12. “曝光偏差”指的是？

A. 图片亮度变化
B. 训练常基于真实历史 token，推理却只能基于模型自身历史预测
C. 源语言和目标语言词表不同
D. 验证集太大

✅ 参考答案

答案：B。 推理中早期错误会成为下一步输入并持续累积；Scheduled Sampling、降低 Teacher Forcing 比例和序列级训练可部分缓解，但都需谨慎验证。

13. 验证 RNN 翻译系统时，最完整的做法是？

A. 只报告训练集 Teacher-Forced token accuracy
B. 只观察最后一个 batch 的 loss
C. 同时记录 Teacher-Forced NLL，并用真实自回归解码计算 BLEU/chrF 等生成指标
D. 使用参考译文作为推理输入后报告 BLEU

✅ 参考答案

答案：C。 NLL 适合稳定选模和定位概率建模问题；自回归指标才能暴露错误累积、过早 EOS、重复和解码策略问题。

14. 自回归解码的正确循环是？

A. 每一步都输入完整参考答案
B. 从 SOS 开始，预测一个 token，将其作为下一步输入，遇 EOS 或长度上限停止
C. 只运行 Encoder
D. 一次 Argmax 得到任意长度序列

✅ 参考答案

答案：B。 RNN Decoder 天然逐步生成；长度上限是防止模型始终不输出 EOS 的安全边界。

15. batch 推理中部分样本已生成 EOS，而其他样本尚未结束，稳妥的处理是？

A. 立即结束整个 batch
B. 删除尚未完成的样本
C. 维护逐样本 finished Mask，冻结已完成样本，直到全部完成或达到上限
D. 把 EOS 改成普通词

✅ 参考答案

答案：C。 只检查“当前步是否所有样本同时预测 EOS”会让已完成样本继续产生垃圾 token；每条序列应独立记录终止状态。

16. 目标词表大小为 V，RNN Decoder 运行 T 个预测步后，堆叠 logits 的常见形状是？

A. [B,T,V]
B. [T,V]
C. [B,V,T,V]
D. [B,S,H]

✅ 参考答案

答案：A。 每个样本、每个目标时间步都有一个 V 类未归一化分数向量；交叉熵前通常 reshape 为 [-1,V]。

17. 训练损失忽略 PAD 的标准做法是？

A. CrossEntropyLoss(ignore_index=pad_id)
B. 把 PAD 标签替换为最高频词
C. 对 logits 先 Argmax 再算损失
D. 删除所有变长样本

✅ 参考答案

答案：A。 交叉熵直接接收 logits 和类别 ID；忽略 PAD 可避免句子较短的样本因补齐数量不同而改变优化目标。

18. 跨 batch 汇总 token loss/accuracy 时，为什么不能简单平均各 batch 指标？

A. batch 指标不能是浮点数
B. 每个 batch 的有效非 PAD token 数可能不同
C. GRU 只支持一个 batch
D. 平均值一定大于 1

✅ 参考答案

答案：B。 应累计 loss_sum、正确 token 数和有效 token 数，最后统一相除；否则短 batch 与长 batch 被赋予相同权重。

19. RNN 翻译训练中使用 clip_grad_norm_ 的主要目的是什么？

A. 增大词表
B. 限制梯度范数，缓解长序列反向传播中的梯度爆炸
C. 自动选择最大长度
D. 替代优化器

✅ 参考答案

答案：B。 正确顺序是 zero_grad → forward → backward → clip → optimizer.step；还应记录裁剪前梯度范数，判断是否频繁触发。

20. 使用 pack_padded_sequence 的直接价值是？

A. 让 GRU 跳过右侧 PAD，减少无效计算并得到正确的最终有效状态
B. 自动计算 BLEU
C. 自动添加 SOS/EOS
D. 允许标签泄漏

✅ 参考答案

答案：A。 传入的 lengths 必须是每条序列的有效长度且通常位于 CPU；batch 未按长度排序时设置 enforce_sorted=False。

21. 若源序列为 [w1,w2,EOS,PAD,PAD]，用于打包的长度应是？

A. 1
B. 2
C. 3
D. 5

✅ 参考答案

答案：C。 有效长度包含 EOS，但不包含 PAD；是否在源侧加入 SOS 是设计选择，只要数据处理与长度定义保持一致。

22. 两层双向 GRU 的最终隐藏状态形状是？

A. [2,B,H]
B. [4,B,H]
C. [B,2H]
D. [B,S,4H]

✅ 参考答案

答案：B。 第一轴按“层 × 方向”排列。送入两层单向 Decoder 前，通常按层拼接正反向状态为 [2,B,2H]，再投影到 Decoder 隐藏维度。

23. Encoder 隐藏维度与 Decoder 隐藏维度不同，应如何初始化 Decoder？

A. 直接相加并依赖广播
B. 使用可学习的线性投影或专门的 bridge 网络对齐维度
C. 丢弃全部 Encoder 状态
D. 用 PAD ID 替代隐藏状态

✅ 参考答案

答案：B。 Bridge 还可处理层数和方向数不一致；不能只看元素总数后随意 reshape，因为轴语义必须保持正确。

24. nn.GRU(..., dropout=0.3, num_layers=1) 的层间 Dropout 实际效果是？

A. 对单层 GRU 内部循环自动生效
B. 只对最后输出生效
C. 层间 Dropout 没有可作用的层间连接，通常不会生效
D. 会删除 30% 的训练样本

✅ 参考答案

答案：C。 PyTorch RNN 模块的 dropout 作用于多层之间；单层模型若需正则化，应显式在 Embedding 或输出等位置加入 Dropout。

25. Attention 相比只传递 Encoder 最终隐藏状态，最关键的改进是？

A. Decoder 每一步可按需聚合所有源位置的信息
B. 不再需要目标词表
C. 可以删除损失函数
D. 推理完全并行

✅ 参考答案

答案：A。 不同目标词可关注不同源位置，从而形成软对齐并缓解固定向量瓶颈；RNN Decoder 本身仍需逐步推理。

26. 相比贪心解码，Beam Search 的主要特点是？

A. 每一步只保留一个最高概率 token
B. 同时保留若干高分前缀，降低局部最优风险但增加计算量
C. 必须使用参考译文
D. 不需要 EOS

✅ 参考答案

答案：B。 Beam 累加序列 log probability；实际还需处理完成候选、长度归一化、重复约束及 batch 化效率。

33. 写出缩放点积 Attention 与 Padding Mask 的计算过程。

✅ 参考答案

对每个解码步，$Q=W_qh_t$，$K=W_kH_{enc}$，$V=W_vH_{enc}$；分数为 $s=QK^T/\sqrt{H}$。在 Softmax 前将源 PAD 位置替换为极小值，$\alpha=\operatorname{softmax}(s)$，最后 $c_t=\alpha V$。应检查有效行权重和为 1、PAD 权重为 0。

49. 实现带源 Padding Mask 的缩放点积 Attention，并验证 PAD 权重为零。

✅ 参考答案

Mask 在 Softmax 前应用；Softmax 后再次清零并归一化，可安全覆盖极端的全 PAD 行。

Python
import math
import torch

def masked_attention(query, keys, values, pad_mask):
    scores = torch.bmm(query.unsqueeze(1), keys.transpose(1, 2)).squeeze(1)
    scores = scores / math.sqrt(keys.size(-1))
    scores = scores.masked_fill(pad_mask, torch.finfo(scores.dtype).min)
    weights = torch.softmax(scores, dim=-1)
    weights = weights.masked_fill(pad_mask, 0.0)
    weights = weights / weights.sum(dim=-1, keepdim=True).clamp_min(1e-9)
    context = torch.bmm(weights.unsqueeze(1), values).squeeze(1)
    return context, weights

query = torch.tensor([[1., 0.], [0., 1.]])
keys = torch.tensor([[[1., 0.], [0., 1.], [9., 9.]],
                     [[1., 0.], [0., 1.], [1., 1.]]])
values = keys.clone()
pad_mask = torch.tensor([[False, False, True], [False, False, False]])
context, weights = masked_attention(query, keys, values, pad_mask)

assert weights.shape == (2, 3) and context.shape == (2, 2)
assert weights[0, 2].item() == 0.0
assert torch.allclose(weights.sum(dim=-1), torch.ones(2))
print(context, weights, sep="\n")

def bridge_bidirectional(hidden, num_layers, projection):
    directions_times_layers, batch, hidden_size = hidden.shape
    assert directions_times_layers == num_layers * 2

52. 实现一个 GRU + Attention Decoder 单步，并检查形状、PAD 权重和梯度。

✅ 参考答案

一步 Decoder 将当前目标词 Embedding 与 Attention context 拼接，再更新循环状态并输出词表 logits。

Python
import math
import torch
from torch import nn

torch.manual_seed(1)
PAD = 0

class AttentionDecoderStep(nn.Module):
    def __init__(self, vocab_size, embed_size, hidden_size):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_size, padding_idx=PAD)
        self.gru = nn.GRU(embed_size + hidden_size, hidden_size, batch_first=True)
        self.output = nn.Linear(hidden_size * 2, vocab_size)

    def forward(self, token, hidden, encoder_output, source_pad_mask):
        query = hidden[-1]
        scores = torch.bmm(query.unsqueeze(1), encoder_output.transpose(1, 2))
        scores = scores.squeeze(1) / math.sqrt(query.size(-1))
        scores = scores.masked_fill(source_pad_mask, -1e9)
        weights = torch.softmax(scores, dim=-1)
        context = torch.bmm(weights.unsqueeze(1), encoder_output).squeeze(1)
        step_input = torch.cat([self.embedding(token), context], dim=-1).unsqueeze(1)
        step_output, next_hidden = self.gru(step_input, hidden)
        logits = self.output(torch.cat([step_output.squeeze(1), context], dim=-1))
        return logits, next_hidden, weights

batch, source_length, hidden_size = 2, 4, 6
decoder = AttentionDecoderStep(vocab_size=11, embed_size=5, hidden_size=hidden_size)
token = torch.tensor([1, 1])
hidden = torch.randn(1, batch, hidden_size, requires_grad=True)
encoder_output = torch.randn(batch, source_length, hidden_size, requires_grad=True)
source_pad_mask = torch.tensor([[False, False, False, True],
                                [False, False, True, True]])
logits, next_hidden, weights = decoder(token, hidden, encoder_output, source_pad_mask)
logits.sum().backward()

assert logits.shape == (batch, 11)
assert next_hidden.shape == (1, batch, hidden_size)
assert torch.equal(weights.masked_select(source_pad_mask), torch.zeros(3))
assert hidden.grad is not None and encoder_output.grad is not None
print(logits.shape, weights)

微型过拟合是端到端单元测试；若它失败，应优先排查目标移位、长度、Mask、隐藏状态、loss 和解码反馈，而不是扩大数据。

import math
import torch
from torch import nn
from torch.nn.utils.rnn import pack_padded_sequence, pad_packed_sequence

torch.manual_seed(7)
torch.set_num_threads(1)
PAD, SOS, EOS, VOCAB_SIZE = 0, 1, 2, 11

class TinyGRUTranslator(nn.Module):
    def __init__(self, embed_size=12, hidden_size=16):
        super().__init__()
        self.source_embedding = nn.Embedding(VOCAB_SIZE, embed_size, padding_idx=PAD)
        self.target_embedding = nn.Embedding(VOCAB_SIZE, embed_size, padding_idx=PAD)
        self.encoder = nn.GRU(embed_size, hidden_size, batch_first=True)
        self.decoder = nn.GRU(embed_size + hidden_size, hidden_size, batch_first=True)
        self.output = nn.Linear(hidden_size * 2, VOCAB_SIZE)

    def encode(self, source):
        lengths = source.ne(PAD).sum(dim=1)
        embedded = self.source_embedding(source)
        packed = pack_padded_sequence(
            embedded, lengths.cpu(), batch_first=True, enforce_sorted=False
        )
        packed_output, hidden = self.encoder(packed)
        encoder_output, _ = pad_packed_sequence(
            packed_output, batch_first=True, total_length=source.size(1)
        )
        return encoder_output, hidden, source.eq(PAD)

    def decode_step(self, token, hidden, encoder_output, source_pad_mask):
        query = hidden[-1]
        scores = torch.bmm(query.unsqueeze(1), encoder_output.transpose(1, 2))
        scores = scores.squeeze(1) / math.sqrt(query.size(-1))
        weights = torch.softmax(scores.masked_fill(source_pad_mask, -1e9), dim=-1)
        context = torch.bmm(weights.unsqueeze(1), encoder_output).squeeze(1)
        decoder_input = torch.cat([self.target_embedding(token), context], dim=-1)
        decoder_output, hidden = self.decoder(decoder_input.unsqueeze(1), hidden)
        logits = self.output(torch.cat([decoder_output.squeeze(1), context], dim=-1))
        return logits, hidden

    def forward(self, source, target_input):
        encoder_output, hidden, source_pad_mask = self.encode(source)
        logits = []
        for step in range(target_input.size(1)):
            step_logits, hidden = self.decode_step(
                target_input[:, step], hidden, encoder_output, source_pad_mask
            )
            logits.append(step_logits)
        return torch.stack(logits, dim=1)

    @torch.no_grad()
    def greedy(self, source, max_new_tokens=3):
        encoder_output, hidden, source_pad_mask = self.encode(source)
        token = torch.full((source.size(0),), SOS, dtype=torch.long)
        generated = []
        finished = torch.zeros(source.size(0), dtype=torch.bool)
        for _ in range(max_new_tokens):
            logits, hidden = self.decode_step(token, hidden, encoder_output, source_pad_mask)
            token = logits.argmax(dim=-1)
            generated.append(token)
            finished |= token.eq(EOS)
            if finished.all():
                break
        return torch.stack(generated, dim=1)

source = torch.tensor([
    [3, 4, EOS, PAD], [3, 5, EOS, PAD],
    [6, 4, EOS, PAD], [6, 5, EOS, PAD],
])
full_target = torch.tensor([
    [SOS, 7, 8, EOS], [SOS, 7, 9, EOS],
    [SOS, 10, 8, EOS], [SOS, 10, 9, EOS],
])
target_input, labels = full_target[:, :-1], full_target[:, 1:]

model = TinyGRUTranslator()
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)
criterion = nn.CrossEntropyLoss(ignore_index=PAD)

for _ in range(200):
    optimizer.zero_grad()
    logits = model(source, target_input)
    loss = criterion(logits.reshape(-1, VOCAB_SIZE), labels.reshape(-1))
    loss.backward()
    nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()

model.eval()
with torch.no_grad():
    teacher_predictions = model(source, target_input).argmax(dim=-1)
    greedy_predictions = model.greedy(source)

assert torch.equal(teacher_predictions, labels)
assert torch.equal(greedy_predictions, labels)
assert loss.item() < 0.01
print({"loss": round(loss.item(), 6), "greedy_exact": True})

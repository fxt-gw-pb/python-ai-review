🎯 注意力机制：Q、K、V 与缩放点积综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：注意力机制：Q、K、V 与缩放点积

55 道练习，按“固定上下文瓶颈 → Q/K/V → 缩放点积 → Self/Cross Attention → Padding/Causal Mask → 多头与稳定实现”组织。重点是能从公式推导张量形状、正确放置 Softmax 与 Mask，并用断言验证实现。

建议先独立作答，再展开参考答案。形状题统一使用 $B$ 表示批量大小、$T_q/T_k$ 表示 Query/Key 序列长度、$d_k/d_v$ 表示 Key/Value 维度。

一、选择题（30 题）

1. 早期无注意力 Encoder–Decoder 使用单一固定上下文向量，主要瓶颈是？

A. 不能使用 GPU
B. 整个源序列的信息被压进一个固定表示，长序列细节容易丢失
C. 输出必须与输入等长
D. 不能计算损失

✅ 参考答案

答案：B。 所有解码时间步共享同一个固定上下文，既形成信息瓶颈，也无法针对当前输出动态选择不同源位置。

2. 注意力中的 Query 最准确的含义是？

A. 被加权汇总的内容
B. 所有候选位置的标签
C. 当前要检索什么信息的查询表示
D. 最终分类概率

✅ 参考答案

答案：C。 Query 表示当前需求；它与各 Key 计算匹配分数，再据此从对应 Value 中汇总内容。

3. Key 的主要作用是？

A. 提供候选位置用于与 Query 匹配的索引特征
B. 直接作为损失函数
C. 保存优化器状态
D. 表示 padding 长度

✅ 参考答案

答案：A。 Key 用于判断“哪里与当前 Query 相关”；Value 则提供被取回的具体内容。二者通常一一对应，但含义不同。

4. Value 的主要作用是？

A. 只计算相似度
B. 生成 Mask
C. 指定 Softmax 轴
D. 作为按注意力权重加权求和的内容

✅ 参考答案

答案：D。 权重由 Q 与 K 的匹配决定，最终输出是对 V 的加权组合，而不是对 K 的必然加权组合。

5. $Q\in\mathbb{R}^{B\times T_q\times d_k}$、$K\in\mathbb{R}^{B\times T_k\times d_k}$，则 $QK^T$ 形状是？

A. [B,d_k,d_k]
B. [B,T_q,T_k]
C. [B,T_k,T_q]
D. [T_q,T_k]

✅ 参考答案

答案：B。 对每个 batch，$Q$ 的每个查询位置都与 $K$ 的每个候选位置打分，因此形成 $T_q\times T_k$ 的分数矩阵。

6. 对分数张量 [B,T_q,T_k] 做标准注意力 Softmax，应沿哪个轴？

A. batch 轴
B. Query 轴
C. 最后的 Key 位置轴
D. 所有轴一起

✅ 参考答案

答案：C。 每个 Query 都要在所有可见 Key 位置上形成一条概率分布，因此通常使用 softmax(scores, dim=-1)。

7. 未出现全 Mask 行时，每个 Query 对 Key 的注意力权重之和应为？

A. 1
B. $d_k$
C. $T_q$
D. 不受约束

✅ 参考答案

答案：A。 Softmax 让每行非负且和为 1，因此输出是 Value 的凸组合；浮点计算中允许极小误差。

8. 若 $V\in\mathbb{R}^{B\times T_k\times d_v}$，注意力输出形状是？

A. [B,T_k,d_k]
B. [B,d_v,T_q]
C. [B,T_q,T_k]
D. [B,T_q,d_v]

✅ 参考答案

答案：D。 权重 [B,T_q,T_k] 乘 V [B,T_k,d_v]，消去 Key 序列轴，保留每个 Query 的 $d_v$ 维上下文表示。

9. 缩放点积注意力对 Q、K、V 的维度要求，哪项正确？

A. 三者最后一维必须完全相同
B. Q 与 K 的最后一维必须同为 $d_k$，V 可使用不同的 $d_v$
C. Q 与 V 的序列长度必须相同
D. K 与 V 的最后一维必须相同

✅ 参考答案

答案：B。 Q 与 K 要做点积，特征维必须一致；K 与 V 需要共享候选位置数 $T_k$，但内容维 $d_v$ 可与 $d_k$ 不同。

10. Transformer 的缩放点积注意力使用哪个公式？

A. $\operatorname{softmax}(Q+K)V$
B. $\operatorname{softmax}(QV^T)K$
C. $\operatorname{softmax}(QK^T/\sqrt{d_k})V$
D. $\operatorname{argmax}(QK^T)V$

✅ 参考答案

答案：C。 顺序是打分、按 $\sqrt{d_k}$ 缩放、在 Key 轴归一化、对 V 加权求和；Mask 应插在 Softmax 之前。

11. 若 Q、K 各维独立、零均值、单位方差，未缩放点积的方差约为？

A. $d_k$
B. $1/d_k$
C. $\sqrt{d_k}$
D. 0

✅ 参考答案

答案：A。 点积是 $d_k$ 个乘积项之和，在简化独立假设下方差随 $d_k$ 线性增长。

12. 除以 $\sqrt{d_k}$ 的主要效果是？

A. 让参数量减半
B. 保证注意力恒为均匀分布
C. 删除负分数
D. 让分数方差大致与维度无关，减轻 Softmax 过度饱和

✅ 参考答案

答案：D。 缩放后方差约从 $d_k$ 降到 1，使 Softmax 不易过早接近 one-hot，从而保留更健康的梯度。

13. Self-Attention 的 Q、K、V 来源是？

A. 分别来自三个无关数据集
B. 来自同一输入序列的不同线性投影
C. Q 来自标签，K/V 来自输入
D. 都必须是相同张量且不能投影

✅ 参考答案

答案：B。 “同源”不等于数值相同；通常用不同的 $W_Q,W_K,W_V$ 把同一序列投影成不同角色表示。

14. Encoder–Decoder Cross-Attention 中常见的来源关系是？

A. Q/K/V 都来自编码器
B. Q/K/V 都来自解码器
C. Q 来自解码器，K/V 来自编码器输出
D. Q 来自编码器，K/V 来自标签

✅ 参考答案

答案：C。 解码器当前状态提出查询，从编码器保存的源序列表示中按相关性取回信息。

15. 加法注意力与点积注意力的核心差异是？

A. 加法注意力用可学习前馈网络打分，点积注意力用向量内积高效矩阵化
B. 加法注意力没有 Softmax
C. 点积注意力不能反向传播
D. 两者都没有参数

✅ 参考答案

答案：A。 二者都遵循“打分 → 归一化 → 加权 V”的统一框架，主要差别在 score 函数的形式与计算效率。

16. 软注意力（Soft Attention）便于端到端训练，主要因为？

A. 每次只随机选一个位置
B. 权重只能取 0 或 1
C. 不需要损失函数
D. 连续权重与加权求和通常可微

✅ 参考答案

答案：D。 连续 Softmax 权重允许梯度流向分数函数和输入表示；硬选择通常需要采样、近似或强化学习方法。

17. 硬注意力的典型困难是？

A. 不能选择任何位置
B. 离散选择不可直接用普通反向传播穿过
C. 一定比软注意力慢
D. 不能处理图像

✅ 参考答案

答案：B。 Argmax 或离散采样对选择本身不可微，训练通常更复杂，因此现代通用架构更常使用软注意力。

18. Padding Mask 在注意力中通常屏蔽什么？

A. 所有 Query
B. 所有真实 token
C. 不应被任何 Query 关注的 PAD Key 位置
D. 模型参数

✅ 参考答案

答案：C。 最基本的 Key padding mask 让各 Query 不给 PAD Key 分配权重；若损失也覆盖序列输出，还需另外屏蔽 PAD Query 或 PAD 标签。

19. Mask 为什么应在 Softmax 之前作用于分数？

A. 这样被屏蔽位置不会进入归一化分母并可得到零权重
B. Softmax 后不能做任何乘法
C. 可以减少 Q 的维度
D. 能自动生成标签

✅ 参考答案

答案：A。 若先 Softmax 再把某些权重置零，剩余权重之和会小于 1；除非再次归一化，否则已不再是正确的条件分布。

20. 对需要完全屏蔽的位置，Softmax 前常把 score 填成？

A. 0
B. 1
C. 正无穷
D. 负无穷或该数据类型足够小的有限值

✅ 参考答案

答案：D。 Softmax 后其指数权重趋近 0。使用负无穷时必须避免整行全部被屏蔽，否则可能产生 NaN。

21. Causal Mask 的作用是？

A. 屏蔽所有过去位置
B. 防止位置 $t$ 关注未来位置 $>t$
C. 删除所有 PAD
D. 限制 batch size

✅ 参考答案

答案：B。 自回归训练可并行输入整段目标序列，但通过上三角 Mask 保证每个位置只读取自己和过去。

22. 关于 Causal Mask 与 Padding Mask，哪项正确？

A. 两者完全相同
B. 有 Causal Mask 就不需要处理 PAD
C. 两者约束不同，实际模型中经常需要组合
D. Padding Mask 只用于图像

✅ 参考答案

答案：C。 Causal Mask 约束时间因果，Padding Mask 约束无效补齐位置；解码器自注意力通常同时需要二者。

23. d_model=512、头数 h=8，均匀拆分时每头维度是多少？

A. 64
B. 8
C. 512
D. 4096

✅ 参考答案

答案：A。 常见设置为 $d_{head}=d_{model}/h=64$，每个头独立计算注意力，再拼接回模型维度。

24. 多头注意力通常如何合并各头输出？

A. 只保留第一头
B. 对所有头做 Argmax
C. 沿序列轴拼接
D. 沿特征轴拼接后再经输出线性投影

✅ 参考答案

答案：D。 各头输出 [B,T,d_head] 在最后一维拼为 [B,T,h*d_head]，再由 $W_O$ 融合不同子空间的信息。

25. 在 Self-Attention 中，同一 token 对另一个 token 的关注权重通常是否对称？

A. 必然对称
B. 不必对称，因为 Query 与 Key 投影及行归一化角色不同
C. 只要长度相同就对称
D. 只有 Padding 时才对称

✅ 参考答案

答案：B。 即便原始输入相同，$W_Q$ 与 $W_K$ 通常不同；Softmax 还按每个 Query 的行分别归一化，所以 $\alpha_{ij}$ 不必等于 $\alpha_{ji}$。

26. 将注意力权重直接当作模型决策的完整解释，最准确的评价是？

A. 权重天然等于因果贡献
B. 权重越大一定越重要
C. 权重可作为诊断线索，但不自动等于因果解释，需要扰动或归因方法交叉验证
D. 权重完全没有任何信息

✅ 参考答案

答案：C。 Value 变换、残差、多层传播和后续非线性都会影响最终决策；可视化权重有价值，但不能过度解读。

27. 若不加入位置编码或其他位置信息，Self-Attention 本身的局限是？

A. 对输入排列具有置换等变性，不能仅凭内容区分绝对或相对顺序
B. 不能计算点积
C. 不能使用 Mask
D. 输出一定全零

✅ 参考答案

答案：A。 Self-Attention 能建立内容间关系，但顺序信息需要位置编码、相对位置偏置或结构化 Mask 等机制注入。

28. 标准全局 Self-Attention 对序列长度 $T$ 的主要内存/计算瓶颈是？

A. $O(1)$
B. $O(\log T)$
C. $O(T)$
D. 注意力分数矩阵通常为 $O(T^2)$

✅ 参考答案

答案：D。 每个 Query 与每个 Key 配对产生 $T\times T$ 分数；长序列可考虑局部、稀疏、线性或分块注意力及 FlashAttention 等实现。

29. 数值稳定 Softmax 常在指数运算前做什么？

A. 除以序列长度
B. 每行减去该行最大值
C. 把所有负数改成 0
D. 对分数取 Argmax

✅ 参考答案

答案：B。 $\operatorname{softmax}(x)=\operatorname{softmax}(x-\max x)$，结果不变但可避免指数溢出；框架 Softmax 通常已采用稳定实现。

30. 验证一个自定义注意力实现，哪组测试最完整？

A. 只打印一次 loss
B. 只检查代码不报错
C. 同时检查形状、每行权重和、Mask 零权重、全 Mask 行、梯度有限性及与可信实现对齐
D. 只观察一张热力图

✅ 参考答案

答案：C。 注意力常见错误集中在转置、Softmax 轴、广播和 Mask 语义；多层断言比“能运行”更能发现静默错误。

二、简答题（15 题）

31. 注意力机制怎样缓解固定上下文 Encoder–Decoder 的信息瓶颈？

✅ 参考答案

无注意力模型把整个源序列压缩为单一向量，所有解码时间步共用它。加入注意力后，编码器保留各源位置表示；每个解码时间步用当前状态作为 Query，与所有 Key 打分，再对 Value 动态加权得到专属 context。这样既缩短了输出与相关输入之间的信息路径，也允许不同输出词关注不同源位置，但不会自动解决数据质量或所有长序列问题。

32. 写出缩放点积注意力公式，并说明每一步的形状。

✅ 参考答案

$$ S=\frac{QK^T}{\sqrt{d_k}},\quad A=\operatorname{softmax}(S),\quad O=AV. $$

若 Q 为 [B,Tq,dk]、K 为 [B,Tk,dk]、V 为 [B,Tk,dv]，则分数 S 和权重 A 都是 [B,Tq,Tk]，输出 O 是 [B,Tq,dv]。Softmax 沿最后的 Key 轴进行；Mask 在 Softmax 前加入 S。

33. 用“数据库检索”类比解释 Q、K、V，并指出这个类比的边界。

✅ 参考答案

Q 类似查询条件，K 类似每条记录用于匹配的索引特征，V 类似匹配后真正取回的内容；注意力根据 Q–K 匹配程度对 V 做软汇总。边界在于神经注意力通常返回多个 Value 的连续加权组合，而不是数据库中的精确离散行；Q/K/V 还是端到端学习的向量表示，不一定具有人工命名字段。

34. 推导为什么缩放因子是 $\sqrt{d_k}$ 而不是 $d_k$。

✅ 参考答案

若 $q_i,k_i$ 独立、零均值、单位方差，则点积 $s=\sum_{i=1}^{d_k}q_ik_i$ 的方差约为 $d_k$，标准差约为 $\sqrt{d_k}$。将 s 除以 $\sqrt{d_k}$ 后，方差变为 $d_k/d_k=1$。若除以 $d_k$，方差会降为 $1/d_k$，可能把分数压得过平。这个推导基于简化分布假设，但给出了稳定尺度的合理依据。

35. Softmax 为什么必须沿 Key 位置轴，而不是 Query 轴？

✅ 参考答案

注意力要回答“对当前这个 Query，应从哪些 Key/Value 位置取信息”。因此固定一个 Query 后，需要在全部 Key 上形成和为 1 的权重分布。若沿 Query 轴归一化，含义会变成“一个 Key 在不同 Query 之间如何分配”，无法保证每个 Query 的上下文是 Value 的规范化加权组合。

36. Padding Mask 常见的形状与广播方式是什么？

✅ 参考答案

若有效 Key 标记为 [B,Tk]，单头分数为 [B,Tq,Tk]，可扩展成 [B,1,Tk] 沿 Query 轴广播；多头分数 [B,H,Tq,Tk] 则常扩展为 [B,1,1,Tk]，同时沿头和 Query 广播。布尔语义必须明确是 True=保留 还是 True=屏蔽，不同 API 可能相反，需用小例子验证。

37. 为什么“整行全部被 Mask”可能产生 NaN？应如何处理？

✅ 参考答案

若整行分数都填为 $-\infty$，稳定 Softmax 中会出现 $-\infty-(-\infty)$，结果为 NaN。应从数据或 Mask 设计上保证每个有效 Query 至少能看到一个 Key；对于确实可能出现的全 Mask 行，可实现安全 masked softmax：只对有效项指数化，分母为 0 时返回全零，并在后续同样屏蔽该 Query。不能让 NaN 静默进入后续层。

38. 比较 Self-Attention 与 Cross-Attention 的来源、形状和用途。

✅ 参考答案

Self-Attention 的 Q/K/V 来自同一序列的不同投影，常见 Tq=Tk，用于建模序列内部关系。Cross-Attention 的 Q 来自目标或查询序列，K/V 来自源序列，Tq 与 Tk 可不同，用于把解码状态、文本查询或图像查询对齐到另一模态/序列。两者都可使用缩放点积、多头和 Mask。

39. 比较加法注意力与点积注意力的打分函数及工程取舍。

✅ 参考答案

点积注意力以 $QK^T$ 或带线性变换的点积打分，矩阵乘法效率高，特别适合 GPU 并行；加法注意力常用 $v^T\tanh(W_qQ+W_kK)$，由前馈网络学习对齐，能自然处理原始维度不同的 Q/K，但计算更复杂。二者都需 Softmax 和 Value 加权，实际选择应结合模型架构、维度与性能测试。

40. 多头注意力为什么可能比单头更有表达力？

✅ 参考答案

不同头拥有独立的 Q/K/V 投影，可在不同子空间学习局部搭配、长距离依赖或不同语义关系；拼接与输出投影再融合这些信息。多头不保证自动形成可解释分工，头数过多会让单头维度过小并增加调度开销，应在保证 d_model 可整除头数的前提下验证。

41. 注意力热力图可以说明什么，不能说明什么？

✅ 参考答案

它能显示某层某头在给定输入下的权重流向，适合发现是否关注 PAD、是否符合 causal 约束以及可能的对齐模式。但权重不是最终输出的完整贡献，更不是因果证据；Value 内容、残差连接、后续层和非线性都会改变影响。应结合遮挡、反事实扰动、梯度/归因方法和任务指标共同判断。

42. 训练时已经输入完整目标序列，Causal Mask 如何防止标签泄漏？

✅ 参考答案

构造下三角可见矩阵，使位置 t 只能关注 <=t 的位置，未来位置分数在 Softmax 前被屏蔽。训练标签通常再右移一位：位置 t 的表示用于预测下一个 token。这样可并行计算所有时间步，却仍满足自回归条件。仅使用 causal mask 还不够，输入/标签移位和 padding loss mask 也必须正确。

43. 为什么 Self-Attention 仍需要位置编码？

✅ 参考答案

若只对内容向量做同样的 Q/K/V 投影，输入 token 的排列与输出排列同步变化，机制本身不知道“第几个”或相对距离。位置编码、相对位置偏置、旋转位置编码或结构化 Mask 为打分注入顺序与距离信息，使模型能够区分相同词集合的不同排列。

44. 标准全局注意力为什么在长序列上昂贵？可采用哪些思路？

✅ 参考答案

长度 T 的 Self-Attention 显式或隐式处理 $T\times T$ 关系，分数矩阵内存通常为 $O(T^2)$，计算也随配对数增长。可使用局部窗口、稀疏/块状模式、低秩或线性注意力、分层检索，以及 FlashAttention 这类避免物化完整中间矩阵的精确 IO 优化。不同方案对精度、硬件和因果 Mask 的支持不同，需实测。

45. 为自定义注意力模块设计一份最小测试清单。

✅ 参考答案

至少包含：不同 Tq/Tk/dv 的输出形状；权重非负且有效行和为 1；Padding/Causal Mask 的被屏蔽权重为 0；全 Mask 行无 NaN 且行为有定义；batch 与多头广播正确；固定小矩阵手算结果；与可信框架实现对齐；反向传播后 Q/K/V 梯度形状正确且有限；float16/bfloat16 下也无溢出。还应测试错误维度能及时报出清晰异常。

46. 手写支持 $T_q\ne T_k$、$d_k\ne d_v$ 的缩放点积注意力，并验证形状与权重和。

✅ 参考答案

下面约定布尔 Mask 中 True 表示允许关注，并要求每个 Query 至少有一个可见 Key。

Python
import math
import torch

def scaled_dot_product_attention(query, key, value, mask=None):
    if query.size(-1) != key.size(-1):
        raise ValueError("query/key feature dimensions must match")
    if key.size(-2) != value.size(-2):
        raise ValueError("key/value sequence lengths must match")
    scores = query @ key.transpose(-2, -1) / math.sqrt(query.size(-1))
    if mask is not None:
        scores = scores.masked_fill(~mask, float("-inf"))
    weights = torch.softmax(scores, dim=-1)
    return weights @ value, weights

torch.manual_seed(0)
Q = torch.randn(2, 3, 5)
K = torch.randn(2, 4, 5)
V = torch.randn(2, 4, 7)
output, weights = scaled_dot_product_attention(Q, K, V)

assert output.shape == (2, 3, 7)
assert weights.shape == (2, 3, 4)
assert torch.allclose(weights.sum(dim=-1), torch.ones(2, 3))
print(output.shape, weights.shape)

51. 实现能安全处理“整行全部被 Mask”的 masked softmax。

✅ 参考答案

使用有限最小值和显式重新归一化；分母为 0 时返回全零，避免 NaN。

Python
import torch

def safe_masked_softmax(scores, mask, dim=-1):
    mask = mask.expand_as(scores)
    fill_value = torch.finfo(scores.dtype).min
    masked_scores = scores.masked_fill(~mask, fill_value)
    row_max = masked_scores.max(dim=dim, keepdim=True).values
    exponentials = torch.exp(masked_scores - row_max) * mask.to(scores.dtype)
    denominator = exponentials.sum(dim=dim, keepdim=True)
    normalized = exponentials / denominator.clamp_min(torch.finfo(scores.dtype).tiny)
    return torch.where(denominator > 0, normalized, torch.zeros_like(normalized))

scores = torch.tensor([[2.0, 1.0, -1.0], [3.0, 4.0, 5.0]])
mask = torch.tensor([[True, False, True], [False, False, False]])
weights = safe_masked_softmax(scores, mask)

assert torch.isfinite(weights).all()
assert torch.allclose(weights[0].sum(), torch.tensor(1.0))
assert weights[0, 1] == 0
assert torch.equal(weights[1], torch.zeros(3))
print(weights)

53. 编写多头张量的拆分与合并函数，并证明它们互为逆操作。

✅ 参考答案

拆分后形状从 [B,T,d_model] 变为 [B,H,T,d_head]，注意 transpose 后合并前需 contiguous()。

Python
import torch

def split_heads(x, num_heads):
    batch, length, model_dim = x.shape
    if model_dim % num_heads != 0:
        raise ValueError("model_dim must be divisible by num_heads")
    head_dim = model_dim // num_heads
    return x.reshape(batch, length, num_heads, head_dim).transpose(1, 2)

def merge_heads(x):
    batch, num_heads, length, head_dim = x.shape
    return x.transpose(1, 2).contiguous().reshape(
        batch, length, num_heads * head_dim
    )

x = torch.arange(2 * 5 * 12, dtype=torch.float32).reshape(2, 5, 12)
heads = split_heads(x, num_heads=3)
restored = merge_heads(heads)

assert heads.shape == (2, 3, 5, 4)
assert restored.shape == x.shape
assert torch.equal(restored, x)
print(heads.shape)

55. 在微型数据上训练一个带 Padding Mask 的注意力池化分类器。

✅ 参考答案

标签由序列中是否出现 token 9 决定，用于验证 Embedding、Mask、注意力池化、损失与优化器的完整闭环。

Python
import torch
from torch import nn

torch.manual_seed(6)

class AttentionPoolClassifier(nn.Module):
    def __init__(self, vocab_size=12, embed_dim=8):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.query = nn.Parameter(torch.randn(embed_dim))
        self.head = nn.Linear(embed_dim, 2)

    def forward(self, token_ids, return_weights=False):
        embedded = self.embedding(token_ids)
        scores = torch.einsum("btd,d->bt", embedded, self.query)
        valid = token_ids.ne(0)
        scores = scores.masked_fill(~valid, float("-inf"))
        weights = torch.softmax(scores, dim=-1)
        pooled = torch.einsum("bt,btd->bd", weights, embedded)
        logits = self.head(pooled)
        return (logits, weights) if return_weights else logits

token_ids = torch.tensor([
    [1, 2, 3, 0], [4, 5, 0, 0], [2, 6, 7, 0], [8, 1, 0, 0],
    [1, 9, 3, 0], [9, 5, 0, 0], [2, 6, 9, 7], [8, 9, 0, 0],
])
labels = torch.tensor([0, 0, 0, 0, 1, 1, 1, 1])
model = AttentionPoolClassifier()
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)
criterion = nn.CrossEntropyLoss()

model.train()
for _ in range(100):
    optimizer.zero_grad()
    loss = criterion(model(token_ids), labels)
    loss.backward()
    optimizer.step()

model.eval()
with torch.no_grad():
    logits, weights = model(token_ids, return_weights=True)
    accuracy = (logits.argmax(dim=-1) == labels).float().mean().item()

assert accuracy == 1.0
assert weights.masked_select(token_ids.eq(0)).eq(0).all()
assert torch.allclose(weights.sum(dim=-1), torch.ones(8))
print({"loss": round(loss.item(), 4), "accuracy": accuracy})

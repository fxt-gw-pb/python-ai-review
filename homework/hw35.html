🗣️ 自然语言处理入门与词向量综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：自然语言处理入门与词向量

55 道练习，按“任务边界 → 文本切分 → 语言学标注 → 数值化 → 分布式词向量 → 下游应用”的数据流组织。代码题尽量使用小语料与断言完成自校验，无需下载大型预训练模型。

建议先完成题目，再展开参考答案。对代码题，请同时观察输出形状、边界条件与失败案例。

一、选择题（30 题）

1. 下列哪一项最准确地描述了自然语言处理（NLP）？

A. 只把文本保存到数据库
B. 让计算机理解、生成并与人类语言交互
C. 只将语音转换成文字
D. 只统计每个字符出现的次数

✅ 参考答案

答案：B。 NLP 覆盖理解、生成和交互等多类语言任务；语音识别、文本存储或字符统计只是可能用到的局部环节。

2. 把“张三明天去上海出差”中的“张三”和“上海”分别标为人名与地名，属于哪项任务？

A. 机器翻译
B. 文本摘要
C. 命名实体识别
D. 语言模型

✅ 参考答案

答案：C。 命名实体识别（NER）的目标是找出人名、地名、机构名等实体，并标注实体类型。

3. 哪项现象最能说明系统具有语义稳健性？

A. 训练集上得分很高
B. 输入换成同义改写或带有少量错别字时，结论仍然合理
C. 每次都输出最长的答案
D. 能记住所有训练样本

✅ 参考答案

答案：B。 稳健性强调对改写、噪声和输入扰动的抵抗能力；记住训练样本反而可能是过拟合。

4. 对一批中文句子建立神经网络输入时，下列顺序最合理的是？

A. Embedding → 分词 → 词表索引 → 清洗
B. 清洗规范化 → 分词 → 词表索引 → Embedding
C. 词表索引 → 分词 → Embedding → 清洗
D. 分词 → 训练模型 → 再建词表

✅ 参考答案

答案：B。 文本需先清洗并切分为 token，再通过词表转成整数 ID，最后由 Embedding 查表得到稠密向量。

5. 中文分词通常比英文按词切分更难，主要原因是？

A. 中文没有语法
B. 中文的词之间通常没有显式空格边界
C. 中文只能按单字处理
D. 英文分词永远不会有歧义

✅ 参考答案

答案：B。 中文的词边界通常需要模型推断。D 也不成立：英文还有标点、缩写、复合词和子词切分等问题。

6. 为商品搜索建立倒排索引，希望长词能被进一步切分以提高召回率，更适合使用 jieba 的哪种模式？

A. 精确模式
B. 全模式
C. 搜索引擎模式
D. 随机模式

✅ 参考答案

答案：C。 搜索引擎模式会在精确分词的基础上对长词再切分，用更多候选 token 换取更高召回率。

7. 关于 jieba.cut(text) 与 jieba.lcut(text)，哪项正确？

A. 两者都返回字符串
B. cut 返回生成器，lcut 返回列表
C. cut 只支持英文
D. lcut 不能使用精确模式

✅ 参考答案

答案：B。 生成器适合惰性遍历；需要多次访问、索引或直接查看全部结果时，lcut 的列表更方便。

8. jieba 用户词典中一行的常用格式是？

A. 词语,词频,词性
B. 词语 词频 词性，其中词频和词性可省略
C. 词性:词语:词频，三项都不能省略
D. 只能写一个单字

✅ 参考答案

答案：B。 常见格式是空格分隔的 token frequency tag；用户词典适合补充领域术语和专有名词。

9. 词性标注（POS）的直接输出是什么？

A. 每个 token 的语法类别，如名词、动词、形容词
B. 每个句子的情感分数
C. 每个词的稠密向量
D. 整篇文档的摘要

✅ 参考答案

答案：A。 POS 以分词结果为基础，为 token 分配词性标签，为句法分析和语义理解提供语法线索。

10. 在 jieba 常用词性标记中，nr 通常表示？

A. 人名
B. 地名
C. 机构名
D. 时间

✅ 参考答案

答案：A。 常见的还有 ns 表示地名、nt 表示机构团体名、nz 表示其他专名。

11. 直接把 POS 结果中的 nr/ns/nt 词抽出当作 NER，最准确的评价是？

A. 这是完整且无误的 NER 方案
B. 这是一种可解释的简化基线，但可能错过多 token 实体和上下文歧义
C. 这与 NER 没有任何关系
D. 这一方法不需要先分词

✅ 参考答案

答案：B。 POS 规则可快速得到基线，但无法完整建模实体边界和语境，也容易被上游分词或词性错误传递。

12. 词表大小为 $V=50{,}000$ 时，每个 token 的 One-hot 向量维度是？

A. 1
B. 50
C. 50,000
D. 由句子长度决定

✅ 参考答案

答案：C。 One-hot 的维度等于词表大小，只有 token 对应索引处为 1，其余为 0，因此大词表下非常稀疏。

13. 在不考虑自身的情况下，任意两个不同 token 的 One-hot 向量余弦相似度是？

A. $-1$
B. $0$
C. $0.5$
D. 取决于它们在语料中的语义

✅ 参考答案

答案：B。 两个不同 One-hot 向量正交，点积为 0。这也说明 One-hot 本身不表达“猫”与“狗”比“猫”与“数据库”更相似。

14. 词表大小为 $V$、嵌入维度为 $D$ 时，nn.Embedding(V, D).weight 的形状是？

A. [D, V]
B. [V, D]
C. [V, V]
D. [D, D]

✅ 参考答案

答案：B。 词表中每个 token 对应一行 $D$ 维向量，所以参数矩阵是 $V\times D$。

15. 词表中“猫”的 ID 为 10、“狗”的 ID 为 11，正确解读是？

A. 狗的语义比猫大 1
B. 两个 ID 是查表地址，大小差不表示语义距离
C. 它们天然比 ID 为 1000 的词更相似
D. 可直接用 ID 的欧式距离表达语义

✅ 参考答案

答案：B。 token ID 是离散索引，不是连续特征。语义相似性应在学习得到的词向量空间中比较。

16. 形状为 [B, T] 的 token ID 张量输入 nn.Embedding(V, D) 后，输出形状是？

A. [B, T, D]
B. [B, D]
C. [B, T, V]
D. [V, D]

✅ 参考答案

答案：A。 每个整数 ID 都被替换成一个 $D$ 维向量，因而在原形状末尾新增嵌入维。输入通常应为 torch.long。

17. 关于“预训练 Word2Vec”与“下游任务中的 Embedding 层”，哪项最准确？

A. Word2Vec 向量加载后绝对不能再更新
B. Embedding 层只能随机初始化
C. Word2Vec 先从通用语料学表示；下游 Embedding 可从随机或预训练向量开始，并选择冻结或微调
D. 两者都不需要词表

✅ 参考答案

答案：C。 关键区别是表示在哪个训练阶段、由什么目标学到；预训练向量并不等于“永久固定”。

18. Word2Vec 能学到语义相似性的核心直觉是？

A. 拼写长度相同的词语义必然相同
B. 出现在相似上下文中的词，往往具有相关的用法或语义
C. 词频越高，词义越接近 0
D. 每个词的向量由人工逐维命名

✅ 参考答案

答案：B。 这是分布式假设的核心：词的用法可从其上下文分布中学习。它是统计关系，不保证完整理解真实世界语义。

19. CBOW 的训练目标是？

A. 用中心词预测上下文词
B. 用上下文词预测中心词
C. 用前一句预测文档标签
D. 用词性预测实体类别

✅ 参考答案

答案：B。 CBOW 会聚合中心词周围的上下文向量，再预测被遮在中间的目标词。

20. Skip-gram 的训练目标是？

A. 用中心词预测周围的上下文词
B. 用所有上下文预测文档长度
C. 用下一句预测上一句
D. 用 One-hot 向量还原字符串

✅ 参考答案

答案：A。 Skip-gram 从一个中心词出发，为窗口内的多个上下文词构造监督信号。

21. 对序列 Hope can set you free，以 can 为中心、左右窗口半径为 1，正确的 Skip-gram 样本是？

A. 输入 [Hope, set]，目标 can
B. 输入 can，目标分别为 Hope 和 set
C. 输入 Hope，目标 free
D. 输入全句，目标是句子长度

✅ 参考答案

答案：B。 Skip-gram 会将一个窗口展开成 (can, Hope) 和 (can, set) 两个中心词—上下文词对。A 是 CBOW 的样本形式。

22. 语料充足且更关心低频词表示时，通常优先尝试？

A. CBOW
B. Skip-gram
C. One-hot
D. 随机向量且永不训练

✅ 参考答案

答案：B。 Skip-gram 为每个中心词产生多个上下文目标，对低频词通常更友好；CBOW 训练更快，适合资源受限或追求快速基线的场景。

23. 将 Word2Vec 的上下文窗口显著增大，最可能带来什么变化？

A. 每个样本可覆盖更远的关联，但也可引入更多弱相关噪声并增加计算量
B. 词表会自动缩小到 2
C. 嵌入维度必然变成窗口大小
D. 模型不再需要语料

✅ 参考答案

答案：A。 窗口控制“上下文”的局部范围。小窗口更偏局部语法和近邻关系，大窗口可学到更宽泛的主题关联，但会产生更多词对。

24. Word2Vec 浅层网络训练后，通常用作词向量表的是？

A. 输入嵌入矩阵中与 token 对应的行
B. 原始字符串的 ASCII 编码
C. 每个词的词频单值
D. 一个不保存参数的 Softmax 层

✅ 参考答案

答案：A。 训练过程会更新嵌入参数；给定 token ID $i$ 时，取嵌入矩阵的第 $i$ 行作为该 token 的向量。

25. 两个非零词向量的余弦相似度接近 1，表示？

A. 向量模长必然相等
B. 两个向量方向接近，在该表示空间中高度相似
C. 两个 token 的拼写完全相同
D. 两个词在现实世界中绝对同义

✅ 参考答案

答案：B。 余弦相似度关注方向而非模长。高相似度只是模型在特定语料与训练目标下学到的统计关系，不是绝对同义证明。

26. 使用单一静态词向量表示“苹果”时，主要局限是？

A. 无法将 token 转成数字
B. 不同上下文中的“苹果”通常共用同一向量，难以区分水果与公司等词义
C. 向量维度只能是 1
D. 完全不能计算相似度

✅ 参考答案

31. 为客服工单构建“投诉类别判定 + 人名、产品名与日期抽取”系统，请列出从原始文本到模型输出的基本数据流。

✅ 参考答案

可按以下顺序设计：

  1. 明确分类标签与实体 Schema，并制定标注规范。
  2. 对工单做脱敏、编码统一、异常字符处理与必要的文本规范化。
  3. 分词或子词切分，并保留 token 到原文的位置映射，以便还原实体边界。
  4. 构建词表、处理 OOV，转成 ID 并做 padding/mask。
  5. 用预训练词向量、可训练 Embedding 或上下文编码器得到数值表示。
  6. 分类头输出工单类别，序列标注头输出实体边界与类型。
  7. 使用独立验证/测试集分别评估分类指标与实体级 Precision、Recall、F1，再做错误分析。

32. 以“研究生命起源”为例，说明中文分词歧义为什么会影响下游任务，并给出两种改进方法。

✅ 参考答案

该字符串可能被切为“研究 / 生命 / 起源”，也可能在特定语境中出现“研究生 / 命 / 起源”等错误边界。边界不同会改变词表 ID、POS 标签、实体边界和最终特征，上游错误因此会级联到分类、检索或 NER。

改进方法例如：

  • 加入领域用户词典、调整词频，并用人工标注的边界集合评估。
  • 使用能利用上下文的统计/神经分词模型，或在下游任务中改用字/子词建模减少对固定词边界的依赖。

33. 用户词典能解决什么问题？为什么不能无限加词？

✅ 参考答案

用户词典能让分词器优先识别品牌、产品型号、医学名词等领域 token，减少专有词被过度切开的情况。但如果加入过多长串、模糊词或过高词频，可能导致过度合并，在其他语境中产生错误边界。应使用领域验证集比较加词前后的边界 Precision/Recall 和下游效果。

34. 说明 POS 与 NER 的区别，并对“李雷在北京大学学习自然语言处理”给出合理的标注示例。

✅ 参考答案

POS 回答“这个 token 在句子中是什么词性”，NER 回答“哪些连续 token 构成什么类型的实体”。例如：

  • POS：李雷/人名，在/介词，北京大学/机构名，学习/动词，自然语言处理/名词。
  • NER：[李雷]PERSON、[北京大学]ORGANIZATION；若业务 Schema 定义了学科实体，还可标注 [自然语言处理]FIELD。

实体类别取决于任务 Schema，不是唯一不变的标签集。

35. 基于 jieba.posseg 的规则 NER 基线容易出现哪些错误？如何评估？

✅ 参考答案

常见错误包括：

  • 分词边界错误导致实体被拆分或合并。
  • POS 误标导致类型错误或漏检。
  • 难以拼接“北京 / 大学”这类多 token 实体。
  • 同一词在不同上下文中类型不同时，固定词性规则无法消歧。
  • 业务实体类型往往超出 nr/ns/nt 的范围。

评估应在人工标注的测试集上做严格实体级匹配，统计每个类型及宏平均/微平均 Precision、Recall、F1，并按边界错误、类型错误和漏检做分层分析。

36. 对比 One-hot 与稠密词向量，至少从维度、存储、语义相似性和学习方式四个角度说明。

✅ 参考答案
角度 One-hot 稠密词向量
维度 等于词表大小 $V$ 通常为人工选定的 $D$ 且 $D\ll V$
存储 高维稀疏；显式存储很浪费 低维稠密，更适合数值计算
相似性 不同 token 两两正交，不含语义远近 可通过方向、距离表达统计相似性
学习 由词表 ID 直接确定，不需训练 从 Word2Vec 目标或下游任务目标中学习

工程上不必显式构造 One-hot；使用 token ID 查嵌入表与 One-hot 乘嵌入矩阵数学等价，但更高效。

37. 为什么 token ID 不能直接当作连续数值特征？

✅ 参考答案

ID 是人为分配的类别地址，排列方式可以随意更换。如果直接当连续值，模型会被迫接受“ID 20 比 ID 10 大一倍”或“ID 相邻就语义相近”的虚假度量关系。正确做法是将 ID 作为查表索引，再转成 One-hot、嵌入或其他类别表示。

38. 传统静态词向量为什么难以处理“我去银行办业务”与“我在河的银行散步”中的多义词？

✅ 参考答案

在传统 Word2Vec 中，词表内的每个 token 通常只对应一个固定向量。不同词义的上下文信号被混合到同一向量中，无法根据当前句子生成不同表示。可使用多原型词向量，或改用会随句子上下文变化的上下文化表示。

39. 请解释分布式假设，并指出只依赖共现统计的两个局限。

✅ 参考答案

分布式假设认为：经常出现在相似语境中的词，往往具有相关的语法功能或语义。Word2Vec 通过预测局部上下文把这种统计结构压缩进向量。

局限包括但不限于：

  • 反义词可能共享相似语境，因而在向量空间中也很接近。
  • 稀有词、新词和 OOV 统计不足。
  • 训练语料中的社会偏见会被向量吸收和放大。
  • 静态表示无法根据当前上下文消解多义词。

40. 对 token 序列 [我, 喜欢, 学习, NLP]，窗口半径为 1 时，写出以“学习”为中心的 CBOW 样本和 Skip-gram 样本。

✅ 参考答案
  • CBOW：输入上下文 [喜欢, NLP]，目标中心词 学习。
  • Skip-gram：输入中心词 学习，展开为两个目标对 (学习, 喜欢) 和 (学习, NLP)。

序列首尾的中心词只有一侧上下文，样本构造函数必须做边界截断，不应越界或用负索引绕回序列尾部。

41. 从训练目标、速度、低频词效果与使用场景对比 CBOW 和 Skip-gram。

✅ 参考答案

CBOW 聚合多个上下文词预测一个中心词，单窗口目标较少、训练快，高频词上通常很稳定，适合资源有限或需要快速基线的场景。Skip-gram 用一个中心词预测多个上下文词，产生的词对更多、计算更慢，但对低频和长尾词往往更友好。最终选择应通过领域验证而非只靠经验结论。

42. 说明窗口大小、嵌入维度、最小词频和训练轮数分别可能带来什么影响。

✅ 参考答案
  • 窗口大小：小窗口强调局部语法/功能相似，大窗口更容易捕捉宽泛主题关联，但词对与噪声增加。
  • 嵌入维度：过小可能容量不足，过大则增加存储、计算与过拟合风险，不保证更好。
  • 最小词频：提高阈值能缩小词表并过滤噪声，但可丢失领域稀有词；降低阈值则相反。
  • 训练轮数：太少会欠拟合，过多则花费更大，小语料上还可能过拟合。

这些参数互相影响，应保持评估协议不变做对照实验。

43. 给定 $a=(1,2)$、$b=(2,4)$、$c=(-2,1)$，分别计算 $\cos(a,b)$ 与 $\cos(a,c)$，并解释。

✅ 参考答案

$$ \cos(a,b)=\frac{1\times2+2\times4}{\sqrt{1^2+2^2}\sqrt{2^2+4^2}} =\frac{10}{\sqrt5\sqrt{20}}=1. $$

$b=2a$，两向量方向完全相同。

$$ \cos(a,c)=\frac{1\times(-2)+2\times1}{\sqrt5\sqrt5}=0. $$

$a$ 与 $c$ 正交。余弦相似度比较方向，所以 $b$ 的模长是 $a$ 的两倍不影响结果。

44. 小样本文本分类任务中，如何在“随机 Embedding”、“预训练向量冻结”和“预训练向量微调”之间选择？

✅ 参考答案

可建立三组只改变嵌入策略的对照实验：

  • 随机 Embedding：不依赖外部语料，但小样本下往往学不充分，可作基线。
  • 预训练 + 冻结：参数少、稳定且不易破坏通用结构，但无法适应领域语义。
  • 预训练 + 微调：可适配任务，但小数据上需要更小的嵌入层学习率、正则化和早停。

三组必须使用同一数据划分、主模型与评估指标，同时检查预训练词表覆盖率。当领域差异大时，还可先在领域无标注语料上继续训练。

45. 设计一份词向量评估报告的结构，要求同时关注效果、覆盖率、稳健性与风险。

✅ 参考答案

一份可用的报告至少包含：

46. 实现“清洗、分词、稳定词表和 ID 映射”管线

对三条中文短句做简单规范化和 jieba 分词，过滤空白与纯标点；按“词频降序、同频时按首次出现顺序”建词表，预留 <pad> 和 <unk>，并将新句子转成 ID。

✅ 参考答案
Python
import re
from collections import Counter

import jieba


def tokenize(text):
    text = re.sub(r"\s+", " ", text.strip().lower())
    return [
        token for token in jieba.lcut(text)
        if token.strip() and not re.fullmatch(r"[^\w]+", token)
    ]


texts = [
    "自然语言处理，让机器理解语言。",
    "词向量把语言转成稠密向量。",
    "机器学习可以学到词向量。",
]
tokenized = [tokenize(text) for text in texts]
all_tokens = [token for sentence in tokenized for token in sentence]

counts = Counter(all_tokens)
first_position = {}
for position, token in enumerate(all_tokens):
    first_position.setdefault(token, position)

ordered_tokens = sorted(
    counts,
    key=lambda token: (-counts[token], first_position[token]),
)
itos = ["<pad>", "<unk>"] + ordered_tokens
stoi = {token: index for index, token in enumerate(itos)}


def encode(text):
    return [stoi.get(token, stoi["<unk>"]) for token in tokenize(text)]


ids = encode("机器理解未知术语")
assert stoi["<pad>"] == 0 and stoi["<unk>"] == 1
assert len(stoi) == len(set(stoi))
assert stoi["<unk>"] in ids                 # “未知术语”不在训练词表时能兜底
print(tokenized)
print(stoi)
print(ids)

评分点是“可复现”：不要用 list(set(tokens)) 直接建立带有已持久化 ID 的词表，否则顺序可在不同环境中改变。生产环境还应把清洗规则、分词器和用户词典版本与词表一起保存。

50. 用 mask 完成变长句子的 Embedding 与平均池化

给定两个已 padding 的 ID 序列，生成 [B,T,D] 词向量并计算忽略 <pad> 的句向量。验证在真实 token 后多加一个 padding 不会改变平均结果。

✅ 参考答案
Python
import torch
from torch import nn

torch.manual_seed(7)
PAD_ID = 0
embedding = nn.Embedding(num_embeddings=8, embedding_dim=6, padding_idx=PAD_ID)


def masked_mean_pool(ids):
    vectors = embedding(ids)                         # [B, T, D]
    mask = ids.ne(PAD_ID).unsqueeze(-1)              # [B, T, 1]
    total = (vectors * mask).sum(dim=1)              # [B, D]
    count = mask.sum(dim=1).clamp_min(1)             # 避免全 padding 时除零
    return vectors, total / count


ids = torch.tensor([[1, 2, 3, 0], [4, 5, 0, 0]], dtype=torch.long)
vectors, sentence_vectors = masked_mean_pool(ids)

extended_ids = torch.tensor(
    [[1, 2, 3, 0, 0], [4, 5, 0, 0, 0]], dtype=torch.long
)
_, extended_sentence_vectors = masked_mean_pool(extended_ids)

assert vectors.shape == (2, 4, 6)
assert sentence_vectors.shape == (2, 6)
assert torch.allclose(sentence_vectors, extended_sentence_vectors, atol=1e-7)
assert torch.allclose(embedding.weight[PAD_ID], torch.zeros(6))
print(sentence_vectors)

51. 实现 CBOW 滑动窗口样本生成器

输入 token 序列和窗口半径，返回 (上下文词列表, 中心词)；正确处理序列两端，上下文不得包含中心词位置。

✅ 参考答案
Python
def make_cbow_samples(tokens, window_radius=1):
    if window_radius < 1:
        raise ValueError("window_radius must be at least 1")

    samples = []
    for center_index, center_token in enumerate(tokens):
        left = max(0, center_index - window_radius)
        right = min(len(tokens), center_index + window_radius + 1)
        context = [
            tokens[index]
            for index in range(left, right)
            if index != center_index
        ]
        if context:
            samples.append((context, center_token))
    return samples


tokens = ["Hope", "can", "set", "you", "free"]
samples = make_cbow_samples(tokens, window_radius=1)

assert samples[0] == (["can"], "Hope")
assert samples[1] == (["Hope", "set"], "can")
assert samples[2] == (["can", "you"], "set")
assert samples[-1] == (["you"], "free")
assert len(samples) == len(tokens)
print(samples)

注意：如果句子内同一 token 重复出现，上下文中完全可以出现与中心 token 相同的字符串；应排除的是中心位置，不是所有同值 token。

52. 实现 Skip-gram 词对生成器

将每个窗口展开为 (中心词, 上下文词) 词对，验证序列长度为 4、窗口半径为 1 时共产生 6 个有向词对。

✅ 参考答案
Python
def make_skipgram_pairs(tokens, window_radius=1):
    if window_radius < 1:
        raise ValueError("window_radius must be at least 1")

    pairs = []
    for center_index, center_token in enumerate(tokens):
        left = max(0, center_index - window_radius)
        right = min(len(tokens), center_index + window_radius + 1)
        for context_index in range(left, right):
            if context_index != center_index:
                pairs.append((center_token, tokens[context_index]))
    return pairs


tokens = ["我", "喜欢", "学习", "NLP"]
pairs = make_skipgram_pairs(tokens, window_radius=1)

assert pairs == [
    ("我", "喜欢"),
    ("喜欢", "我"),
    ("喜欢", "学习"),
    ("学习", "喜欢"),
    ("学习", "NLP"),
    ("NLP", "学习"),
]
print(pairs)

53. 实现基于余弦相似度的最近邻检索

给定一个小型词向量表，返回查询词之外的 Top-k 最相似词，并对零向量做安全处理。

✅ 参考答案
Python
import torch
import torch.nn.functional as F

words = ["猫", "狗", "宠物", "汽车", "零向量"]
vectors = torch.tensor([
    [1.00, 0.90, 0.05],
    [0.95, 1.00, 0.05],
    [0.90, 0.85, 0.10],
    [0.05, 0.10, 1.00],
    [0.00, 0.00, 0.00],
])
stoi = {word: index for index, word in enumerate(words)}


def nearest_neighbors(query, top_k=2):
    query_index = stoi[query]
    normalized = F.normalize(vectors, p=2, dim=1, eps=1e-12)
    scores = normalized @ normalized[query_index]
    scores[query_index] = -torch.inf             # 排除查询词自身
    values, indices = torch.topk(scores, k=min(top_k, len(words) - 1))
    return [(words[index], score.item()) for index, score in zip(indices, values)]


result = nearest_neighbors("猫", top_k=2)
assert result[0][0] in {"狗", "宠物"}
assert torch.isfinite(torch.tensor(result[0][1]))
print(result)

eps 避免零向量归一化时除零。真实系统还应对查询词 OOV、top_k 越界和并列分数制定明确策略。

54. 验证“Embedding 查表”与“One-hot 乘权重矩阵”等价

用已知词向量矩阵创建 nn.Embedding，对一批 ID 分别使用查表和矩阵乘法取向量，比较结果；再展示冻结与可微调两种设置。

✅ 参考答案
Python
import torch
import torch.nn.functional as F
from torch import nn

weights = torch.tensor([
    [0.0, 0.0, 0.0],   # <pad>
    [1.0, 0.2, 0.1],   # 猫
    [0.9, 0.3, 0.1],   # 狗
    [0.1, 0.2, 1.0],   # 汽车
], dtype=torch.float32)
ids = torch.tensor([[1, 2], [3, 1]], dtype=torch.long)

frozen_embedding = nn.Embedding.from_pretrained(
    weights, freeze=True, padding_idx=0
)
lookup_result = frozen_embedding(ids)

one_hot = F.one_hot(ids, num_classes=weights.shape[0]).to(torch.float32)
matrix_result = one_hot @ weights

assert lookup_result.shape == (2, 2, 3)
assert torch.allclose(lookup_result, matrix_result)
assert frozen_embedding.weight.requires_grad is False

trainable_embedding = nn.Embedding.from_pretrained(
    weights.clone(), freeze=False, padding_idx=0
)
assert trainable_embedding.weight.requires_grad is True
print(lookup_result)

显式 One-hot 乘法需构造末维为 $V$ 的稀疏张量；Embedding 直接按 ID 取行，结果等价但更节省存储和计算。

55. 用小语料训练一个最小 Skip-gram 模型

在无需下载外部数据的小型英文语料上，生成 Skip-gram 词对，用 Embedding + Linear 预测上下文词，验证损失下降并查看近邻词。

✅ 参考答案
Python
import torch
import torch.nn.functional as F
from torch import nn

torch.manual_seed(7)

sentences = [
    "cat kitten pet animal".split(),
    "dog puppy pet animal".split(),
    "cat dog pet animal".split(),
    "kitten puppy pet animal".split(),
    "car bus vehicle road".split(),
    "truck car vehicle road".split(),
    "bus truck vehicle road".split(),
]
itos = sorted({token for sentence in sentences for token in sentence})
stoi = {token: index for index, token in enumerate(itos)}


def make_pairs(sentence, radius=1):
    result = []
    for center_index, center in enumerate(sentence):
        left = max(0, center_index - radius)
        right = min(len(sentence), center_index + radius + 1)
        for context_index in range(left, right):
            if context_index != center_index:
                result.append((stoi[center], stoi[sentence[context_index]]))
    return result


pairs = [pair for sentence in sentences for pair in make_pairs(sentence)]
centers = torch.tensor([center for center, _ in pairs], dtype=torch.long)
contexts = torch.tensor([context for _, context in pairs], dtype=torch.long)


class TinySkipGram(nn.Module):
    def __init__(self, vocab_size, embedding_dim):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, embedding_dim)
        self.output = nn.Linear(embedding_dim, vocab_size, bias=False)

    def forward(self, center_ids):
        return self.output(self.embedding(center_ids))


model = TinySkipGram(len(itos), embedding_dim=8)
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)
initial_loss = None

for step in range(401):
    logits = model(centers)
    loss = F.cross_entropy(logits, contexts)
    if step == 0:
        initial_loss = loss.item()
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

final_loss = loss.item()
assert final_loss < initial_loss * 0.8

with torch.no_grad():
    normalized = F.normalize(model.embedding.weight, dim=1)
    query_index = stoi["cat"]
    scores = normalized @ normalized[query_index]
    scores[query_index] = -torch.inf
    neighbor_indices = torch.topk(scores, k=3).indices.tolist()

print(f"loss: {initial_loss:.4f} -> {final_loss:.4f}")
print("neighbors of cat:", [itos[index] for index in neighbor_indices])

这个实验只用来验证数据流和优化闭环。语料过小，近邻结果会受随机初始化和少量共现强烈影响，不应当作通用语义模型。大词表训练时还需要负采样或层次 Softmax 等效率优化。

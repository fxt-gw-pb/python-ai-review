🔥 PyTorch 基础综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：PyTorch 基础：张量、自动微分与训练

55 道练习，覆盖 Tensor 创建与属性、NumPy/设备互操作、运算与索引、形状变换、自动微分、Dataset/DataLoader 和训练闭环。

一、选择题（30 题）

1. 下列哪一项不属于 PyTorch Tensor 的核心属性？

A. 数据文件原始路径
B. dtype
C. device
D. shape

✅ 参考答案

答案：A。 shape 描述逻辑尺寸，dtype 描述元素类型，device 描述存储设备；张量不会自动保存其来源文件路径。

2. torch.tensor(numpy_array) 与 torch.from_numpy(numpy_array) 的主要区别是什么？

A. 两者行为完全相同
B. 前者只能创建整数张量
C. 后者一定在 GPU 上创建张量
D. 前者通常复制数据，后者通常共享 CPU 内存

✅ 参考答案

答案：D。 torch.tensor 通常创建独立副本；from_numpy 与满足条件的 NumPy 数组共享 CPU 内存，一方原地修改可能影响另一方。

3. 一个默认 float32 模型接收到 float64 输入时，最可能出现什么问题？

A. 自动永久修改模型类型
B. 输入自动转为字符串
C. 类型不匹配错误
D. 模型自动移动到 CPU

✅ 参考答案

答案：C。 参与同一运算的浮点类型应兼容。NumPy 默认浮点常为 float64，交给默认 float32 模型前通常要 .float() 或显式转换。

4. 模型参数已经移动到 CUDA，而输入仍在 CPU，会发生什么？

A. 自动使用两个设备并行计算
B. 自动把模型移回 CPU
C. 只会降低一点精度
D. 通常报设备不一致错误

✅ 参考答案

答案：D。 同一次算子中的张量必须位于兼容设备。应统一使用 device，同时移动模型和批次数据。

5. torch.arange(0, 10, 2) 的结果是什么？

A. [0, 2, 4, 6, 8, 10]
B. [0, 2, 4, 6, 8]
C. 长度为 2 的随机张量
D. [2, 4, 6, 8, 10]

✅ 参考答案

答案：B。 arange 采用左闭右开区间，第三个参数是步长。

6. torch.linspace(0, 1, steps=5) 中 steps=5 表示什么？

A. 步长固定为 5
B. 生成 5 个包含首尾端点的等距值
C. 生成 5 行数据
D. 只生成整数

✅ 参考答案

答案：B。 结果是 [0, 0.25, 0.5, 0.75, 1]。linspace 的第三个参数是点数，不是步长。

7. 连续两次调用 torch.manual_seed(42) 后分别生成同形状随机张量，通常会怎样？

A. GPU 自动关闭
B. 第二次必定全为 0
C. 第一次张量被删除
D. 两个张量相同

✅ 参考答案

答案：D。 重置相同种子会让相同环境中的随机序列重新开始。但跨硬件、算子和版本仍不保证逐位一致。

8. 浮点张量 torch.tensor([-1.9, 1.9]).long() 的结果是什么？

A. [-1, 1]
B. [-2, 2]
C. [-2, 1]
D. 报错

✅ 参考答案

答案：A。 浮点转整数采用向 0 截断，而不是统一向下取整。

9. tensor.item() 最适合用于什么情况？

A. 任意多元素张量转列表
B. 张量转 GPU
C. 单元素张量转 Python 标量
D. 创建梯度

✅ 参考答案

答案：C。 .item() 只适用于恰好包含一个元素的张量，常用于读取标量损失。

10. 对两个形状均为 [2, 3] 的张量 a、b，a * b 表示什么？

A. 逐元素乘法
B. 矩阵乘法
C. 沿第 0 维拼接
D. 外积

✅ 参考答案

答案：A。 * 是 Hadamard 逐元素乘法；矩阵乘法应使用 @、torch.matmul 或特定情形下的 mm。

11. 若 a.shape <mark> [2, 3]、b.shape </mark> [3, 4]，则 (a @ b).shape 是什么？

A. [4, 2]
B. [3, 3]
C. [2, 3, 4]
D. [2, 4]

✅ 参考答案

答案：D。 内部维度 3 相消，输出保留外部维度 2 和 4。

12. torch.var(x, correction=0) 计算的是什么？

A. 分母为 $N$ 的总体方差
B. 分母为 $N-1$ 的样本方差
C. 标准差
D. 均值

✅ 参考答案

答案：A。 correction=0 不做贝塞尔校正，分母是 $N$；correction=1 才对应常见无偏样本方差。

13. 为什么不建议对参与自动微分的叶子张量随意执行原地操作？

A. 原地操作只能运行在 GPU
B. 原地操作一定更慢
C. 可能破坏计算图所需的历史值并触发错误
D. 会自动改变维度

✅ 参考答案

答案：C。 反向传播可能需要前向时保存的值。原地覆盖会让版本检查失败或造成难以追踪的梯度问题。

14. 对二维连续张量执行 x.transpose(0, 1) 后，最准确的说法是什么？

A. 一定复制出新的连续存储
B. 元素值全部取反
C. 通常只改变 stride，结果可能不连续
D. 自动删除原张量

✅ 参考答案

答案：C。 转置通常创建共享存储的视图，只改变逻辑解释和步长，因此后续 view 可能失败。

15. view 与 reshape 的差异是什么？

A. reshape 永远共享内存
B. view 要求底层布局兼容；reshape 必要时可复制
C. view 可以改变元素总数
D. 两者只能用于一维张量

✅ 参考答案

答案：B。 view 更严格；reshape 会尽量返回视图，不兼容时创建副本，因此不能假设 reshape 一定共享存储。

16. 基础切片 x[1:4] 通常具有什么存储特性？

A. 一定复制
B. 通常与原张量共享存储
C. 一定转到 CPU
D. 自动开启梯度

✅ 参考答案

答案：B。 基础索引和切片通常返回视图；修改视图可能影响原张量。高级索引通常返回副本。

17. 对二维张量使用 x[[0, 1], [1, 2]] 表示什么？

A. 配对取 (0,1) 与 (1,2) 两个元素
B. 取第 0、1 行和第 1、2 列的笛卡尔子矩阵
C. 只取第 0 行
D. 取主对角线全部元素

✅ 参考答案

答案：A。 两个高级索引按位置配对。若要笛卡尔选择，可使用 x[rows[:, None], cols] 或 torch.meshgrid。

18. 对二维张量应用同形状布尔掩码 x[mask]，结果通常是什么形状？

A. 一维，包含所有掩码为 True 的元素
B. 与原张量完全相同
C. 固定为 [1, 1]
D. 自动补 0

✅ 参考答案

答案：A。 同形状布尔掩码会按存储顺序抽出满足条件的元素，结果通常是一维。

19. x.unsqueeze(1) 的作用是什么？

A. 在索引 1 处插入大小为 1 的维度
B. 删除第 1 维
C. 把第 1 维扩大一倍
D. 转置第 0、1 维

✅ 参考答案

答案：A。 例如 [3, 4] 会变成 [3, 1, 4]，元素总数不变。

20. squeeze() 会删除哪些维度？

A. 所有维度
B. 大小为 1 的维度
C. 所有偶数维度
D. 最后一维无论大小

✅ 参考答案

答案：B。 无参数 squeeze() 删除全部单例维；squeeze(dim) 只有在指定维大小为 1 时才生效。

21. 对形状 [2, 3, 4] 的张量执行 permute(2, 0, 1)，输出形状是什么？

A. [3, 4, 2]
B. [4, 2, 3]
C. [2, 4, 3]
D. [24]

✅ 参考答案

答案：B。 新第 0 维来自旧第 2 维，新第 1 维来自旧第 0 维，新第 2 维来自旧第 1 维。

22. torch.cat([a, b], dim=0) 的约束是什么？

A. 只能拼接标量
B. 两个张量必须完全相同
C. 必须先增加新维度
D. 除拼接维外，其余维度必须一致

✅ 参考答案

答案：D。 cat 沿已有维延长尺寸，不创建新轴。

23. torch.stack([a, b], dim=0) 与 cat 的核心区别是什么？

A. stack 不要求形状一致
B. stack 会求和
C. stack 新建一个维度
D. stack 只支持整数

✅ 参考答案

答案：C。 stack 要求每个输入形状相同，并在指定位置插入新维度。

24. 两个形状都是 [N, 8] 的特征张量沿特征维拼接后，形状是什么？

A. [N, 8]
B. [N, 2, 8]
C. [2N, 8]
D. [N, 16]

✅ 参考答案

答案：D。 torch.cat([a, b], dim=1) 保留样本数，把特征数相加。逐元素相加则仍为 [N, 8]。

25. torch.chunk(x, 3, dim=0) 的块大小必须完全相等吗？

A. 必须，否则报错
B. 不必，不能整除时块大小可能不等，返回块数也受维度长度约束
C. 一定返回 4 块
D. 只支持一维张量

✅ 参考答案

答案：B。 chunk 尝试按数量分块；不能整除时大小不完全相等。若要求指定每块长度，可使用 split。

26. 下列哪个通常是叶子张量？

A. q = torch.relu(y)
B. y = x * 3
C. z = y.sum()
D. x = torch.tensor(2.0, requires_grad=True)

✅ 参考答案

答案：D。 由用户直接创建且需要梯度的 x 是叶子张量；计算产生的 y、z、q 通常是非叶子张量并带有 grad_fn。

27. 为什么多元素输出 y 通常不能直接调用无参数 y.backward()？

A. PyTorch 不支持多元素张量
B. 只有整数能反传
C. 需要指定上游梯度，或先把输出规约为标量
D. 必须先转 NumPy

✅ 参考答案

答案：C。 反向传播实际计算向量—雅可比积。标量输出隐含上游梯度 1，多元素输出则需显式传入同形状梯度。

28. 连续两次执行 loss.backward()，中间没有清空叶子张量的 .grad，结果会怎样？

A. 梯度被新值覆盖
B. 梯度累加
C. 梯度自动归一化
D. 计算图永久保存

✅ 参考答案

答案：B。 PyTorch 默认累加到 .grad。训练中通常每个 batch 都调用 optimizer.zero_grad() 或把梯度置为 None。

29. 对 CPU 张量执行 y = x.detach()，最准确的说法是什么？

A. 与计算图断开，但通常仍与 x 共享存储
B. 一定深拷贝
C. 自动移动到 GPU
D. 删除 x

✅ 参考答案

答案：A。 detach 解决的是梯度关系，不等于复制。要同时断图并获得独立数据，应使用 x.detach().clone()。

30. Dataset 与 DataLoader 的职责如何区分？

A. 两者完全相同
B. Dataset 只负责 GPU，DataLoader 只负责 CPU

二、简答题（15 题）

31. Tensor 与 NumPy ndarray 的共同点和主要差异是什么？

✅ 参考答案

两者都支持多维数组、广播、索引和向量化运算。Tensor 额外支持自动微分、GPU/加速设备和深度学习模块生态；NumPy 更偏通用 CPU 数值计算。二者转换时还要注意设备、梯度状态、数据类型和是否共享内存。

32. 解释张量的 shape、stride 与物理存储之间的关系

✅ 参考答案

shape 描述每个逻辑轴的长度；stride 描述沿某个轴移动一个位置需要跨过多少个底层元素；底层存储是一段线性内存。转置可以只改变 shape/stride 而不移动元素，因此逻辑顺序改变但物理数据仍共享，也可能变成非连续张量。

33. 为什么不能简单断言 reshape 一定复制或一定共享内存？

✅ 参考答案

reshape 会优先返回兼容布局的视图；如果原张量的 stride 无法支持目标形状，它会创建连续副本。是否共享取决于输入布局和目标形状，应在需要独立存储时显式 clone()，而不是依赖偶然行为。

34. 把模型输出安全转换为 NumPy 的通用步骤是什么？为什么？

✅ 参考答案

常用写法是 output.detach().cpu().numpy()：detach 与计算图断开，cpu 把 CUDA 张量移回 CPU，numpy 才能创建 NumPy 视图。若结果还需与原张量完全隔离，再调用 .copy()。

35. 什么是广播？判断两个形状能否广播的规则是什么？

✅ 参考答案

广播让不同形状张量在不显式复制完整数据的情况下进行逐元素运算。比较形状时从末维向前：对应维必须相等，或其中一个为 1，或某一方缺少该维；否则不能广播。

36. 为什么方差 API 需要 correction 参数？

✅ 参考答案

总体方差通常除以 $N$，从有限样本估计总体方差时常使用贝塞尔校正除以 $N-1$。correction 明确自由度修正，避免把不同统计定义混为一谈；小样本时差异尤其明显。

37. 基础索引与高级索引在“是否共享存储”方面通常有什么差异？

✅ 参考答案

切片、单个整数等基础索引通常返回共享存储的视图；列表/LongTensor/布尔掩码等高级索引通常收集元素并返回副本。赋值语句是特殊情形：高级索引放在左侧时会把值写回原张量。

38. 用一个形状例子说明 cat 与 stack 的区别

✅ 参考答案

设 a、b 都是 [2, 3]。cat([a,b], dim=0) 得到 [4,3]，沿已有行轴延长；stack([a,b], dim=0) 得到 [2,2,3]，新增一个“来源张量”轴。cat 除拼接维外一致，stack 要求所有形状完全一致。

39. 多元素输出的 backward(gradient) 在数学上计算了什么？

✅ 参考答案

若输出为向量 $y$，传入上游向量 $v$，PyTorch 计算 $v^T J$，其中 $J=\partial y/\partial x$ 是雅可比矩阵。这是向量—雅可比积，不会默认构造并保存完整雅可比，因而更高效。

40. 为什么更新参数时不应使用 parameter.data -= lr * parameter.grad？

✅ 参考答案

.data 会绕过自动微分的安全检查，可能造成难以发现的图错误。手写更新应放在 with torch.no_grad(): 中执行原地减法；实际训练更推荐优化器的 step()。

41. detach()、clone() 和 detach().clone() 分别解决什么问题？

✅ 参考答案

detach() 断开梯度关系但通常共享存储；clone() 复制数据，同时保留从原张量到副本的梯度关系；detach().clone() 同时断开计算图并获得独立存储，适合保存不会反向影响原张量的快照。

42. 比较全批量、单样本 SGD 与 mini-batch 训练

✅ 参考答案

全批量每步用全部数据，梯度稳定但成本高；单样本 SGD 更新频繁、噪声大且硬件利用率低；mini-batch 在梯度噪声、内存和并行效率之间折中，是深度学习最常用方式。

43. 自定义 Dataset 最少需要实现哪些方法？DataLoader 又提供了什么？

✅ 参考答案

映射式 Dataset 通常继承 Dataset 并实现 __len__ 与 __getitem__。DataLoader 负责按 batch_size 组批、shuffle、drop_last、并行 worker、采样策略和批次迭代。

44. 为什么统计 epoch 平均损失时应按 batch 样本数加权？

✅ 参考答案

损失函数通常返回当前 batch 的平均损失。最后一个 batch 可能更小，若简单平均各 batch，会让小 batch 与大 batch 权重相同。应累加 loss.item() * batch_size，最后除以总样本数。

45. 描述一个规范训练、验证与保存流程

✅ 参考答案

训练阶段调用 model.train()，逐批执行清梯度、前向、算损失、反向和更新。验证阶段调用 model.eval() 并配合 torch.inference_mode()，汇总指标而不更新参数。依据验证指标选择模型，把 state_dict 保存到明确路径；恢复时按相同结构实例化模型，使用 map_location 和 weights_only=True 加载，再切换到评估模式。

from torch.utils.data import DataLoader, Dataset


class LineDataset(Dataset):
    def __init__(self, size=100):
        self.x = torch.linspace(-1, 1, size).unsqueeze(1)
        self.y = 3 * self.x + 1

    def __len__(self):
        return self.x.size(0)

    def __getitem__(self, index):
        return self.x[index], self.y[index]


loader = DataLoader(LineDataset(), batch_size=16, shuffle=True)
for features, targets in loader:

54. 训练一个一元线性回归模型

import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

torch.manual_seed(42)

y = 3 * x + 1 + 0.1 * torch.randn_like(x)

    for features, targets in loader:

        predictions = model(features)
        loss = criterion(predictions, targets)
        loss.backward()
        optimizer.step()

print(model.weight.item(), model.bias.item())

55. 完成评估、保存与恢复

对一个已训练的 nn.Linear(1, 1) 计算验证 MSE，保存参数，再加载到新模型中验证输出一致。

✅ 参考答案
Python
from pathlib import Path

import torch
import torch.nn as nn

model = nn.Linear(1, 1)
with torch.no_grad():
    model.weight.fill_(3.0)
    model.bias.fill_(1.0)

x_valid = torch.tensor([[-1.0], [0.0], [2.0]])
y_valid = 3 * x_valid + 1

model.eval()
with torch.inference_mode():
    predictions = model(x_valid)
    mse = nn.functional.mse_loss(predictions, y_valid)
print("validation mse:", mse.item())

path = Path("checkpoints/linear.pt")
path.parent.mkdir(parents=True, exist_ok=True)
torch.save(model.state_dict(), path)

restored = nn.Linear(1, 1)
state = torch.load(path, map_location="cpu", weights_only=True)
restored.load_state_dict(state)
restored.eval()

with torch.inference_mode():
    print(torch.allclose(model(x_valid), restored(x_valid)))

🔗 关系抽取：规则、Pipeline 与 CasRel 综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：关系抽取：规则、Pipeline 与 CasRel

55 道练习，按“SPO 输出合同 → 规则关系抽取 → Pipeline 实体对分类 → BiLSTM+Attention → Joint 与 CasRel → 重叠三元组 → 解码、评估与医学安全”组织。做题时始终区分三层含义：文本表达了什么、模型抽取了什么、知识图谱最终采信了什么。文本中的关系 mention 不是自动成立的临床事实，否定、条件、时间、主体和证据来源必须随结果保留。

建议先独立作答，再展开参考答案。本练习依据原课件第四章和站内重组讲义设计，并参考 BiLSTM+Attention 关系分类论文、CopyRE 重叠三元组论文、CasRel 原始论文、Pipeline 与 Joint 实证研究及独立编码器 Pipeline 研究校准。注意：CasRel 论文基准把关系建模为从主体到客体的函数，并在原实验中采用“关系与主体、客体 head 正确”的部分匹配；本练习面向医学建图，除特别说明外使用更严格的“主体/客体完整 span 与类型、方向、关系全部正确”口径。10 道代码题只使用 Python 标准库离线运行，不下载模型。

一、选择题（30 题）

1. 封闭关系抽取的直接输出通常是什么？

A. 预定义关系集合中的 (subject, predicate, object) 及其文本证据 \ B. 已经由医生确认的诊断结论 \ C. 任意新谓词的自然语言解释 \ D. Neo4j 的全部节点

✅ 参考答案

答案：A。 封闭关系抽取在既定 Schema 内识别主体、关系和客体。模型输出仍是文本级抽取结果，不能跳过断言识别、实体链接、证据审核和入图策略。

2. 封闭关系抽取与开放信息抽取的关键区别是？

A. 前者预先定义关系集合，后者可从文本发现谓词表达 \ B. 前者只能处理英文，后者只能处理中文 \ C. 前者不需要实体，后者不需要文本 \ D. 二者只有模型参数量不同

✅ 参考答案

答案：A。 医学知识图谱通常以封闭抽取保证语义统一，也可用开放抽取发现候选关系，再经归一化与人工审核扩展 Schema。

3. 为什么 Drug-TREATS-Disease 不能随意交换为 Disease-TREATS-Drug？

A. 多数关系具有方向，交换主体和客体会改变或破坏语义 \ B. 三元组只能按字符串长度排序 \ C. 客体不能有实体类型 \ D. 交换后会自动变成开放关系

✅ 参考答案

答案：A。 Schema 应明确主体类型、客体类型、方向，以及是否存在命名清楚的逆关系。无向关系也应在存储合同中规定规范化顺序。

4. 关系抽取结果保留字符偏移和来源定位的主要价值是？

A. 可回到原文复核实体边界、语境和证据 \ B. 自动提高模型参数量 \ C. 取代实体链接 \ D. 使任何关系都变为真

✅ 参考答案

答案：A。 doc_id 与 [start,end)、句段定位、规则或模型版本等信息支持审计、纠错、去重和重新处理。只保存三个字符串会丢失证据链。

5. 在生产级严格三元组评估中，什么情况才能计为 TP？

A. 主体和客体完整边界与类型、关系类型、方向全部正确 \ B. 只要主体字符串出现过 \ C. 只要关系类别正确，不看实体 \ D. 预测置信度超过 0.5

✅ 参考答案

答案：A。 阈值不是正确性的定义。若论文基准采用 head match 或忽略实体类型，必须另报该口径，不能与严格指标直接混用。

6. 为什么除了总体 micro F1 还应报告按关系类型的指标？

A. 高频关系可能主导 micro F1，掩盖稀有但重要关系的失败 \ B. macro F1 一定高于 micro F1 \ C. 每类指标不需要 gold 标签 \ D. micro F1 只能用于回归

✅ 参考答案

答案：A。 还应按重叠类型、实体距离、否定语境和来源切片，避免一个汇总数掩盖关键风险。

7. 规则关系抽取最适合哪种起点？

A. 样本少、句式稳定、关系高价值且需要可解释审计 \ B. 关系集无限且文本风格完全未知 \ C. 不允许维护规则版本 \ D. 任何任务都要求零误差

✅ 参考答案

答案：A。 规则能提供高精度冷启动和回归基线，但召回、冲突、跨域迁移和维护成本必须持续度量。

8. “患者否认头痛”被规则匹配为 HAS_SYMPTOM(患者, 头痛)，首要问题是？

A. 没有处理否定范围，把被否认的提及当成肯定事实 \ B. 主体字符串太短 \ C. 必须改用无向关系 \ D. 需要删除原文

✅ 参考答案

答案：A。 可保留“头痛” mention，但关系证据应带 negated 断言或被入图策略拒绝。还要处理假设、家族史、历史状态和条件句。

9. 一个可审计的规则命中至少应额外记录什么？

A. rule_id、规则版本、命中片段、来源和必要的过滤理由 \ B. 只有最终三个字符串 \ C. 训练轮数 \ D. GPU 型号

✅ 参考答案

答案：A。 这些字段能回答“为什么抽到了这条关系”，也便于规则变更后的定向回归和失效分析。

10. 一句话识别出 (m) 个实体，枚举所有有向实体对的数量级通常是？

A. (O(m^2)) \ B. (O(\log m)) \ C. (O(1)) \ D. (O(2^{-m}))

✅ 参考答案

答案：A。 排除自环后有 (m(m-1)) 个有向对。类型约束、距离、句界和候选召回器可以减少无效组合，但不能无依据删掉可能的正例。

11. 在候选生成阶段使用 Disease → Symptom 类型约束的主要作用是？

A. 提前排除与 Schema 不兼容的有向实体对 \ B. 自动证明剩余实体对一定有关系 \ C. 让 NER 不再出错 \ D. 把多标签任务变成聚类

✅ 参考答案

答案：A。 类型约束提高效率并缓解负类不平衡，但剩余候选仍需分类；若 NER 类型错，也可能误删真正候选。

12. Pipeline 关系分类器为什么通常需要 NO_RELATION 类或等价机制？

A. 大多数候选实体对没有目标关系，需要显式拒绝 \ B. 每个实体对都至少有一个关系 \ C. 它用于替代 padding \ D. 它表示开放关系的谓词文本

✅ 参考答案

答案：A。 负例数量和采样策略会影响校准与召回。未标注不一定等于真正负例，弱监督数据尤其要警惕假负例。

13. 训练关系分类器使用 gold 实体，推理使用 predicted 实体，主要产生什么问题？

A. 训练与推理输入分布不一致，并出现上游边界和类型错误 \ B. 自动消除误差传播 \ C. 关系集合变为空 \ D. 模型不再需要实体标记

✅ 参考答案

答案：A。 应同时报告 gold-entity（oracle）与 predicted-entity 的端到端结果，并用带噪实体或实际预测实体构造训练样本做对照。

14. 若某个 gold 三元组的主体或客体被 NER 漏掉，后续实体对分类器能否恢复它？

A. 标准 Pipeline 中不能，因为该实体对不会进入关系分类 \ B. 一定能，Attention 会补齐实体 \ C. 只要提高 softmax 温度就能 \ D. 只要换成 macro F1 就能

✅ 参考答案

答案：A。 这就是误差传播的硬上限之一。可通过更强 NER、联合建模、候选 span 补充或端到端召回器改善，但必须实测。

15. 在关系分类输入中加入 SUBJ 与 OBJ 边界标记的主要原因是？

A. 明确当前待分类的有向实体对，并让编码器利用实体边界信息 \ B. 自动完成知识库消歧 \ C. 强制注意力权重等于因果证据 \ D. 把所有 token 变成实体

✅ 参考答案

答案：A。 同一句往往有多个候选对；没有实体标记、位置编码或 span 表示，模型难以知道当前分类目标和方向。

16. 原课件 BiLSTM+Attention 关系分类器中，BiLSTM 与 Attention 分别做什么？

A. BiLSTM 编码双向上下文，Attention 对位置表示加权汇总 \ B. BiLSTM 生成 Neo4j 节点，Attention 写 Cypher \ C. BiLSTM 只做分词，Attention 只做去重 \ D. 二者都只负责实体链接

✅ 参考答案

答案：A。 最终句子/实体对表示再送入关系分类层。实现时 padding 位置必须在注意力 softmax 前屏蔽。

17. “Attention 权重最高的 token 就是模型的可靠解释”为什么不严谨？

A. 注意力是模型内部加权机制，不自动等价于充分、忠实的因果解释 \ B. 注意力权重从不归一化 \ C. Attention 不能用于文本 \ D. 最高权重一定在 padding

✅ 参考答案

答案：A。 可把权重用于诊断，但生产审计仍应保存原文证据，并结合遮挡、反事实、规则核验或人工复核。

18. 关于 Pipeline 与 Joint 的说法，哪项最稳妥？

A. Joint 允许实体与关系表示或解码协同，但并不保证一定优于精心设计的 Pipeline \ B. Joint 永远没有错误传播 \ C. Pipeline 永远不需要 NER \ D. Joint 必须使用同一个标签给每个 token

✅ 参考答案

答案：A。 Joint 也会受数据、解码结构和目标冲突影响；Pipeline 则有模块化、可替换和易诊断的优势。选型要靠同口径实验。

19. CasRel 对关系的核心重新表述是什么？

A. 对每个关系学习从给定主体到可能客体的关系特定标注器 \ B. 把每个关系随机映射为一个字符 \ C. 对每个实体对只允许一个关系 \ D. 先生成 Cypher 再识别实体

✅ 参考答案

答案：A。 论文写作 (f_r(s)\rightarrow o)：先找主体，再在每个主体条件下由各关系通道找客体或输出空客体。

20. CasRel 的低层主体标注器通常预测什么？

A. 每个 token 是主体 head / tail 的二元概率 \ B. 每个 token 的关系类别 softmax \ C. 每个文档唯一主体 ID \ D. 关系图的最短路径

✅ 参考答案

答案：A。 head 和 tail 是两个二元序列；解码时还要处理阈值、配对、子词偏移、特殊 token 与最大 span 长度。

21. 若批大小为 (B)、序列长为 (L)、关系数为 (R)，CasRel 客体 head 概率常见形状是？

A. [B, L, R] \ B. [B] \ C. [R, R] \ D. [L, vocab_size]

✅ 参考答案

答案：A。 每个主体条件下，每种关系都在每个位置预测客体 head；客体 tail 同形。训练损失需用 [B,L,R] 的有效 token mask。

22. CasRel 为什么要把指定主体表示融合进文本表示后再预测客体？

A. 让客体和关系预测条件于当前主体，区分同句中的不同三元组 \ B. 让所有主体共享同一客体 \ C. 取消关系通道 \ D. 只为增加序列长度

✅ 参考答案

答案：A。 对每个检测到的主体分别运行高层标注器，才能保留同一句中“一主体多客体”和多个主体的不同关系。

23. CasRel 某个关系通道的 head/tail 全为 0，通常表示什么？

A. 给定主体在该关系下没有检测到客体，即空客体 \ B. 该主体一定不存在 \ C. 该关系是无向关系 \ D. 模型已经达到满分

✅ 参考答案

答案：A。 论文目标同时学习有效关系的客体和其他关系的 null object。阈值与类别不平衡会影响空/非空判断。

24. 三元组 A-r1-B 与 A-r2-C 属于哪种重叠？

A. SEO（SingleEntityOverlap） \ B. EPO（EntityPairOverlap） \ C. Normal \ D. 没有实体

✅ 参考答案

35. 一句中有 Drug、Disease、Symptom、Food 四个实体，Pipeline 候选如何缩减？

全有向枚举有 (4\times3=12) 对。根据 Schema 只保留如 Drug→Disease、Disease→Symptom、Disease→Food 等兼容方向；再根据句界、最大距离、依存路径或候选召回器缩减。每个过滤器要在开发集评估候选召回率，不能只追求候选数量少。

36. 如何构造关系分类训练样本并处理 NO_RELATION 不平衡？

✅ 参考答案

为每个候选保留句子、带方向的实体标记、边界和类型；gold 实体对赋一个或多个关系，其他候选为 NO_RELATION，但需考虑漏标造成的假负例。可用类型约束、分层/困难负采样、类权重或 focal loss；验证集和测试集不应通过采样改变真实分布。若同一实体对可有多个关系，应使用多标签目标而非单一 softmax。

37. 设计一个量化 Pipeline 误差传播的 oracle 实验。

✅ 参考答案

固定同一关系分类器和测试集，至少跑两组：gold 实体输入与 predicted 实体输入。前者近似关系分类上限，后者是端到端性能；再统计 gold 三元组中主体/客体可达率、边界错、类型错和关系错。若 gold-entity F1 高而端到端低，优先修 NER/候选；两者都低则关系表示、标签或 Schema 更可疑。

38. 写出 BiLSTM+Attention 的一种加权汇总形式，并说明 mask。

✅ 参考答案

设 BiLSTM 输出 (H=[h_1,\ldots,h_L])，可计算 [ u_i=\tanh(Wh_i+b),\quad \alpha_i=\operatorname{softmax}(u_i^\top q),\quad r=\sum_i\alpha_i h_i. ] padding 和特殊 token 的 logit 应在 softmax 前置为负无穷，使其权重为 0；实体标记、相对位置或 span 表示要让模型知道当前有向实体对。(r) 再送入单标签 softmax 或多标签 sigmoid 分类器。

39. 规则、Pipeline 与 CasRel 应如何选型？

✅ 参考答案

句式稳定、数据少、要求高可解释性时先做规则；已有可靠 NER、模块需独立替换或关系集频繁变化时选 Pipeline；标注充分、重叠三元组多且端到端召回重要时评估 CasRel/其他 Joint。生产中常组合三者：规则作保护与高精度通道，模型扩大召回，低置信度和高风险结果人工复核。最终以同一数据切分、Schema 和严格指标比较。

40. “阿司匹林可缓解头痛并降低发热”的 CasRel 训练标签如何构造？

✅ 参考答案

若 gold 为 (阿司匹林, RELIEVES, 头痛) 与 (阿司匹林, RELIEVES, 发热)，主体 head/tail 序列在“阿司匹林”首尾置 1。选定该主体后，在 RELIEVES 关系通道把“头痛”“发热”的 head/tail 都置 1，其他关系通道全 0；padding 全部 mask。若 Schema 将两种谓词区分为不同关系，则分别在对应通道标注，不能由模型自行合并语义。

41. CasRel 推理时为什么必须逐个预测主体运行客体标注器？

✅ 参考答案

客体概率是 (p_r(o\mid s,x))，依赖当前主体。应先解码全部主体，对每个主体构造其表示，然后遍历关系通道解码全部客体；最终做三元组级去重。只取第一个主体或把多个主体平均在一起，会把不同主体的客体错配，也会降低 SEO 场景召回。

46. 校验严格 SPO 输出合同

✅ 参考答案

def validate_spo(text: str, spo: dict, schema: dict) -> bool:
    required = {"subject", "predicate", "object", "source", "confidence"}
    missing = required - set(spo)
    if missing:
        raise ValueError(f"缺少字段: {sorted(missing)}")
    if not isinstance(spo["source"], str) or not spo["source"].strip():
        raise ValueError("source 必须是非空字符串")
    if not isinstance(spo["confidence"], (int, float)) or not 0 <= spo["confidence"] <= 1:
        raise ValueError("confidence 必须在 [0,1]")

    def validate_entity(name: str) -> None:
        entity = spo[name]
        for field in ("id", "text", "type", "start", "end"):
            if field not in entity:
                raise ValueError(f"{name} 缺少 {field}")
        start, end = entity["start"], entity["end"]
        if not isinstance(start, int) or not isinstance(end, int):
            raise ValueError("偏移必须是整数")
        if not 0 <= start < end <= len(text):
            raise ValueError(f"{name} 偏移越界")
        if text[start:end] != entity["text"]:
            raise ValueError(f"{name} 文本与 [start,end) 不一致")

    validate_entity("subject")
    validate_entity("object")
    predicate = spo["predicate"]
    if predicate not in schema:
        raise ValueError("未知关系")
    expected_subject, expected_object = schema[predicate]
    actual_types = spo["subject"]["type"], spo["object"]["type"]
    if actual_types != (expected_subject, expected_object):
        raise ValueError("实体类型或关系方向不符合 Schema")
    return True

def generate_candidates(entities: list[dict], schema: dict) -> list[dict]:
    candidates = {}
    for subject in entities:
        for obj in entities:
            if subject["mention_id"] == obj["mention_id"]:
                continue
            for relation, (subject_type, object_type) in schema.items():
                if (subject["type"], obj["type"]) != (subject_type, object_type):
                    continue
                key = subject["mention_id"], relation, obj["mention_id"]
                candidates[key] = {
                    "subject_id": subject["mention_id"],
                    "predicate": relation,
                    "object_id": obj["mention_id"],
                }
    return [candidates[key] for key in sorted(candidates)]

48. 实现带否定过滤和证据溯源的规则关系抽取

✅ 参考答案

def extract_by_rules(text: str, entities: list[dict], rules: list[dict], source: str) -> list[dict]:
    results = []
    for rule in rules:
        for subject in entities:
            for obj in entities:
                if subject["end"] > obj["start"]:
                    continue
                if (subject["type"], obj["type"]) != (rule["subject_type"], rule["object_type"]):
                    continue
                between = text[subject["end"]:obj["start"]]
                trigger_at = between.find(rule["trigger"])
                if trigger_at < 0:
                    continue
                left_context = between[max(0, trigger_at - 2):trigger_at]
                if any(cue in left_context for cue in rule["negation_cues"]):
                    continue
                results.append({
                    "subject_id": subject["mention_id"],
                    "predicate": rule["predicate"],
                    "object_id": obj["mention_id"],
                    "rule_id": rule["rule_id"],
                    "source": source,
                    "evidence": {
                        "start": subject["start"],
                        "end": obj["end"],
                        "text": text[subject["start"]:obj["end"]],
                    },
                    "assertion": "affirmed",
                })
    return results


rules = [{
    "rule_id": "disease_manifest_v2",
    "subject_type": "Disease",
    "object_type": "Symptom",
    "trigger": "表现为",
    "predicate": "HAS_SYMPTOM",
    "negation_cues": ["不", "未", "无"],
}]

49. 量化 Pipeline 的可达上限与误差传播

def pipeline_recall_report(gold_triples: int, ner_reachable: int, relation_correct: int) -> dict:
    if not 0 <= relation_correct <= ner_reachable <= gold_triples:
        raise ValueError("必须满足 0 <= relation_correct <= ner_reachable <= gold_triples")
    if gold_triples == 0:
        return {"ner_pair_recall": 0.0, "conditional_re_recall": 0.0, "end_to_end_recall": 0.0}
    ner_pair_recall = ner_reachable / gold_triples
    conditional_re_recall = relation_correct / ner_reachable if ner_reachable else 0.0
    end_to_end_recall = relation_correct / gold_triples
    return {
        "ner_pair_recall": ner_pair_recall,
        "conditional_re_recall": conditional_re_recall,
        "end_to_end_recall": end_to_end_recall,
    }


report = pipeline_recall_report(100, 72, 54)

    "ner_pair_recall": 0.72,
    "conditional_re_recall": 0.75,
    "end_to_end_recall": 0.54,
}
assert abs(
    report["ner_pair_recall"] * report["conditional_re_recall"]
    - report["end_to_end_recall"]
) < 1e-12

50. 构造支持 EPO 的多标签关系分类样本

同一有向实体对可有多个关系；无正关系时标为 NO_RELATION，并确定性截取负例。

✅ 参考答案
Python
def build_relation_examples(
    entities: list[dict],
    schema: dict,
    gold_triples: set[tuple[str, str, str]],
    negative_ratio: int = 2,
) -> list[dict]:
    positive, negative = [], []
    for subject in entities:
        for obj in entities:
            if subject["mention_id"] == obj["mention_id"]:
                continue
            allowed = sorted(
                relation for relation, type_pair in schema.items()
                if (subject["type"], obj["type"]) == type_pair
            )
            if not allowed:
                continue
            labels = [
                relation for relation in allowed
                if (subject["mention_id"], relation, obj["mention_id"]) in gold_triples
            ]
            example = {
                "subject_id": subject["mention_id"],
                "object_id": obj["mention_id"],
                "labels": labels or ["NO_RELATION"],
            }
            (positive if labels else negative).append(example)

    key = lambda item: (item["subject_id"], item["object_id"])
    positive.sort(key=key)
    negative.sort(key=key)
    negative_limit = negative_ratio * max(1, len(positive))
    return positive + negative[:negative_limit]


entities = [
    {"mention_id": "m1", "type": "Drug"},
    {"mention_id": "d1", "type": "Disease"},
    {"mention_id": "d2", "type": "Disease"},
]
schema = {
    "TREATS": ("Drug", "Disease"),
    "PREVENTS": ("Drug", "Disease"),
}
gold = {
    ("m1", "TREATS", "d1"),
    ("m1", "PREVENTS", "d1"),
}
examples = build_relation_examples(entities, schema, gold, negative_ratio=1)
assert examples[0]["labels"] == ["PREVENTS", "TREATS"]
assert examples[1]["labels"] == ["NO_RELATION"]
print(examples)

def overlap_patterns(triples: list[tuple[str, str, str]]) -> set[str]:
    patterns = set()
    for index, first in enumerate(triples):
        first_subject, _, first_object = first
        for second in triples[index + 1:]:
            second_subject, _, second_object = second
            same_pair = (
                first_subject == second_subject
                and first_object == second_object
            )
            shared_entities = (
                {first_subject, first_object}
                & {second_subject, second_object}
            )
            if same_pair:
                patterns.add("EPO")
            elif shared_entities:
                patterns.add("SEO")
    return patterns or {"Normal"}


assert overlap_patterns([("A", "r1", "B"), ("C", "r2", "D")]) == {"Normal"}
assert overlap_patterns([("A", "r1", "B"), ("A", "r2", "C")]) == {"SEO"}
assert overlap_patterns([("A", "r1", "B"), ("A", "r2", "B")]) == {"EPO"}
mixed = [("A", "r1", "B"), ("A", "r2", "B"), ("A", "r3", "C")]
assert overlap_patterns(mixed) == {"EPO", "SEO"}
print(overlap_patterns(mixed))

52. 将 head/tail 二元概率解码为合法 span

每个 head 选择其后最近且未使用的合法 tail，返回半开区间，并支持 mask 与最大长度。

✅ 参考答案
Python
def decode_spans(
    head_scores: list[float],
    tail_scores: list[float],
    threshold: float = 0.5,
    max_span_length: int = 20,
    valid_mask=None,
) -> list[tuple[int, int]]:
    if len(head_scores) != len(tail_scores):
        raise ValueError("head/tail 长度不一致")
    if valid_mask is None:
        valid_mask = [True] * len(head_scores)
    if len(valid_mask) != len(head_scores):
        raise ValueError("mask 长度不一致")

    heads = [
        index for index, score in enumerate(head_scores)
        if valid_mask[index] and score >= threshold
    ]
    tails = [
        index for index, score in enumerate(tail_scores)
        if valid_mask[index] and score >= threshold
    ]
    used_tails = set()
    spans = []
    for head in heads:
        candidates = [
            tail for tail in tails
            if tail not in used_tails
            and head <= tail
            and tail - head + 1 <= max_span_length
        ]
        if candidates:
            tail = min(candidates)
            used_tails.add(tail)
            spans.append((head, tail + 1))
    return spans


heads = [0.1, 0.9, 0.2, 0.8, 0.1, 0.99]
tails = [0.1, 0.2, 0.9, 0.1, 0.8, 0.99]
mask = [True, True, True, True, True, False]
assert decode_spans(heads, tails, 0.5, 3, mask) == [(1, 3), (3, 5)]
assert decode_spans([0.9, 0.0], [0.0, 0.9], 0.5, 1) == []
print("ok")

53. 实现 CasRel 的级联三元组解码

先解码全部主体，再对每个主体的每个关系通道解码全部客体；必须保留 SEO 与 EPO。

✅ 参考答案
Python
def decode_spans(head_scores: list[float], tail_scores: list[float], threshold: float):
    heads = [i for i, score in enumerate(head_scores) if score >= threshold]
    tails = [i for i, score in enumerate(tail_scores) if score >= threshold]
    spans, used = [], set()
    for head in heads:
        possible = [tail for tail in tails if tail >= head and tail not in used]
        if possible:
            tail = min(possible)
            used.add(tail)
            spans.append((head, tail + 1))
    return spans


def decode_casrel(
    subject_heads: list[float],
    subject_tails: list[float],
    object_scores_by_subject: dict,
    threshold: float = 0.5,
) -> list[dict]:
    triples = {}
    subjects = decode_spans(subject_heads, subject_tails, threshold)
    for subject_span in subjects:
        relation_channels = object_scores_by_subject.get(subject_span, {})
        for relation, scores in relation_channels.items():
            object_spans = decode_spans(scores["heads"], scores["tails"], threshold)
            for object_span in object_spans:
                key = subject_span, relation, object_span
                triples[key] = {
                    "subject_span": subject_span,
                    "predicate": relation,
                    "object_span": object_span,
                }
    return [triples[key] for key in sorted(triples)]

54. 合并重复三元组但保留全部证据

用标准实体 ID 和关系作为事实键；证据按 evidence_id 去重，不能只保留最高分句子。

✅ 参考答案

def merge_evidence(claims: list[dict]) -> list[dict]:
    merged = {}
    for claim in claims:
        key = claim["subject_id"], claim["predicate"], claim["object_id"]
        if key not in merged:
            merged[key] = {
                "subject_id": key[0],
                "predicate": key[1],
                "object_id": key[2],
                "evidence": {},
            }
        evidence = claim["evidence"]
        evidence_id = evidence["evidence_id"]
        previous = merged[key]["evidence"].get(evidence_id)
        if previous is None or evidence["confidence"] > previous["confidence"]:
            merged[key]["evidence"][evidence_id] = dict(evidence)

    result = []
    for key in sorted(merged):
        item = merged[key]
        evidence_list = [item["evidence"][eid] for eid in sorted(item["evidence"])]
        result.append({
            "subject_id": item["subject_id"],
            "predicate": item["predicate"],
            "object_id": item["object_id"],
            "max_confidence": max(ev["confidence"] for ev in evidence_list),
            "evidence": evidence_list,
        })
    return result

55. 计算严格三元组指标并进行错误归因

严格键包含文档、主体 span/type、关系、客体 span/type；诊断方向错、关系错、边界/类型错和虚假预测。

✅ 参考答案
Python
def evaluate_triples(gold: list[dict], predicted: list[dict]) -> dict:
    def key(item):
        return (
            item["doc_id"],
            tuple(item["subject"]),
            item["predicate"],
            tuple(item["object"]),
        )

    gold_map = {key(item): item for item in gold}
    pred_map = {key(item): item for item in predicted}
    exact = set(gold_map) & set(pred_map)
    unmatched_gold = [gold_map[k] for k in gold_map.keys() - exact]
    unmatched_pred = [pred_map[k] for k in pred_map.keys() - exact]
    remaining = unmatched_gold[:]
    errors = {
        "direction_error": [],
        "relation_confusion": [],
        "entity_boundary_or_type": [],
        "spurious": [],
        "missed": unmatched_gold,
    }

    def overlaps(first, second):
        return max(first[0], second[0]) < min(first[1], second[1])

    for prediction in unmatched_pred:
        paired = None
        for truth in remaining:
            if prediction["doc_id"] != truth["doc_id"]:
                continue
            if (
                prediction["predicate"] == truth["predicate"]
                and tuple(prediction["subject"]) == tuple(truth["object"])
                and tuple(prediction["object"]) == tuple(truth["subject"])
            ):
                errors["direction_error"].append({"gold": truth, "predicted": prediction})
                paired = truth
                break
            if (
                tuple(prediction["subject"]) == tuple(truth["subject"])
                and tuple(prediction["object"]) == tuple(truth["object"])
                and prediction["predicate"] != truth["predicate"]
            ):
                errors["relation_confusion"].append({"gold": truth, "predicted": prediction})
                paired = truth
                break
            same_relation = prediction["predicate"] == truth["predicate"]
            subject_overlap = overlaps(prediction["subject"], truth["subject"])
            object_overlap = overlaps(prediction["object"], truth["object"])
            if same_relation and subject_overlap and object_overlap:
                errors["entity_boundary_or_type"].append({
                    "gold": truth, "predicted": prediction
                })
                paired = truth
                break
        if paired is None:
            errors["spurious"].append(prediction)
        else:
            remaining.remove(paired)

    tp = len(exact)
    fp = len(pred_map) - tp
    fn = len(gold_map) - tp
    precision = tp / (tp + fp) if tp + fp else 0.0
    recall = tp / (tp + fn) if tp + fn else 0.0
    f1 = 2 * precision * recall / (precision + recall) if precision + recall else 0.0
    return {
        "tp": tp, "fp": fp, "fn": fn,
        "precision": precision, "recall": recall, "f1": f1,
        "errors": errors,
    }

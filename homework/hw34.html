🔄 循环神经网络综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：循环神经网络：词嵌入、RNN 与文本生成

55 道练习，覆盖序列数据与 NLP 管线、分词与词表、词嵌入、RNN 隐藏状态与形状系统、BPTT 与梯度问题、滑动窗口数据集和自回归文本生成。

一、选择题（30 题）

1. 序列数据区别于普通独立样本的关键特征是什么？

A. 序列数据一定与钟表时间有关
B. 元素的排列顺序本身携带信息
C. 序列数据不能转成张量
D. 序列数据没有标签

✅ 参考答案

答案：B。 “我爱你”和“你爱我”包含相同的词但含义不同；顺序可以是词序、帧顺序或事件顺序，不一定是物理时间。

2. 文本进入神经网络前的常见管线顺序是什么？

A. 词嵌入 → 分词 → 清洗 → 词表映射
B. 词表映射 → 清洗 → 分词 → 词嵌入
C. 分词 → 词嵌入 → 清洗 → 词表映射
D. 清洗规范化 → 分词 → 词表映射为整数 ID → 词嵌入 → 序列模型

✅ 参考答案

答案：D。 文字必须先数值化：清洗、切分成 token、映射成整数索引，再由嵌入层转成稠密向量后进入模型。

3. 词表（Vocabulary）指的是什么？

A. token 与整数 ID 之间的双向映射
B. 语料中所有句子的列表
C. 词向量矩阵本身
D. 模型输出的概率分布

✅ 参考答案

答案：A。 词表负责 token↔ID 的互查；语料按词表替换后得到整数索引序列，嵌入矩阵才是向量参数。

4. 用 list(set(tokens)) 构建词表的主要问题是什么？

A. 会包含重复词
B. 内存占用过大
C. 集合顺序不稳定，不同运行的 ID 可能变化，导致已保存模型与词表错位
D. 无法处理中文

✅ 参考答案

答案：C。 模型参数与词表 ID 一一对应，映射一变权重就失效；应使用 dict.fromkeys 等保持首次出现顺序的方法。

5. 特殊 token <unk> 的作用是什么？

A. 补齐批次内不同长度的序列
B. 表示词表中没有出现的 token
C. 表示句子结束
D. 表示句子开始

✅ 参考答案

答案：B。 <pad> 负责补齐长度，<eos>/<bos> 表示边界，<unk> 兜底词表外的词，避免查表失败。

6. “猫”的 ID 是 27、“狗”的 ID 是 28，两个 ID 的正确解读是什么？

A. 它们只是查表地址，不含大小或相似度含义
B. 狗比猫“大 1”
C. 猫和狗比 ID 为 900 的词更相似
D. ID 可以直接作为浮点特征参与运算

✅ 参考答案

答案：A。 整数 ID 是离散索引；要表达语义相似性需要经过 Embedding 等编码，不能把 ID 当连续数值。

7. One-hot 表示词的两个主要问题是什么？

A. 无法区分不同的词
B. 需要先训练才能得到
C. 维度太低、信息太少
D. 维度等于词表大小而非常高，且任意两个词的向量正交、无法表达相似度

✅ 参考答案

答案：D。 词嵌入用 $E\ll V$ 的稠密向量解决这两个问题，语义相近的词在训练后可以获得相近的向量。

8. 词表大小 $V$、嵌入维度 $E$ 时，嵌入矩阵的形状是什么？

A. [E, V]
B. [V, V]
C. [V, E]
D. [E, E]

✅ 参考答案

答案：C。 $W_{\mathrm{emb}}\in\mathbb{R}^{V\times E}$；输入 ID $i$ 时返回第 $i$ 行，等价于 one-hot 左乘该矩阵，但直接查表更高效。

9. nn.Embedding 的输入张量应使用什么数据类型？

A. torch.float32
B. torch.long 整数索引
C. torch.bool
D. 任意类型均可

✅ 参考答案

答案：B。 嵌入层按整数索引查表；传入浮点张量会报错。

10. 形状 [B, T] 的 token ID 批次经过 nn.Embedding(V, E) 后的输出形状是什么？

A. [B, T, E]
B. [B, E]
C. [B, T, V]
D. [T, E]

✅ 参考答案

答案：A。 每个 ID 被替换成 $E$ 维向量，形状规律为 $[B,T]\to[B,T,E]$。

11. nn.Embedding(..., padding_idx=0) 的效果是什么？

A. 删除所有 ID 为 0 的输入
B. 输入不允许出现 0
C. 词表大小减 1
D. ID 0 对应的嵌入行不参与梯度更新

✅ 参考答案

答案：D。 padding 位置的向量保持固定（默认初始化为 0 且梯度为 0），避免填充符干扰学习。

12. nn.Embedding.from_pretrained(weights, freeze=False) 的含义是什么？

A. 随机初始化并冻结
B. 加载预训练向量且永远不再更新
C. 加载预训练向量，并在训练中继续微调
D. 只加载词表不加载向量

✅ 参考答案

答案：C。 freeze=True 才是冻结；使用预训练向量时，词表 ID 必须与向量行号严格对齐，否则语义会错位。

13. RNN 展开图里画出的多个 Cell，实质是什么？

A. 多个互相独立、各有参数的网络
B. 同一个循环计算用同一组参数沿时间轴重复执行
C. 多层堆叠的不同层
D. 集成学习中的多个基模型

✅ 参考答案

答案：B。 展开只是把时间步画开；所有时间步共享 $W_{ih}$、$W_{hh}$ 和偏置，这就是 RNN 的参数共享。

14. PyTorch 普通 tanh RNN 的隐藏状态更新公式是哪一个？

A. $h_t=\tanh(W_{ih}x_t+b_{ih}+W_{hh}h_{t-1}+b_{hh})$
B. $h_t=W_{ih}x_t$
C. $h_t=\mathrm{softmax}(W_{hh}h_{t-1})$
D. $h_t=\mathrm{ReLU}(x_t+h_{t-1})$

✅ 参考答案

答案：A。 当前输入与旧状态分别线性变换后相加，再经过 tanh；任务输出还需额外的线性层从 $h_t$ 计算。

15. “拼接式”写法 $h_t=\tanh(W_{\mathrm{combined}}[h_{t-1};x_t]+b)$ 与分开写法的关系是什么？

A. 拼接式的参数更少
B. 两者结果不同，不能混用
C. 拼接式不需要偏置
D. 数学等价，$W_{\mathrm{combined}}=[W_{hh}, W_{ih}]$

✅ 参考答案

答案：D。 分块矩阵乘法展开即得两个线性变换相加；PyTorch 内部保留分开的 weight_ih 与 weight_hh，便于实现和优化。

16. 单层、单向、带偏置、input_size=I、hidden_size=H 的 nn.RNN 参数量是多少？

A. $HI$
B. $H^2$
C. $HI+H^2+2H$
D. $(H+I)^2$

✅ 参考答案

答案：C。 weight_ih 为 $H\times I$、weight_hh 为 $H\times H$，再加 bias_ih 和 bias_hh 各 $H$ 个。

17. 输入序列从 10 个时间步增加到 100 个，RNN 的参数量如何变化？

A. 不变，只增加计算次数和中间状态
B. 变为 10 倍
C. 按 $\sqrt{10}$ 倍增长
D. 需要重新初始化模型

✅ 参考答案

答案：A。 所有时间步共享同一组参数，序列长度只影响展开的计算量，不影响参数量。

18. nn.RNN 的 input_size 参数表示什么？

A. 序列的时间步数
B. 每个时间步输入特征的维度，文本任务中通常等于词嵌入维度
C. 批量大小
D. 词表大小

✅ 参考答案

答案：B。 它与 seq_len 是最容易混淆的一对：前者是单步特征维度 $I$，后者是时间步数 $T$。

19. batch_first=True 会改变哪些张量的布局？

A. 所有输入输出，包括 $h_0$ 和 $h_n$
B. 只有 $h_0$ 和 $h_n$
C. 什么都不改变
D. 只有输入 $X$ 和 output 变为 $[B,T,\cdot]$，$h_0$/$h_n$ 仍是 $[DL,B,H]$

✅ 参考答案

答案：D。 隐藏状态布局不受 batch_first 影响，这是最常见的形状陷阱之一。

20. output 与 h_n 的区别是什么？

A. 两者完全相同
B. output 是最终预测概率，h_n 是损失值
C. output 是最后一层在每个时间步的隐藏状态；h_n 是每层（每方向）在最终时间步的隐藏状态
D. output 只有一个时间步

✅ 参考答案

答案：C。 单层、单向、无 padding 时 output[:, -1, :] 与 h_n[-1] 数值相同；多层、双向或变长序列时不能机械套用。

21. hidden_size=H 的双向 RNN，output 的特征维是多少？

A. $2H$
B. $H$
C. $H/2$
D. $H^2$

✅ 参考答案

答案：A。 正向与反向的隐藏状态在特征维拼接；双向结构适合离线分类标注，不适合严格的因果生成。

22. 构造 nn.RNN(128, 256, num_layers=1, dropout=0.2)，dropout 的实际效果是什么？

A. 每个时间步随机屏蔽 20% 输入
B. 不生效——层间 Dropout 只在 num_layers > 1 时作用于层与层之间
C. 隐藏状态直接乘 0.8
D. 训练时长增加 20%

✅ 参考答案

答案：B。 RNN 的 dropout 参数作用于多层之间的传递；单层时 PyTorch 会给出警告且不生效。

23. 用 RNN 预测词表中的下一个词，还需要什么组件？

A. 不需要，output 就是概率
B. 一个池化层
C. 再加一层 Embedding
D. 一个线性层把隐藏状态投影成词表上的 $V$ 维 logits

✅ 参考答案

答案：D。 RNN 层的职责是更新隐藏状态；$z_t=W_{hy}h_t+b_y$ 才是词表 logits，训练时交给交叉熵。

24. 关于模型输出与 CrossEntropyLoss 的配合，正确的是？

A. 模型末尾必须加 Softmax
B. 模型末尾必须加 Sigmoid
C. 直接输出 raw logits，损失内部已含数值稳定的 LogSoftmax
D. 需要先手动取对数

✅ 参考答案

答案：C。 预先做 Softmax 会重复归一化并损失数值稳定性。

25. 语言模型训练时，目标序列通常怎样构造？

A. 与输入完全相同
B. 把真实序列右移一位：位置 $t$ 的目标是位置 $t+1$ 的真实 token
C. 把输入倒序
D. 随机打乱输入

✅ 参考答案

答案：B。 若目标不右移，模型会被训练成复读当前输入，而不是预测下一个词。

26. 语料索引为 [0, 1, 2, 3, 40, 0, 4, 5, 6]，窗口长度 5、输入 x = [1, 2, 3, 40, 0]，对应的目标 y 是什么？

A. [1, 2, 3, 40, 0]
B. [0, 1, 2, 3, 40]
C. [3, 40, 0, 4, 5]
D. [2, 3, 40, 0, 4]

✅ 参考答案

答案：D。 目标是输入整体右移一位的序列，每个位置都预测自己的下一个 token。

27. 普通 RNN 梯度消失的根本原因是什么？

A. 时间反向传播中反复连乘循环权重与激活函数导数，范数持续小于 1 时梯度指数衰减
B. 学习率太大
C. 批量太小
D. 词表太大

✅ 参考答案

答案：A。 tanh 饱和区导数很小，连乘让远距离时间步的梯度趋近 0，这就是长期依赖难学的原因；LSTM/GRU 用门控缓解。

28. 梯度范数裁剪（clip_grad_norm_）的作用边界是什么？

A. 同时解决梯度消失和爆炸
B. 只解决梯度消失
C. 主要抑制梯度爆炸，不能从根本上解决梯度消失
D. 让损失单调下降

✅ 参考答案

答案：C。 裁剪把过大的梯度范数缩回阈值内，防止一步更新摧毁参数；消失问题要靠结构（门控）或更短的依赖路径解决。

29. 分块训练超长序列并跨块传递隐藏状态时，hidden = hidden.detach() 的目的是什么？

A. 把隐藏状态清零
B. 把隐藏状态从计算图中分离，避免反向传播穿越所有历史块导致图无限延长
C. 把隐藏状态移动到 CPU
D. 提高数值精度

✅ 参考答案

答案：B。 这就是截断 BPTT：保留数值作为记忆，但梯度只在当前块内传播。若各 batch 是独立随机窗口，直接重置隐藏状态更清晰。

30. 生成文本时把温度设为 $\tau > 1$，输出分布会怎样变化？

A. 分布更尖锐，几乎变成贪心
B. 概率和不再为 1
C. 一定生成更长的句子
D. 分布更平坦，采样更多样，也更容易出错

✅ 参考答案

答案：D。 $p_i=\mathrm{softmax}(z_i/\tau)$：$\tau<1$ 更保守，$\tau>1$ 更发散；Top-k 则先截断候选集合再采样。

二、简答题（15 题）

31. 序列数据与普通独立样本有什么区别？RNN 靠什么机制利用历史信息？

✅ 参考答案

普通前馈网络把每个输入当作彼此独立的固定长度样本，打乱样本顺序不影响任务含义；序列数据的元素顺序本身携带信息，词序、帧序改变则语义改变。RNN 显式维护一个随时间更新的隐藏状态 $h_t$：每个时间步用当前输入 $x_t$ 和上一时刻状态 $h_{t-1}$ 计算新状态，让过去的信息以压缩形式参与当前计算。

32. 区分 token、词表和语料索引序列，并说明 <pad>、<unk>、<eos> 各自的作用。

✅ 参考答案

token 是分词后的基本单位；词表是 token 与整数 ID 的双向映射；语料索引序列是整篇语料按词表替换后的整数序列，才是模型真正接收的输入。<pad> 把不同长度的序列补齐成同一批次的矩形张量；<unk> 兜底词表外的 token，避免查表失败；<eos> 显式标记句子边界，生成时也可作为停止信号（更完整的系统还有 <bos> 表示开始）。

33. 对比 One-hot 与词嵌入两种词表示方法。

✅ 参考答案

One-hot 用 $V$ 维向量表示词，只有对应位置为 1：表示无歧义，但维度等于词表大小、非常稀疏，且任意两个词的向量正交，完全无法表达“猫和狗比猫和汽车更相似”。词嵌入把每个词映射为 $E\ll V$ 维稠密向量，参数可训练；在合适目标下，共现环境相似的词会获得相近的向量，常用余弦相似度衡量。查表取行在数学上等价于 one-hot 左乘嵌入矩阵，但实现上更高效。

34. “嵌入的 128 维就是语义、词性、褒贬等 128 个属性”这种说法应如何澄清？

✅ 参考答案

这是帮助建立直觉的类比，不是事实。实际训练出的嵌入中，单独某一维通常没有可直接命名的独立含义，语义信息分布在整个向量空间里，由训练目标共同塑造。能观察到的是整体几何关系——相似词彼此靠近、某些方向大致对应某类变化；二维可视化图只是高维空间降维后的近似投影。

35. output 和 h_n 分别是什么？“取 output[:, -1, :] 等于 h_n[-1]”这条规则什么时候失效？

✅ 参考答案

output 是最后一层在每个时间步的隐藏状态（batch_first=True 时形状 $[B,T,DH]$）；h_n 是每一层、每个方向在最终时间步的隐藏状态（$[DL,B,H]$）。两者在单层、单向、无 padding 时数值一致。失效场景：多层时 h_n 含每层状态而 output 只有最后一层；双向时 output 拼接了正反两个方向且反向的“最终”在序列开头；变长 padding 序列时张量最后一个位置可能只是填充，必须按真实长度取最后有效状态。

36. batch_first=True 改变哪些张量布局、不改变哪些？为什么这是常见陷阱？

✅ 参考答案

它只把输入 $X$ 和 output 的前两维从 $[T,B,\cdot]$ 换成 $[B,T,\cdot]$；$h_0$ 和 $h_n$ 永远保持 $[D\times L, B, H]$。陷阱在于使用者容易假定“所有张量都变成 batch 在前”，于是按 $[B,L,H]$ 初始化 $h_0$ 或错误索引 $h_n$，形状恰好可能广播或转置后不报错，导致隐蔽的语义错误。

37. RNN 的参数共享有什么意义？写出嵌入层、单层 RNN、输出层三部分的参数量表达式。

✅ 参考答案

共享让参数量与序列长度解耦：不管序列多长，都用同一组 $W_{ih}$、$W_{hh}$ 处理每个时间步，还使模型能把学到的转移规律应用到任意位置。参数量：嵌入层 $VE$；单层单向带偏置 RNN 为 $HI+H^2+2H$；输出线性层 $VH+V$。大词表任务中嵌入层和输出层常占参数大头。

38. 序列到单值、序列到序列、自回归生成三类任务分别怎样从 RNN 的输出中取结果？

✅ 参考答案

序列到单值（情感分析等）：取最后一个有效时间步的隐藏状态，或对全部有效时间步做平均/最大池化、注意力加权汇总，再接分类头。序列到序列（词性标注等逐位输出）：把整个 output 送入任务头，每个时间步都有监督。自回归生成：训练时用右移目标让每个位置预测下一个 token；推理时把上一步生成的 token 作为下一步输入，循环进行并持续传递隐藏状态。

39. 什么是 teacher forcing？它带来的 exposure bias 指什么？

✅ 参考答案

训练语言模型时，位置 $t$ 的输入是真实序列中的 token 而不是模型自己生成的 token，这种“右移监督”即 teacher forcing，它让训练稳定且可并行计算所有位置的损失。exposure bias 指训练与推理的输入分布不一致：推理时模型只能以自己上一步的输出为条件，一旦出错，后续输入落入训练时没见过的分布，误差可能逐步累积放大。

40. 普通 RNN 为什么会梯度消失和梯度爆炸？各自的常用对策是什么？

✅ 参考答案

沿时间反向传播时，梯度要反复乘以循环权重和激活导数：连乘范数持续小于 1 时指数衰减（tanh 饱和区导数很小加剧此问题），远处时间步几乎收不到梯度，即梯度消失，长期依赖难学；持续大于 1 时指数增长，即梯度爆炸，表现为损失震荡或 NaN。对策：爆炸用梯度范数裁剪、减小学习率；消失主要靠结构改进——LSTM/GRU 的门控与近似线性记忆通路、更短的依赖路径（注意力/Transformer），普通技巧无法根治。

41. 什么是截断 BPTT？跨块传递隐藏状态时为什么要 detach()？独立随机窗口的场景又该怎么处理？

✅ 参考答案

超长连续序列难以整体反向传播，截断 BPTT 把序列切块训练：前向时隐藏状态跨块传递以保留记忆，反向传播只在当前块内进行。hidden.detach() 把状态从计算图分离，否则图会随块数无限延长，显存与计算都不可承受。若每个 batch 是彼此独立的随机滑窗（如本讲歌词项目），块间没有连续性，直接在每个 batch 开始时按 x.size(0) 重置全零隐藏状态更清晰。

42. 序列总长 $N$、窗口长 $T$、步长 $S$ 时，滑窗样本数应是多少？课件用 word_count // num_chars 当数据集长度的问题在哪里？

✅ 参考答案

输入窗口起点 start 需满足 start + T + 1 <= N（目标要右移一位），起点最大为 $N-T-1$，样本数为 $\lfloor (N-T-1)/S\rfloor+1$。课件写法用整除声明长度、却把 idx 直接当起点，两套规则不一致：可能漏掉大量重叠窗口、覆盖不到序列尾部，甚至在边界处发生越界读取。正确做法是先按统一规则生成合法起点列表，__len__ 和 __getitem__ 都以它为准。

43. 对比贪心解码、温度采样和 Top-k 采样。

✅ 参考答案

贪心每步取最大 logit：确定、稳定，但容易陷入重复和平庸。温度采样先把 logits 除以 $\tau$ 再 softmax 后按概率抽样：$\tau<1$ 分布更尖锐、更保守，$\tau>1$ 更平坦、更多样但更易出错。Top-k 先只保留得分最高的 $k$ 个候选，再在其中归一化采样，砍掉长尾防止采到明显不合理的词。实践中温度与 Top-k（或 Top-p）常组合使用，在多样性与合理性之间折中。

44. 文本生成模型的 checkpoint 除了 state_dict 还应保存什么？为什么？

✅ 参考答案

还应保存：index_to_token 与 token_to_index 词表、结构超参数（词表大小、嵌入维度、隐藏维度、层数）、分词与文本规范化规则、特殊 token 的 ID。因为权重的每一行嵌入、输出层的每一列都与特定 ID 绑定，词表丢失或重建顺序不同，加载的权重就无法正确解释输入输出；超参数不符则连模型都实例化不出来。加载端要用相同的类与配置实例化后再 load_state_dict。

45. 困惑度（PPL）是怎么定义的？它能完整衡量生成质量吗？

✅ 参考答案

$\mathrm{PPL}=\exp(\mathcal{L}_{\mathrm{CE}})$，即平均交叉熵的指数。它衡量模型对真实下一个词的平均“惊讶程度”，越低说明模型给真实 token 分配的概率越高。但 PPL 只评估概率拟合，不能完整衡量文本的事实正确性、全局连贯性、创造性和安全性；两个 PPL 相近的模型生成体验可能差别很大，实际评价还需人工评审或任务指标配合。

46. 构建顺序稳定、带特殊 token 的词表

用 dict.fromkeys 实现可复现的词表构建，包含 <pad>/<unk>/<eos>，并验证：重复词只保留一次、未知词映射到 <unk>。

✅ 参考答案
Python
SPECIAL_TOKENS = ["<pad>", "<unk>", "<eos>"]


def build_vocab(sentences):
    corpus_tokens = [
        token
        for sentence in sentences
        for token in sentence + ["<eos>"]
    ]
    # 语料里也会出现 <eos>，去重时要排除特殊 token，否则词表出现重复项
    index_to_token = SPECIAL_TOKENS + [
        token
        for token in dict.fromkeys(corpus_tokens)
        if token not in SPECIAL_TOKENS
    ]
    token_to_index = {
        token: index for index, token in enumerate(index_to_token)
    }
    unk_id = token_to_index["<unk>"]
    corpus_ids = [
        token_to_index.get(token, unk_id) for token in corpus_tokens
    ]
    return index_to_token, token_to_index, corpus_ids


sentences = [["我", "爱", "学习"], ["我", "爱", "音乐"]]
index_to_token, token_to_index, corpus_ids = build_vocab(sentences)

assert index_to_token[:3] == SPECIAL_TOKENS
assert len(index_to_token) == len(set(index_to_token))   # 无重复项
assert index_to_token.count("我") == 1
assert token_to_index["<eos>"] == 2
assert token_to_index.get("不存在", token_to_index["<unk>"]) == 1
print(index_to_token)
print(corpus_ids)

51. 实现滑动窗口 Dataset

✅ 参考答案

import torch
from torch.utils.data import Dataset


class NextTokenDataset(Dataset):
    def __init__(self, corpus_ids, seq_len, stride=1):
        self.corpus = torch.as_tensor(corpus_ids, dtype=torch.long)
        self.seq_len = seq_len
        self.starts = list(
            range(0, len(self.corpus) - seq_len, stride)
        )

    def __len__(self):
        return len(self.starts)

    def __getitem__(self, index):
        start = self.starts[index]
        stop = start + self.seq_len
        return self.corpus[start:stop], self.corpus[start + 1 : stop + 1]

52. 搭建 TextGenerator 并验证形状链

实现 Embedding→RNN→Linear 三层结构，验证 [B,T]→logits [B,T,V]、hidden [L,B,H] 的完整形状链。

✅ 参考答案
Python
import torch
from torch import nn


class TextGenerator(nn.Module):
    def __init__(self, vocab_size, embedding_dim=32,
                 hidden_size=64, num_layers=1):
        super().__init__()
        self.hidden_size = hidden_size
        self.num_layers = num_layers
        self.embedding = nn.Embedding(vocab_size, embedding_dim,
                                      padding_idx=0)
        self.rnn = nn.RNN(embedding_dim, hidden_size,
                          num_layers=num_layers, batch_first=True)
        self.output = nn.Linear(hidden_size, vocab_size)

    def forward(self, token_ids, hidden=None):
        embedded = self.embedding(token_ids)
        states, hidden = self.rnn(embedded, hidden)
        return self.output(states), hidden

    def init_hidden(self, batch_size, device):
        return torch.zeros(self.num_layers, batch_size,
                           self.hidden_size, device=device)


model = TextGenerator(vocab_size=50)
x = torch.randint(0, 50, (8, 12))
hidden = model.init_hidden(batch_size=8, device="cpu")
logits, hidden_out = model(x, hidden)

assert logits.shape == (8, 12, 50)      # [B, T, V]
assert hidden_out.shape == (1, 8, 64)   # [L, B, H]
print(logits.shape, hidden_out.shape)

54. 实现温度 + Top-k 采样函数

实现 sample_next_token(logits, temperature, top_k)，验证：Top-k 采样结果一定落在得分最高的 k 个候选中，温度必须为正。

✅ 参考答案
Python
import torch


def sample_next_token(logits, temperature=1.0, top_k=20):
    if temperature <= 0:
        raise ValueError("temperature must be positive")
    logits = logits / temperature
    if top_k is not None:
        top_k = min(top_k, logits.numel())
        values, indices = torch.topk(logits, top_k)
        probs = torch.softmax(values, dim=-1)
        choice = torch.multinomial(probs, 1)
        return indices[choice].item()
    probs = torch.softmax(logits, dim=-1)
    return torch.multinomial(probs, 1).item()


torch.manual_seed(0)
logits = torch.randn(50)
top3 = set(torch.topk(logits, 3).indices.tolist())

for _ in range(200):
    assert sample_next_token(logits, temperature=0.8, top_k=3) in top3

try:
    sample_next_token(logits, temperature=0.0)
except ValueError as error:
    print("caught:", error)
print("all passed")

55. 自回归生成循环

用未训练的 TextGenerator 实现生成循环：逐步传递 hidden、用 .item() 取采样结果，遇到 <eos>（ID 2）停止或到达最大步数。

✅ 参考答案
Python
import torch

torch.manual_seed(0)
model = TextGenerator(vocab_size=50)    # 沿用第 52 题的类定义
model.eval()

EOS_ID = 2
max_new_tokens = 20


@torch.inference_mode()
def generate(model, start_id, max_new_tokens):
    hidden = model.init_hidden(batch_size=1, device="cpu")
    current_id = start_id
    generated = [current_id]

    for _ in range(max_new_tokens):
        x = torch.tensor([[current_id]], dtype=torch.long)
        logits, hidden = model(x, hidden)      # hidden 必须迭代传递
        next_logits = logits[0, -1]
        probs = torch.softmax(next_logits / 0.9, dim=-1)
        current_id = torch.multinomial(probs, 1).item()
        generated.append(current_id)
        if current_id == EOS_ID:
            break

    return generated


ids = generate(model, start_id=5, max_new_tokens=max_new_tokens)
assert 2 <= len(ids) <= max_new_tokens + 1
assert all(isinstance(i, int) for i in ids)
print(ids)

🧭 RAG 项目背景、原理与工程结构综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：RAG 项目背景、原理与工程结构🚀 所属项目：EduRAG：企业私有知识智能问答

40 道练习，围绕 EduRAG 第一章：企业私有知识缺口与幻觉、微调 vs RAG、检索增强原理、FAQ→RAG 五步流程、工程目录与依赖、以及 Ollama / LangChain / Milvus 的分工。重点练「何时走 FAQ、何时走 RAG、上下文如何拼进大模型」。

建议先独立作答，再展开参考答案。代码题只用纯 Python，不依赖真实数据库或模型服务。

一、选择题（20 题）

1. 多数通用大模型在企业场景里最常见的局限是什么？

A. 完全无法生成自然语言
B. 主要依赖训练时见过的数据，难以动态获得最新知识与企业私有知识
C. 只能做图像分类
D. 必须联网才能启动 Python

✅ 参考答案

答案：B。 通用模型的参数里固化的是训练截止前的公开分布；企业内部文档、制度、产品细节通常不在其中，也难以及时更新。

2. 模型「幻觉」在问答场景中通常指什么？

A. GPU 过热自动关机
B. 生成内容流畅，但与事实、文档或业务现状不符
C. 检索结果永远为空
D. 向量维度必须是偶数

✅ 参考答案

答案：B。 幻觉的危险在于：回答看起来合理，却可能编造政策、价格或操作步骤，因此企业问答需要可核对的知识来源。

3. 课件中为缓解私有知识缺口给出的两条主路径是？

A. 只加大模型参数量；只增加上下文窗口
B. 基于企业私有知识的垂直微调；基于私有知识库的 RAG 问答
C. 只用正则；只用规则引擎
D. 删除全部日志；关闭缓存

✅ 参考答案

答案：B。 微调把领域知识写进（或对齐进）模型行为；RAG 在推理时检索外部知识再生成。二者可组合，但成本、更新频率与可追溯性不同。

4. RAG 的英文全称是？

A. Random Attention Graph
B. Retrieval-Augmented Generation
C. Recurrent Autoencoder Gradient
D. Remote API Gateway

✅ 参考答案

答案：B。 核心思想是：先检索相关知识，再把检索结果作为条件增强生成，而不是只靠参数记忆。

5. 相对「把全部企业知识都微调进模型」，RAG 更直接解决的是哪类问题？

A. 彻底消除所有计算成本
B. 知识可外部更新、可按需检索，并减少对训练语料覆盖不到的事实的臆造
C. 让模型不再需要 Prompt
D. 取消向量检索

✅ 参考答案

答案：B。 文档更新后通常只需重建/增量更新索引，而不必每次全量微调；同时回答可引用检索片段，便于核对。

6. EduRAG 项目流程的第一步是什么？

A. 立刻把所有 PDF 丢进大模型上下文
B. 将已有 FAQ 数据写入 MySQL
C. 先训练一个新的 Tokenizer
D. 先部署 Kubernetes

✅ 参考答案

答案：B。 高频、口径稳定的问答适合沉淀为 FAQ；命中时可短路昂贵的向量检索与生成。

7. 课件中 FAQ 相似度命中阈值取 0.85 的工程含义是？

A. 任意 query 都必须返回同一答案
B. query 与库中问题足够相似时，直接返回对应答案；否则进入 RAG
C. 只允许返回 85 条结果
D. 向量维度固定为 85

✅ 参考答案

答案：B。 阈值是路由开关：高置信 FAQ 走确定性答复，低置信或复杂问题交给知识库检索增强。

8. 当 FAQ 相似度低于阈值时，系统下一步应进入？

A. 直接拒绝回答且不记录日志
B. RAG 检索链路（知识库加载/向量检索/生成）
C. 随机返回一条无关 FAQ
D. 强制把阈值改成 0

✅ 参考答案

答案：B。 FAQ 未命中不代表无答案，而是需要开放域知识库检索与生成来处理更复杂或长尾问题。

9. 搭建本地知识库时，课件列出的正确顺序是？

A. 先生成答案，再删除文档
B. 文档加载 → 文档分割 → 向量化 → 写入向量库（Milvus）
C. 先微调，再关闭数据库
D. 只存文件名，不存文本

✅ 参考答案

答案：B。 切分控制检索粒度；向量化把语义变成可相似检索的表示；Milvus 负责持久化与 top-k 召回。

10. 基于 query 做 Milvus 检索时，典型动作是？

A. 把整库原文一次性塞进 Prompt，不做检索
B. 将 query 向量化，检索相似的 top-k 文本段
C. 只按文件修改时间排序
D. 随机抽样 1 条即可，无需相似度

✅ 参考答案

答案：B。 top-k 在召回覆盖与上下文长度/噪声之间折中；过大易引入无关段，过小易漏证据。

11. RAG 生成阶段送入大模型的输入通常是？

A. 仅有系统时间
B. query 与检索到的 top-k 文本段拼接后的提示
C. 仅有空字符串
D. 仅有 GPU 型号

✅ 参考答案

答案：B。 「检索增强」落在 Prompt/上下文构造：模型应优先依据检索证据作答，并在证据不足时学会拒答或澄清。

12. 项目根目录 integrated_qa_system/ 中，mysql_qa/ 与 rag_qa/ 的职责划分更合理的是？

A. 二者完全重复，可随便删一个
B. mysql_qa 侧重 FAQ/MySQL/BM25/缓存；rag_qa 侧重向量检索与 RAG 核心逻辑
C. 二者都只负责画前端页面
D. 二者都只写单元测试

✅ 参考答案

答案：B。 目录按能力边界拆分，便于独立调试 FAQ 短路与 RAG 全链路，再由根目录 main.py 集成路由。

13. base/config.py 与 config.ini 在工程中的主要作用是？

A. 把密钥和连接信息硬编码进每个业务函数
B. 集中加载配置，供日志、数据库、向量库、模型服务等模块复用
C. 替代全部业务代码
D. 专门生成假数据

✅ 参考答案

答案：B。 配置外置便于环境切换（开发/测试/生产），也减少密钥散落在源码中的风险。

14. rag_qa/core/strategy_selector.py 从命名与职责上看，最可能负责什么？

A. 选择 CPU 风扇转速
B. 根据查询意图/场景选择检索或问答策略
C. 编译 C++ 扩展
D. 清空 Milvus 全部集合且无日志

✅ 参考答案

答案：B。 EduRAG 后续章节会展开意图识别与检索策略；第一章目录已把策略选择器放在 RAG 核心路径上。

15. rank_bm25、jieba 出现在依赖中，更直接服务于哪条能力？

A. 图像超分辨率
B. 中文分词与稀疏检索（BM25）相关能力
C. 只做音频转写
D. 只做 GPU 驱动安装

✅ 参考答案

答案：B。 FAQ/文本匹配侧常用分词 + BM25；与稠密向量检索形成互补，后续章节会专项展开。

16. 课件对 Ollama 的定位是？

A. 关系型数据库
B. 帮助在本地快速运行与管理大模型的服务工具
C. 前端 CSS 框架
D. 仅用于打包 Docker 镜像的替代品

✅ 参考答案

答案：B。 本地模型运行、拉模型、提供 API，是 EduRAG 调用 Qwen 等模型的基础设施之一。

17. 课件对 LangChain 的定位是？

A. 专门替代 Git
B. 为 LLM 组件提供通用接口与编排能力，降低复杂应用开发成本
C. 只做操作系统内核调度
D. 只能训练 CNN

✅ 参考答案

答案：B。 Models、Prompts、Chains、Agents、Memory、Indexes 等抽象，用来把检索、提示与生成「链接」成可维护流水线。

18. Milvus 在本项目中的角色是？

A. 浏览器插件商店
B. 开源向量数据库，用于向量存储与相似检索
C. 仅用于存储用户密码明文
D. 仅用于渲染 Markdown

✅ 参考答案

答案：B。 知识片段向量化后写入 Milvus，在线阶段按 query 向量召回 top-k。

19. 课件要求的 Python 版本下限是？

A. Python 2.7
B. Python 3.10 及以上

20. 根目录 main.py 相对 rag_qa/main.py、mysql_qa/main.py 的更准确描述是？

A. 三者必须互相覆盖对方文件
B. 子模块可独立入口调试；根 main.py 负责集成 FAQ 与 RAG 的统一入口
C. 根 main.py 只能打印 Hello World
D. 子模块入口禁止存在

答案：B。 独立入口利于分阶段验收；集成入口落实「先 FAQ、再 RAG」的产品路径。

二、简答题（12 题）

21. 用两句话说明：为什么「模型很强」仍不足以支撑企业私有知识问答？

✅ 参考答案

通用大模型擅长语言与推理模式，但不保证掌握某企业最新制度、内部术语与未公开文档。缺少可更新的外部知识源时，它容易用「看起来合理」的内容填补空白，也就是幻觉风险上升。

22. 对比「垂直微调」与「RAG」：各举一个更适合的业务条件。

✅ 参考答案
  • 更适合微调：需要稳定改变风格/格式/任务头行为，或领域表达方式长期固定，且能承担训练与评测成本。
  • 更适合 RAG：知识频繁更新、必须可引用原文、希望控制训练成本，并把事实来源外置到知识库。
    实际系统常组合：用微调提升指令遵循，用 RAG 注入可变事实。

23. 按「检索 → 增强 → 生成」简述 RAG 原理，并指出每一步失败会怎样。

✅ 参考答案
  • 检索：按 query 找相关文档片段；失败则上下文缺失，模型只能靠参数瞎编或拒答。
  • 增强：把证据写入 Prompt/上下文；失败（拼接混乱、超长截断不当）会导致模型用错证据或忽略约束。
  • 生成：模型基于证据作答；若未约束「依据上下文」，仍可能无视检索结果产生幻觉。

24. 写出 EduRAG 课件中的五步主流程，并标明每一步的数据落点。

✅ 参考答案
  1. FAQ 入 MySQL；
  2. query 与 FAQ 相似度匹配，≥0.85 直接返回答案，否则转 RAG；
  3. 本地文档加载、切分、向量化，写入 Milvus；
  4. query 向量化后从 Milvus 取 top-k 文本段；
  5. query + top-k 拼接送入大模型生成回答。

25. 为什么要「先 FAQ、后 RAG」，而不是所有问题都走向量检索？

✅ 参考答案

FAQ 对高频标准问法延迟更低、答案口径更可控、成本更小；向量检索 + 生成更适合长尾与复杂表述。先短路高置信 FAQ，能把算力留给真正需要检索增强的问题，也减少生成漂移。

26. 若把 FAQ 阈值从 0.85 调到 0.50 或 0.98，分别可能出现什么现象？

✅ 参考答案
  • 降到 0.50：更容易「假命中」，把不太像的问题当成标准 FAQ，返回答非所问。
  • 升到 0.98：更保守，大量本可走 FAQ 的问题涌入 RAG，成本上升，且可能引入检索噪声。
    阈值应结合线上命中率、人工抽检与延迟/成本一起调。

27. 文档为什么要「分割」后再向量化？只存整篇长文档有什么问题？

✅ 参考答案

整篇过长时，相似检索粒度粗糙，命中章节可能夹带大量无关段落，Prompt 也更容易超窗。合理切分（并保留元数据/重叠）能提高召回精度，并让生成聚焦证据句段。

28. 解释 rag_qa/core/ 下 prompts.py、vector_store.py、rag_system.py 的协作关系。

✅ 参考答案

vector_store.py 负责向量写入与检索；prompts.py 管理提示模板（如何把证据与问题组织起来）；rag_system.py 编排「取证 → 填模板 → 调模型」的核心流程。分层后可单独替换检索器或模板而不重写整条链路。

29. 用一句话区分 Ollama、LangChain、Milvus 在本项目中的分工。

✅ 参考答案

Ollama 负责本地模型服务；LangChain 负责 LLM 应用编排；Milvus 负责向量存取与相似检索。

30. requirements.txt 里同时出现 pymysql、redis、pymilvus、fastapi——分别对应系统的哪一层？

✅ 参考答案
  • pymysql：FAQ/业务关系数据访问；
  • redis：缓存加速（如热点问答）；
  • pymilvus：向量知识库；
  • fastapi（及 uvicorn 等）：对外 API/服务交付层。

31. 若 FAQ 命中但业务规则已变更，系统应如何避免「高置信错误答案」？

✅ 参考答案

FAQ 也需要版本与生效期治理：更新答案、下线过期条目、变更审计，并监控用户负反馈。必要时对命中结果再做规则校验，或在关键字段变更后降低自动短路、强制走知识库最新文档。

32. 为什么企业 RAG 回答最好带来源片段/文档定位，而不仅给最终句子？

✅ 参考答案

来源让人工可复核，降低幻觉不被发现的风险；也便于追责、纠错与持续优化检索。没有溯源时，用户只能盲目信任生成文本。

33. 实现 FAQ 路由：按最高相似度与阈值决定返回答案或进入 RAG

给定 FAQ 列表 (question, answer) 与相似度函数结果，实现 route_faq(query, faqs, scores, threshold=0.85)：
- 若最高分 ≥ 阈值，返回 {"path": "faq", "answer": ..., "score": ...}；
- 否则返回 {"path": "rag", "answer": None, "score": ...}。

✅ 参考答案
Python
def route_faq(query, faqs, scores, threshold=0.85):
    if len(faqs) != len(scores):
        raise ValueError("faqs 与 scores 长度必须一致")
    if not faqs:
        return {"path": "rag", "answer": None, "score": 0.0}

    best_i = max(range(len(scores)), key=lambda i: (scores[i], -i))
    best_score = float(scores[best_i])
    if best_score >= threshold:
        return {"path": "faq", "answer": faqs[best_i][1], "score": best_score}
    return {"path": "rag", "answer": None, "score": best_score}


faqs = [("退费多久到账", "3个工作日"), ("如何改课", "联系班主任")]
assert route_faq("退费多久到账", faqs, [0.92, 0.10])["path"] == "faq"
assert route_faq("向量库怎么建", faqs, [0.20, 0.15])["path"] == "rag"
print(route_faq("退费多久到账", faqs, [0.92, 0.10]))

34. 实现简易字符级 Jaccard 相似度，模拟「query 与 FAQ 问题匹配」

对字符串按字符集合计算 Jaccard；空串相似度为 0。用它给候选 FAQ 打分并排序。

✅ 参考答案
Python
def char_jaccard(a: str, b: str) -> float:
    sa, sb = set(a), set(b)
    if not sa and not sb:
        return 1.0
    if not sa or not sb:
        return 0.0
    return len(sa & sb) / len(sa | sb)


def rank_faqs(query: str, faqs: list[tuple[str, str]]) -> list[tuple[float, str, str]]:
    ranked = [(char_jaccard(query, q), q, a) for q, a in faqs]
    ranked.sort(key=lambda x: (-x[0], x[1]))
    return ranked


faqs = [("如何退费", "见退费政策"), ("如何开票", "见发票流程"), ("向量检索", "见 RAG 章节")]
top = rank_faqs("怎么退费", faqs)
assert top[0][1] == "如何退费"
print(top[:2])

这是教学用弱基线；真实 FAQ 匹配通常用分词、BM25 或句向量，并配合阈值与人工抽检。

35. 实现「加载 → 切分」的最小文本切分器

按固定窗口 size 与重叠 overlap 切分；最后一块若过短也保留。拒绝 overlap >= size。

✅ 参考答案
Python
def split_text(text: str, size: int = 20, overlap: int = 5) -> list[str]:
    if size <= 0:
        raise ValueError("size 必须为正")
    if overlap < 0 or overlap >= size:
        raise ValueError("overlap 须满足 0 <= overlap < size")
    if not text:
        return []

    chunks, start, n = [], 0, len(text)
    while start < n:
        end = min(start + size, n)
        chunks.append(text[start:end])
        if end == n:
            break
        start = end - overlap
    return chunks


parts = split_text("abcdefghijKLMNOPQRST", size=8, overlap=2)
assert parts[0] == "abcdefgh"
assert parts[1].startswith("gh")
print(parts)

36. 实现检索结果到 Prompt 的上下文拼接（含长度预算）

给定 query 与按相关度降序的 passages，按总字符预算 budget 依次加入，返回最终 Prompt。预算不足时跳过过长片段（不截断半句也可，本题采用「整段放不下就跳过」）。

✅ 参考答案
Python
def build_rag_prompt(query: str, passages: list[str], budget: int = 120) -> str:
    header = f"问题：{query}\n证据：\n"
    used = len(header)
    lines = []
    for i, p in enumerate(passages, 1):
        block = f"[{i}] {p}\n"
        if used + len(block) > budget:
            continue
        lines.append(block)
        used += len(block)
    body = "".join(lines) if lines else "[无可用证据]\n"
    return header + body + "请仅依据证据回答；证据不足时说明未知。"


prompt = build_rag_prompt("退费时效", ["退费三个工作日到账", "开票需三个工作日"], budget=80)
assert "退费三个工作日到账" in prompt
print(prompt)

def label_module(path: str) -> str:

    for suffix, label in RULES:
        if norm.endswith(suffix) or norm == suffix:
            return label
    return "unknown"

38. 实现 top-k 去重：按文本归一化后保留高分、去重

检索结果可能出现重复切段。对 (text, score) 列表：文本 strip 后相同只保留分数最高者，再取 top-k。

✅ 参考答案
Python
def dedup_topk(items: list[tuple[str, float]], k: int = 3) -> list[tuple[str, float]]:
    best = {}
    for text, score in items:
        key = text.strip()
        if not key:
            continue
        if key not in best or score > best[key]:
            best[key] = float(score)
    ranked = sorted(best.items(), key=lambda x: (-x[1], x[0]))
    return ranked[:k]


items = [(" 退费政策 ", 0.9), ("退费政策", 0.8), ("开票流程", 0.7), ("向量库", 0.6)]
assert dedup_topk(items, 2) == [("退费政策", 0.9), ("开票流程", 0.7)]
print(dedup_topk(items, 3))

40. 实现迷你端到端编排：FAQ 优先，未命中则检索生成

用内存 FAQ 与内存知识片段，串起：打分 → 路由 →（可选）关键词 top-k → 拼 Prompt。模型用回显函数代替。

✅ 参考答案
Python
def char_jaccard(a: str, b: str) -> float:
    sa, sb = set(a), set(b)
    if not sa and not sb:
        return 1.0
    if not sa or not sb:
        return 0.0
    return len(sa & sb) / len(sa | sb)


def search_passages(store: list[str], query: str, k: int = 2) -> list[str]:
    ranked = sorted(store, key=lambda text: (-char_jaccard(query, text), text))
    return ranked[:k]


def mini_pipeline(query, faqs, store, threshold=0.85):
    faq_scores = [char_jaccard(query, q) for q, _ in faqs]
    best_i = max(range(len(faqs)), key=lambda i: faq_scores[i])
    if faq_scores[best_i] >= threshold:
        return {"path": "faq", "output": faqs[best_i][1]}

    passages = search_passages(store, query, k=2)
    prompt = "问题：" + query + "\n证据：" + " | ".join(passages)
    answer = passages[0] if passages else "未知"
    return {"path": "rag", "output": answer, "prompt": prompt}


faqs = [("退费多久到账", "三个工作日到账")]
store = [
    "知识库：退费申请提交后三个工作日到账",
    "知识库：开票在每月最后一个周五批量处理",
]

assert mini_pipeline("退费多久到账", faqs, store)["path"] == "faq"
rag = mini_pipeline("开票什么时候处理", faqs, store, threshold=0.99)
assert rag["path"] == "rag"
assert "开票" in rag["output"]
print(rag)

该函数把第一章的主路径压缩成可运行骨架；后续章节会把关键词检索替换为真实向量模型与 Milvus，并把回显 LLM 换成 Ollama/LangChain 调用。

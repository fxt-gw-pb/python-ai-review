⚡ Redis：缓存数据库与 Python 操作综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：Redis：缓存数据库与 Python 操作🚀 所属项目：EduRAG：企业私有知识智能问答

40 道练习，覆盖 Redis 定位与核心特性（内存、RDB/AOF、多数据结构）、典型场景，以及 Python StrictRedis 工程封装（配置注入、JSON 存取、answer: 缓存键、异常与日志）。重点练「缓存命中如何减少重复计算」和「decode_responses / JSON 编解码边界」。

建议先独立作答，再展开参考答案。代码题用纯 Python 模拟迷你 Redis 与客户端封装，不连接真实 Redis 进程。

一、选择题（20 题）

1. Redis 全称更接近？

A. Remote Dictionary Server
B. Random Disk SQL
C. Relational Data Schema
D. Rapid Django Service

✅ 参考答案

答案：A。 高性能键值数据库。

2. Redis 最突出的性能来源是？

A. 每次查询都全表扫描磁盘 CSV
B. 数据主要在内存中，读写极快
C. 只能单线程爬取网页

C. 仅 Git commit
D. 仅 JPEG 压缩

✅ 参考答案

二、简答题（12 题）

21. 用一句话说明 Redis 适合当缓存的原因。

✅ 参考答案

热数据放内存，按 key 极速读写，可挡住对慢后端（如 MySQL/重计算）的重复请求，降低延迟与压力。

22. RDB 与 AOF 在直觉上如何分工？（不必背细节参数）

✅ 参考答案

RDB 偏快照式落盘；AOF 偏追加写命令/变更。可单独或组合使用，在性能、体积与可恢复性之间权衡。

23. 为什么复杂结构要用 JSON 再存 Redis 字符串？

✅ 参考答案

Redis string 存的是字节/文本；dict 等 Python 对象需序列化。JSON 通用、可读，loads 后恢复结构；也可用 hash 等类型，课件选 JSON 字符串路径。

24. get_data 与 get_answer 的差异是什么？

✅ 参考答案

get_data 通用：按 key 取 JSON 对象。get_answer 面向问答缓存：固定前缀 answer: + query，返回答案字符串（课件未再 json.loads）。

25. 缓存键设计为何常加前缀（如 user: / answer:）？

✅ 参考答案

避免不同业务键冲突，便于按前缀排查/批量管理，语义清晰（用户资料 vs 问答答案）。

26. 连接失败时为什么要打日志并向上抛出？

✅ 参考答案

初始化失败说明基础设施不可用，继续跑会产生更难查的连锁错误；日志保留现场，调用方可快速失败或降级。

27. decode_responses=False 时 get 后直接 json.loads 可能怎样？

✅ 参考答案

拿到的是 bytes，部分环境下 json.loads 仍可处理，但混用 str/bytes 易出错；课件用 True 统一成 str 更省心。

28. 缓存未命中时应如何与「真实计算」配合？（工程直觉）

✅ 参考答案

未命中则走 MySQL/BM25/模型等计算路径，得到结果后 set 回 Redis（可带 TTL），下次命中直接返回。注意更新源数据时的失效策略。

29. 列出 RedisClient.__init__ 需要的 4 个连接参数。

✅ 参考答案

host、port、password、db（外加 decode_responses=True）。

30. 示例中存 user:1 后查 answer:test_query 为何未命中？

✅ 参考答案

两个不同 key；演示只写入用户资料，没有写入该 query 的缓存答案，故 get_answer 返回 None。

31. 将 Redis 接入 FAQ 系统时，适合缓存什么？

✅ 参考答案

高频重复问题的最终答案、或检索中间结果（需权衡一致性）；避免缓存易变且无失效策略的敏感个性化数据。

32. 密码、主机为何不应硬编码进公开仓库？

✅ 参考答案

泄露导致未授权访问；应用环境变量/私密配置注入，仓库只保留本地演示默认值或占位符。

三、代码题（8 题）

33. 迷你内存 KV：set/get 与 decode 语义

✅ 参考答案
Python
class MiniRedis:
    def __init__(self):
        self.store = {}

    def set(self, key, value):
        self.store[key] = value
        return True

    def get(self, key):
        return self.store.get(key)


r = MiniRedis()
r.set("k", "v")
assert r.get("k") == "v" and r.get("missing") is None
print(r.get("k"))

34. JSON 序列化存取（对齐 set_data/get_data）

✅ 参考答案
Python
import json


class MiniRedis:
    def __init__(self):
        self.store = {}

    def set(self, key, value):
        self.store[key] = value

    def get(self, key):
        return self.store.get(key)


def set_data(client, key, value):
    client.set(key, json.dumps(value, ensure_ascii=False))


def get_data(client, key):
    data = client.get(key)
    return json.loads(data) if data else None


c = MiniRedis()
set_data(c, "user:1", {"name": "Alice", "age": 25})
assert get_data(c, "user:1")["name"] == "Alice"
print(get_data(c, "user:1"))

def get_answer(client, query: str):

✅ 参考答案

def cached_answer(client, query: str, compute):
    key = f"answer:{query}"
    hit = client.get(key)
    if hit is not None:
        return hit, True
    val = compute(query)
    client.set(key, val)
    return val, False


class MiniRedis:
    def __init__(self):
        self.store = {}

    def set(self, k, v):
        self.store[k] = v

    def get(self, k):
        return self.store.get(k)


c = MiniRedis()
calls = {"n": 0}

def compute(q):
    calls["n"] += 1
    return f"ans:{q}"


v1, hit1 = cached_answer(c, "hi", compute)
v2, hit2 = cached_answer(c, "hi", compute)
assert (v1, hit1) == ("ans:hi", False) and hit2 is True and calls["n"] == 1
print(v2, calls["n"])

class RedisError(Exception):
    pass


class FlakyClient:
    def __init__(self, fail=False):
        self.fail = fail
        self.store = {}

    def set(self, k, v):
        if self.fail:
            raise RedisError("down")
        self.store[k] = v

    def get(self, k):
        if self.fail:
            raise RedisError("down")
        return self.store.get(k)


def safe_get(client, key):
    try:
        return client.get(key)
    except RedisError:
        return None

40. 迷你 RedisClient：串起连接参数、JSON、答案缓存

✅ 参考答案

import json


class MiniStrictRedis:
    def __init__(self, **kwargs):
        self.kwargs = kwargs
        self.store = {}

    def set(self, k, v):
        self.store[k] = v

    def get(self, k):
        return self.store.get(k)


class RedisClient:
    def __init__(self, backend):
        self.client = backend

    def set_data(self, key, value):
        self.client.set(key, json.dumps(value))

    def get_data(self, key):
        data = self.client.get(key)
        return json.loads(data) if data else None

    def get_answer(self, query):
        return self.client.get(f"answer:{query}")


backend = MiniStrictRedis(host="localhost", port=6379, decode_responses=True)
cli = RedisClient(backend)
cli.set_data("user:1", {"name": "Alice", "age": 25})
backend.set("answer:q1", "A1")
assert cli.get_data("user:1")["age"] == 25
assert cli.get_answer("q1") == "A1"
assert cli.get_answer("missing") is None
print(cli.get_data("user:1"))

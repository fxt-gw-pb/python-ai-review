🦙 Ollama：本地大模型运行与调用综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：Ollama：本地大模型运行与调用🚀 所属项目：EduRAG：企业私有知识智能问答

40 道练习，覆盖 Ollama 的定位与安装、OLLAMA_MODELS 目录配置、ollama run 拉模、显存经验值，以及 Python 三种调用方式（原生 ollama / LangChain / requests）。重点练「本地服务默认端口」和「聊天式 vs 生成式 vs 流式」。

建议先独立作答，再展开参考答案。代码题用纯 Python 模拟 Client / 流式拼接 / HTTP 请求体，不启动真实 Ollama 进程。

二、简答题（12 题）

21. 为什么要把模型目录改到非系统盘？

✅ 参考答案

模型文件大、数量多；默认用户目录（常在 C 盘）易占满系统盘，拖慢整机。用 OLLAMA_MODELS 指到空间更大的盘更稳妥。

22. 简述「下载安装 → 配环境变量 → 再启动」的顺序原因。

✅ 参考答案

安装后先退出 Ollama，再设 OLLAMA_MODELS，最后重新启动，否则进程可能仍用旧默认路径，变量不生效。

23. ollama run 首次执行时会发生什么？

✅ 参考答案

若本地无该模型，会从模型库下载；完成后进入可对话状态。耗时取决于网速与模型体积。

24. 无独立 GPU 时 Ollama 怎么跑？对选型有何提示？

✅ 参考答案

默认走 CPU，速度通常更慢。应优先选更小参数量（如 1.5B/7B 量级）或接受更长延迟，并关注内存占用。

25. 写出 chat 调用中一条 user message 的最小字典结构。

✅ 参考答案

{'role': 'user', 'content': '为什么天空是蓝色的？'}，再包进 messages=[...] 传给 ollama.chat / Client。

26. Client 远程调用相对默认本机调用多了什么？

✅ 参考答案

显式 host='http://<IP>:11434'，指向另一台跑着 Ollama 的机器；本机则常用 127.0.0.1:11434。

27. 流式输出对聊天 UI 的价值是什么？

✅ 参考答案

边生成边显示，降低「干等整段」的体感延迟；实现上迭代 chunk，把 message.content 片段拼到界面。

28. LangChain 封装相对原生 ollama 库的典型好处？

✅ 参考答案

更容易接入 LC 的 Prompt、Chain、Agent、检索流水线；invoke 风格统一，便于和项目其他组件组合。代价是多一层依赖。

29. 用 requests 调用时，headers 为何常设 Content-Type: application/json？

✅ 参考答案

服务端按 JSON 解析 body（model、options、messages 等）；类型不对可能导致 4xx 或解析失败。

30. temperature 与「评测可复现」之间如何权衡？

✅ 参考答案

评测/回归常用 temperature=0（或很低）降低随机性；创意写作可适当调高。完全可复现还受采样、版本等影响，但低温是第一步。

31. 对比三种调用：何时优先选 requests？

✅ 参考答案

不想引入 ollama/LangChain 依赖、只需 HTTP、或要精细控制超时/代理/鉴权头时，用 requests 打 /api/chat 更直接。

32. 本地 Ollama 在 EduRAG 一类项目中扮演什么角色？

✅ 参考答案

作为私有/本地推理后端，为问答、改写、路由等提供 LLM 能力，减少对外部云 API 的依赖与数据外发风险（仍需注意本机安全与资源）。

def assistant_text(response: dict) -> str:
    return response["message"]["content"]

34. 拼接流式 chunk 为完整回复

✅ 参考答案
Python
def join_stream(chunks: list[dict]) -> str:
    return "".join(c["message"]["content"] for c in chunks)


chunks = [
    {"message": {"content": "天空"}},
    {"message": {"content": "是"}},
    {"message": {"content": "蓝色的"}},
]
assert join_stream(chunks) == "天空是蓝色的"
print(join_stream(chunks))

35. 迷你 Client：按 host 记录调用，返回假响应

✅ 参考答案
Python
class MiniOllamaClient:
    def __init__(self, host: str = "http://127.0.0.1:11434"):
        self.host = host.rstrip("/")
        self.calls = []

    def chat(self, model: str, messages: list[dict]) -> dict:
        self.calls.append({"host": self.host, "model": model, "messages": messages})
        last = messages[-1]["content"]
        return {"message": {"role": "assistant", "content": f"[{model}@{self.host}] 收到：{last}"}}


c = MiniOllamaClient()
out = c.chat("qwen2:1.5b", [{"role": "user", "content": "你好"}])
assert "你好" in out["message"]["content"] and len(c.calls) == 1
print(out["message"]["content"])

class MiniLCOllama:
    def __init__(self, model: str, base_url: str | None = None, temperature: float = 0.0):
        self.model = model
        self.base_url = base_url or "http://127.0.0.1:11434"
        self.temperature = temperature

    def invoke(self, prompt: str) -> str:
        # 教学：不发真实 HTTP，只体现参数落点
        return f"{self.model}|t={self.temperature}|{self.base_url}|{prompt}"


llm = MiniLCOllama("qwen2:1.5b", temperature=0)
res = llm.invoke("你是谁")
assert res.startswith("qwen2:1.5b|t=0|") and res.endswith("你是谁")
print(res)

def min_vram_gb(size_tag: str) -> int | None:
    # size_tag 如 "7B" / "13B"
    table = {"7B": 8, "13B": 16}
    key = size_tag.upper().replace(" ", "")
    return table.get(key)

def call_llm(backend: str, prompt: str, *, model="qwen2:1.5b") -> str:
    if backend == "ollama":
        return f"chat:{model}:{prompt}"
    if backend == "langchain":
        return f"invoke:{model}:{prompt}"
    if backend == "requests":
        return f"POST /api/chat:{model}:{prompt}"
    raise ValueError(f"unknown backend: {backend}")


assert call_llm("ollama", "hi").startswith("chat:")
assert "api/chat" in call_llm("requests", "hi")
print(call_llm("langchain", "你是谁"))

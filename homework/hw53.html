🧬 医学实体抽取：规则、BiLSTM-CRF 与 Viterbi 综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：医学实体抽取：规则、BiLSTM-CRF 与 Viterbi

55 道练习，按“NER 任务与输出合同 → BIO/BIOES → 规则基线 → BiLSTM 发射分数 → 线性链 CRF → 前向算法与 Viterbi → Mask、约束和严格实体级评估”组织。做题时始终区分三个对象：文本中的 mention、模型输出的 span、知识图谱中的标准实体。NER 只完成前两者，不能跳过实体链接、断言识别和证据审核直接写入图谱。

建议先独立作答，再展开参考答案。本练习依据原课件第三章和站内重组讲义设计，并参考 BiLSTM-CRF 原始论文、CRF 原始论文、CoNLL-2003 NER 任务与 Forney 的 Viterbi 教程论文校准。注意：BiLSTM 产生上下文相关的发射分数，CRF 对完整标签序列建分数；训练分母使用 logsumexp 汇总所有路径，Viterbi 使用 max 和回指针寻找单条最优路径；padding 必须同时从损失和解码中排除；严格实体级指标要求边界与类型全部正确。10 道代码题只使用 Python 标准库离线运行，不下载模型。

一、选择题（30 题）

1. 医学命名实体识别（NER）的直接输出通常是什么？

A. 文本中实体 mention 的边界、类别及可选分数
B. 已确认的患者诊断和处方
C. Neo4j 中唯一标准实体 ID
D. 任意两个实体之间的全部关系

✅ 参考答案

答案：A。 NER 找到“哪里提到了什么类型的实体”。把 mention 对齐到标准概念属于实体链接；判断否定、假设和时间属于断言/语境识别；形成图谱事实还需关系、来源与审核。

2. 统一输出 start=0, end=6 表示文本切片 [0,6)，最重要的原因是？

A. 明确 end 为开区间，使 text[start:end] 可直接复现 mention
B. 让所有实体固定为 6 个字符
C. 自动完成实体消歧
D. 让模型不需要标签

✅ 参考答案

答案：A。 偏移合同必须写清字符计数和区间语义。若标注、训练、推理一方使用闭区间，容易产生系统性的差一位错误。

3. BIOES 中 S-Disease 表示什么？

A. 单个 token 构成的 Disease 实体
B. Disease 实体的第二个 token
C. 句子开始符
D. padding 标签

✅ 参考答案

答案：A。 B、I、E 分别表示多 token 实体的开始、内部、结束，S 表示单 token 实体，O 表示目标实体之外。

4. 下列哪条 BIOES 路径合法？

A. B-Disease, I-Disease, E-Disease, O, S-Symptom
B. O, I-Disease, O
C. B-Disease, I-Symptom, E-Disease
D. START, E-Disease, O

✅ 参考答案

答案：A。 I/E-X 必须延续同类型的 B/I-X；序列不能从 I 或 E 开始。实现中应同时约束 START 和 STOP 转移。

5. 字级 NER 改用子词 tokenizer 后，最需要补充什么？

A. 子词到原文字符偏移的对齐与特殊 token/padding 的 mask
B. 把所有标签改成 O
C. 删除原始文本
D. 只保留 token ID 不保存 tokenizer 版本

✅ 参考答案

答案：A。 训练标签、预测 token 和原文 span 必须可逆映射。[CLS]、[SEP]、wordpiece 延续片段等不能凭位置猜测，需使用 offset mapping 和明确的标签对齐策略。

6. 单层 BIO/BIOES 序列标注对嵌套实体的典型限制是？

A. 同一 token 通常只能有一个标签，难以同时表示内外两层实体
B. 无法表示任何单层实体
C. 只能识别英文
D. 必须把每个字符建成图节点

✅ 参考答案

答案：A。 如“右肺上叶癌”既标整体疾病又标“右肺上叶”部位，单标签序列无法同时表达。可改用 span 分类、多层标签或超图等方案，评测合同也需相应改变。

7. 为什么应长期保留词典/正则 NER 基线？

A. 它可解释、易回归，在规范文本和高精度场景中仍有价值
B. 规则在任何数据上一定优于模型
C. 规则不需要版本管理
D. 规则自动理解所有否定语境

✅ 参考答案

答案：A。 规则基线可提供冷启动、回归比较和保护层，但召回、冲突与维护成本需要度量。模型上线不意味着规则应被删除。

8. 词典同时匹配“支气管炎”和“慢性支气管炎”，正确处理方式是？

A. 按任务规范执行确定性的重叠策略，并保留被舍弃候选和原因
B. 两个 span 无条件都输出，即使规范禁止重叠
C. 随机选一个
D. 永远选最短字符串

✅ 参考答案

答案：A。 最长优先只是常用策略，不保证全局最优。若候选有分数，可用加权区间选择；嵌套是否保留由标注合同决定。

9. 用正则抽取剂量时，哪项最重要？

A. 同时校验数值、单位、边界、范围与上下文，避免匹配到编号或无关数字
B. 只写 \d+ 即可覆盖所有情况
C. 把所有数字都标成 Drug
D. 正则不需要测试集

✅ 参考答案

答案：A。 医学数值表达可能包含小数、范围、复合单位和中文数字。规则应有正反例测试、版本与来源，而非只看一个演示句。

10. “患者否认发热”中规则或模型识别出“发热”，下一步应该？

A. 保留 mention，同时识别其 negated 断言，不能直接当肯定症状事实
B. 删除“否认”并写入肯定关系
C. 把“患者”标成疾病
D. 自动生成处方

✅ 参考答案

答案：A。 NER 正确不等于事实判断正确。知识图谱管道还要处理否定、假设、家族史、条件和时间范围。

11. 先用完整词典自动标全量语料，再随机切分训练/测试，主要风险是？

A. 规则来源与重复文本可能让测试集不独立，并掩盖未登录实体能力
B. 标签数量一定减少
C. CRF 无法训练
D. Viterbi 复杂度变为指数级

✅ 参考答案

答案：A。 应按患者/文档/来源分组切分，记录自动标注与人工修订，单独报告词典内、词典外和困难子集，避免把规则覆盖率误当模型泛化。

12. BiLSTM 在 BiLSTM-CRF 中的主要作用是？

A. 融合每个位置左右两侧上下文，生成上下文表示
B. 保证所有标签转移合法
C. 计算全部路径的归一化常数
D. 把 mention 链接到知识库 ID

✅ 参考答案

答案：A。 线性层再把 BiLSTM 隐状态映射为每个位置对各标签的发射分数；序列级标签依赖主要由 CRF 层建模。

13. 批量发射分数张量常见形状是？

A. [batch, sequence_length, num_tags]
B. [num_tags, vocab_size, vocab_size]
C. [batch] 且没有位置维
D. [sequence_length, sequence_length] 且没有标签维

✅ 参考答案

答案：A。 不同库可能使用 batch-first 或 time-first，但每个有效位置都必须有 K 个标签分数，mask 的形状也必须与批次和序列维对齐。

14. 线性链 CRF 的转移分数通常表达什么？

A. 相邻标签从前一状态到当前状态的兼容性
B. token 在词表中的频率
C. 文档与知识库实体的相似度
D. 字符在原文中的像素位置

✅ 参考答案

答案：A。 发射分数依赖输入位置，转移分数描述标签序列结构。还应显式处理 START/STOP 或等价的起止参数。

15. 一条 CRF 路径的完整分数通常包含？

A. 每个位置的发射分数、相邻标签转移分数及起止转移
B. 只有最后一个 token 的发射分数
C. 只有标签计数
D. 只有模型参数数量

✅ 参考答案

答案：A。 若训练和解码对起止状态的处理不一致，路径概率与最优路径都会出错。padding 位置不应贡献发射或转移。

16. 为什么逐位置取发射分数 argmax 不等价于 Viterbi？

A. 局部最大不考虑标签转移，组合后可能不是全局最高分或合法路径
B. Viterbi 不使用发射分数
C. argmax 会自动枚举所有路径
D. 二者只在 batch 大小时不同

✅ 参考答案

答案：A。 Viterbi 同时考虑到当前位置的最佳累计分和前驱，通过动态规划得到完整序列的全局最优路径。

17. CRF 中 $p(y\mid x)$ 的分母是什么？

A. 给定输入下所有允许标签路径分数指数和
B. 词表大小
C. gold 路径长度
D. 测试集样本数

✅ 参考答案

答案：A。 分母是配分函数 $Z(x)=\sum_{y'}\exp s(x,y')$。不能显式枚举 $K^T$ 条路径，应使用前向动态规划计算。

18. 单样本 CRF 负对数似然可写为？

A. $\log Z(x)-s(x,y^{gold})$
B. $s(x,y^{gold})+\log Z(x)$
C. 只对每个 token 独立做 MSE
D. $K^T$

✅ 参考答案

答案：A。 最小化它会提高 gold 路径相对全部候选路径的分数。不同库返回 log-likelihood 或 negative log-likelihood，调用时要确认符号和 reduction。

19. 前向算法与 Viterbi 的关键差异是？

A. 前向用 logsumexp 汇总全部路径，Viterbi 用 max 并保存回指针
B. 前向只用于删除节点
C. Viterbi 会更新模型参数
D. 二者一个用标签、一个不用标签

✅ 参考答案

答案：A。 两者利用相同的 Markov 分解和动态规划结构，但求解目标不同：一个计算归一化常数，一个求最优路径。

20. Viterbi 为什么必须保存 backpointer？

A. 终点只给出最佳末状态，回指针用于恢复此前每一步的最优前驱
B. 用来保存原始病历全文
C. 用来计算 tokenizer 词表
D. 用来随机采样标签

✅ 参考答案

答案：A。 只保存每层最大分数无法还原完整路径。到达 STOP 后从最佳末标签沿 backpointer 逆序回溯。

21. 长度为 $T$、标签数为 $K$ 的一阶线性链 CRF，朴素动态规划复杂度通常是？

A. $O(TK^2)$
B. $O(K^T)$
C. $O(T+K)$ 且不看转移
D. $O(1)$

✅ 参考答案

答案：A。 每个位置的每个当前标签要比较/汇总 K 个前驱。动态规划把指数枚举降为多项式，但标签集很大时转移计算仍需关注。

22. 为什么 CRF 前向计算应在 log 空间使用稳定 logsumexp？

A. 长序列概率连乘易下溢，直接指数大分数又可能上溢
B. log 会让所有路径分数相同
C. logsumexp 等于 max 且永远无误差
D. 为了删除 transition 参数

✅ 参考答案

答案：A。 常用公式是 $m+\log\sum_i\exp(a_i-m)$。logsumexp 平滑汇总全部路径，不应在训练中随意替换成 max。

23. batch 中第二个句子长度为 3，却把第 4、5 个 padding 位置计入 CRF 损失，会怎样？

A. 模型学习无意义的 padding 转移，损失和梯度被污染
B. 自动提高实体级 F1

32. BIO 与 BIOES 各有什么取舍？

✅ 参考答案

BIO 标签更少、实现简单；BIOES 显式区分实体结束和单 token 实体，能提供更细的边界信号和更严格的合法转移。二者都能表达非嵌套连续实体，不能天然解决嵌套或不连续实体。选择后必须固定编码、解码、约束和评测规则，不能训练用 BIO、解码却按 BIOES 解释。

33. 如何建模“右肺上叶癌”中的嵌套实体？

✅ 参考答案

先由业务定义是否需要同时输出 Disease“右肺上叶癌”和 BodyPart“右肺上叶”。若需要，单层 BIOES 不够，可使用按类型多层序列标签、候选 span 分类、biaffine/span matrix 或其他支持嵌套的结构。输出合同应允许重叠 span，评测按每个 (start,end,type) 严格匹配；不要用冲突消解静默删除业务需要的内层实体。

34. 设计一个可上线的规则 NER 管道。

✅ 参考答案

维护带版本和来源的标准词典、别名与正则；先做规范化但保留原文偏移映射；生成全部候选；按边界、词性/后缀、黑白名单和上下文过滤；使用确定性重叠策略；识别否定、假设、家族史和时间；输出 span、类型、分数、规则 ID、版本和证据；用固定测试集回归，并监控覆盖率、冲突率和人工抽检误差。

35. 为什么“最长匹配优先”不总是最优？

✅ 参考答案

最长候选可能低置信、类型错误，且一个长 span 可能阻塞两个总价值更高的相邻实体。若规范禁止重叠，可以把候选看成带权区间，使用确定性的加权区间调度最大化总分；若允许嵌套，则不能简单互斥。策略和 tie-break 必须版本化并留存被舍弃候选。

36. 给出 BiLSTM-CRF 从输入到输出的张量流。

✅ 参考答案

输入 token IDs [B,T] 经 Embedding 得 [B,T,D]；双向 LSTM 每个方向隐藏维为 H 时拼接为 [B,T,2H]；Linear 映射为发射分数 [B,T,K]；CRF 用 [B,T] mask、转移矩阵和起止分数计算每条序列 log-likelihood；推理时 Viterbi 为每个样本返回长度等于有效 token 数的标签 ID 路径，再映射回原文 span。

37. 手工说明一条标签路径如何计分。

✅ 参考答案

长度 T 的路径分数为 START 到首标签转移，加每个位置对应标签的发射分数，加相邻标签转移，最后加末标签到 STOP 的转移。即 $s(x,y)=A_{START,y_1}+\sum_i P_{i,y_i}+\sum_{i=2}^T A_{y_{i-1},y_i}+A_{y_T,STOP}$。 若有硬约束，非法转移不进入路径集合或赋极小分数；padding 不计分。

38. 为什么训练用前向算法而推理用 Viterbi？

✅ 参考答案

训练负对数似然需要配分函数，即所有候选路径的总质量，所以递推时对前驱做 logsumexp；推理需要分数最高的单条路径，所以对前驱做 max，并保存达到最大值的前驱供回溯。二者共享状态分解，但不能互换目标。

39. 如何设计 BIOES 的 START/STOP 与合法转移？

✅ 参考答案

START 只能到 O、B-X、S-X；O、E-X、S-X 可到 O、B-Y、S-Y 或 STOP；B-X、I-X 只能到 I-X 或 E-X，不能换类型；START 不能作为普通当前位置，STOP 不能再转出。训练数据先验证，解码时用 mask 或极小分数约束。若库已内置起止参数，应避免重复计算。

40. 设计一个能发现 mask 错误的最小测试。

42. 一份实体抽取错误分析报告至少包含什么？

44. 超长病历采用滑窗推理时如何合并实体？

46. 将字符 span 转为 BIOES 标签

def spans_to_bioes(text: str, spans: list[dict]) -> list[str]:
    tags = ["O"] * len(text)
    previous_end = 0
    for item in sorted(spans, key=lambda x: (x["start"], x["end"], x["type"])):
        start, end, entity_type = item["start"], item["end"], item["type"]
        if not (0 <= start < end <= len(text)):
            raise ValueError("span 越界")
        if start < previous_end:
            raise ValueError("本标签体系不允许重叠 span")
        if item.get("text") is not None and text[start:end] != item["text"]:
            raise ValueError("mention 与原文切片不一致")
        length = end - start
        if length == 1:
            tags[start] = f"S-{entity_type}"
        else:
            tags[start] = f"B-{entity_type}"
            for index in range(start + 1, end - 1):
                tags[index] = f"I-{entity_type}"
            tags[end - 1] = f"E-{entity_type}"
        previous_end = end
    return tags

47. 严格解码 BIOES 标签为实体 span

✅ 参考答案

def bioes_to_spans(text: str, tags: list[str]) -> list[dict]:
    if len(text) != len(tags):
        raise ValueError("文本与标签长度不一致")
    spans = []
    index = 0
    while index < len(tags):
        tag = tags[index]
        if tag == "O":
            index += 1
            continue
        if "-" not in tag:
            raise ValueError(f"非法标签: {tag}")
        prefix, entity_type = tag.split("-", 1)
        if prefix == "S":
            spans.append({"start": index, "end": index + 1,
                          "type": entity_type, "text": text[index:index + 1]})
            index += 1
            continue
        if prefix != "B":
            raise ValueError(f"位置 {index} 出现孤立 {prefix}")
        start = index
        index += 1
        while index < len(tags) and tags[index] == f"I-{entity_type}":
            index += 1
        if index >= len(tags) or tags[index] != f"E-{entity_type}":
            raise ValueError(f"从位置 {start} 开始的实体未合法闭合")
        end = index + 1
        spans.append({"start": start, "end": end,
                      "type": entity_type, "text": text[start:end]})
        index = end
    return spans

    {"start": 0, "end": 2, "type": "Disease", "text": "肺炎"},

48. 生成词典规则的全部候选 span

def dictionary_candidates(text: str, lexicon: list[dict]) -> list[dict]:
    candidates = []
    for entry in lexicon:
        mention = entry["mention"]
        if not mention:
            raise ValueError("词典 mention 不能为空")
        start = 0
        while True:
            start = text.find(mention, start)
            if start == -1:
                break
            candidates.append({
                "start": start,
                "end": start + len(mention),
                "text": mention,
                "type": entry["type"],
                "rule_id": entry["rule_id"],
                "score": entry.get("score", 1.0),
            })
            start += 1
    return sorted(candidates, key=lambda x: (x["start"], x["end"], x["type"], x["rule_id"]))

49. 用加权区间调度消解重叠候选

候选不能重叠，目标是最大化总分；证明简单“取单个最高分”可能不是全局最优。

✅ 参考答案

from bisect import bisect_right


def select_non_overlapping(candidates: list[dict]) -> list[dict]:
    items = sorted(candidates, key=lambda x: (x["end"], x["start"], -x["score"], x["text"]))
    ends = [item["end"] for item in items]
    previous = [bisect_right(ends, item["start"], hi=i) - 1 for i, item in enumerate(items)]
    dp = [(0.0, [])]
    for i, item in enumerate(items, start=1):
        without_score, without_items = dp[i - 1]
        base_score, base_items = dp[previous[i - 1] + 1]
        with_score = base_score + item["score"]
        with_items = base_items + [item]
        if with_score > without_score:
            dp.append((with_score, with_items))
        elif with_score < without_score:
            dp.append((without_score, without_items))
        else:
            key_with = [(x["start"], x["end"], x["text"]) for x in with_items]
            key_without = [(x["start"], x["end"], x["text"]) for x in without_items]
            dp.append((with_score, with_items if key_with < key_without else without_items))
    return sorted(dp[-1][1], key=lambda x: (x["start"], x["end"]))


candidates = [
    {"start": 0, "end": 4, "text": "长候选", "score": 6.0},
    {"start": 0, "end": 2, "text": "左实体", "score": 4.0},
    {"start": 2, "end": 4, "text": "右实体", "score": 4.0},
]
selected = select_non_overlapping(candidates)
assert [item["text"] for item in selected] == ["左实体", "右实体"]
assert sum(item["score"] for item in selected) == 8.0
print(selected)

50. 为规则候选添加简单断言分类

在 mention 前的有限窗口识别否定、假设与家族史；这是可回归基线，不应被误当完整临床语境模型。

✅ 参考答案

def classify_assertion(text: str, start: int, end: int, window: int = 8) -> dict:
    if not (0 <= start < end <= len(text)):
        raise ValueError("span 越界")
    context = text[max(0, start - window):start]
    cue_groups = [
        ("family_history", ["家族史", "母亲", "父亲"]),
        ("negated", ["否认", "未见", "无"]),
        ("hypothetical", ["考虑", "疑似", "若出现"]),
    ]
    for assertion, cues in cue_groups:
        matched = [cue for cue in cues if cue in context]
        if matched:
            return {"assertion": assertion, "cue": matched[-1], "context": context}
    return {"assertion": "affirmed", "cue": None, "context": context}


cases = [
    ("患者否认发热", "发热", "negated"),
    ("考虑肺炎可能", "肺炎", "hypothetical"),
    ("母亲患糖尿病", "糖尿病", "family_history"),
    ("患者出现咳嗽", "咳嗽", "affirmed"),
]
for text, mention, expected in cases:
    start = text.index(mention)
    result = classify_assertion(text, start, start + len(mention))
    assert result["assertion"] == expected, (text, result)
print("ok")

51. 校验 BIOES 合法转移与完整路径

实现 START/STOP 规则，检查类型一致性。

✅ 参考答案
Python
def split_tag(tag: str):
    if tag in {"START", "STOP", "O"}:
        return tag, None
    if "-" not in tag:
        raise ValueError(f"非法标签: {tag}")
    prefix, entity_type = tag.split("-", 1)
    if prefix not in {"B", "I", "E", "S"} or not entity_type:
        raise ValueError(f"非法标签: {tag}")
    return prefix, entity_type


def legal_transition(previous: str, current: str) -> bool:
    prev_prefix, prev_type = split_tag(previous)
    curr_prefix, curr_type = split_tag(current)
    if previous == "STOP" or current == "START":
        return False
    if previous == "START":
        return curr_prefix in {"O", "B", "S"}
    if current == "STOP":
        return prev_prefix in {"O", "E", "S"}
    if prev_prefix in {"O", "E", "S"}:
        return curr_prefix in {"O", "B", "S"}
    return prev_prefix in {"B", "I"} and curr_prefix in {"I", "E"} and prev_type == curr_type


def valid_path(tags: list[str]) -> bool:
    states = ["START"] + tags + ["STOP"]
    return all(legal_transition(a, b) for a, b in zip(states, states[1:]))


assert valid_path(["B-Disease", "E-Disease", "O", "S-Symptom"])
assert not valid_path(["O", "I-Disease"])
assert not valid_path(["B-Disease", "E-Symptom"])
assert valid_path(["O"])
print("ok")

52. 计算一条 CRF 路径的完整分数

发射分数为 [T,K]，转移字典含 START 和 STOP。

✅ 参考答案

def sequence_score(emissions: list[list[float]], tags: list[str], labels: list[str], transitions: dict) -> float:
    if len(emissions) != len(tags) or not tags:
        raise ValueError("序列长度必须一致且非空")
    label_to_id = {label: index for index, label in enumerate(labels)}
    if any(len(row) != len(labels) for row in emissions):
        raise ValueError("发射分数维度错误")
    if any(tag not in label_to_id for tag in tags):
        raise ValueError("未知标签")
    score = transitions[("START", tags[0])] + emissions[0][label_to_id[tags[0]]]
    for position in range(1, len(tags)):
        score += transitions[(tags[position - 1], tags[position])]
        score += emissions[position][label_to_id[tags[position]]]
    score += transitions[(tags[-1], "STOP")]
    return score


labels = ["O", "B-X"]
emissions = [[0.2, 1.0], [0.8, 0.1]]
transitions = {
    ("START", "O"): 0.0, ("START", "B-X"): 0.3,
    ("O", "O"): 0.2, ("O", "B-X"): -0.1,
    ("B-X", "O"): 0.5, ("B-X", "B-X"): -1.0,
    ("O", "STOP"): 0.1, ("B-X", "STOP"): -0.2,
}
score = sequence_score(emissions, ["B-X", "O"], labels, transitions)
assert abs(score - (0.3 + 1.0 + 0.5 + 0.8 + 0.1)) < 1e-9
print(score)

53. 用前向算法计算 log 配分函数

实现稳定 logsumexp，并用穷举小样例验证动态规划结果。

✅ 参考答案
Python
import itertools
import math


def logsumexp(values: list[float]) -> float:
    if not values:
        return -math.inf
    maximum = max(values)
    if maximum == -math.inf:
        return maximum
    return maximum + math.log(sum(math.exp(value - maximum) for value in values))


def log_partition(emissions: list[list[float]], labels: list[str], transitions: dict) -> float:
    if not emissions:
        return transitions[("START", "STOP")]
    alpha = {
        label: transitions[("START", label)] + emissions[0][index]
        for index, label in enumerate(labels)
    }
    for row in emissions[1:]:
        alpha = {
            current: row[current_index] + logsumexp([
                alpha[previous] + transitions[(previous, current)] for previous in labels
            ])
            for current_index, current in enumerate(labels)
        }
    return logsumexp([alpha[label] + transitions[(label, "STOP")] for label in labels])


labels = ["A", "B"]
emissions = [[0.2, 0.7], [0.4, -0.1], [0.0, 0.3]]
transitions = {
    ("START", "A"): 0.1, ("START", "B"): -0.2,
    ("A", "A"): 0.3, ("A", "B"): -0.4,
    ("B", "A"): 0.2, ("B", "B"): 0.1,
    ("A", "STOP"): 0.0, ("B", "STOP"): 0.2,
}

def brute_score(path):
    score = transitions[("START", path[0])]
    for position, tag in enumerate(path):
        score += emissions[position][labels.index(tag)]
        if position:
            score += transitions[(path[position - 1], tag)]
    return score + transitions[(path[-1], "STOP")]


all_scores = [brute_score(path) for path in itertools.product(labels, repeat=len(emissions))]
expected = logsumexp(all_scores)
actual = log_partition(emissions, labels, transitions)
assert abs(actual - expected) < 1e-12
print(actual)

54. 实现带回指针的 Viterbi 解码

返回全局最优标签路径和分数，并用穷举验证。

✅ 参考答案

import itertools


def viterbi_decode(emissions: list[list[float]], labels: list[str], transitions: dict):
    if not emissions:
        return [], transitions[("START", "STOP")]
    scores = {
        label: transitions[("START", label)] + emissions[0][index]
        for index, label in enumerate(labels)
    }
    backpointers = []
    for row in emissions[1:]:
        new_scores = {}
        step_back = {}
        for current_index, current in enumerate(labels):
            choices = [
                (scores[previous] + transitions[(previous, current)], previous)
                for previous in labels
            ]
            best_score, best_previous = max(choices, key=lambda item: (item[0], item[1]))
            new_scores[current] = best_score + row[current_index]
            step_back[current] = best_previous
        scores = new_scores
        backpointers.append(step_back)
    best_final_score, best_last = max(
        ((scores[label] + transitions[(label, "STOP")], label) for label in labels),
        key=lambda item: (item[0], item[1]),
    )
    path = [best_last]
    for step_back in reversed(backpointers):
        path.append(step_back[path[-1]])
    path.reverse()
    return path, best_final_score


labels = ["A", "B"]
emissions = [[0.2, 0.7], [0.4, -0.1], [0.0, 0.3]]
transitions = {
    ("START", "A"): 0.1, ("START", "B"): -0.2,
    ("A", "A"): 0.3, ("A", "B"): -0.4,
    ("B", "A"): 0.2, ("B", "B"): 0.1,
    ("A", "STOP"): 0.0, ("B", "STOP"): 0.2,
}

def score(path):
    total = transitions[("START", path[0])] + transitions[(path[-1], "STOP")]
    for position, tag in enumerate(path):
        total += emissions[position][labels.index(tag)]
        if position:
            total += transitions[(path[position - 1], tag)]
    return total


path, best_score = viterbi_decode(emissions, labels, transitions)
brute_path = max(itertools.product(labels, repeat=len(emissions)), key=lambda p: (score(p), p))
assert tuple(path) == brute_path
assert abs(best_score - score(brute_path)) < 1e-12
print(path, best_score)

55. 计算严格实体级指标并分类错误

严格 TP 要求文档、边界和类型全部相同；把非严格命中拆成类型错、边界错、虚假实体，剩余 gold 为漏检。

✅ 参考答案
Python
def evaluate_entities(gold: list[dict], predicted: list[dict]) -> dict:
    def key(item):
        return item["doc_id"], item["start"], item["end"], item["type"]

    gold_by_key = {key(item): item for item in gold}
    predicted_by_key = {key(item): item for item in predicted}
    exact_keys = set(gold_by_key) & set(predicted_by_key)
    unmatched_gold = [item for item in gold if key(item) not in exact_keys]
    unmatched_predicted = [item for item in predicted if key(item) not in exact_keys]
    errors = {"type_confusion": [], "boundary_error": [], "spurious": [], "missed": []}

    remaining_gold = unmatched_gold[:]
    for prediction in unmatched_predicted:
        same_boundary = next((item for item in remaining_gold
                              if item["doc_id"] == prediction["doc_id"]
                              and item["start"] == prediction["start"]
                              and item["end"] == prediction["end"]), None)
        if same_boundary is not None:
            errors["type_confusion"].append({"gold": same_boundary, "predicted": prediction})
            remaining_gold.remove(same_boundary)
            continue
        same_type_overlap = next((item for item in remaining_gold
                                  if item["doc_id"] == prediction["doc_id"]
                                  and item["type"] == prediction["type"]
                                  and max(item["start"], prediction["start"])
                                  < min(item["end"], prediction["end"])), None)
        if same_type_overlap is not None:
            errors["boundary_error"].append({"gold": same_type_overlap, "predicted": prediction})
            remaining_gold.remove(same_type_overlap)
        else:
            errors["spurious"].append(prediction)
    errors["missed"] = remaining_gold

    tp = len(exact_keys)
    fp = len(predicted_by_key) - tp
    fn = len(gold_by_key) - tp
    precision = tp / (tp + fp) if tp + fp else 0.0
    recall = tp / (tp + fn) if tp + fn else 0.0
    f1 = 2 * precision * recall / (precision + recall) if precision + recall else 0.0
    return {"tp": tp, "fp": fp, "fn": fn,
            "precision": precision, "recall": recall, "f1": f1, "errors": errors}


gold = [
    {"doc_id": "1", "start": 0, "end": 6, "type": "Disease"},
    {"doc_id": "1", "start": 8, "end": 10, "type": "Symptom"},
    {"doc_id": "2", "start": 0, "end": 2, "type": "Drug"},
]
predicted = [
    {"doc_id": "1", "start": 0, "end": 6, "type": "Disease"},
    {"doc_id": "1", "start": 8, "end": 9, "type": "Symptom"},
    {"doc_id": "2", "start": 0, "end": 2, "type": "Disease"},
    {"doc_id": "3", "start": 0, "end": 2, "type": "Disease"},
]

assert (report["tp"], report["fp"], report["fn"]) == (1, 3, 2)
assert round(report["precision"], 4) == 0.25
assert round(report["recall"], 4) == 0.3333
assert round(report["f1"], 4) == 0.2857
assert len(report["errors"]["boundary_error"]) == 1
assert len(report["errors"]["type_confusion"]) == 1
assert len(report["errors"]["spurious"]) == 1
print(report)

🧩 知识融合：实体统一、链接与消歧综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：知识融合：实体统一、链接与消歧

55 道练习，按“概念与 mention → 实体统一/链接/消歧 → 候选生成 → TF-IDF 与稠密检索 → 候选排序 → NIL/澄清 → 关系对齐与冲突融合 → 分模块评估和医学治理”组织。做题时始终记住：名称是可变字符串，实体是有稳定标识和语义边界的概念；相似度是候选证据，不是同一性的证明。

建议先独立作答，再展开参考答案。本练习依据原课件第五章和站内重组讲义设计，并参考 UMLS 官方概念与 CUI 说明、AIDA 全局实体消歧论文、BLINK 两阶段实体链接论文、生物医学概念 Generate-and-Rank 论文、生物医学实体链接统一评估研究与部分知识库下的 NIL 研究校准。10 道代码题只使用 Python 标准库离线运行，不访问真实患者数据或外部术语服务。

一、选择题（30 题）

1. 实体链接（Entity Linking）的直接目标是什么？

A. 将文本中的 mention 映射到知识库中的标准实体 ID，或判定为 NIL \ B. 只识别 mention 的字符边界 \ C. 为所有同名字符串创建同一节点 \ D. 自动生成医学诊断

✅ 参考答案

答案：A。 NER 解决“哪里提到了什么类型”，实体链接进一步回答“这个 mention 指向知识库中的哪个概念”。知识库无正确概念时必须允许 NIL。

2. 下列哪组最准确地区分 mention 与实体？

A. mention 是原文中的具体出现及偏移；实体是由稳定 ID 表示的概念 \ B. mention 与实体永远是同一个字符串 \ C. mention 只存在数据库，实体只存在文本 \ D. mention 不需要上下文

✅ 参考答案

答案：A。 同一实体可以有多个 mention 和别名；同一 surface form 也可能在不同上下文指向不同实体。

3. “慢支”“慢性支气管炎”统一到标准写法，主要属于什么？

A. 实体统一/概念归一 \ B. 关系抽取 \ C. 句法分析 \ D. 文本生成

✅ 参考答案

答案：A。 若进一步映射到 disease:000123 等知识库 ID，就是实体链接；在多个候选中依据上下文选择则涉及实体消歧。

4. “苹果发布新手机”中的“苹果”判为公司而非水果，主要属于什么？

A. 实体消歧 \ B. 字符编码转换 \ C. 关系对齐 \ D. 图数据库删除

✅ 参考答案

答案：A。 名称相同而意义不同，需要结合局部上下文、实体类型、描述和图邻域选择正确候选。

5. 把“患者”“他”“该男性”判定为同一文本对象，主要属于什么？

A. 指代消解 \ B. 实体候选排序 \ C. TF-IDF 拟合 \ D. 关系 Schema 删除

✅ 参考答案

答案：A。 指代消解把同一篇文本中的共指 mention 聚成簇；实体链接再把 mention 或簇映射到知识库概念。二者相关但输出合同不同。

6. 将 has_sign 与 has_symptom 统一为 HAS_SYMPTOM，主要属于什么？

A. 关系对齐 \ B. NER 边界解码 \ C. 文档分词 \ D. NIL 检测

✅ 参考答案

答案：A。 对齐不能只看名称，还要比较定义、方向、domain/range、基数、逆关系和适用范围。

7. 为什么标准实体 ID 不宜直接由 canonical_name 的哈希生成？

A. 标准名称可能更正、改写或多语言化，而概念身份应尽量稳定 \ B. 哈希不能处理任何字符串 \ C. 实体不需要名称 \ D. 名称永远不会变化

✅ 参考答案

答案：A。 名称是可更新属性。概念发生 merge、split、retire 时还应维护版本和重定向历史，而不是静默重建 ID。

8. UMLS 中 CUI 的主要语义是什么？

A. 唯一表示一个概念意义，可关联表达同一意义的多个名称 \ B. 只表示一个字符位置 \ C. 只表示某次 API 请求 \ D. 是关系置信度

✅ 参考答案

答案：A。 UMLS 还区分概念、术语、字符串和来源 atom。名称相同不自动意味着 CUI 相同，字符串不同也可能属于同一概念。

9. 直接用实体名称作为图数据库唯一主键，最典型的风险是？

A. 同义词造成重复节点，同名异义词造成错误合并 \ B. 自动提高候选召回率 \ C. 自动保留来源版本 \ D. 让 NIL 不再存在

✅ 参考答案

答案：A。 应以稳定概念 ID 为主键，把标准名、别名、源代码和历史名称作为带来源的属性管理。

10. 医学文本规范化时哪种策略最稳妥？

A. 只做有合同的全半角、大小写、空白和已知变体处理，保留有语义的数字、单位、部位与否定词 \ B. 删除所有数字和方向词 \ C. 删除所有括号内容 \ D. 把所有疾病词尾都去掉

✅ 参考答案

答案：A。 “左/右”“I/II 型”“5 mg/50 mg”等差异可能改变概念。每条规范化规则都要有正反例和可逆记录。

11. 候选生成模块的首要优化目标通常是什么？

A. 高 Recall@K，尽量让正确实体进入候选集合 \ B. 只返回一个候选 \ C. 让每个候选分数等于 1 \ D. 直接覆盖知识库

✅ 参考答案

答案：A。 排序器无法选出未被召回的正确实体。候选数量还要控制延迟，因此应同时报告 Recall@K、平均候选数和时延。

12. Recall@5 为 0.92 表示什么？

A. 在可链接样本中，92% 的正确实体出现在前 5 个候选内 \ B. 最终 top-1 Accuracy 一定是 0.92 \ C. NIL Precision 是 0.92 \ D. 每个样本恰有 5 个正确实体

✅ 参考答案

答案：A。 Recall@K 衡量候选生成上限，不代表排序器已把正确实体放到第一名。

13. 在候选生成时使用实体类型过滤的正确做法是？

A. 用可靠类型约束排除不兼容候选，并单独测量因类型错误造成的召回损失 \ B. 把所有 Disease 候选都删掉 \ C. 假定 NER 类型永远正确 \ D. 类型冲突时随机选择

✅ 参考答案

答案：A。 硬过滤效率高但会放大上游类型错误；可比较硬过滤、软特征和类型未知三种策略。

14. 编辑距离用于实体统一的主要边界是什么？

A. 能衡量字面接近，但不理解同义词、缩写和同名异义 \ B. 能自动读取全文上下文 \ C. 能证明两个医学概念完全相同 \ D. 只适用于图结构

✅ 参考答案

答案：A。 “心梗”和“心肌梗死”字面差异大却可能同义；完全相同的“苹果”也可能指不同概念。

15. 中文短 mention 使用字符 n-gram 的常见优势是？

A. 对分词边界和局部拼写变化较稳健 \ B. 自动获得世界知识 \ C. 自动完成 NIL 聚类 \ D. 不需要任何知识库

✅ 参考答案

答案：A。 n-gram 仍是词法特征，不理解深层语义。应与别名、上下文、类型或稠密表示组合。

16. TF-IDF 实体链接基线中，vectorizer 应如何使用？

A. 在知识库描述或训练允许的语料上 fit，查询和测试样本只 transform \ B. 对每条测试样本重新 fit 全部测试集 \ C. 用 gold ID 作为输入词 \ D. 删除 IDF

✅ 参考答案

答案：A。 在测试集上拟合词表和 IDF 会泄漏分布信息，也让线上重现困难。需保存词表、IDF、规范化器和知识库版本。

17. 查询向量在 TF-IDF 词表中全为 0 时，正确处理是什么？

A. 返回低信息/无法比较状态，进入其他召回通道、NIL 或澄清 \ B. 默认链接到第一个实体 \ C. 把余弦相似度设为无穷大 \ D. 随机扩展 gold 描述

✅ 参考答案

答案：A。 零向量没有可用词法证据；实现还要避免除零，并记录该失败模式。

18. 余弦相似度主要比较什么？

A. 两个向量方向的接近程度，而非原始长度大小 \ B. 两个 ID 的数字差 \ C. 两个图节点的创建时间 \ D. 文档字节数

✅ 参考答案

答案：A。 (\cos(\mathbf q,\mathbf d)=\frac{\mathbf q^\top\mathbf d}{|\mathbf q||\mathbf d|})。高余弦只是相似证据，仍需类型、阈值和上下文校验。

19. BLINK 式 bi-encoder 与 cross-encoder 的典型分工是？

A. bi-encoder 高效召回，cross-encoder 联合编码 mention 与候选做精排 \ B. 二者都只生成随机 ID \ C. cross-encoder 只能做 NER \ D. bi-encoder 不使用文本

✅ 参考答案

答案：A。 独立编码便于预计算实体向量和近邻搜索；交互编码通常更准确但对每个候选都需前向计算，成本更高。

20. 为什么候选排序训练需要 hard negatives？

A. 与 mention 很相似但错误的候选能迫使模型学习真正的区分特征 \ B. 随机负例永远比困难负例更接近 \ C. hard negatives 就是把 gold 改错 \ D. 只为减少训练数据

✅ 参考答案

答案：A。 应避免把可能漏标的同义概念当负例，也不能从测试标签构造训练负例。

21. AIDA 一类全局消歧方法利用什么信息？

A. 同一文档中候选实体之间的主题与图一致性 \ B. 只看单个字符 \ C. 只看数据库行号 \ D. 自动删除局部证据

✅ 参考答案

答案：A。 全局一致性可帮助局部歧义，但也可能把一个早期错误传播给整篇文档，因此应保留局部分数与全局增益。

22. 知识库没有正确实体时，系统应输出什么？

A. NIL/未收录，而不是强行链接最高分候选 \ B. 排名第一的任意实体 \ C. 删除原 mention \ D. 把 mention 变成关系

✅ 参考答案

答案：A。 NIL 可能表示真正新概念、知识库子集之外实体、类型不支持或证据不足，后续处理应区分这些原因。

23. top-1 分数较高但与 top-2 分差很小，较合理的决策是？

A. 进入澄清或人工复核，避免把细小分差当确定结论 \ B. 永远选择 top-1 \ C. 同时覆盖两个实体 ID \ D. 删除 top-2 的知识库记录

✅ 参考答案

答案：A。 三态决策可结合绝对阈值、top1-top2 margin、类型一致性和校准不确定性，输出 LINK、CLARIFY 或 NIL。

24. 链接阈值和 margin 应在哪里确定？

A. 在独立验证集按目标风险和指标校准，测试集只做最终评估 \ B. 反复查看测试结果后调整 \ C. 每个请求随机生成 \ D. 只根据实体 ID 大小

✅ 参考答案

答案：A。 医学高风险应用往往更重视误链接成本，可提高拒绝率，并报告 coverage-risk 曲线而非只给一个 Accuracy。

25. 若正确实体在候选排名第 4，该样本的 reciprocal rank 是？

A. (1/4) \ B. 4 \ C. 0 \ D. (1/16)

✅ 参考答案

答案：A。 MRR 是可评估样本 reciprocal rank 的平均值。应说明 NIL 样本是否排除，以及正确实体未召回时按 0 处理。

26. 排序器的 top-1 Accuracy 明显低于候选 Recall@10，最可能说明什么？

A. 正确实体多已召回，但排序或特征仍不足 \ B. 候选生成一定完全失败 \ C. NER 一定满分 \ D. NIL 不需要评估

✅ 参考答案

47. 构建别名索引并生成多通道候选

精确别名和字符 n-gram 模糊召回并行，类型过滤可选；同一别名允许映射多个 ID。

✅ 参考答案
Python
import re
import unicodedata


def normalize(text: str) -> str:
    return re.sub(r"\s+", "", unicodedata.normalize("NFKC", text).casefold())


def char_ngrams(text: str) -> set[str]:
    text = normalize(text)
    if len(text) < 2:
        return {text} if text else set()
    grams = set()
    for size in (2, 3):
        grams.update(text[i:i + size] for i in range(len(text) - size + 1))
    return grams


def build_alias_index(entities: list[dict]) -> dict[str, list[str]]:
    index = {}
    for entity in entities:
        for name in [entity["canonical_name"]] + entity.get("aliases", []):
            key = normalize(name)
            index.setdefault(key, set()).add(entity["entity_id"])
    return {key: sorted(ids) for key, ids in index.items()}


def generate_candidates(
    mention: str,
    entities: list[dict],
    entity_type=None,
    top_k: int = 5,
) -> list[dict]:
    alias_index = build_alias_index(entities)
    by_id = {entity["entity_id"]: entity for entity in entities}
    query_key = normalize(mention)
    query_grams = char_ngrams(mention)
    candidates = {}

    for entity_id in alias_index.get(query_key, []):
        entity = by_id[entity_id]
        if entity_type is None or entity["type"] == entity_type:
            candidates[entity_id] = {"entity_id": entity_id, "score": 1.0, "channels": {"exact"}}

    for entity in entities:
        if entity_type is not None and entity["type"] != entity_type:
            continue
        best = 0.0
        for name in [entity["canonical_name"]] + entity.get("aliases", []):
            grams = char_ngrams(name)
            union = query_grams | grams
            similarity = len(query_grams & grams) / len(union) if union else 0.0
            best = max(best, similarity)
        if best > 0:
            item = candidates.setdefault(
                entity["entity_id"],
                {"entity_id": entity["entity_id"], "score": 0.0, "channels": set()},
            )
            item["score"] = max(item["score"], best)
            item["channels"].add("char_ngram")

    ranked = sorted(candidates.values(), key=lambda item: (-item["score"], item["entity_id"]))
    for item in ranked:
        item["channels"] = sorted(item["channels"])
    return ranked[:top_k]


entities = [
    {"entity_id": "D1", "canonical_name": "慢性支气管炎", "aliases": ["慢支"], "type": "Disease"},
    {"entity_id": "D2", "canonical_name": "急性支气管炎", "aliases": [], "type": "Disease"},
    {"entity_id": "S1", "canonical_name": "支气管痉挛", "aliases": [], "type": "Symptom"},
]

assert generate_candidates("慢支", entities, "Disease")[0]["entity_id"] == "D1"
assert generate_candidates("慢性支气管发炎", entities, "Disease")[0]["entity_id"] == "D1"
assert all(item["entity_id"] != "S1" for item in generate_candidates("支气管", entities, "Disease"))
print(generate_candidates("慢性支气管发炎", entities, "Disease"))

48. 实现 Levenshtein 距离与规范化相似度

使用滚动数组把空间复杂度降为 (O(\min(m,n)))。

✅ 参考答案

import unicodedata


def levenshtein(first: str, second: str) -> int:
    first = unicodedata.normalize("NFKC", first).casefold()
    second = unicodedata.normalize("NFKC", second).casefold()
    if len(first) < len(second):
        first, second = second, first
    previous = list(range(len(second) + 1))
    for row, char_first in enumerate(first, start=1):
        current = [row]
        for column, char_second in enumerate(second, start=1):
            insert = current[column - 1] + 1
            delete = previous[column] + 1
            replace = previous[column - 1] + (char_first != char_second)
            current.append(min(insert, delete, replace))
        previous = current
    return previous[-1]


def normalized_similarity(first: str, second: str) -> float:
    maximum = max(len(first), len(second))
    if maximum == 0:
        return 1.0
    return 1 - levenshtein(first, second) / maximum


assert levenshtein("kitten", "sitting") == 3
assert levenshtein("高血压病", "高血压") == 1
assert normalized_similarity("", "") == 1.0
assert normalized_similarity("高血压病", "高血压") == 0.75
assert normalized_similarity("心梗", "心肌梗死") <= 0.5
print("ok")

49. 用标准库实现字符 TF-IDF 与余弦候选排序

IDF 只在知识库描述上拟合；查询只使用已冻结词表。

✅ 参考答案
Python
from collections import Counter
import math
import unicodedata


def grams(text: str) -> list[str]:
    text = "".join(unicodedata.normalize("NFKC", text).casefold().split())
    result = []
    for size in (2, 3):
        result.extend(text[i:i + size] for i in range(len(text) - size + 1))
    return result


def fit_idf(documents: list[str]) -> dict[str, float]:
    document_frequency = Counter()
    for document in documents:
        document_frequency.update(set(grams(document)))
    count = len(documents)
    return {
        term: math.log((count + 1) / (frequency + 1)) + 1
        for term, frequency in document_frequency.items()
    }


def vectorize(text: str, idf: dict[str, float]) -> dict[str, float]:
    counts = Counter(term for term in grams(text) if term in idf)
    return {term: count * idf[term] for term, count in counts.items()}


def cosine(first: dict[str, float], second: dict[str, float]) -> float:
    dot = sum(value * second.get(term, 0.0) for term, value in first.items())
    norm_first = math.sqrt(sum(value * value for value in first.values()))
    norm_second = math.sqrt(sum(value * value for value in second.values()))
    return dot / (norm_first * norm_second) if norm_first and norm_second else 0.0


def rank_descriptions(query: str, descriptions: dict[str, str]) -> list[tuple[str, float]]:
    ids = sorted(descriptions)
    idf = fit_idf([descriptions[entity_id] for entity_id in ids])
    query_vector = vectorize(query, idf)
    ranked = [
        (entity_id, cosine(query_vector, vectorize(descriptions[entity_id], idf)))
        for entity_id in ids
    ]
    return sorted(ranked, key=lambda item: (-item[1], item[0]))


descriptions = {
    "E_COMPANY": "科技公司，研发并发布手机、电脑和操作系统",
    "E_FRUIT": "水果，果实可食用，含维生素和膳食纤维",
}
ranking = rank_descriptions("这家公司发布了新款手机和操作系统", descriptions)
assert ranking[0][0] == "E_COMPANY"
assert ranking[0][1] > ranking[1][1]
assert cosine({}, {"x": 1.0}) == 0.0
print(ranking)

50. 实现可解释的多特征候选排序

按验证集确定的权重融合名称、上下文、类型和图邻域特征，并用 entity_id 确定性打破平局。

def rank_candidates(candidates: list[dict], weights: dict[str, float]) -> list[dict]:
    required = {"name", "context", "type", "graph"}
    if set(weights) != required or any(weight < 0 for weight in weights.values()):
        raise ValueError("权重字段错误或存在负权重")
    ranked = []
    for candidate in candidates:
        features = candidate["features"]
        if set(features) != required:
            raise ValueError("候选特征字段不完整")
        if any(not 0 <= value <= 1 for value in features.values()):
            raise ValueError("特征必须在 [0,1]")
        contributions = {
            name: weights[name] * features[name] for name in required
        }
        ranked.append({
            "entity_id": candidate["entity_id"],
            "score": sum(contributions.values()),
            "contributions": contributions,
        })
    return sorted(ranked, key=lambda item: (-item["score"], item["entity_id"]))

     "features": {"name": 0.9, "context": 0.95, "type": 1.0, "graph": 0.8}},
]

51. 实现 LINK、CLARIFY、NIL 三态决策

结合绝对分数、top1-top2 margin 和类型冲突返回可审计 reason code。

✅ 参考答案

def decide_link(
    ranked: list[dict],
    minimum_score: float = 0.70,
    minimum_margin: float = 0.10,
) -> dict:
    if not ranked:
        return {"status": "NIL", "entity_id": None, "reason": "no_candidate"}
    top = ranked[0]
    if not top.get("type_ok", True):
        return {"status": "CLARIFY", "entity_id": None, "reason": "type_conflict"}
    if top["score"] < minimum_score:
        return {"status": "NIL", "entity_id": None, "reason": "low_score"}
    second_score = ranked[1]["score"] if len(ranked) > 1 else 0.0
    if top["score"] - second_score < minimum_margin:
        return {"status": "CLARIFY", "entity_id": None, "reason": "small_margin"}
    return {"status": "LINK", "entity_id": top["entity_id"], "reason": "confident"}


assert decide_link([])["reason"] == "no_candidate"
assert decide_link([{"entity_id": "E1", "score": 0.60, "type_ok": True}])["status"] == "NIL"
ambiguous = [
    {"entity_id": "E1", "score": 0.86, "type_ok": True},
    {"entity_id": "E2", "score": 0.82, "type_ok": True},
]
assert decide_link(ambiguous)["status"] == "CLARIFY"
confident = [
    {"entity_id": "E1", "score": 0.91, "type_ok": True},
    {"entity_id": "E2", "score": 0.70, "type_ok": True},
]
assert decide_link(confident) == {
    "status": "LINK", "entity_id": "E1", "reason": "confident"
}

52. 计算候选 Recall@K、MRR 与端到端 Accuracy

def evaluate_linking(records: list[dict], k: int) -> dict:
    if k <= 0:
        raise ValueError("k 必须为正整数")
    linkable = [record for record in records if record["gold_id"] is not None]
    hits = 0
    reciprocal_sum = 0.0
    for record in linkable:
        candidate_ids = record["candidate_ids"]
        if record["gold_id"] in candidate_ids[:k]:
            hits += 1
        if record["gold_id"] in candidate_ids:
            reciprocal_sum += 1 / (candidate_ids.index(record["gold_id"]) + 1)
    accuracy = (
        sum(record["predicted_id"] == record["gold_id"] for record in records) / len(records)
        if records else 0.0
    )
    return {
        "samples": len(records),
        "linkable": len(linkable),
        f"recall@{k}": hits / len(linkable) if linkable else 0.0,
        "mrr": reciprocal_sum / len(linkable) if linkable else 0.0,
        "accuracy_all": accuracy,
    }


records = [
    {"gold_id": "D1", "candidate_ids": ["D1", "D3"], "predicted_id": "D1"},
    {"gold_id": "D2", "candidate_ids": ["D3", "D2"], "predicted_id": "D3"},
    {"gold_id": None, "candidate_ids": ["D4"], "predicted_id": None},
]

at_two = evaluate_linking(records, 2)
assert at_one["recall@1"] == 0.5
assert at_two["recall@2"] == 1.0

53. 计算 NIL 指标与选择性链接表现

把 NIL 作为正类计算 P/R/F1，同时报告系统实际自动链接的 coverage 和选择性准确率。

✅ 参考答案
Python
def evaluate_nil(gold_ids: list, predicted_ids: list) -> dict:
    if len(gold_ids) != len(predicted_ids):
        raise ValueError("gold 与 predicted 长度不一致")
    tp = sum(gold is None and pred is None for gold, pred in zip(gold_ids, predicted_ids))
    fp = sum(gold is not None and pred is None for gold, pred in zip(gold_ids, predicted_ids))
    fn = sum(gold is None and pred is not None for gold, pred in zip(gold_ids, predicted_ids))
    precision = tp / (tp + fp) if tp + fp else 0.0
    recall = tp / (tp + fn) if tp + fn else 0.0
    f1 = 2 * precision * recall / (precision + recall) if precision + recall else 0.0
    linked = [(gold, pred) for gold, pred in zip(gold_ids, predicted_ids) if pred is not None]
    selective_accuracy = (
        sum(gold == pred for gold, pred in linked) / len(linked) if linked else 0.0
    )
    return {
        "nil_tp": tp,
        "nil_fp": fp,
        "nil_fn": fn,
        "nil_precision": precision,
        "nil_recall": recall,
        "nil_f1": f1,
        "link_coverage": len(linked) / len(gold_ids) if gold_ids else 0.0,
        "selective_link_accuracy": selective_accuracy,
    }


gold = ["D1", None, "D2", None]
predicted = ["D1", None, None, "D3"]
report = evaluate_nil(gold, predicted)
assert (report["nil_tp"], report["nil_fp"], report["nil_fn"]) == (1, 1, 1)
assert report["nil_precision"] == report["nil_recall"] == report["nil_f1"] == 0.5
assert report["link_coverage"] == 0.5
assert report["selective_link_accuracy"] == 0.5
print(report)

54. 校验并执行带方向的关系 Schema 对齐

支持 same 与 inverse 映射，校验 source/target 的 domain 和 range，并保留原始谓词与 mapping_id。

✅ 参考答案
Python
def normalize_relation(triple: dict, mapping: dict, source: str) -> dict:
    predicate = triple["predicate"]
    if predicate not in mapping:
        raise ValueError("不存在关系映射")
    rule = mapping[predicate]
    direction = rule["direction"]
    if direction not in {"same", "inverse"}:
        raise ValueError("direction 只能是 same 或 inverse")
    if (triple["subject_type"], triple["object_type"]) != (
        rule["source_domain"], rule["source_range"]
    ):
        raise ValueError("原关系 domain/range 不匹配")

    if direction == "same":
        if (rule["source_domain"], rule["source_range"]) != (
            rule["target_domain"], rule["target_range"]
        ):
            raise ValueError("same 映射的 domain/range 不一致")
        subject_id, subject_type = triple["subject_id"], triple["subject_type"]
        object_id, object_type = triple["object_id"], triple["object_type"]
    else:
        if (rule["source_domain"], rule["source_range"]) != (
            rule["target_range"], rule["target_domain"]
        ):
            raise ValueError("inverse 映射的 domain/range 未交换")
        subject_id, subject_type = triple["object_id"], triple["object_type"]
        object_id, object_type = triple["subject_id"], triple["subject_type"]

    return {
        "subject_id": subject_id,
        "subject_type": subject_type,
        "predicate": rule["target_predicate"],
        "object_id": object_id,
        "object_type": object_type,
        "provenance": {
            "source": source,
            "source_predicate": predicate,
            "mapping_id": rule["mapping_id"],
        },
    }

    "treated_by": {

        "direction": "inverse",
        "source_domain": "Disease",
        "source_range": "Drug",
        "target_predicate": "TREATS",
        "target_domain": "Drug",
        "target_range": "Disease",
    }
}

55. 融合实体属性声明并保留冲突证据

同一 entity_id 和属性形成一个声明组；按来源等级、时间、置信度生成当前视图，但不删除其他值。

✅ 参考答案
Python
def fuse_attribute_claims(claims: list[dict], source_priority: dict[str, int]) -> list[dict]:
    groups = {}
    for claim in claims:
        required = {
            "entity_id", "attribute", "value", "source",
            "observed_at", "confidence", "evidence_id",
        }
        if required - set(claim):
            raise ValueError("声明字段不完整")
        if claim["source"] not in source_priority:
            raise ValueError("未知来源等级")
        if not 0 <= claim["confidence"] <= 1:
            raise ValueError("置信度越界")
        key = claim["entity_id"], claim["attribute"]
        group = groups.setdefault(key, {})
        previous = group.get(claim["evidence_id"])
        if previous is None or claim["confidence"] > previous["confidence"]:
            group[claim["evidence_id"]] = dict(claim)

    result = []
    for key in sorted(groups):
        evidence = list(groups[key].values())
        evidence.sort(key=lambda item: item["evidence_id"])
        current = max(
            evidence,
            key=lambda item: (
                source_priority[item["source"]],
                item["observed_at"],
                item["confidence"],
                item["evidence_id"],
            ),
        )
        values = {item["value"] for item in evidence}
        result.append({
            "entity_id": key[0],
            "attribute": key[1],
            "current_value": current["value"],
            "current_evidence_id": current["evidence_id"],
            "has_conflict": len(values) > 1,
            "all_evidence": evidence,
        })
    return result

🐼 Pandas 综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关讲义：Pandas 进阶：读写、清洗与分组透视

40 道练习，覆盖 Series 与 DataFrame、索引对齐、筛选、清洗、合并、分组聚合、透视表、时间序列和业务指标分析。

一、选择题（20 题）

1. 两个索引不同的 Series 相加时，Pandas 默认怎样处理？

A. 只按当前位置相加
B. 按索引标签对齐，无法匹配的位置产生缺失值
C. 自动删除全部索引
D. 强制按较短 Series 的长度截断

✅ 参考答案

答案：B。 Pandas 的核心语义是按标签对齐。需要把缺失位置视为 0 时，可使用 s1.add(s2, fill_value=0)。

2. df.shape 返回什么？

A. (行数, 列数)
B. 只有行数
C. 所有列名
D. 每列的数据类型

✅ 参考答案

答案：A。 shape 是二维尺寸元组；df.size 才是单元格总数，df.dtypes 返回各列类型。

3. loc 与 iloc 的主要区别是？

A. loc 按标签，iloc 按整数位置
B. loc 只能取列
C. iloc 的切片包含右端点
D. 二者都只能读取不能赋值

✅ 参考答案

答案：A。 loc['a':'c'] 的标签切片通常包含右端标签；iloc[0:3] 遵循 Python 位置切片的左闭右开规则。

4. 同时筛选 score >= 80 且 city == '北京'，正确写法是？

A. df[df.score >= 80 and df.city == '北京']
B. df[(df['score'] >= 80) & (df['city'] == '北京')]
C. df[df['score'] >= 80, df['city'] == '北京']
D. df.where('score >= 80')

✅ 参考答案

答案：B。 Series 条件要使用逐元素运算符 &、|、~，并给每个条件加括号，不能使用 Python 标量逻辑词 and、or。

5. 下面哪种赋值方式最能避免链式赋值问题？

A. df[df['x'] > 0]['y'] = 1
B. df.loc[df['x'] > 0, 'y'] = 1
C. df['x']['y'] = 1
D. df.values[0] = 'all'

✅ 参考答案

答案：B。 .loc[行条件, 列] 在一步中明确定位并赋值。链式索引可能操作临时对象，导致警告或修改不生效。

6. df.isna().sum() 常用于什么？

A. 统计每列缺失值数量
B. 删除重复行
C. 计算列均值
D. 设置索引

✅ 参考答案

答案：A。 isna() 生成布尔表，默认 sum(axis=0) 统计每列 True 的数量。

7. 只想删除 user_id 或 amount 缺失的行，其他列缺失可以保留，应使用？

A. df.dropna()
B. df.dropna(subset=['user_id', 'amount'])
C. df.drop_duplicates()
D. df.fillna(0)

✅ 参考答案

答案：B。 subset 把删除依据限定在关键字段，避免因非核心列缺失而误删整行。

8. 对偏态严重且含异常值的收入列填补缺失，通常比均值更稳健的是？

A. 中位数
B. 最大值
C. 字符串空格
D. 行号

✅ 参考答案

答案：A。 中位数对极端值不敏感。但填补策略仍应结合缺失机制、业务含义与分组结构，而不是机械套用。

9. 哪种 dtype 适合取值集合较小、重复很多的字符串类别列？

A. category
B. complex128
C. timedelta64
D. object 永远最省内存

✅ 参考答案

答案：A。 category 用编码和类别表表示重复值，常能节省内存，并能表达有序类别。

10. df.sort_values(['city', 'score'], ascending=[True, False]) 表示？

A. 城市降序、分数升序
B. 先按城市升序，同城内按分数降序
C. 只按分数排序
D. 修改索引为城市

✅ 参考答案

答案：B。 多列排序按列表顺序确定主次键；ascending 可为每一列单独指定方向。

11. groupby('city')['sales'].agg(['count', 'sum', 'mean']) 的结果是什么？

A. 每个城市的数量、总额和均值
B. 原表逐行累计值
C. 只保留销售额最大行
D. 把城市拆成多列

✅ 参考答案

答案：A。 agg 将每个分组压缩成汇总结果。需要让统计值回到原行时，应考虑 transform。

12. groupby(...).transform('mean') 与 agg('mean') 的关键区别是？

A. transform 返回与原对象对齐的同长度结果
B. agg 一定返回同长度结果
C. transform 只能处理字符串
D. 二者没有区别

✅ 参考答案

答案：A。 transform 适合计算组内占比、组内中心化等逐行特征；agg 适合生成分组摘要表。

13. pivot_table 默认如何处理同一行列组合对应多条记录？

A. 使用聚合函数汇总，默认常为均值
B. 必然报重复错误
C. 随机保留一条
D. 自动删除数值列

✅ 参考答案

答案：A。 pivot_table 支持 aggfunc；严格要求组合唯一且不聚合时可使用 pivot。

14. pd.merge(left, right, on='id', how='left') 会保留哪些键？

A. 仅两表交集
B. 左表全部键，并匹配右表信息
C. 右表全部键
D. 两表全部键且一定不重复

✅ 参考答案

答案：B。 左连接以左表为基准；右侧无匹配时新增列为缺失值。

15. 合并订单表和用户表时，用户表的 user_id 应唯一。哪个参数可主动验证这一关系？

A. validate='many_to_one'
B. sort=False
C. suffixes=None
D. copy=False

✅ 参考答案

答案：A。 多笔订单可对应一个用户，因此是 many-to-one。validate 能及早发现维表键重复导致的行数膨胀。

16. pd.concat([jan, feb], axis=0, ignore_index=True) 的典型用途是？

A. 按行合并相同结构的月度数据并重建索引
B. 按键做数据库连接
C. 计算相关系数
D. 把全部列转成日期

✅ 参考答案

答案：A。 concat 适合沿某轴堆叠对象；涉及业务键匹配时通常使用 merge。

17. 将字符串日期列可靠转换为日期类型，推荐？

A. pd.to_datetime(df['date'], errors='coerce')
B. df['date'].astype(int)
C. df['date'].str.len()
D. df['date'].sum()

✅ 参考答案

答案：A。 无法解析的值会在 errors='coerce' 下变为 NaT，便于后续检查，而不是静默保留脏字符串。

18. 日期列转换后，提取年份的常用写法是？

A. df['date'].dt.year
B. df['date'].year()
C. df.year['date']
D. np.year(df)

✅ 参考答案

答案：A。 Pandas 的日期时间 Series 通过 .dt 访问年月日、星期等向量化属性。

19. 一列标签内容为 'AI|Python'，想拆成列表并展开为多行，常用组合是？

A. .str.split('|').explode()
B. .sum().mean()
C. .dropna().pivot()
D. .astype(float)

✅ 参考答案

答案：A。 需注意 str.split 的 pat 是否按正则解释；可写 str.split('|', regex=False) 明确按字面量分隔。

20. 为什么数值列运算通常应优先使用向量化表达式，而不是逐行 apply(axis=1)？

A. 向量化通常更快、更清晰，也更容易利用底层优化
B. apply 永远不能运行
C. 向量化会自动修复所有业务错误
D. apply 只能处理日期

✅ 参考答案

答案：A。 apply(axis=1) 常回到 Python 逐行调用，性能较差。复杂、无法向量化的业务逻辑才考虑 apply，并应先评估数据规模。

二、简答题（8 题）

21. 什么是 Pandas 的索引对齐？它为什么既强大又可能带来风险？

✅ 参考答案

Pandas 在算术、赋值和拼接时常按索引标签对齐，而不是只看位置。这使不同来源的数据能自然对接；但重复索引、错位索引或意外保留的旧索引也会产生 NaN、重复或错误匹配。关键步骤应检查索引唯一性、集合差异和结果行数。

22. 什么是链式赋值？怎样写出更可靠的修改代码？

✅ 参考答案

链式赋值是连续使用两次索引后再赋值，如 df[df.x > 0]['y'] = 1。中间结果可能是视图也可能是副本，修改不一定回到原表。应写成 df.loc[df.x > 0, 'y'] = 1；若确实要创建独立子表，则先 .copy()。

23. agg、transform 和 filter 在分组操作中的职责分别是什么？

✅ 参考答案

agg 把每组压缩成一个或多个统计值；transform 生成与原行对齐的同长度结果；filter 根据整组条件决定保留或删除该组。选择时先问：我要摘要表、逐行特征，还是筛掉整组？

24. 内连接、左连接、右连接和外连接应该如何选择？

✅ 参考答案

内连接只保留两侧匹配键；左连接保留左侧业务主体；右连接相反；外连接保留两侧所有键，适合核对差异。选择取决于谁是分析主体以及未匹配数据是否有业务价值。合并后要检查行数、键唯一性和 _merge 来源标记。

25. 缺失值处理为什么不能只用统一的 fillna(0)？

✅ 参考答案

0 可能是有效业务值，也可能改变分布和统计含义。处理前要区分结构性缺失、随机缺失、采集失败和“不适用”；再选择删除、常量填补、总体或分组统计填补、模型填补，或保留缺失指示列。关键是记录规则并验证影响。

26. pivot、pivot_table 与 groupby 各适合什么场景？

✅ 参考答案

pivot 只做长宽转换，要求行列组合唯一；pivot_table 在转换时允许对重复组合聚合，适合报表；groupby 的分组、聚合和变换更灵活，适合分析流水线。三者本质都围绕键和数值重组数据。

27. 什么是长表和宽表？为什么分析和展示可能偏好不同形态？

✅ 参考答案

长表通常一行表示一次观测，变量名和数值纵向重复，适合分组、统计和可视化；宽表把类别展开成列，便于报表阅读或某些模型输入。可使用 melt 从宽转长，使用 pivot 或 pivot_table 从长转宽。

28. 请给出一个可靠数据清洗流水线的检查顺序。

✅ 参考答案

先确认行列规模、字段含义和主键；统一列名与类型；检查缺失、重复、非法范围和异常类别；按明确规则清洗；再做表连接并验证关系；最后核对行数、唯一性、汇总金额和抽样记录。每一步尽量保留日志和断言，避免静默错误。

三、代码题（12 题）

29. 创建订单 DataFrame，修正金额类型，并把无法转换的金额标记为缺失。

✅ 参考答案
Python
import pandas as pd

orders = pd.DataFrame({
    "order_id": [101, 102, 103, 104],
    "user_id": [1, 2, 1, 3],
    "amount": ["99.5", "invalid", "120", "88.0"],
})

orders["amount"] = pd.to_numeric(orders["amount"], errors="coerce")
print(orders)
print(orders.dtypes)

errors='coerce' 让脏值变成 NaN，后续必须决定填补、删除还是回查来源。

30. 从学生表中筛选北京或上海、分数不低于 85 且出勤率高于 0.9 的记录，只保留姓名和分数。

✅ 参考答案
Python
import pandas as pd

students = pd.DataFrame({
    "name": ["小林", "小周", "小陈", "小郭"],
    "city": ["北京", "广州", "上海", "北京"],
    "score": [88, 92, 85, 80],
    "attendance": [0.95, 0.96, 0.91, 0.98],

import pandas as pd

    "order_id": [1, 2, 3, 4, 5, 5],
    "user_id": [10, 11, 10, 12, 13, 13],
    "order_date": ["2026-01-03", "2026-01-20", "2026-02-02", "bad", "2026-02-15", "2026-02-15"],

🪵 Python Logging：项目日志体系综合训练

🏷️ 选择 + 简答 + 代码🏷️ 40 题📖 相关章节：Python Logging：项目日志体系🚀 所属项目：EduRAG：企业私有知识智能问答

40 道练习，覆盖日志作用（调试/监控/审计）、级别阶梯、basicConfig、Formatter 占位符、FileHandler 与双通道输出，以及工程化 setup_logger（建目录、防重复 handler、按处理器分流级别）。重点练「记录器级别 vs 处理器级别」和「为何别用 print 硬扛生产排障」。

建议先独立作答，再展开参考答案。代码题用纯 Python 模拟级别过滤与双 handler 路由，不依赖真实日志框架副作用。

二、简答题（12 题）

21. 为什么项目里要坚持打日志而不是只靠复现时的 print？

✅ 参考答案

线上问题难复现；日志带时间、级别与上下文，可事后审计与监控。分级还能在生产降噪、在调试加明细。

22. 解释 WARNING 与 ERROR 的语义差别。

✅ 参考答案

WARNING：可能有问题但仍可继续；ERROR：已发生错误并影响功能。CRITICAL 更严重，可能面临崩溃/不可用。

23. basicConfig 适合什么阶段？有何局限？

✅ 参考答案

适合快速入门/小脚本。一旦需要多 handler、不同级别分流、防重复配置，应显式 getLogger + Handler + Formatter（或封装 setup_logger）。

24. 为何文件用 DEBUG、控制台用 INFO 是常见组合？

✅ 参考答案

控制台给运维/开发即时看关键路径；文件保留更细调试线索。避免控制台被 DEBUG 淹没，又不丢排障信息。

25. filemode='w' 有什么风险？

✅ 参考答案

每次启动覆盖旧日志，历史排查线索丢失。长期运行多用追加 'a'，并结合轮转/归档策略控制体积。

26. 简述「记录器级别」和「处理器级别」如何共同过滤。

✅ 参考答案

先看 logger：低于 logger 级别的直接丢。通过后交给各 handler，再按 handler 自己的级别决定是否写出。两边都过才输出到该目的地。

27. setup_logger 为什么要 exist_ok=True 建目录？

✅ 参考答案

日志路径父目录可能不存在；重复启动也不应因目录已存在而报错，保证幂等初始化。

28. 多模块项目如何避免日志重复打印？

✅ 参考答案

同一命名 logger 只添加一次 handlers（if not logger.handlers）；避免到处 basicConfig；注意传播到 root 造成双份时可关 propagate（进阶）。

29. 格式串里同时放 asctime、levelname、name、message 各解决什么？

✅ 参考答案

何时、多严重、哪个组件、发生了什么——四要素支撑检索与定位。

30. 空数据触发 logger.error 后返回 None，这种模式的价值？

✅ 参考答案

把失败变成可观察事件 + 可控返回值，调用方可降级/告警，而不是静默错误或只抛未记录异常。

31. 生产日志为何要谈「脱敏」？

✅ 参考答案

日志常进集中平台/多人可读；token、密码、身份证等泄露风险高。应遮罩或禁止落盘，只留必要业务主键。

32. 章节 chips 提到「日志轮转」时，工程上通常想解决什么？（概念）

✅ 参考答案

长期追加会导致单文件过大、占盘难检索。轮转按大小/时间切分并保留有限份数，平衡可观测性与磁盘。具体可用 RotatingFileHandler 等（课件封装以 FileHandler 为主）。

三、代码题（8 题）

33. 级别过滤：给定阈值，哪些消息会输出？

✅ 参考答案
Python
LEVELS = {"DEBUG": 10, "INFO": 20, "WARNING": 30, "ERROR": 40, "CRITICAL": 50}


def passes(msg_level: str, threshold: str) -> bool:
    return LEVELS[msg_level] >= LEVELS[threshold]


assert not passes("DEBUG", "INFO")
assert passes("ERROR", "INFO")
print([lv for lv in LEVELS if passes(lv, "WARNING")])

34. 模拟 Formatter：按模板渲染记录

✅ 参考答案
Python
def format_record(template: str, **fields) -> str:
    # 教学：用 str.format_map；真实 logging 用 LogRecord 属性
    mapping = {k.strip("%()s"): v for k, v in {
        "%(asctime)s": fields["asctime"],
        "%(levelname)s": fields["levelname"],
        "%(message)s": fields["message"],
    }.items()}
    out = template
    for k, v in mapping.items():
        out = out.replace(f"%({k})s", str(v))
    return out


s = format_record(
    "%(asctime)s - %(levelname)s - %(message)s",
    asctime="2025-04-01",
    levelname="INFO",
    message="程序启动",
)
assert s == "2025-04-01 - INFO - 程序启动"
print(s)

35. 双 handler 分流：console INFO+ / file DEBUG+

✅ 参考答案
Python
def route(msg_level: str, logger_level="DEBUG") -> dict:
    LEVELS = {"DEBUG": 10, "INFO": 20, "WARNING": 30, "ERROR": 40, "CRITICAL": 50}
    if LEVELS[msg_level] < LEVELS[logger_level]:
        return {"console": False, "file": False}
    return {
        "console": LEVELS[msg_level] >= LEVELS["INFO"],
        "file": LEVELS[msg_level] >= LEVELS["DEBUG"],
    }


assert route("DEBUG") == {"console": False, "file": True}
assert route("INFO") == {"console": True, "file": True}
print(route("ERROR"))

def write_log(buf: list[str], line: str, mode: str = "a") -> list[str]:
    if mode == "w":
        return [line]
    if mode == "a":
        return buf + [line]
    raise ValueError(mode)

b = write_log(b, "a2", "a")

b = write_log(b, "only", "w")

37. 防重复 addHandler

✅ 参考答案
Python
class MiniLogger:
    def __init__(self, name):
        self.name = name
        self.handlers = []

    def addHandler(self, h):
        self.handlers.append(h)


def setup(logger, handlers):
    if not logger.handlers:
        for h in handlers:
            logger.addHandler(h)
    return logger


lg = MiniLogger("MainApp")
setup(lg, ["console", "file"])
setup(lg, ["console", "file"])  # 第二次不应再加
assert lg.handlers == ["console", "file"]
print(len(lg.handlers))

from pathlib import Path


def ensure_log_dir(log_file: str, existing: set[str]) -> set[str]:
    parent = str(Path(log_file).parent)
    if parent in (".", ""):
        return existing
    return existing | {parent}


dirs = ensure_log_dir("logs/app.log", set())

40. 迷你 setup_logger：返回可调用的记录接口

✅ 参考答案
Python
LEVELS = {"DEBUG": 10, "INFO": 20, "WARNING": 30, "ERROR": 40, "CRITICAL": 50}


class MiniSetupLogger:
    def __init__(self, name, console_level="INFO", file_level="DEBUG"):
        self.name = name
        self.console_level = console_level
        self.file_level = file_level
        self.console = []
        self.file = []

    def _log(self, level, msg):
        if LEVELS[level] < LEVELS["DEBUG"]:
            return
        line = f"{level} - {self.name} - {msg}"
        if LEVELS[level] >= LEVELS[self.file_level]:
            self.file.append(line)
        if LEVELS[level] >= LEVELS[self.console_level]:
            self.console.append(line)

    def debug(self, msg): self._log("DEBUG", msg)
    def info(self, msg): self._log("INFO", msg)
    def error(self, msg): self._log("ERROR", msg)


log = MiniSetupLogger("MainApp")
log.debug("仅文件")
log.info("程序启动")
assert any("仅文件" in x for x in log.file) and not any("仅文件" in x for x in log.console)
assert any("程序启动" in x for x in log.console)
print(len(log.file), len(log.console))

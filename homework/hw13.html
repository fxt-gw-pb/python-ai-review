🌐 网络编程

🏷️ 简答 + 实操🏷️ 5 题📖 相关讲义：网络编程、进程与线程入门

网络编程练习从 IP、端口和协议三个基本要素出发，覆盖 TCP/UDP 对比、连接过程、字符编码，以及一组可以在本机直接运行的 TCP 客户端与服务端。

一、网络基础

1. 网络编程的三要素是什么？各自有什么作用？

✅ 参考答案

网络编程的三要素通常指：

  • IP 地址：定位网络中的一台主机，解决“数据发到哪台设备”。
  • 端口号：定位主机中的具体应用进程，解决“交给哪个程序”。端口范围是 0～65535。
  • 通信协议：约定数据传输与解析规则，解决“双方如何交流”。常见传输层协议有 TCP 和 UDP。

可把它类比为寄快递：IP 是收件地址，端口是具体收件人，协议是双方约定的包装和交接规则。

2. TCP 和 UDP 有哪些主要区别？

✅ 参考答案
对比维度 TCP UDP
连接 面向连接，通信前先建立连接 无连接，直接发送数据报
可靠性 可靠、有序，具备确认、重传等机制 不保证到达、顺序或去重
数据形式 字节流，没有天然消息边界 一个个独立数据报，有消息边界
开销与速度 开销较大，通常相对慢 开销小，通常相对快
典型场景 文件传输、网页、登录 直播、语音、在线游戏、广播

选择时不能只看“谁更快”：若业务不能接受数据丢失或乱序，通常优先 TCP；若更关注实时性且能容忍少量丢包，可以考虑 UDP。

3. 简述 TCP 的三次握手和四次挥手

✅ 参考答案

三次握手用于建立连接：

  1. 客户端发送 SYN，请求建立连接。
  2. 服务端回复 SYN + ACK，确认客户端请求，同时发出自己的连接请求。
  3. 客户端回复 ACK，双方确认收发能力正常，连接建立。

四次挥手用于关闭连接：

  1. 主动关闭方发送 FIN，表示自己不再发送数据。
  2. 被动关闭方回复 ACK，确认收到关闭请求。
  3. 被动关闭方处理完剩余数据后发送 FIN。
  4. 主动关闭方回复 ACK，连接最终释放。

TCP 是全双工通信，两个方向需要分别关闭，因此断开连接通常需要四次交互。

二、编码与 TCP 实操

1. 在字符串和 bytes 之间转换

把字符串 AI 人工智能进阶班 转换为二进制 bytes；再把 b"AI python" 转换为字符串 str。

✅ 参考答案
Python
text = "AI 人工智能进阶班"
binary_data = text.encode("utf-8")
print(binary_data)
print(type(binary_data))  # <class 'bytes'>

raw_data = b"AI python"
decoded_text = raw_data.decode("utf-8")
print(decoded_text)       # AI python
print(type(decoded_text)) # <class 'str'>
  • str.encode(编码)：字符串编码为字节数据。
  • bytes.decode(编码)：字节数据解码为字符串。

发送方和接收方必须约定相同编码，中文场景通常统一使用 UTF-8。

2. 编写 TCP 客户端和服务端完成一次消息传输

客户端发送“欢迎来到我的世界！”，服务端接收、解码并打印消息。

✅ 参考答案

先创建并运行服务端 server.py：

Python
import socket


HOST = "127.0.0.1"
PORT = 9000

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as server_socket:
    server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
    server_socket.bind((HOST, PORT))
    server_socket.listen(1)
    print(f"服务端正在监听 {HOST}:{PORT}")

    client_socket, client_address = server_socket.accept()
    with client_socket:
        data = client_socket.recv(1024)
        print("客户端地址：", client_address)
        print("收到消息：", data.decode("utf-8"))

再另开一个终端运行客户端 client.py：

Python
import socket


HOST = "127.0.0.1"
PORT = 9000
message = "欢迎来到我的世界！"

with socket.socket(socket.AF_INET, socket.SOCK_STREAM) as client_socket:
    client_socket.connect((HOST, PORT))
    client_socket.sendall(message.encode("utf-8"))

运行顺序必须是先服务端、后客户端。127.0.0.1 表示本机回环地址；若要跨设备通信，需要使用服务端在局域网中的地址，并确认端口未被防火墙阻止。

🏗️ Transformer 架构与工程实现综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：Transformer 架构与工程实现

55 道练习，按“输入表示 → 多头注意力 → Add & Norm → FFN → Encoder → Decoder → 输出与训练”组织。重点是从数据流、张量形状和 Mask 语义理解 Transformer，而不是只背架构图。

建议先独立作答，再展开参考答案。代码题统一采用 batch_first=True，布尔 Mask 遵循 PyTorch 高层模块常用约定：True 表示该位置被屏蔽。

一、选择题（30 题）

1. Transformer 相比传统 RNN 最核心的序列建模变化是？

A. 删除所有非线性函数
B. 以注意力为主建立位置间依赖，不再依靠隐藏状态逐时间步递归
C. 只能处理固定长度输入
D. 不再需要训练数据

✅ 参考答案

答案：B。 Transformer 用自注意力让各位置直接交互，训练时可并行处理时间步；但自回归推理仍需逐 token 生成。

2. 原始 Encoder–Decoder Transformer 的四个宏观部分是？

A. 卷积、池化、循环、聚类
B. 分词、词云、翻译、部署
C. 输入表示、编码器、解码器、词表输出
D. Query、Key、Value、标签

✅ 参考答案

答案：C。 输入表示包含 token embedding 与位置编码；编码器产生 memory，解码器结合目标前缀与 memory，输出层映射到目标词表 logits。

3. 一个标准 Transformer Encoder 层通常包含哪两个主要子层？

A. 多头自注意力与位置前馈网络
B. 掩码自注意力与交叉注意力
C. Embedding 与 Softmax
D. 卷积与循环层

✅ 参考答案

答案：A。 每个子层还配有残差连接、LayerNorm 和 Dropout；Encoder 不需要标准的因果 Mask，除非任务特意要求因果编码。

4. 一个标准 Transformer Decoder 层比 Encoder 层多出的核心子层是？

A. 第二个 Embedding
B. BatchNorm
C. 最大池化
D. 对 Encoder memory 的交叉注意力

✅ 参考答案

答案：D。 Decoder 通常依次包含带因果约束的目标自注意力、对 Encoder memory 的 Cross-Attention，以及位置前馈网络。

5. Token Embedding 后加入位置编码的主要原因是？

A. 注意力无法处理浮点数
B. 纯自注意力本身不携带序列顺序，需要显式注入位置
C. 位置编码负责输出类别
D. 可自动删除 PAD

✅ 参考答案

答案：B。 没有位置相关信号时，自注意力对输入排列具有置换等变性，难以区分相同 token 集合的不同顺序。

6. 原始 Transformer 常将 token embedding 乘以什么尺度？

A. $d_{model}$
B. $1/d_{model}$
C. $\sqrt{d_{model}}$
D. 序列长度

✅ 参考答案

答案：C。 该缩放使 embedding 与位置编码的尺度更匹配。它与注意力分数除以 $\sqrt{d_k}$ 是两个不同位置、不同目的的缩放。

7. 正弦—余弦位置编码的一个重要性质是？

A. 不含可训练参数，并可为训练长度之外的位置按公式计算
B. 每个位置都必须通过标签学习
C. 所有位置向量完全相同
D. 只能用于 Decoder

✅ 参考答案

答案：A。 固定位置编码用不同频率的正弦和余弦表达位置，可外推计算；但模型是否真正泛化到更长序列仍需验证。

8. 可学习绝对位置 Embedding 的直接限制是？

A. 无法反向传播
B. 不能与 token embedding 相加
C. 不占用参数
D. 需要预设最大位置数，超出表范围不能直接查表

✅ 参考答案

答案：D。 工程中要校验 seq_len <= max_len；扩展上下文时可能需要插值、重新训练或改用其他位置方案。

9. 使用 d_model=510、nhead=8 的标准均匀多头注意力，主要问题是？

A. 序列长度太短
B. d_model 不能被头数整除，无法均匀拆成整数维子空间
C. 不能使用 Softmax
D. Value 维度必须是 8

✅ 参考答案

答案：B。 常规实现要求 $d_{head}=d_{model}/nhead$ 为整数，框架会直接报错或要求更改配置。

10. d_model=768、nhead=12 时，每个头的维度通常是？

A. 12
B. 32
C. 64
D. 9216

✅ 参考答案

答案：C。 $768/12=64$。头数增加并不必然增加注意力投影总参数量，因为总模型维度仍为 768。

11. 标准多头注意力中，四个主要线性投影通常是？

A. $W_Q,W_K,W_V,W_O$
B. $W_x,W_h,W_c,W_y$
C. 卷积核、池化核、偏置、标签
D. 只有一个 $W$

✅ 参考答案

答案：A。 前三个生成各头的 Q/K/V，拼接各头后由 $W_O$ 融合回 d_model。

12. 输入与输出均采用 d_model 时，多头注意力层输出形状通常是？

A. [B,H,T,d_model]
B. [B,T,nhead]
C. [T,T]
D. [B,T,d_model]

✅ 参考答案

答案：D。 头维在内部拆分和合并，对外接口恢复模型维度，才能与输入做残差相加。

13. 残差连接 x + Sublayer(x) 对形状的基本要求是？

A. 两个张量必须可对齐相加，通常保持相同的 d_model
B. 子层输出必须是标量
C. batch size 必须为 1
D. 序列长度必须等于词表大小

✅ 参考答案

答案：A。 注意力输出投影和 FFN 第二层都会回到 d_model，部分原因就是要维持残差路径的形状。

14. Transformer 中 LayerNorm 通常沿哪个维度归一化？

A. 整个数据集
B. batch 维
C. 每个 token 表示的最后一个特征维 d_model
D. 词表维

✅ 参考答案

答案：C。 对 [B,T,D] 使用 LayerNorm(D) 时，每个样本每个位置独立按 D 个特征归一化，不依赖其他 batch 样本。

15. LayerNorm 相比 BatchNorm 更适合变长序列建模的一个原因是？

A. 不依赖 batch 统计量，小 batch 与变长输入下行为更稳定
B. LayerNorm 没有参数
C. BatchNorm 不能反向传播
D. LayerNorm 会自动生成位置编码

✅ 参考答案

答案：A。 BatchNorm 依赖同一特征在 batch/空间上的统计，容易受 batch 大小和 padding 影响；LayerNorm 在单个 token 特征内部计算。

16. Transformer 的位置前馈网络（FFN）如何作用？

A. 在序列维做全局平均
B. 每个位置使用不同参数
C. 把所有位置拼成一个向量
D. 对每个位置独立应用同一组两层 MLP 参数

✅ 参考答案

答案：D。 注意力负责位置间混合，FFN 负责每个位置内部的通道变换；同一层中所有时间步共享 FFN 权重。

17. 标准 FFN 的常见形式是？

A. $\operatorname{softmax}(X)$
B. $W_2\,\sigma(W_1X+b_1)+b_2$
C. $QK^T$
D. $XW_1W_1^T$

✅ 参考答案

答案：B。 第一层从 d_model 扩展到 d_ff，激活后第二层投影回 d_model；现代模型也常用 GELU、SwiGLU 等变体。

18. 输入 [B,T,d_model] 经过标准 FFN 后，输出形状是？

A. [B,T,d_ff]
B. [B,d_model]
C. [B,T,d_model]
D. [T,T]

✅ 参考答案

答案：C。 中间层是 d_ff，最终必须回到 d_model 以便残差相加和层堆叠。

19. 双向 Encoder Self-Attention 通常需要哪种 Mask？

A. 需要源序列 Padding Mask，通常不需要 Causal Mask
B. 只需要 Causal Mask
C. 完全不允许 Mask
D. 必须屏蔽过去位置

✅ 参考答案

答案：A。 Encoder 通常允许每个真实 token 看完整源序列，但不能关注 PAD；若做因果编码等特殊任务，才另加结构化 Mask。

20. Decoder Self-Attention 通常需要？

A. 只需要源序列 Padding Mask
B. 只需要 Encoder 输出
C. 不允许任何 Mask
D. 目标 Padding Mask 与 Causal Mask

✅ 参考答案

答案：D。 前者屏蔽目标 PAD，后者屏蔽未来目标位置；训练中两者常需同时存在。

21. Decoder Cross-Attention 中 Q、K、V 的来源是？

A. Q/K/V 都来自 Encoder
B. Q 来自 Decoder 当前表示，K/V 来自 Encoder memory
C. Q 来自标签，K/V 来自位置编码
D. Q/K/V 都来自目标词表

✅ 参考答案

答案：B。 Decoder 用当前目标侧表示查询源侧编码结果，从而将生成位置与相关源 token 对齐。

22. memory_key_padding_mask 应对应什么？

A. 目标序列未来位置
B. 目标序列标签
C. Encoder memory 中源序列的 PAD 位置
D. 模型参数中的零值

✅ 参考答案

答案：C。 memory 与源序列位置一一对应，因此 Cross-Attention 要复用源侧 padding mask，避免 Decoder 关注源 PAD。

23. Teacher Forcing 训练中，目标完整序列为 [BOS, y1, y2, EOS]，合理的输入—标签移位是？

A. Decoder 输入 [BOS,y1,y2]，标签 [y1,y2,EOS]
B. 输入与标签完全相同且不加 Mask
C. Decoder 输入只含 EOS
D. 标签向右再补一个 BOS

✅ 参考答案

答案：A。 每个位置根据目标前缀预测下一个 token；Causal Mask 防止训练时并行输入的后续 token 泄漏。

24. Decoder 输出经线性层得到词表 logits，配合 CrossEntropyLoss 时应该？

A. 先 Argmax 再计算损失
B. 先转成字符串
C. 对每个 batch 求和后再分类
D. 直接传原始 logits，不在模型中提前 Softmax

✅ 参考答案

答案：D。 交叉熵内部使用数值稳定的 LogSoftmax；提前 Softmax 会改变接口语义并降低数值稳定性。

25. “权重绑定（weight tying）”在语言模型输出中常指？

A. 所有层共享一个 LayerNorm
B. 输出投影矩阵与 token embedding 权重共享或转置共享
C. Q/K/V 完全使用同一矩阵
D. 冻结所有参数

✅ 参考答案

答案：B。 当输入/输出词表与维度兼容时，权重绑定可减少参数并利用输入输出语义结构；仍需正确处理偏置和缩放约定。

26. Pre-LN 与原始 Post-LN 的正确对比是？

A. 两者都没有残差
B. Pre-LN 是 LN(x + F(x))
C. Pre-LN 常写 x + F(LN(x))，Post-LN 常写 LN(x + F(x))
D. Post-LN 不使用 LayerNorm

✅ 参考答案

答案：C。 Pre-LN 往往让深层优化更稳定；原始论文使用 Post-LN。实际完整堆栈还可能在末尾放一个最终 LayerNorm。

27. Dropout 在 model.eval() 后应如何表现？

A. 停止随机丢弃，使同一输入输出确定
B. 丢弃比例翻倍
C. 只丢弃 PAD
D. 自动删除残差连接

✅ 参考答案

二、简答题（15 题）

31. 按数据流描述一个完整 Encoder–Decoder Transformer 的前向过程。

✅ 参考答案

源 token ID 经源 Embedding、尺度调整与位置编码后进入 N 层 Encoder，得到与源长度对齐的 memory。目标前缀经目标 Embedding 和位置编码进入 Decoder：先做带 Causal/目标 Padding Mask 的自注意力，再以 Decoder 表示为 Q、memory 为 K/V 做交叉注意力，最后经 FFN；三个子层都有残差、归一化和 Dropout。Decoder 输出经线性层映射到目标词表 logits，训练时与右移一位的目标标签计算忽略 PAD 的交叉熵。

32. 写出正弦—余弦位置编码公式，并说明偶数/奇数通道的分工。

✅ 参考答案

$$ PE(pos,2i)=\sin\left(pos/10000^{2i/d_{model}}\right), $$

$$ PE(pos,2i+1)=\cos\left(pos/10000^{2i/d_{model}}\right). $$

相邻偶/奇通道共享频率，分别使用 sin/cos；不同 i 对应不同波长。位置 0 的 sin 通道为 0、cos 通道为 1。该编码无训练参数，可按公式生成任意位置，但超长外推效果仍取决于训练分布。

33. 区分 Embedding 乘 $\sqrt{d_{model}}$ 与注意力分数除以 $\sqrt{d_k}$。

✅ 参考答案

前者发生在输入端，用于放大初始化较小的 token embedding，使其与位置编码和残差流尺度更匹配；后者发生在每个注意力头内部，用于抑制高维 Q·K 点积方差随 $d_k$ 增大，防止 Softmax 过度饱和。二者公式相似但作用对象、方向和目的完全不同，不能互相抵消或省略。

34. 描述多头自注意力从 [B,T,D] 到 [B,T,D] 的完整形状变化。

✅ 参考答案

输入分别经 $W_Q/W_K/W_V$ 得到三个 [B,T,D]，再 reshape/transpose 为 [B,H,T,D/H]。每头分数为 [B,H,T,T]，Softmax 后乘 V 得 [B,H,T,D/H]。将头轴转回并沿特征拼接为 [B,T,D]，最后经 $W_O$ 仍为 [B,T,D]，从而可与输入残差相加。

35. 忽略偏置时，d_model=D 的标准多头注意力投影参数量是多少？头数会改变它吗？

✅ 参考答案

$W_Q,W_K,W_V,W_O$ 通常都是 $D\times D$，总计约 $4D^2$；若四个线性层都有偏置，再加约 $4D$。在总维度 D 固定且各头均匀切分时，改变头数只改变内部 reshape 与每头维度，不改变这四个大矩阵的总参数量，但会影响计算调度、注意力模式和单头容量。

36. Attention 与 FFN 在 Transformer 层中分别承担什么作用？

✅ 参考答案

Attention 在序列位置之间混合信息，让一个 token 从其他位置读取内容；FFN 对每个位置独立使用同一 MLP，在特征通道内部做非线性变换与扩展压缩。前者解决“和谁交互”，后者解决“交互后的表示如何变换”。两者缺一会显著限制表达能力。

37. 残差连接、LayerNorm 与 Dropout 各自解决什么问题？

✅ 参考答案

残差为信息与梯度提供直接路径，便于堆叠深层网络；LayerNorm 稳定每个 token 的特征尺度，降低优化敏感性；Dropout 在训练时随机丢弃部分激活或注意力权重以正则化。三者协同但不可互相替代，且 Dropout 在 eval() 时必须关闭。

38. 精确比较 Pre-LN 与 Post-LN，并说明工程取舍。

✅ 参考答案

原始 Post-LN 子层常写为 $LN(x+Dropout(F(x)))$；Pre-LN 常写为 $x+Dropout(F(LN(x)))$，整堆层后通常还有最终 LN。Pre-LN 的残差主路径更直接，深层训练往往更稳定；Post-LN 与原论文一致，有时最终效果或表示尺度不同。选择需与初始化、学习率和模型深度共同验证，不能把两者混成“前后各一次 LN”。

39. 列出 Encoder Self-Attention、Decoder Self-Attention、Decoder Cross-Attention 所需 Mask。

✅ 参考答案

Encoder Self-Attention 使用源 src_key_padding_mask；Decoder Self-Attention 使用目标 tgt_key_padding_mask 与目标 tgt_mask（Causal Mask）；Decoder Cross-Attention 使用源侧 memory_key_padding_mask，通常就是源 Padding Mask。若任务本身有局部窗口或块状约束，还可叠加结构 Mask。必须确认 API 中布尔值 True 的语义。

40. 为什么 Teacher Forcing 同时需要目标移位与 Causal Mask？

✅ 参考答案

移位定义监督目标：输入 [BOS,y1,y2] 分别预测 [y1,y2,EOS]。训练为了并行把整个目标输入一次送入 Decoder，若没有 Causal Mask，早期位置可直接看到后续真值 token，造成未来泄漏。只有 Mask 没有移位则学习目标错位；只有移位没有 Mask 仍可看到更远未来，因此两者都需要。

41. Encoder memory 是什么？为什么 Cross-Attention 的 K/V 可缓存？

✅ 参考答案

memory 是 Encoder 对每个源位置生成的上下文化表示 [B,Ts,D]。在一次自回归翻译中源句固定，因此 memory 及其 K/V 投影对所有解码步都不变，可预先计算并复用；Decoder 自注意力历史 K/V 也可随生成增量缓存。缓存减少重复计算，但要正确管理 batch、beam 和序列位置。

42. 输出层、Softmax 与训练损失应怎样衔接？

✅ 参考答案

Decoder 隐状态经 Linear(D, vocab_size) 得到 logits [B,T,V]。训练时将其与目标标签展平或保持框架支持的布局，直接交给 CrossEntropyLoss(ignore_index=pad_id)，不提前 Softmax；推理展示概率或采样时才用 Softmax。还应确认 BOS/PAD 不作为不合理的生成候选，并按有效 token 数统计损失。

43. Transformer 一层的主要计算与参数复杂度来自哪里？

✅ 参考答案

Self-Attention 的分数与加权计算约为 $O(T^2D)$，长序列时 $T^2$ 注意力矩阵成为瓶颈；Q/K/V/O 投影约 $O(TD^2)$。FFN 约 $O(TD\,d_{ff})$，且参数主要是 $2D\,d_{ff}$。短序列大模型常受投影/FFN 主导，长序列则注意力二次项更突出，需结合具体 T、D 分析。

44. Encoder-only、Decoder-only 与 Encoder–Decoder 架构分别适合什么任务？

✅ 参考答案

Encoder-only 使用双向自注意力，适合理解、分类、检索和序列标注；Decoder-only 使用因果自注意力，适合自回归生成并可通过提示统一任务；Encoder–Decoder 先编码完整输入、再条件生成，适合翻译、摘要和结构化变换。任务并非只能由某一类完成，选择还取决于预训练目标、数据与部署成本。

45. 给出完整 Transformer 的最小工程验收方案。

✅ 参考答案

用微型固定 batch 检查所有中间/输出形状、Mask 的 True/False 语义和未来权重为 0；验证目标移位、PAD loss 忽略与词表范围；前向/反向无 NaN，所有应训练参数有有限梯度；关闭 Dropout 后输出确定；保存加载后输出一致；尝试过拟合几十条样本；最后测试不同源/目标长度、全 PAD 异常输入、最大位置边界、CPU/GPU 与混合精度。

46. 实现正弦—余弦位置编码模块，并验证位置 0、形状和 buffer 行为。

✅ 参考答案

位置编码不是参数，但应注册为 buffer，使其进入 state_dict 并随模型移动设备。

Python
import math
import torch
from torch import nn

class SinusoidalPositionalEncoding(nn.Module):
    def __init__(self, d_model, max_len=128):
        super().__init__()
        position = torch.arange(max_len, dtype=torch.float32).unsqueeze(1)
        frequencies = torch.exp(
            torch.arange(0, d_model, 2, dtype=torch.float32)
            * (-math.log(10000.0) / d_model)
        )
        pe = torch.zeros(max_len, d_model)
        pe[:, 0::2] = torch.sin(position * frequencies)
        pe[:, 1::2] = torch.cos(position * frequencies[:pe[:, 1::2].shape[1]])
        self.register_buffer("pe", pe.unsqueeze(0))

    def forward(self, x):
        if x.size(1) > self.pe.size(1):
            raise ValueError("sequence exceeds max_len")
        return x + self.pe[:, :x.size(1)].to(dtype=x.dtype)

module = SinusoidalPositionalEncoding(d_model=8, max_len=16)
x = torch.zeros(2, 5, 8)
y = module(x)

assert y.shape == x.shape
assert torch.equal(y[0, 0, 0::2], torch.zeros(4))
assert torch.equal(y[0, 0, 1::2], torch.ones(4))
assert "pe" in module.state_dict()
assert all(name != "pe" for name, _ in module.named_parameters())
print(y.shape)

def make_masks(src_ids, tgt_input_ids, pad_id=0):
    src_key_padding_mask = src_ids.eq(pad_id)

from torch import nn

class PreNormFFNBlock(nn.Module):
    def __init__(self, d_model, d_ff):
        super().__init__()

53. 使用 nn.Transformer 搭建一个最小 Encoder–Decoder 前向模型。

✅ 参考答案

nn.Transformer 只负责 Encoder/Decoder 主体，Embedding、位置表示与词表输出层仍需自行提供。

Python
import torch
from torch import nn

class TinyTransformer(nn.Module):
    def __init__(self, src_vocab, tgt_vocab, d_model=16, max_len=20):
        super().__init__()
        self.src_embedding = nn.Embedding(src_vocab, d_model, padding_idx=0)
        self.tgt_embedding = nn.Embedding(tgt_vocab, d_model, padding_idx=0)
        self.src_position = nn.Embedding(max_len, d_model)
        self.tgt_position = nn.Embedding(max_len, d_model)
        self.transformer = nn.Transformer(
            d_model=d_model,
            nhead=4,
            num_encoder_layers=1,
            num_decoder_layers=1,
            dim_feedforward=32,
            dropout=0.0,
            batch_first=True,
            norm_first=False,
        )
        self.output = nn.Linear(d_model, tgt_vocab)

    def forward(self, src, tgt):
        src_pos = torch.arange(src.size(1), device=src.device).unsqueeze(0)
        tgt_pos = torch.arange(tgt.size(1), device=tgt.device).unsqueeze(0)
        src_x = self.src_embedding(src) + self.src_position(src_pos)
        tgt_x = self.tgt_embedding(tgt) + self.tgt_position(tgt_pos)
        src_pad = src.eq(0)
        tgt_pad = tgt.eq(0)
        causal = torch.triu(
            torch.ones(tgt.size(1), tgt.size(1), dtype=torch.bool, device=tgt.device),
            diagonal=1,
        )
        hidden = self.transformer(
            src_x,
            tgt_x,
            tgt_mask=causal,
            src_key_padding_mask=src_pad,
            tgt_key_padding_mask=tgt_pad,
            memory_key_padding_mask=src_pad,
        )
        return self.output(hidden)

model = TinyTransformer(src_vocab=30, tgt_vocab=40)
src = torch.tensor([[3, 4, 5, 0], [6, 7, 0, 0]])
tgt = torch.tensor([[1, 8, 9], [1, 10, 0]])
logits = model(src, tgt)

assert logits.shape == (2, 3, 40)
assert torch.isfinite(logits).all()
print(logits.shape)

55. 统计两组 Transformer Encoder 参数量，验证 d_model 翻倍时约增长 4 倍。

✅ 参考答案

保持 d_ff=4*d_model、层数不变，主要矩阵参数随 $d_model^2$ 增长。

Python
import torch
from torch import nn

def make_encoder(d_model, nhead, layers=2):
    layer = nn.TransformerEncoderLayer(
        d_model=d_model,
        nhead=nhead,
        dim_feedforward=4 * d_model,
        dropout=0.0,
        batch_first=True,
    )
    return nn.TransformerEncoder(layer, num_layers=layers)

small = make_encoder(d_model=32, nhead=4)
large = make_encoder(d_model=64, nhead=8)
small_params = sum(parameter.numel() for parameter in small.parameters())
large_params = sum(parameter.numel() for parameter in large.parameters())
ratio = large_params / small_params

small_output = small(torch.randn(2, 5, 32))
large_output = large(torch.randn(2, 5, 64))
assert small_output.shape == (2, 5, 32)
assert large_output.shape == (2, 5, 64)
assert 3.8 < ratio < 4.2
print({"small": small_params, "large": large_params, "ratio": round(ratio, 3)})

二、简答题（12 题）

21. 为什么「先验知识强」仍可能不够用？

✅ 参考答案

通用 LLM 能答很多题，但在特定任务上常出现格式漂移、不严格按输入约束输出，后处理/解析困难。微调（本项目）用于对齐目标输出格式与任务协议。

22. 用自己的话解释 Deep Norm + post layer norm 改动的工程意图。

✅ 参考答案

层归一化位置与归一化策略影响深层训练稳定性。课件采用基于 Deep Norm 的 post-LN，属于为更稳训练做的结构选择（与原始 Decoder 配置不同）。

23. 列出 6B 配置表中你认为最关键的 4 个数字并说明用途。

✅ 参考答案

示例：参数 6.2B（容量）、隐藏维 4096 / 层数 28 / 头数 32（算力与结构宽度深度）、词表 130528（分词空间）、最大长度 2048（上下文与显存）。任选合理即可。

24. 对照解释 FP16 / INT8 / INT4 推理显存阶梯。

✅ 参考答案

无量化 FP16 约 13GB；INT8 约 10GB；INT4 约 6GB。量化降低权重精度以换显存与部署门槛，可能伴随一定质量折损，需按场景权衡。

25. 「高效参数微调」显存为何仍接近或略高于同级推理？

✅ 参考答案

除前向外，训练还需梯度、优化器状态等额外占用（即便 LoRA 只训少量参数）。课件表：FP16 微调约 14GB，INT4 约 7GB。

26. 优点里「较低部署门槛」具体指什么？

✅ 参考答案

INT4 下约 6GB 显存即可推理，使 ChatGLM-6B 有机会跑在消费级 GPU 上，而不是只能上数据中心大卡。

27. 缺点「模型容量小」会带来哪些可观察现象？

✅ 参考答案

相对更大模型，知识记忆、复杂推理与语言表现可能更弱；课件还提到多轮对话能力相对较弱。应用上需靠检索/工具或更大模型补齐。

28. 为什么 requirements 要钉死 transformers / peft 版本？

✅ 参考答案

ChatGLM 自定义建模文件与 PEFT 注入接口随版本变化大；钉版本降低「能 import 但不能 load / 不能训练」的兼容风险。复现新环境时仍要核对。

29. 简述从空机器到可加载权重的最小步骤。

✅ 参考答案

准备 GPU 环境与 Python → 建 conda 环境 → pip install -r requirements.txt → 建模型目录 → modelscope 下载 ChatGLM-6B → 核对关键 py/权重文件齐全。

30. 项目架构（概念）上，本章之后通常接什么？

✅ 参考答案

混合任务数据预处理（指令化 jsonl），再 LoRA 注入与训练验证、推理预测。本章解决「用什么模型、什么环境」。

31. embedding 梯度缩小 10 倍，直观上如何理解「范数变小」？

✅ 参考答案

更新步里 embedding 参数变动幅度被压小，减轻早期不稳定或梯度爆炸/震荡风险，换更稳的训练动态（具体实现以官方/建模代码为准）。

32. 若只有 8GB 显卡，优先考虑哪种精度路径？风险是什么？

✅ 参考答案

优先 INT4 推理（表称约 6GB）；微调约 7GB 仍可能吃紧，需降序列长度、开梯度检查点或换更大显存。风险是量化带来的质量下降与实现兼容问题。

✅ 参考答案

ORDER = [("INT4", 6), ("INT8", 10), ("FP16", 13)]


def recommend_quant(avail_gb: float) -> str | None:
    # 选满足最低需求的最高精度
    ok = [name for name, need in ORDER if avail_gb >= need]
    if not ok:
        return None
    # ORDER 从低显存到高；取最后一个即可用的最高精度
    return ok[-1]


assert recommend_quant(6) == "INT4"
assert recommend_quant(12) == "INT8"
assert recommend_quant(16) == "FP16"
print(recommend_quant(5))

def context_risk(base_gb: float, max_len: int, ref_len: int = 2048) -> str:
    # 教学：长度超过参考则标记风险升高（非精确公式）
    if max_len <= ref_len:
        return f"约>={base_gb}GB（长度未超参考 {ref_len}）"
    return f"可能显著高于 {base_gb}GB（max_len={max_len}>{ref_len}）"


print(context_risk(6, 2048))
print(context_risk(6, 8192))
assert "显著" in context_risk(6, 8192)

def pinned_version(line: str) -> tuple[str, str] | None:
    line = line.strip()
    if "==" in line:
        name, ver = line.split("==", 1)
        return name.strip(), ver.strip()
    return None


assert pinned_version("transformers==4.33") == ("transformers", "4.33")
assert pinned_version("peft==0.3.0") == ("peft", "0.3.0")
print(pinned_version("accelerate>=0.20.3"))

🏛️ 主流 LLM 架构：BERT、GPT 与 T5 综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：主流 LLM 架构：BERT、GPT 与 T5

55 道练习，按“Transformer 张量流 → Encoder-only 双向理解 → Decoder-only 因果生成 → Encoder–Decoder 条件生成 → T5 文本到文本与 Span Corruption → Teacher Forcing、Cross-Attention 与缓存 → 成本评估和架构选型”组织。重点是从 Attention 的可见范围和训练目标推导模型行为，而不是用模型名称代替分析。

建议先独立作答，再展开参考答案。练习按 Attention Is All You Need、T5 论文、Hugging Face T5 与 Encoder–Decoder Models资料校准；10 道代码题使用纯 Python/PyTorch，不下载外部 checkpoint。

一、选择题（30 题）

1. 三类架构的自注意力可见范围，哪项正确？

A. Encoder 通常双向；Decoder-only 与 Encoder–Decoder 的 Decoder 通常因果
B. BERT 与 GPT 都能无条件看到未来 token
C. T5 Encoder 只能看左侧
D. 三类架构完全没有 Mask

✅ 参考答案

答案：A。 Encoder 的有效 token 通常可相互关注；生成 Decoder 的位置 $t$ 只能访问不晚于 $t$ 的目标前缀。Padding Mask 还要与这些结构性约束共同使用。

2. Encoder-only 最自然的输出是什么？

A. 每个输入 token 的上下文化表示
B. 必然无限续写文本
C. 只输出一个文件名
D. 不产生隐藏状态

✅ 参考答案

答案：A。 分类可池化为句级表示，NER 使用逐 token 表示，抽取式问答预测输入 span；是否有 pooler_output 或特定任务头取决于模型实现。

3. Decoder-only 预训练的标准因果目标是？

A. 根据前缀预测 next token
B. 同时读取未来答案后预测当前位置
C. 只判断句子对是否相邻
D. 预测图片尺寸

✅ 参考答案

答案：A。 训练可对所有位置并行计算 loss，但因果 Mask 保证第 $t$ 个隐藏状态不能使用 $t$ 之后的 token。

4. Encoder–Decoder 的 Cross-Attention 中，Query、Key、Value 通常来自哪里？

A. Query 来自 Decoder；Key/Value 来自 Encoder 输出
B. 三者都来自标签文件名
C. Query 来自 Encoder，Key/Value 必须来自未来目标
D. Cross-Attention 不使用 Q/K/V

✅ 参考答案

答案：A。 Decoder 以当前目标前缀表示发起查询，从源序列的 Encoder 表示中读取相关信息；这形成“已生成什么→源文本哪里相关”的条件通道。

5. Cross-Attention 权重的典型形状是？

A. [B, heads, target_length, source_length]
B. [B, classes]
C. [source_length]
D. [vocab, vocab]

✅ 参考答案

答案：A。 每个目标 query 位置对所有源 key 位置形成一行分布；源 Padding 位置应被 Mask，不参与加权求和。

6. 条件生成时 Encoder 输出为什么通常只计算一次？

A. 源序列在逐 token 解码期间保持不变，可被所有目标步复用
B. Encoder 没有参数
C. Decoder 不读取 Encoder
D. 每一步源文本都会自动改变

✅ 参考答案

答案：A。 目标 Decoder self-attention cache 随生成增长；Cross-Attention 对固定 Encoder 输出的 K/V 也可缓存，从而避免重复投影。

7. BERT MLM 与 T5 Span Corruption 的主要区别是？

A. BERT 在原序列位置预测被选 token；T5 用 sentinel 替换连续 span，并由 Decoder 生成被删内容
B. 二者都只做 next-token prediction
C. T5 没有 Encoder
D. BERT 必须生成整篇摘要

✅ 参考答案

答案：A。 T5 的去噪任务天然训练了 Encoder–Decoder 条件生成接口，并用一组 sentinel token 标记多个被破坏片段的边界。

8. T5 Span Corruption 中 <extra_id_0> 等 sentinel token 的作用是？

A. 标记输入中被移除 span 的位置，并在目标中标识各 span 的恢复顺序
B. 表示分类置信度
C. 替代所有普通词表
D. 表示 GPU 编号

✅ 参考答案

答案：A。 不同 span 使用不同 sentinel；目标通常按 sentinel + 原 span 内容的顺序串联，并以新的 sentinel 或 EOS 收尾。

9. T5 的“所有任务统一为 text-to-text”表示？

A. 输入与目标都序列化为文本，分类也可生成标签字符串
B. 每个任务都必须输出长文章
C. 不再需要任务定义
D. 指标可以全部使用 BLEU

✅ 参考答案

答案：A。 统一的是接口和训练形式，不是业务语义；分类目标可以只是一个短标签，仍应使用准确率、F1 等任务指标。

10. T5 输入前的任务前缀如 summarize: 或 translate ...: 主要用于？

A. 告诉共享模型当前要执行的任务/方向
B. 增加模型参数量
C. 替代 tokenizer
D. 屏蔽所有源 token

✅ 参考答案

答案：A。 前缀本身也是普通输入 token；训练和推理必须保持格式一致，具体 checkpoint 是否依赖某类前缀应查看模型卡。

11. 原始 T5 是否依赖 BERT 的 token_type_ids 区分句子 A/B？

A. 不应假设；T5 通常通过文本格式/前缀表达任务，并不使用 BERT 式 Segment Embedding
B. 必须全部设置为 1
C. 用它替代 Cross-Attention
D. 所有 Transformer 都必须提供

✅ 参考答案

答案：A。 通用代码应传 tokenizer 实际返回且模型支持的字段，不能把 BERT 输入协议套到 T5 或 Decoder-only 模型上。

12. T5 的位置建模与原始 BERT 的典型差异是？

A. T5 使用相对位置 bias/bucket；原始 BERT 使用可学习绝对 Position Embedding
B. T5 完全不编码位置
C. BERT 使用 Cross-Attention 位置
D. 二者只靠文件顺序

✅ 参考答案

答案：A。 位置机制属于 checkpoint 架构的一部分；不能仅凭“Transformer”推断 Padding 方向、最大长度外推或位置 ID 行为都相同。

13. Seq2Seq 训练中为什么要将目标标签右移形成 decoder_input_ids？

A. 让 Decoder 在第 $t$ 步读取起始符与此前真值，预测当前目标 token
B. 让模型看到未来标签
C. 删除 EOS
D. 让 Encoder 预测标签

✅ 参考答案

答案：A。 输入通常为 [START, y_1, ..., y_{T-1}]，监督为 [y_1, ..., y_T]；许多任务模型在传入 labels 时内部完成右移，不能再手工重复位移。

14. 目标序列中的 Padding 标签通常设为？

A. -100，使交叉熵忽略这些位置
B. 最大词表 ID
C. 随机类别
D. 源序列长度

✅ 参考答案

答案：A。 decoder_attention_mask 控制注意力可见性，labels=-100 控制 loss 是否计入；二者解决不同问题。

15. Teacher Forcing 训练与自回归推理的主要差异是？

A. 训练 Decoder 看到真值前缀，推理看到模型自己先前生成的前缀
B. 推理可看到未来真值
C. 训练不计算 loss
D. 二者都不需要 Decoder

✅ 参考答案

答案：A。 这种条件差异可能带来 exposure bias；验证必须使用真实生成流程，不能只看 teacher-forced loss。

16. 对 T5 翻译或摘要任务，优先使用哪个 Auto 类？

A. AutoModelForSeq2SeqLM
B. AutoModelForMaskedLM
C. AutoModelForTokenClassification
D. 只用 AutoModel 即可生成

✅ 参考答案

答案：A。 它包含 Encoder–Decoder 与词表生成头，并与 generate() 接口配合；AutoModel 通常只返回隐藏状态。

17. Seq2Seq 推理为什么应调用 generate() 而非只做一次前向传播？

A. 目标序列未知，需迭代生成并处理缓存、EOS、beam/sampling 与停止条件
B. 前向传播不能产生 logits
C. generate() 会自动训练模型
D. 只有分类任务需要生成

✅ 参考答案

答案：A。 一次 forward 适合给定完整 decoder 输入的打分或训练；开放目标推理存在“输出反馈为下一步输入”的循环。

18. 同一 QA 任务中，BERT 抽取式与 T5 生成式方案的区别是？

A. BERT 常预测 context 内 start/end；T5 可生成答案文本，不天然受原文 span 限制
B. 二者输出必然完全一致
C. T5 只能输出位置索引
D. BERT 必须生成长回答

✅ 参考答案

答案：A。 抽取式可追溯、幻觉空间较小，但不能改写；生成式灵活，需额外评估事实支持、规范化和不可回答处理。

19. Decoder-only 如何完成条件生成而不使用独立 Encoder？

A. 将指令/源文本与目标前缀放在同一因果序列中，通过格式与 loss mask 区分区域
B. 完全不输入源文本
C. 删除 causal mask
D. 使用 BERT 的 NSP 代替生成

✅ 参考答案

答案：A。 源 token 自身只能看更早位置，但目标 token 可以关注全部源前缀；训练时常只对目标/assistant 区域计算 loss。

20. “Encoder–Decoder 的训练和推理成本必然是 Decoder-only 的两倍”为什么错误？

A. 成本取决于层数、宽度、参数共享、源/目标长度、Attention 模式、缓存和硬件实现
B. 因为所有模型成本恒为零
C. Encoder–Decoder 没有 Attention
D. Decoder-only 一定有两个 Encoder

✅ 参考答案

答案：A。 相同模型名称或参数量也不代表相同 FLOPs；条件生成还要比较 Encoder $S^2$、Decoder $T^2$、Cross-Attention $ST$ 与统一因果序列的计算方式。

21. 对“很长输入、很短输出”的摘要/分类式生成，Encoder–Decoder 的潜在优势是？

A. 源序列只编码一次，短 Decoder 通过 Cross-Attention 读取源表示
B. 每生成一个 token 必须重训 Encoder
C. 自动不需要显存
D. 输出一定没有事实错误

✅ 参考答案

答案：A。 是否更快仍取决于实现和规模，但将源理解与目标生成分工可减少长源前缀在每个 Decoder layer 中重复参与因果计算。

22. 能否只凭“12 层、768 隐藏维”准确推出模型参数量？

A. 不能，还需词表、FFN 维度、QKV/输出投影、层数定义、bias、Embedding/LM Head 共享等
B. 能，参数量恒为 12×768
C. 参数量只由 head 数决定
D. 参数量与配置无关

✅ 参考答案

答案：A。 BERT、GPT、T5 即使某些维度相同，模块组成和共享策略不同，参数量仍会不同；应读取配置或直接统计 Parameter。

23. T5 的 Encoder 和 Decoder 是否必须具有完全相同层数？

A. 不必须；配置可分别指定，具体 checkpoint 才决定实际值
B. 必须永远都是 12+12
C. T5 没有 Decoder
D. 层数由输入句数决定

✅ 参考答案

答案：A。 官方实现中可用 num_layers 与 num_decoder_layers 表达不同深度；模型选型应查看实际 config，不能把某个规模的表格当成全家族常量。

24. 输入 Embedding、Decoder Embedding 与 LM Head 能否共享参数？

A. 可以，许多 Seq2Seq 模型会共享部分或全部词嵌入/输出投影以减少参数
B. 数学上绝不可能
C. 共享后不再需要词表
D. 共享一定让损失为零

✅ 参考答案

答案：A。 是否共享由架构配置决定；词表不同的异构 Encoder–Decoder 可能无法直接共享。

25. 关于 T5 LayerNorm/FFN，哪项更可靠？

A. 应查看具体 T5/T5v1.1 配置，不能假设所有变体激活与归一化细节完全一致
B. 所有 T5 变体内部逐位相同
C. T5 没有 FFN
D. LayerNorm 决定任务前缀

✅ 参考答案

答案：A。 原始 T5 与后续变体可能使用不同 FFN 投影等细节；同为 Encoder–Decoder 只描述宏观结构，不保证 block 实现一致。

26. T5 句子分类是否必须读取 [CLS]？

A. 不必须；文本到文本方案可生成标签，且 T5 tokenizer 不应套用 BERT 的 [CLS] 假设
B. 必须硬编码 ID 101
C. T5 只支持 [MASK] 分类
D. 分类不需要输入

✅ 参考答案

二、简答题（15 题）

31. 给定源表示 [B,S,H] 与目标表示 [B,T,H]，说明 Cross-Attention 的张量流。

✅ 参考答案

Decoder 目标表示投影为 $Q:[B,h,T,d]$，Encoder 源表示投影为 $K,V:[B,h,S,d]$；计算 $QK^T/\sqrt d$ 得分 [B,h,T,S]，屏蔽源 PAD 后沿 $S$ 做 Softmax，再乘 $V$ 得 [B,h,T,d]，拼接各头并投影回 [B,T,H]。

32. 分别写出 Encoder self-attention、Decoder self-attention、Cross-Attention 的可见规则。

✅ 参考答案

Encoder：有效源 query 可看全部有效源 key；Decoder self：有效目标 query 只能看不晚于自身的有效目标 key；Cross：每个有效目标 query 可看全部有效源 key，不受目标因果顺序限制，但必须屏蔽源 PAD。三者的 Padding Mask 不能混用错轴。

33. 将 token 序列 A B C D E F 的 span B C 与 E 做成 T5 corruption 样本。

✅ 参考答案

一种标准形式是输入 A <extra_id_0> D <extra_id_1> F，目标 <extra_id_0> B C <extra_id_1> E <extra_id_2>（再按 tokenizer 规则加 EOS）。Sentinel 顺序必须在输入与目标中对应，span 不能重叠或乱序。

34. T5 用文本标签做分类有什么优势与风险？

✅ 参考答案

优势是统一 Seq2Seq 接口、多任务可共用模型与任务前缀，标签还可具有自然语言语义；风险是生成同义词、大小写、空格或额外解释导致解析歧义，标签 token 长度影响 loss。应定义封闭标签集合、规范化解析、约束解码，并报告非法输出率。

35. 解释 Teacher Forcing 中目标右移、因果 Mask 和 loss labels 三者如何配合。

✅ 参考答案

右移后的 Decoder 输入为 start + 真值目标前缀；因果 Mask 防止位置读取右侧未来目标；未右移 labels 是当前位置应预测的真值，PAD 设 -100。若任务模型已根据 labels 自动右移，外部不要再次右移，否则监督错一位。

36. 为什么验证 Seq2Seq 模型不能只看 teacher-forced token loss？

✅ 参考答案

Teacher forcing 每步都获得正确历史，即使模型某步犯错也不会传播；真实生成使用自身输出，错误会改变后续上下文。验证还需运行 generate()，根据任务计算 BLEU/ROUGE/Exact Match/字段准确率，并检查 EOS、长度、重复、事实性和失败样本。

37. 解释 Encoder–Decoder 生成中的两类 Cache。

✅ 参考答案

Decoder self-attention K/V 随每个新目标 token 增长，避免反复计算既有目标前缀；Cross-Attention K/V 是固定 Encoder 输出的投影，可在源编码后复用。两类缓存都按层保存，但形状的序列轴分别对应目标长度与源长度。

38. 用 $S$、$T$ 表示源和目标长度，粗略比较 Encoder–Decoder 与拼接式 Decoder-only 的 Attention score 数量。

✅ 参考答案

忽略层数和常数，Encoder–Decoder 每组相应层包含源 self $S^2$、目标 self 约 $T^2/2$、Cross $ST$；将源和目标拼成单一因果序列的 Decoder-only 每层约为 $(S+T)^2/2$ 个可见 score。实际总成本还乘不同层数/宽度，且源 Encoder 只计算一次，不能据此简单断言固定倍数。

39. 为什么参数量相等也不保证三类架构时延相等？

✅ 参考答案

模块分布、源/目标长度、是否生成、Cross-Attention、KV Cache、并行度、激活大小、算子融合、Padding 比例和硬件利用率都不同。Encoder 分类通常一次并行前向；Decoder 生成受串行 token 延迟约束；Seq2Seq 还分 Encoder prefill 与 Decoder decode 两阶段。

46. 构造 Encoder self、Decoder causal self 与 Cross-Attention 三种可见矩阵。

✅ 参考答案

True 表示 query 可关注 key；三个矩阵的最后两维分别是 S×S、T×T、T×S。

import torch

def architecture_masks(source_mask, target_mask):
    source_valid = source_mask.bool()
    target_valid = target_mask.bool()
    source_self = source_valid[:, :, None] & source_valid[:, None, :]
    length = target_mask.size(1)
    causal = torch.tril(torch.ones(length, length, dtype=torch.bool))
    target_self = (
        causal.unsqueeze(0)
        & target_valid[:, :, None]
        & target_valid[:, None, :]
    )
    cross = target_valid[:, :, None] & source_valid[:, None, :]
    return source_self, target_self, cross

target_mask = torch.tensor([[1, 1, 0]])

assert target_self.shape == (1, 3, 3)
assert cross.shape == (1, 3, 4)
assert target_self[0, 1].tolist() == [True, True, False]

assert not cross[0, 2].any()

47. 从零实现多头 Cross-Attention，并验证输出与权重形状。

✅ 参考答案

本题省略投影层，直接接收已按 head 划分的 Q/K/V；源 Mask 在 Softmax 前施加。

Python
import math
import torch

def cross_attention(query, key, value, source_mask):
    # query: [B,H,T,D], key/value: [B,H,S,D]
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(query.size(-1))
    scores = scores.masked_fill(~source_mask[:, None, None, :].bool(), float("-inf"))
    weights = torch.softmax(scores, dim=-1)
    output = torch.matmul(weights, value)
    return output, weights

torch.manual_seed(0)
query = torch.randn(2, 3, 4, 5)
key = torch.randn(2, 3, 6, 5)
value = torch.randn(2, 3, 6, 5)
source_mask = torch.tensor([[1, 1, 1, 1, 0, 0], [1, 1, 1, 1, 1, 0]])
output, weights = cross_attention(query, key, value, source_mask)
assert output.shape == (2, 3, 4, 5)
assert weights.shape == (2, 3, 4, 6)
assert torch.allclose(weights.sum(-1), torch.ones(2, 3, 4))
assert (weights[0, :, :, 4:] == 0).all() and (weights[1, :, :, 5] == 0).all()
print(output.shape, weights.shape)

48. 将多个不重叠 span 转换为 T5 sentinel corruption 的输入与目标。

✅ 参考答案

def t5_span_corruption(tokens, spans):
    spans = sorted(spans)
    previous_end = 0
    corrupted, target = [], []
    for index, (start, end) in enumerate(spans):
        if not (previous_end <= start < end <= len(tokens)):
            raise ValueError("spans must be valid, ordered and non-overlapping")
        sentinel = f"<extra_id_{index}>"
        corrupted.extend(tokens[previous_end:start])
        corrupted.append(sentinel)
        target.append(sentinel)
        target.extend(tokens[start:end])
        previous_end = end
    corrupted.extend(tokens[previous_end:])
    target.append(f"<extra_id_{len(spans)}>")
    return corrupted, target

assert target == ["<extra_id_0>", "B", "C", "<extra_id_1>", "E", "<extra_id_2>"]
try:
    t5_span_corruption(tokens, [(1, 4), (3, 5)])

    pass
else:

50. 计算只依赖目标 labels 的 Seq2Seq token loss，并忽略目标 PAD。

✅ 参考答案

源长度影响 Encoder/Cross-Attention 计算，却不改变 logits 与 labels 必须在目标轴对齐这一契约。

import torch
import torch.nn.functional as F

def seq2seq_loss(logits, labels):
    if logits.shape[:2] != labels.shape:
        raise ValueError("logits target axes must match labels")
    return F.cross_entropy(
        logits.reshape(-1, logits.size(-1)),
        labels.reshape(-1),
        ignore_index=-100,
    )

logits = torch.full((2, 3, 8), -5.0)
labels = torch.tensor([[3, 4, -100], [5, 1, -100]])
for batch, row in enumerate(labels):
    for step, token in enumerate(row):
        if token != -100:
            logits[batch, step, token] = 5.0
loss = seq2seq_loss(logits, labels)
assert loss.item() < 0.001

changed = logits.clone()
changed[:, 2] = 999.0
assert torch.allclose(loss, seq2seq_loss(changed, labels))
print(round(loss.item(), 8))

51. 用计数器验证自回归生成期间 Encoder 只执行一次。

✅ 参考答案

Decoder 每一步读取同一个 encoder_output，直到生成 EOS 或达到长度上限。

Python
class ToyEncoder:
    def __init__(self):
        self.calls = 0

    def __call__(self, source):
        self.calls += 1
        return tuple(source)

def toy_decoder_next(encoder_output, generated):
    position = len(generated) - 1  # generated 起始含 START
    if position < len(encoder_output):
        return encoder_output[position]
    return 0  # EOS

def generate_seq2seq(source, encoder, start_id, eos_id, max_new_tokens):
    encoder_output = encoder(source)
    generated = [start_id]
    for _ in range(max_new_tokens):
        next_token = toy_decoder_next(encoder_output, generated)
        generated.append(next_token)
        if next_token == eos_id:
            break
    return generated

encoder = ToyEncoder()
result = generate_seq2seq([4, 5, 0], encoder, start_id=9, eos_id=0, max_new_tokens=10)
assert result == [9, 4, 5, 0]
assert encoder.calls == 1
print(result, "encoder_calls=", encoder.calls)

55. 训练一个微型 Encoder–Decoder 完成复制任务，并用自回归解码验证闭环。

✅ 参考答案

训练输入使用右移真值，推理只从 START 开始；模型必须通过 Cross-Attention 读取源序列。

Python
import torch
from torch import nn
import torch.nn.functional as F

torch.manual_seed(7)
vocab_size, start_id, eos_id = 10, 2, 1
source = torch.tensor([
    [3, 4, eos_id], [5, 6, eos_id],
    [3, 6, eos_id], [5, 4, eos_id],
])
labels = source.clone()
decoder_inputs = torch.cat([
    torch.full((source.size(0), 1), start_id), labels[:, :-1]
], dim=1)

class TinySeq2Seq(nn.Module):
    def __init__(self):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, 16)
        self.transformer = nn.Transformer(
            d_model=16, nhead=4, num_encoder_layers=1,
            num_decoder_layers=1, dim_feedforward=32,
            dropout=0.0, batch_first=True,
        )
        self.output = nn.Linear(16, vocab_size)

    def encode(self, source_ids):
        return self.transformer.encoder(self.embedding(source_ids))

    def decode(self, target_ids, memory):
        causal = nn.Transformer.generate_square_subsequent_mask(target_ids.size(1))
        hidden = self.transformer.decoder(
            self.embedding(target_ids), memory, tgt_mask=causal
        )
        return self.output(hidden)

model = TinySeq2Seq()
optimizer = torch.optim.Adam(model.parameters(), lr=0.03)
model.train()
for _ in range(220):
    memory = model.encode(source)
    logits = model.decode(decoder_inputs, memory)
    loss = F.cross_entropy(logits.reshape(-1, vocab_size), labels.reshape(-1))
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

model.eval()
with torch.no_grad():
    memory = model.encode(source)
    generated = torch.full((source.size(0), 1), start_id)
    for _ in range(labels.size(1)):
        next_logits = model.decode(generated, memory)[:, -1]
        generated = torch.cat([generated, next_logits.argmax(-1, keepdim=True)], dim=1)
prediction = generated[:, 1:]
accuracy = (prediction == labels).float().mean().item()
assert accuracy == 1.0

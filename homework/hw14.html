🧵 多任务、进程与线程

🏷️ 简答 + 综合实操🏷️ 6 题📖 相关讲义：网络编程、进程与线程入门

这套综合题同时覆盖多任务、多进程和多线程。建议先画出“哪些任务并行、哪些步骤必须等待”的关系，再开始创建 Process、Thread 和锁。

一、并发基础

1. 什么是任务？什么是多任务？

✅ 参考答案
  • 任务：程序中需要完成的一个具体操作或功能单元，例如播放音乐、读取文件、发送网络请求。
  • 多任务：操作系统或程序在同一时间段内推进多个任务。单核 CPU 通过快速切换让任务并发执行，多核 CPU 还可能让任务真正并行。

多任务的目标通常是提高资源利用率和程序响应能力，例如下载文件时界面仍能响应用户操作。

2. 进程和线程有什么区别？什么是多进程和多线程？

✅ 参考答案
对比维度 进程 线程
基本定位 操作系统分配资源的基本单位 CPU 调度执行的基本单位
内存资源 每个进程有独立地址空间 同一进程内的线程共享资源
创建与切换 开销较大 开销较小
通信 需要队列、管道等进程间通信机制 可直接共享变量，但要处理竞争
故障影响 一个进程崩溃通常不直接影响其他进程 一个线程异常可能导致整个进程退出

多进程是在多个独立进程中执行不同任务，隔离性较好，适合 CPU 密集型任务；多线程是在一个进程内创建多个线程，适合文件读写、网络等待等 I/O 密集型任务。

在 CPython 中，受 GIL 影响，多个线程通常不能同时执行 Python CPU 密集型字节码。因此 CPU 密集型工作一般优先多进程，I/O 密集型工作一般优先多线程。

3. 完成一个多任务程序通常有哪些步骤？

✅ 参考答案
  1. 拆分任务：把整体功能划分成可以独立执行的子任务。
  2. 判断类型：CPU 密集型优先考虑多进程，I/O 密集型优先考虑多线程。
  3. 封装函数：把每个子任务写成独立函数，明确参数和返回结果。
  4. 创建执行单元：用 multiprocessing.Process 或 threading.Thread 绑定任务函数。
  5. 启动任务：调用每个对象的 start()。
  6. 协调与等待：按需要使用 join()、锁、队列或事件协调执行顺序和共享资源。
  7. 处理异常并清理资源：确保文件、网络连接、锁等资源能够正确释放。

多进程入口通常还需要放在 if __name__ == "__main__": 保护中，避免子进程重复执行创建逻辑。

二、多进程与多线程实操

1. 同时吃水果、听音乐和聊天

分别使用多进程和多线程，让“吃水果”“听音乐”“和同事聊天”三个任务并发执行。

✅ 参考答案
Python
import time
import threading
from multiprocessing import Process


def eat_fruit():
    for i in range(3):
        print(f"【吃水果】第 {i + 1} 口")
        time.sleep(0.5)


def listen_music():
    for i in range(3):
        print(f"【听音乐】第 {i + 1} 首")
        time.sleep(0.5)


def chat_with_colleague():
    for i in range(3):
        print(f"【聊天】第 {i + 1} 句")
        time.sleep(0.5)


def run_with_processes():
    workers = [
        Process(target=eat_fruit),
        Process(target=listen_music),
        Process(target=chat_with_colleague),
    ]
    for worker in workers:
        worker.start()
    for worker in workers:
        worker.join()


def run_with_threads():
    workers = [
        threading.Thread(target=eat_fruit),
        threading.Thread(target=listen_music),
        threading.Thread(target=chat_with_colleague),
    ]
    for worker in workers:
        worker.start()
    for worker in workers:
        worker.join()


if __name__ == "__main__":
    print("===== 多进程 =====")
    run_with_processes()

    print("===== 多线程 =====")
    run_with_threads()

先集中 start()，再集中 join()。如果启动一个任务后立刻 join()，三个任务会退化为依次执行。

2. 编写一个线程死锁案例并解释原因

✅ 参考答案

下面的代码故意制造死锁，但把两个工作线程设为守护线程，并给主线程的 join() 设置超时，演示后程序仍可退出：

Python
import threading


lock_a = threading.Lock()
lock_b = threading.Lock()
barrier = threading.Barrier(2)


def task_one():
    with lock_a:
        print("任务一拿到锁 A")
        barrier.wait()
        print("任务一等待锁 B")
        with lock_b:
            print("任务一完成")


def task_two():
    with lock_b:
        print("任务二拿到锁 B")
        barrier.wait()
        print("任务二等待锁 A")
        with lock_a:
            print("任务二完成")


t1 = threading.Thread(target=task_one, daemon=True)
t2 = threading.Thread(target=task_two, daemon=True)
t1.start()
t2.start()
t1.join(timeout=1)
t2.join(timeout=1)

print("是否发生死锁：", t1.is_alive() and t2.is_alive())

死锁原因：任务一持有锁 A 并等待锁 B，任务二持有锁 B 并等待锁 A，双方都不释放当前资源，形成循环等待。

常见预防方法包括：所有线程按相同顺序获取锁、缩小加锁范围、使用超时获取锁，或避免在持有一把锁时等待另一把锁。

3. 组合使用多进程和多线程 ⭐

实现以下架构并计算整体时间：

  • 大任务 1：视频解码，耗时 4 秒。
  • 大任务 2：用两个线程并发读取两半数据，分别耗时 1.5 秒、2 秒；读取完成后聚合计算 3 秒。
  • 大任务 3：用两个线程并发加载两个训练批次，各耗时 1 秒；加载完成后模型训练 5 秒。
  • 三个大任务使用多进程并行。
✅ 参考答案
Python
import time
import threading
from multiprocessing import Process


def simulate(name, seconds):
    print(f"{name}开始")
    time.sleep(seconds)
    print(f"{name}完成")


def run_threads(tasks):
    threads = [
        threading.Thread(target=simulate, args=(name, seconds))
        for name, seconds in tasks
    ]
    for thread in threads:
        thread.start()
    for thread in threads:
        thread.join()


def video_decode():
    simulate("视频解码", 4)


def big_data_process():
    run_threads([
        ("读取前 0.5G 数据", 1.5),
        ("读取后 0.5G 数据", 2),
    ])
    simulate("数据聚合计算", 3)
    print("大数据处理完成")


def machine_learning_training():
    run_threads([
        ("加载训练批次 1", 1),
        ("加载训练批次 2", 1),
    ])
    simulate("模型训练", 5)
    print("机器学习训练完成")


if __name__ == "__main__":
    start = time.perf_counter()

    processes = [
        Process(target=video_decode),
        Process(target=big_data_process),
        Process(target=machine_learning_training),
    ]
    for process in processes:
        process.start()
    for process in processes:
        process.join()

    elapsed = time.perf_counter() - start
    print(f"整体执行时间：{elapsed:.2f} 秒")

忽略进程创建开销时：

  • 视频解码约 4 秒；
  • 数据处理约 max(1.5, 2) + 3 = 5 秒；
  • 机器学习任务约 max(1, 1) + 5 = 6 秒。

三个大任务并行，所以整体耗时由最慢的大任务决定，约为 6 秒，而不是三者相加的 15 秒。这里用 sleep() 模拟耗时，真实 CPU 密集型计算应替换为实际计算逻辑。

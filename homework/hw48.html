💹 金融 Prompt 实战：分类、抽取与匹配综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：金融 Prompt 实战：分类、抽取与匹配

55 道练习，按“金融任务定义 → 文本分类与 Few-Shot → 金额、日期、主体和证据抽取 → 语义匹配与阈值 → 结构化后处理 → 隐私、安全、评测与部署”组织。重点是把模型输出约束为可验证的数据产品，而不是让一个看似专业的回答直接进入金融决策。

建议先独立作答，再展开参考答案。本练习仅用于 NLP 与 Prompt 工程训练，不构成投资建议。内容按 NIST AI 600-1 生成式 AI 风险管理框架、OWASP Prompt Injection Prevention、JSON Schema、FinBERT 与 Sentence-BERT资料校准；10 道代码题均离线运行，不调用模型 API，也不使用真实客户或交易数据。

一、选择题（30 题）

1. “判断一段文本属于新闻报道、公司公告、财务报告还是分析师报告”属于什么任务？

A. 单标签文本分类
B. 信息抽取
C. 文本匹配
D. 时间序列预测

✅ 参考答案

答案：A。 每条文本从互斥标签集合中选择一个类别。若一条文本允许同时属于多个类别，则应重新定义为多标签分类，并相应调整输出 Schema、损失和评测指标。

2. 文档类型分类与金融情绪分类的区别是？

A. 前者判断文本来源/体裁，后者判断针对特定主体或事件的正负中性倾向
B. 二者标签永远相同
C. 文档类型只能由股价决定
D. 情绪分类不需要目标主体

✅ 参考答案

答案：A。 “公司公告”可以披露利好或利空，“新闻报道”也可能保持中性。Prompt 必须写清任务语义，不能把体裁、主题、情绪和市场影响混为一谈。

3. Prompt 中标签写“财务报告”，后处理白名单却写“财务公告”，最可能导致什么？

A. 本来正确的输出被判为非法，或统计出两个含义重叠的类别
B. 自动合并两个标签
C. 自动提高召回率
D. 对结果没有影响

✅ 参考答案

答案：A。 标签名称、定义、示例、Schema、解析器和评测集必须使用同一版本的枚举。显示名称与内部 ID 最好分离，例如内部固定 financial_report，界面再映射中文。

4. 一条输入不属于四个既定文档类型时，较合理的设计是？

A. 提供 unknown/out_of_scope 标签和清晰使用条件
B. 强迫模型随机选择最接近的一类
C. 把输入内容用作新标签
D. 删除原文后返回空字符串

✅ 参考答案

答案：A。 开放世界输入不一定落入闭集。未知类应进入单独评测，避免“永远四选一”制造虚假确定性；高风险场景还可转人工复核。

5. 金融文本为什么需要领域化示例与评测？

A. “负债、减值、降息”等词在金融语境中的含义和影响依赖主体、事件与上下文
B. 通用语言模型完全不认识任何汉字
C. 所有金融文本都只有负面情绪
D. 领域词汇能自动预测未来价格

✅ 参考答案

答案：A。 领域语言、标签边界和数据分布与通用语料不同。领域适配可以改善任务表现，但文本分类结果仍不能直接推导收益或交易动作。

6. 四分类 Few-Shot 示例如何选择更合理？

A. 各类覆盖典型样本和易混边界，格式一致，并加入未知类示例
B. 只展示“新闻报道”
C. 每类复制同一句话
D. 示例答案故意使用不同标签拼写

✅ 参考答案

答案：A。 示例会同时教给模型类别先验、边界和输出格式。应检查类别平衡、代表性、正确性、时间泄漏以及示例顺序敏感性。

7. 从测试集直接挑选与每条待测样本最相似的已标注示例，会有什么风险？

A. 若使用了测试标签或未来信息，会形成数据泄漏并高估效果
B. 必然降低所有指标
C. 会自动变成模型微调
D. 与评测公平性无关

✅ 参考答案

答案：A。 示例检索库必须来自训练/开发数据，测试集只能用于最终评估。时间敏感金融数据还应按时间切分，防止未来公告或修订信息泄漏到过去样本。

8. 哪个字符串是合法 JSON？

A. {"label":"company_announcement"}
B. {'label':'company_announcement'}
C. {label: company_announcement}
D. json {label}

✅ 参考答案

答案：A。 JSON 的键和字符串值使用双引号。Python 字典的单引号表示法不是 JSON；模型输出还应经过真正的 JSON 解析器和 Schema 校验。

9. Prompt 要求“严格输出 JSON”之后，为什么仍需代码校验？

A. 模型可能输出代码围栏、漏字段、错类型、非法枚举或额外解释
B. 因为 JSON 无法表示字符串
C. 因为解析器会自动做金融事实核验
D. 只要模型语气专业就不必校验

✅ 参考答案

答案：A。 格式约束提高遵从概率，验证器才建立确定性接口。即使 Schema 合法，也还需核验事实与证据，因为格式正确不等于抽取正确。

10. 信息抽取中保存 evidence_text、start、end 的主要价值是？

A. 可以验证值确实来自原文，并支持审计与高亮
B. 自动补齐原文没有的字段
C. 自动判断投资价值
D. 替代字段 Schema

✅ 参考答案

答案：A。 应检查 source[start:end] == evidence_text。规范化值如金额数值可与原文 span 分开保存，既便于计算，又不丢失可追溯证据。

11. 原文没有收盘价时，结构化输出更推荐？

A. 使用 JSON null 或空数组，并明确缺失原因
B. 猜测一个合理价格
C. 写字符串 ['原文中未提及'] 充当真实值
D. 复制开盘价

✅ 参考答案

答案：A。 缺失状态应与真实文本值区分。若需要更细，可使用 {"value": null, "status": "not_mentioned"}；不要让自然语言占位符污染数值字段。

12. 抽取“1.25亿元”时，最稳妥的存储方式是？

A. 同时保存原文、规范化数值 125000000 和币种 CNY
B. 只保存浮点数 1.25
C. 删除“亿”和币种
D. 自动换算为美元但不记录汇率

✅ 参考答案

答案：A。 原文用于追溯，规范化值用于计算，币种和单位不能丢失。金额运算优先使用十进制数，避免二进制浮点带来的精度问题。

13. 同一句出现“开盘 23 元，收盘 26 美元”，系统应如何处理？

A. 分别保留币种并标记币种不一致，不能静默比较涨跌幅
B. 假设二者币种相同
C. 把美元字符串删除
D. 自动按任意汇率换算

✅ 参考答案

答案：A。 这可能是原文错误或真实跨币种表达。若需换算，必须有明确汇率来源、时间点、方向和精度规则；否则应报告异常并转核验。

14. 抽取“昨日成交量增加”时，为什么需要 as_of_date？

A. 相对日期必须绑定明确参考时间才能规范化
B. 因为“昨日”永远指 2025-01-01
C. 为了增加 Prompt 长度
D. 日期与金融数据无关

✅ 参考答案

答案：A。 参考日期还应包含时区和资料发布时间。没有锚点时应保留原始表达或标记无法规范化，不能根据当前运行时间悄悄猜测。

15. 一段文本披露两只股票的不同价格，合适的输出结构是？

A. events 数组，每个事件绑定自己的主体、价格、日期和证据
B. 把所有值放进几个无对应关系的列表
C. 只保留第一只股票
D. 把两个主体拼成一个名称

✅ 参考答案

答案：A。 平行列表容易失去“哪个价格属于哪个主体”的对应关系。事件级对象能表达一对多关系并便于逐事件验证。

16. 下列哪个后处理检查最重要？

A. 字段、类型、枚举、数值范围、证据 span 和跨字段一致性全部检查
B. 只检查输出长度
C. 只检查回答是否包含“金融”
D. 直接使用 eval()

✅ 参考答案

答案：A。 例如成交量应为非负整数、价格应带币种、日期应可解析、证据应落在原文内。eval() 会执行任意代码，不应用于模型输出。

17. “金融文本相似”的定义为什么必须具体化？

A. 同主题、同事件、同立场、同影响方向和近义改写是不同匹配标准
B. 任意出现“市场”都表示相似
C. 文本长度相同就语义相同
D. 相似度没有业务含义

✅ 参考答案

答案：A。 在去重、事件对齐、检索和矛盾检测中，正类定义不同。Prompt、标注指南和评测集必须使用同一个操作性定义。

18. “利率上升冲击房地产”与“利率下降利好房地产”属于？

A. 主题相近但关键方向相反，若任务要求语义等价应判不相似
B. 必然完全相同
C. 只要都出现房地产就判相似
D. 无法进行任何标注

✅ 参考答案

答案：A。 关键词重叠不能替代事件论元、数值和极性的比较。这类高词面重合但含义不同的样本是重要 hard negative。

19. “A 公司收购 B 公司”与“B 公司收购 A 公司”为什么通常不等价？

A. 交易角色方向相反
B. 两句字符集合相同所以必然等价
C. 公司名称不影响语义
D. 收购关系是无向关系

✅ 参考答案

答案：A。 匹配系统应关注主体、客体、动作、时间、数值和否定。对方向敏感任务，可先抽取结构化事件再比较。

20. 用 embedding 余弦相似度做大规模候选召回，再用 Cross-Encoder 或 LLM 精排，主要优势是？

A. 兼顾双塔召回效率与成对建模精度
B. 保证没有任何漏召回
C. 不需要阈值或评测
D. 自动获得真实标签

✅ 参考答案

答案：A。 Bi-Encoder 可预计算向量，适合大规模检索；Cross-Encoder 联合读取句对，通常更精细但计算更贵。最终效果仍依赖领域数据和任务定义。

21. 相似度阈值应该如何确定？

A. 在独立验证集上按业务代价与目标指标选择，再到测试集只评一次
B. 永远固定为 0.5
C. 在测试集上反复调到最高

二、简答题（15 题）

31. 为“新闻报道、公司公告、财务报告、分析师报告、未知”设计标签规范。

✅ 参考答案

每类应写内部 ID、正定义、排除条件、优先级和边界示例：新闻报道侧重媒体对事件的报道；公司公告是公司正式披露但不一定是财务报表；财务报告以财务期间、报表或经营数据为核心；分析师报告包含研究判断、估值或行业分析；其余或证据不足为 unknown。若一段文本同时包含多种体裁，应规定按来源元数据、主要内容还是多标签处理，不能让标注员临场猜。

32. 如何设计四分类 Few-Shot 样例集？

✅ 参考答案

从训练数据选择真实、脱敏、标注一致的样本；各类既有典型例也有易混例，如“媒体转述公司公告”和“公告中的财务数据”。加入 unknown、短文本、长文本和格式噪声，示例输出统一用内部标签 ID。检查顺序、类别频率和关键词捷径，禁止使用测试集或未来公告。通过消融实验比较 Zero-Shot、固定 Few-Shot 与检索 Few-Shot。

33. “央行发布降息公告”应归新闻报道还是公司公告？如何避免歧义？

✅ 参考答案

仅凭这句话无法可靠判断体裁：若原文来自央行官方公告，可归政策/监管公告；若来自媒体转述，则是新闻报道，而“公司公告”通常不适用于央行。解决方法是把来源、发布主体和文档类型元数据作为输入，并在标签体系中明确是否需要“监管/政策公告”类别。现有闭集中证据不足时应输出 unknown，而不是根据“公告”一词机械分类。

34. 设计金融事件抽取 Schema。

✅ 参考答案

可使用 document_id、as_of、events[]；每个事件含 event_type、subject、date、open_price、close_price、volume。实体字段保存 raw_text、规范化 value、currency/unit、start/end、status。规定 null 与空数组语义、额外字段策略、数值范围和跨字段规则，并让每个事件内部绑定对应主体，避免用无对应关系的平行列表。

35. 金额规范化需要记录哪些信息？

✅ 参考答案

至少记录原始文本、十进制数值、原单位倍数、币种、是否含税/估算、证据位置和解析状态。例如“约 1.25 亿元”可保存 raw、125000000、CNY、approximate=true。跨币种换算另存汇率值、币对、来源、时间和舍入规则；没有这些信息时不换算，也不直接比较涨跌。

36. 为什么规范化值不能替代原文证据？

✅ 参考答案

规范化过程可能解析错误、舍入或丢失限定词。保留证据 span 能复查 source[start:end]、展示高亮、定位模型错误和重新运行新解析器。可靠系统应同时保存 raw 与 normalized，并把“模型抽到什么”和“代码如何解释”分成两个可审计步骤。

37. 如何表达“原文未提及”“模型无法解析”和“字段不适用”？

✅ 参考答案

不要都塞进一个字符串。可使用 value: null 搭配状态枚举：not_mentioned 表示原文没有，parse_error 表示有证据但规范化失败，not_applicable 表示该事件不需要该字段，redacted 表示因权限隐藏。不同状态对应不同重试、统计与人工处理策略。

38. 一段文本包含多个金融事件时如何抽取？

✅ 参考答案

先识别事件触发词和主体，再为每个事件建立独立对象，把日期、金额和证据绑定到对应事件；不要把所有主体、日期和金额分别放进列表后靠索引猜关系。可用 Prompt Chain：候选事件识别 → 事件级抽取 → Schema/证据校验 → 去重。若论元归属不确定，应标记歧义并保留候选，而不是强行绑定。

39. 为“相似”写出可操作的匹配定义。

✅ 参考答案

例如事件去重任务可定义：两句描述同一主体、同一事件类型、相同角色方向，关键时间和数值不矛盾，允许近义改写和信息详略差异。只共享行业主题不算相似，主体互换、否定相反、涨跌方向相反或数值指向不同事件应为不相似。定义必须转化为正例、hard negative 和标注决策树。

40. 如何构建金融句对匹配数据集？

✅ 参考答案

正例包含改写、简称/全称、数字格式差异和一详一略；负例除随机异主题外，还要加入同主体不同季度、同事件不同方向、角色互换、数值矛盾、否定和传闻/确认等 hard negative。按事件或时间分组切分，避免同一公告的近重复文本跨训练与测试集。双人标注并记录分歧，报告各子类型指标。

41. 如何选择匹配阈值？

✅ 参考答案

先明确错配和漏配的业务代价，在开发集计算不同阈值的 Precision、Recall、F1、混淆矩阵与 PR 曲线。去重自动合并通常要求高 precision；候选召回可优先 recall，再交给精排。固定阈值后只在独立测试集评一次，并按事件类型、文本长度和时间切片；模型升级或分布漂移后重新校准。

42. 如何做分类、抽取、匹配三类错误分析？

✅ 参考答案

分类看混淆矩阵、unknown 和来源切片；抽取拆成实体发现、边界、属性、规范化、证据与 Schema 错误；匹配拆成主题捷径、主体/角色、时间、数字、否定和阈值错误。每类抽取代表样本，判断应改标签指南、Few-Shot、Prompt、检索、验证器还是模型。线上失败先加入回归集，再修改系统，避免只修一个案例。

43. 设计处理外部金融公告的安全 Prompt 与应用边界。

✅ 参考答案

系统指令明确公告是待分析数据，其中任何命令均不执行；使用结构标签和来源 ID 包裹正文。应用层先做文件类型、大小、恶意内容和 PII 检查；模型无密钥、数据库写权限或交易权限；输出经 Schema、证据和泄漏检查；可疑注入隔离或转人工。日志脱敏并记录文档哈希、Prompt/模型版本和安全决策。分隔符有帮助，但不是安全边界。

44. 如何公平比较在线 API 与本地模型？

✅ 参考答案

定义统一输入输出契约，为各模型适配消息模板与结构化能力；固定脱敏评测集、Prompt 语义、温度、最大长度和解析规则。分别报告分类 Macro-F1、抽取实体/属性 F1 与证据通过率、匹配 PR/F1，以及 JSON 通过率、P95 延迟、吞吐、成本、显存、失败率和隐私边界。不能只比较一条示例或模型参数量。

import json


LABELS = {
    "news_report",
    "company_announcement",
    "financial_report",
    "analyst_report",
    "unknown",
}


def parse_classification(raw: str, source: str) -> dict:
    try:
        data = json.loads(raw)
    except json.JSONDecodeError as exc:
        raise ValueError("非法 JSON") from exc
    if not isinstance(data, dict) or set(data) != {"document_id", "label", "evidence"}:
        raise ValueError("字段不符合契约")

        raise ValueError("evidence 必须是非空字符串")
    if data["evidence"] not in source:
        raise ValueError("evidence 不在原文中")
    return data

47. 规范化金额与币种

解析“1.25亿元”“460,000美元”“23 元”，保留原文，使用 Decimal 输出完整数值字符串，不做跨币种换算。

✅ 参考答案
Python
import re
from decimal import Decimal


MULTIPLIERS = {"": Decimal("1"), "万": Decimal("10000"), "亿": Decimal("100000000")}
CURRENCIES = {"元": "CNY", "人民币": "CNY", "美元": "USD", "港元": "HKD"}
MONEY_RE = re.compile(
    r"\s*(?P<number>\d{1,3}(?:,\d{3})*(?:\.\d+)?|\d+(?:\.\d+)?)"
    r"\s*(?P<unit>[万亿]?)\s*(?P<currency>人民币|美元|港元|元)\s*"
)


def parse_money(raw: str) -> dict:
    match = MONEY_RE.fullmatch(raw)
    if not match:
        raise ValueError("金额格式无法解析")
    number = Decimal(match.group("number").replace(",", ""))
    value = number * MULTIPLIERS[match.group("unit")]
    if value < 0:
        raise ValueError("金额不能为负")
    return {
        "raw": raw,
        "value": format(value, "f"),
        "currency": CURRENCIES[match.group("currency")],
    }


assert parse_money("1.25亿元")["value"] == "125000000.00"
assert parse_money("460,000美元") == {"raw": "460,000美元", "value": "460000", "currency": "USD"}
assert parse_money("23 元")["currency"] == "CNY"
print([parse_money(x) for x in ("1.25亿元", "460,000美元", "23 元")])

48. 校验证据 span

每个抽取实体包含 field、text、start、end；检查类型、边界、原文一致性和字段白名单。

✅ 参考答案
Python
FIELDS = {"date", "stock_name", "open_price", "close_price", "volume"}


def validate_spans(source: str, entities: list[dict]) -> list[dict]:
    if not isinstance(entities, list):
        raise TypeError("entities 必须是列表")
    validated = []
    for entity in entities:
        if not isinstance(entity, dict) or set(entity) != {"field", "text", "start", "end"}:
            raise ValueError("实体字段非法")
        if entity["field"] not in FIELDS:
            raise ValueError("field 不在白名单")
        start, end = entity["start"], entity["end"]
        if not isinstance(start, int) or not isinstance(end, int) or not 0 <= start < end <= len(source):
            raise ValueError("span 越界")
        if source[start:end] != entity["text"]:
            raise ValueError("span 与原文不一致")
        validated.append(dict(entity))
    return validated


source = "2026-07-16，甲公司开盘价23元。"
text = "23元"
start = source.index(text)
entities = [{"field": "open_price", "text": text, "start": start, "end": start + len(text)}]
assert validate_spans(source, entities) == entities

bad = [{"field": "open_price", "text": "25元", "start": start, "end": start + len(text)}]
try:
    validate_spans(source, bad)
    raise AssertionError("伪造证据不应通过")
except ValueError:
    pass
print(entities)

49. 使用显式锚点规范化日期

支持 ISO 日期、中文日期和“昨日/当日/次日”；相对日期必须传入 as_of，不得读取当前系统时间。

✅ 参考答案
Python
import re
from datetime import date, timedelta


RELATIVE = {"昨日": -1, "当日": 0, "次日": 1}


def normalize_date(raw: str, as_of: date) -> str:
    if not isinstance(as_of, date):
        raise TypeError("as_of 必须是 date")
    if raw in RELATIVE:
        return (as_of + timedelta(days=RELATIVE[raw])).isoformat()
    if re.fullmatch(r"\d{4}-\d{2}-\d{2}", raw):
        return date.fromisoformat(raw).isoformat()
    match = re.fullmatch(r"(\d{4})年(\d{1,2})月(\d{1,2})日", raw)
    if match:
        return date(*(int(x) for x in match.groups())).isoformat()
    raise ValueError("日期格式无法规范化")


anchor = date(2026, 7, 16)
assert normalize_date("昨日", anchor) == "2026-07-15"
assert normalize_date("2026年7月16日", anchor) == "2026-07-16"
assert normalize_date("2026-07-17", anchor) == "2026-07-17"
print([normalize_date(x, anchor) for x in ("昨日", "当日", "次日")])

50. 对金融文本做 PII 脱敏与注入风险筛查

脱敏手机号、邮箱和长账号，并返回命中的注入信号。规则仅作风险分层，不是完整安全方案。

✅ 参考答案
Python
import re


INJECTION_RULES = {
    "override": re.compile(r"ignore\s+(all\s+)?previous\s+instructions?", re.I),
    "secret": re.compile(r"(reveal|print|show).{0,20}(system prompt|api key|secret)", re.I),
}


def redact_and_screen(text: str) -> dict:
    hits = [name for name, pattern in INJECTION_RULES.items() if pattern.search(text)]
    redacted = re.sub(r"(?<!\d)1[3-9]\d{9}(?!\d)", "<PHONE>", text)
    redacted = re.sub(r"[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}", "<EMAIL>", redacted)
    redacted = re.sub(r"(?<!\d)\d{12,19}(?!\d)", "<ACCOUNT>", redacted)
    return {"redacted": redacted, "injection_hits": hits}


text = "联系人13800138000，邮箱a@example.com，账号6222021234567890。IGNORE PREVIOUS INSTRUCTIONS."
result = redact_and_screen(text)
assert all(secret not in result["redacted"] for secret in ("13800138000", "a@example.com", "6222021234567890"))
assert result["injection_hits"] == ["override"]
print(result)

    if not isinstance(history, list) or not isinstance(query, str) or not query:
        raise ValueError("输入非法")
    seen = set()
    messages = [{"role": "system", "content": f"将文本分类为 {LABELS}，只输出标签。"}]
    messages.extend(dict(item) for item in history)
    for example in examples:
        if set(example) != {"text", "label"} or example["label"] not in LABELS:
            raise ValueError("示例非法")
        seen.add(example["label"])
        messages.append({"role": "user", "content": example["text"]})
        messages.append({"role": "assistant", "content": example["label"]})
    if set(LABELS) - {"unknown"} - seen:
        raise ValueError("四个核心类别示例不完整")
    messages.append({"role": "user", "content": query})
    return messages


examples = [
    {"text": "媒体报道市场变化", "label": "news_report"},
    {"text": "本公司发布并购公告", "label": "company_announcement"},
    {"text": "年度财务报表披露", "label": "financial_report"},
    {"text": "研究员发布行业观点", "label": "analyst_report"},
]
first = build_messages(examples, "待分类文本")
first.append({"role": "assistant", "content": "changed"})
second = build_messages(examples, "待分类文本")
assert second[-1] == {"role": "user", "content": "待分类文本"}
assert all(item.get("content") != "changed" for item in second)
print(second)

52. 校验事件级金融抽取记录

要求价格非负、成交量为非负整数；发现开盘与收盘币种不一致时保留记录并返回警告，而不是静默比较。

✅ 参考答案
Python
from decimal import Decimal, InvalidOperation


def validate_event(event: dict) -> dict:
    required = {"subject", "date", "open_price", "close_price", "volume"}
    if not isinstance(event, dict) or set(event) != required:
        raise ValueError("事件字段不完整")
    if not isinstance(event["subject"], str) or not event["subject"]:
        raise ValueError("subject 非法")
    warnings = []

    for field in ("open_price", "close_price"):
        price = event[field]
        if not isinstance(price, dict) or set(price) != {"value", "currency"}:
            raise ValueError(f"{field} 结构非法")
        try:
            value = Decimal(price["value"])
        except (InvalidOperation, TypeError):
            raise ValueError(f"{field} 数值非法")
        if value < 0 or price["currency"] not in {"CNY", "USD", "HKD"}:
            raise ValueError(f"{field} 范围或币种非法")

    if event["open_price"]["currency"] != event["close_price"]["currency"]:
        warnings.append("currency_mismatch")
    if not isinstance(event["volume"], int) or event["volume"] < 0:
        raise ValueError("volume 必须是非负整数")
    return {"event": event, "warnings": warnings}


event = {
    "subject": "盘古(0021)",
    "date": "2023-04-05",
    "open_price": {"value": "23", "currency": "CNY"},
    "close_price": {"value": "26", "currency": "USD"},
    "volume": 310000,
}
result = validate_event(event)
assert result["warnings"] == ["currency_mismatch"]
assert result["event"]["volume"] == 310000
print(result)

53. 在验证集上选择匹配阈值

计算 Precision、Recall、F1；若 F1 相同，优先 Precision 更高，再优先更高阈值。

✅ 参考答案
Python
def metrics(scores: list[float], labels: list[int], threshold: float) -> dict:
    predictions = [int(score >= threshold) for score in scores]
    tp = sum(p == 1 and y == 1 for p, y in zip(predictions, labels))
    fp = sum(p == 1 and y == 0 for p, y in zip(predictions, labels))
    fn = sum(p == 0 and y == 1 for p, y in zip(predictions, labels))
    precision = tp / (tp + fp) if tp + fp else 0.0
    recall = tp / (tp + fn) if tp + fn else 0.0
    f1 = 2 * precision * recall / (precision + recall) if precision + recall else 0.0
    return {"threshold": threshold, "precision": precision, "recall": recall, "f1": f1}


def choose_threshold(scores: list[float], labels: list[int]) -> dict:
    if len(scores) != len(labels) or not scores or any(label not in {0, 1} for label in labels):
        raise ValueError("验证数据非法")
    candidates = sorted(set(scores), reverse=True)
    results = [metrics(scores, labels, threshold) for threshold in candidates]
    return max(results, key=lambda item: (item["f1"], item["precision"], item["threshold"]))


scores = [0.95, 0.80, 0.70, 0.60, 0.30]
labels = [1, 0, 1, 0, 0]
best = choose_threshold(scores, labels)
assert best["threshold"] == 0.70
assert best["precision"] == 2 / 3 and best["recall"] == 1.0
assert round(best["f1"], 3) == 0.8
print(best)

54. 实现向量余弦相似度与 Top-K 召回

使用预先给定的离线向量，不依赖外部模型；检查维度、零向量和稳定排序。

✅ 参考答案
Python
import math


def cosine(a: list[float], b: list[float]) -> float:
    if len(a) != len(b) or not a:
        raise ValueError("向量维度非法")
    norm_a = math.sqrt(sum(x * x for x in a))
    norm_b = math.sqrt(sum(x * x for x in b))
    if norm_a == 0 or norm_b == 0:
        raise ValueError("不允许零向量")
    return sum(x * y for x, y in zip(a, b)) / (norm_a * norm_b)


def top_k(query: list[float], corpus: list[dict], k: int) -> list[dict]:
    if not 1 <= k <= len(corpus):
        raise ValueError("k 超出范围")
    ranked = []
    for index, item in enumerate(corpus):
        score = cosine(query, item["vector"])
        ranked.append({"id": item["id"], "score": score, "index": index})
    ranked.sort(key=lambda item: (-item["score"], item["index"]))
    return [{"id": item["id"], "score": item["score"]} for item in ranked[:k]]


corpus = [
    {"id": "profit_up", "vector": [1.0, 0.0, 0.0]},
    {"id": "profit_down", "vector": [0.8, 0.2, 0.0]},
    {"id": "oil_market", "vector": [0.0, 0.0, 1.0]},
]
result = top_k([1.0, 0.0, 0.0], corpus, 2)
assert [item["id"] for item in result] == ["profit_up", "profit_down"]
assert result[0]["score"] == 1.0
print(result)

向量召回只产生候选；profit_up 与 profit_down 可能因主题相同而相似，但关键方向相反，需用 hard-negative 训练、精排或结构化事件比较处理。

55. 构建带校验、重试和幂等缓存的离线批处理

假模型第一次返回非法 JSON、第二次返回合法结果；相同文档重复处理时命中内容哈希缓存，不再次调用模型。

✅ 参考答案
Python
import hashlib
import json


LABELS = {"news_report", "company_announcement", "financial_report", "analyst_report", "unknown"}


class FakeClient:
    def __init__(self):
        self.calls = 0

    def complete(self, _text: str) -> str:
        self.calls += 1
        if self.calls == 1:
            return "not json"
        return '{"label":"company_announcement","evidence":"发布收购公告"}'


def parse_output(raw: str, source: str) -> dict:
    data = json.loads(raw)
    if not isinstance(data, dict) or set(data) != {"label", "evidence"}:
        raise ValueError("Schema 非法")
    if data["label"] not in LABELS:
        raise ValueError("标签非法")
    if not isinstance(data["evidence"], str) or data["evidence"] not in source:
        raise ValueError("证据非法")
    return data


def process_document(text: str, client: FakeClient, cache: dict, max_attempts: int = 2) -> dict:
    document_id = hashlib.sha256(text.encode("utf-8")).hexdigest()[:16]
    if document_id in cache:
        return {"document_id": document_id, "result": dict(cache[document_id]), "cached": True}

    last_error = None
    for attempt in range(1, max_attempts + 1):
        try:
            result = parse_output(client.complete(text), text)
            cache[document_id] = dict(result)
            return {
                "document_id": document_id,
                "result": result,
                "cached": False,
                "attempts": attempt,
            }
        except (json.JSONDecodeError, ValueError) as exc:
            last_error = exc
    raise RuntimeError("模型输出在重试后仍无效") from last_error


client = FakeClient()
cache = {}
text = "甲公司发布收购公告，交易尚待审批。"
first = process_document(text, client, cache)
second = process_document(text, client, cache)

assert first["attempts"] == 2 and not first["cached"]
assert second["cached"] and client.calls == 2
assert first["result"]["evidence"] in text
print({"first": first, "second": second, "client_calls": client.calls})

真实服务还应区分可重试错误与安全/业务拒绝，设置超时、退避、限流和监控；若调用具有副作用，必须使用业务幂等键，不能仅依赖进程内缓存。

🪄 Prompt-Tuning：PET、软提示与 P-tuning 综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：Prompt-Tuning：从 NLP 四范式到 PET 与软提示

55 道练习，按“NLP 四范式与术语边界 → ICL、Instruction Tuning 与 CoT → PET/PVP → Verbalizer 与校准 → Soft Prompt 与 P-tuning → 冻结参数、梯度流、评测和部署”组织。重点不是背方法名，而是能从输入构造、训练参数、目标函数和推理工件四个维度准确判断一种方法。

建议先独立作答，再展开参考答案。本练习依据 PET 原始论文、Prompt Tuning 原始论文、P-tuning 原始论文、P-tuning v2 论文 与 Hugging Face PEFT Prompt Tuning 文档校准。注意：课程中的“四范式”适合作为理解技术演进的教学框架，并非互斥的正式分类；Instruction Tuning 会进行监督式参数更新；“提示微调可媲美全量微调”是依赖模型规模、任务和实验设置的经验结论，不是无条件保证。10 道代码题均可离线运行，不下载预训练权重。

一、选择题（30 题）

1. 将 NLP 演进概括为“传统机器学习 → 深度学习 → 预训练后微调 → 预训练后提示”的正确理解是？

A. 便于比较表示、监督信号和任务接口的教学框架，不代表旧范式已经消失或四类严格互斥
B. 所有 NLP 系统都必须依次执行四个阶段
C. Prompt 出现后不再需要训练数据
D. 每个阶段只能使用一种模型架构

✅ 参考答案

答案：A。 实际系统可以混合使用规则、检索、全量微调、参数高效微调和上下文示例。所谓“四范式”帮助理解任务知识如何注入模型，但不能代替对具体训练流程的判断。

2. 预训练目标与下游分类目标之间的“任务形式差距”是指？

A. 模型预训练可能学习预测 token，而下游直接新增分类头并预测类别 ID
B. 训练集和验证集文件名不同
C. CPU 与 GPU 的品牌不同
D. 类别名称必须使用英文

✅ 参考答案

答案：A。 Prompt 方法常把分类改写成填空或文本生成，使下游输入输出形式更接近 MLM、因果语言建模或 Seq2Seq 预训练目标；但“形式对齐”不等于自动解决数据偏差和标签歧义。

3. 下列哪项最准确地区分 Prompt Engineering 与 Prompt Tuning？

A. 前者通常人工或程序选择离散文本且不做梯度更新，后者通常用梯度学习连续提示参数
B. 二者都必须更新全部模型权重
C. 前者只能用于分类，后者只能用于翻译
D. 二者完全同义

✅ 参考答案

答案：A。 “Prompt-Tuning”在宽泛材料中有时泛指提示方法，但在参数高效微调语境里，通常特指学习 soft prompt。判断时应明确提示是否可训练、是否为连续向量、基座是否冻结。

4. GPT 类模型的 Few-Shot In-Context Learning（ICL）在一次普通推理中会？

A. 把示例放入上下文，但不通过反向传播更新模型权重
B. 自动训练一个新分类头
C. 永久修改模型参数
D. 必须生成 [MASK]

✅ 参考答案

答案：A。 ICL 的“学习”发生在当前上下文条件下，不是优化器意义上的参数学习。示例会占用上下文窗口，换一个请求若不再次提供示例，模型不会保留这次上下文。

5. Instruction Tuning 最典型的训练行为是？

A. 用“指令—输入—目标输出”监督数据更新模型或适配参数
B. 只改一段系统提示，完全不训练
C. 只在推理时加入三个示例
D. 只调整 temperature

✅ 参考答案

答案：A。 指令微调是一种训练过程，可以全量更新，也可以结合 LoRA 等 PEFT。它与推理时写指令 Prompt 有联系，但不能归为“完全无参数更新”的 ICL。

6. Chain-of-Thought（CoT）最适合被描述为？

A. 诱导或训练模型产生中间推理步骤的一类策略，单独使用 CoT Prompt 不必更新权重
B. 一种固定的优化器
C. 一种 tokenizer
D. 一定只训练输入 embedding

✅ 参考答案

答案：A。 Zero-Shot/Few-Shot CoT 可以纯推理使用，也可以与带推理数据的训练结合。看到“CoT”不能直接推断参数是否更新，要看完整流程。

7. PET（Pattern-Exploiting Training）的核心任务改写是？

A. 用 Pattern 把样本变成完形填空，再用 Verbalizer 把类别映射到标签词
B. 删除标签并随机聚类
C. 把所有 token 平均后做 KNN
D. 只增加模型层数

✅ 参考答案

答案：A。 Pattern 与 Verbalizer 合称 PVP。原始 PET 还包含多个 PVP 模型、在无标注数据上生成软标签以及最终分类器训练等半监督流程，因此“一个 [MASK] 模板”只是 PET 思想的重要部件，不是全部流程。

8. 在情感任务模板“这家餐厅的味道很棒。总体来说，它很 [MASK]。”中，Pattern 是？

A. 将原文本嵌入带 [MASK] 的完整输入形式
B. positive -> 好 的标签映射
C. 交叉熵优化器
D. 模型预测的概率

✅ 参考答案

答案：A。 Pattern 决定模型看到的上下文；positive -> 好、negative -> 差 属于 Verbalizer。二者应分别设计和消融。

9. Verbalizer 的作用是？

A. 建立任务标签与词表 token（标签词）之间的映射
B. 把所有词切成字符
C. 冻结优化器
D. 计算位置编码

✅ 参考答案

答案：A。 例如 positive -> 好、negative -> 差。分类时可从 [MASK] 位置的词表 logits 取出标签词得分，再在类别维度归一化。

10. 一个单 [MASK] 模板把类别映射为“非常满意”时，首先要检查？

A. tokenizer 是否把标签词切成多个 token，模板的预测位数与打分规则是否匹配
B. 字符串是否超过一个汉字
C. GPU 是否有两个风扇
D. 标签是否按字母排序

✅ 参考答案

答案：A。 “一个词”在自然语言中可能对应多个子词 token。单 mask 的 MLM 不能直接一次预测任意长度短语，需要多 mask、改用单 token 标签词，或明确定义自回归/多 token 聚合方案。

11. 两个类别使用同一个标签词，最直接的问题是？

A. 从该词的得分无法唯一还原类别
B. 参数量会自动翻倍
C. 输入长度会变为零
D. 模型会自动创建新 token

✅ 参考答案

答案：A。 Verbalizer 通常应保证类别映射可区分。若每类有多个标签词，还需规定类内聚合方式，并检查不同类标签词是否语义重叠。

12. 下列哪项更可能造成 Verbalizer 的先验偏差？

A. 某个标签词在预训练语料中极常见或与模板天然搭配，即使输入信息很弱也得到高分
B. 训练集文件使用 UTF-8
C. 类别 ID 从 0 开始
D. 使用了 Python 列表

✅ 参考答案

答案：A。 应在内容为空、无意义或类别平衡的校准输入上检查标签词先验，并比较多个标签词、模板或做 contextual calibration，而不是把原始词表概率直接当作无偏类别概率。

13. Hard Prompt 的典型特征是？

A. 由词表中的离散 token 构成，通常可直接读成人类语言
B. 必定是不可解释的浮点向量
C. 必定更新全部 Transformer 层
D. 只能放在输出末尾

✅ 参考答案

答案：A。 Hard Prompt 可以人工编写，也可以通过搜索得到。它可读但对措辞敏感；“hard”描述离散性，不表示训练一定困难或参数一定全量更新。

14. Soft Prompt 的典型特征是？

A. 若干可训练的连续 embedding，未必对应词表中的真实 token
B. 一段不可修改的自然语言说明
C. 一组测试集标签
D. 模型输出的概率表

✅ 参考答案

答案：A。 Soft Prompt 位于模型 embedding 空间，可通过反向传播优化。虚拟 token 的编号只是索引这些向量，不意味着能逐个解码成有意义的自然语言词。

15. 基座模型冻结后，Soft Prompt 还能学习的原因是？

A. 损失梯度仍可穿过基座计算图传回 Prompt，只是基座参数不接收或不应用更新
B. 冻结会阻止一切梯度传播
C. Prompt 不需要损失函数
D. 优化器会随机修改测试标签

✅ 参考答案

答案：A。 requires_grad=False 表示不为基座参数累积梯度，通常不会切断对输入 embedding 的梯度。不要把“冻结参数”误解为在基座前后执行 detach()。

16. 朴素 Prompt Tuning 使用 (m) 个虚拟 token、隐藏维度 (d)，仅提示表的参数量是？

A. (m\times d)
B. (m+d)
C. (m^d)
D. 与 (m,d) 都无关

✅ 参考答案

答案：A。 若使用 P-tuning 的 MLP/LSTM Prompt Encoder，还要加上编码器参数；若在多层注入深提示，参数量也会随层数和具体实现变化。

17. 增大 Soft Prompt 长度最准确的判断是？

A. 表达容量和训练参数增加，但会占用序列预算、增加计算，效果不保证单调提升
B. 越长必然越准确
C. 不影响显存与上下文
D. 参数量反而减少

✅ 参考答案

答案：A。 Prompt 长度是超参数，应在开发集比较，并同时报告截断率、吞吐和显存。长 Prompt 可能挤掉原始文本或目标 token。

18. 关于 Soft Prompt 初始化，哪项最合理？

A. 随机、从词表采样或用任务文本 embedding 初始化都可作为候选，应通过多随机种子实验比较
B. 只有全零初始化永远最优
C. 初始化不可能影响优化
D. 必须从测试集答案初始化

✅ 参考答案

答案：A。 官方 PEFT 实现支持 RANDOM、SAMPLE_VOCAB 和 TEXT 等初始化。初始化可能影响小数据训练的收敛与方差，不能用测试集选最优方案。

19. Prompt Tuning 原始论文中“规模的力量”主要说明？

A. 在其 T5 实验设置中，模型变大后 Prompt Tuning 与全量 Model Tuning 的差距缩小
B. 任意小模型、任意任务上都严格优于全量微调
C. Prompt 长度必须等于模型层数
D. Soft Prompt 不需要训练数据

✅ 参考答案

答案：A。 这是特定模型族、任务和设置下的经验发现。复现或迁移到中文、小模型、序列标注等场景时仍需与全量微调及其他 PEFT 基线公平比较。

20. P-tuning v1 相比直接学习一张 Soft Prompt 表，常见的额外结构是？

A. 用 MLP 或 LSTM Prompt Encoder 对连续提示进行重参数化
B. 删除所有输入 token
C. 固定每个标签的概率
D. 只训练输出 tokenizer

✅ 参考答案

答案：A。 Prompt Encoder 生成或变换虚拟 token embedding，可增强提示之间的依赖建模；训练后是否保留编码器或导出其生成的提示取决于实现。

21. P-tuning v2 的“Deep Prompt Tuning”主要变化是？

A. 在 Transformer 多层注入可训练提示，而不只在输入层加入虚拟 token
B. 只把输入文本重复两遍
C. 取消所有可训练参数
D. 把分类改为 KMeans

✅ 参考答案

二、简答题（15 题）

31. 用“输入、训练、保存、推理”四列区分 Prompt Engineering、ICL、Instruction Tuning 与 Prompt Tuning。

✅ 参考答案

Prompt Engineering 输入离散指令，通常不训练，只保存模板，推理时每次携带；ICL 还在输入中放演示，同样无梯度更新，保存示例选择策略；Instruction Tuning 用指令—输出监督数据更新全量或 PEFT 参数，保存新模型或适配器，推理时可只给新指令；Prompt Tuning 学习连续虚拟 token，通常冻结基座，保存 prompt 权重与配置，推理时系统在 embedding 层自动拼接。四者可以组合，例如先做指令微调，再在特定任务上训练 Soft Prompt。

32. 把二分类情感任务改写为一个 PVP，并指出潜在缺陷。

✅ 参考答案

Pattern 可写为“评论：{text}。总的来说，这次体验很 [MASK]。”；Verbalizer 为 positive→“好”、negative→“差”。需检查“好/差”在当前 tokenizer 中均为单 token，模板语法自然且不提前泄漏答案；再测试替代模板和标签词，观察顺序与先验偏差。若中性样本存在，二分类标签体系本身也需修订。

33. 为什么 PVP 的 Pattern 和 Verbalizer 要分别做消融？

✅ 参考答案

Pattern 决定上下文及 [MASK] 与原文的关系，Verbalizer 决定词表概率如何映射为类别。只替换其中一项才能判断误差来自模板措辞还是标签词先验。建议构造多个 Pattern×Verbalizer 组合，在相同切分、种子和预算下报告均值、方差、每类指标及空内容校准结果。

34. 如何处理类别名称需要多个 token 的情况？

✅ 参考答案

不要假设自然语言“一个词”等于 tokenizer 的一个 token。可选择语义合适的单 token 标签词；为每个 token 放置对应数量的 mask 并定义联合概率；改为自回归/Seq2Seq 生成标签字符串；或给每类配置多个单 token verbalizer 并聚合。所有方案都需保证训练与推理打分一致，并测试长度偏置。

35. 设计一个 Verbalizer 质量诊断流程。

✅ 参考答案

先验证 tokenization、类别间不冲突、词义与模板搭配自然；再用无信息输入测标签词先验，在训练集看类内覆盖和每类召回；比较多个标签词与聚合规则，加入否定、反事实和 hard negative；最后在独立开发集按模板、类别、长度和领域切片评估。若做校准，保存校准集和算法版本，禁止用测试集估计偏置。

36. 解释“基座冻结但梯度仍穿过基座”的计算过程。

✅ 参考答案

Prompt embedding 是需要梯度的叶子参数，随后经过冻结 Transformer 的矩阵运算得到 logits 和损失。反向传播为计算 Prompt 梯度，仍使用这些冻结权重参与链式法则；只是权重本身 requires_grad=False，不会累积 .grad 或被优化器更新。如果在 Prompt 输出处错误 detach()，Prompt 才会失去梯度。

37. 如何审计一个“只训练 Soft Prompt”的实现？

✅ 参考答案

列出所有命名参数的 requires_grad，断言只有 prompt 模块可训练；优化器参数 ID 必须与可训练参数 ID 集合相同；训练一步前后保存基座参数哈希或逐张量比较，确认完全不变；确认 prompt 梯度非零且权重发生变化；同时检查 dropout 模式、LM head、LayerNorm 和额外分类头是否符合预先声明的方法定义。

38. Prompt 长度如何影响训练与部署？

✅ 参考答案

长度 (m) 使朴素提示参数约为 (m d)，也让有效序列变长，增加注意力计算、KV cache、显存和延迟，并减少原文本可用窗口。过短可能容量不足，过长可能更难优化或挤掉输入。应在开发集对多个长度做消融，同时报告质量、截断率、吞吐、峰值显存和 P95 延迟，而非只看准确率。

39. 比较随机初始化、词表采样初始化和文本初始化。

✅ 参考答案

随机连续向量不受自然语言限制，但可能远离已有 embedding 流形；从词表采样位于已有 token embedding 上但任务语义随机；文本初始化用任务描述或标签相关词，可能更快收敛但引入措辞偏置。应固定数据与预算，用多个种子报告收敛曲线和最终指标；不能由测试集选择初始化。

40. 比较 Prompt Tuning、P-tuning v1 与 P-tuning v2。

✅ 参考答案

朴素 Prompt Tuning 通常直接学习输入层的一段虚拟 token embedding；P-tuning v1 使用 MLP/LSTM Prompt Encoder 重参数化连续提示，并可与离散 anchor 组合；P-tuning v2 将提示扩展到多层的 deep prompts，以增强较小模型和复杂 NLU 任务的容量。三者在不同代码库中的命名可能不完全一致，比较时应以“注入位置、生成结构、冻结范围”描述，而非只看方法名。

41. 如何验证“Prompt Tuning 可媲美全量微调”的主张？

✅ 参考答案

在同一基座、数据切分和指标上建立全量微调、线性探测、Prompt Tuning 及其他 PEFT 基线；为每种方法使用公开且合理的调参预算，运行多个种子。报告平均值、标准差、统计不确定性、可训练参数、训练时间、峰值显存、检查点大小和推理开销，并在不同数据量、任务类型和模型规模上分层。结论限定在实际实验范围内。

42. 小样本 Prompt Tuning 出现训练集高分、验证集低分，如何诊断？

✅ 参考答案

检查数据泄漏、标签噪声、类别不平衡和切分；比较训练/验证损失及每类指标；降低 Prompt 长度或训练步数，尝试权重衰减、早停和更稳初始化；检查学习率与梯度范数；增加种子与交叉验证；与零样本、线性分类头和全量/LoRA 基线比较。Prompt 参数少不代表不会过拟合。

43. 设计 PET 的半监督训练流程，并指出伪标签风险。

✅ 参考答案

先为标注小样本设计多个 PVP，分别训练对应 MLM；再对大量无标注样本预测并集成为软标签；用这些软标签训练最终分类器。风险包括模板共同偏差、错误自我强化、类别分布失真和无标注域偏移。应设置置信度/一致性过滤、类别配额、人工抽检，并在只用真标签的基线上比较；测试集绝不能参与伪标签生成或筛选。

44. Prompt 检查点要怎样设计才能复现和多任务部署？

✅ 参考答案

每个任务保存 prompt/encoder 权重、PEFT 类型、虚拟 token 数、token_dim、注入层、初始化方式、模板与 Verbalizer、标签表、基座模型 ID 与不可变 revision、tokenizer、依赖版本和评测摘要。服务端共享只读基座，按任务加载或路由小型 Prompt 工件，并做兼容性校验、缓存、版本回滚和并发隔离。不能只凭张量形状猜兼容性。

MASK = "[MASK]"


def build_pvp(text: str, verbalizer: dict[str, int]) -> str:
    if not isinstance(text, str) or not text.strip():
        raise ValueError("text 必须是非空字符串")
    if MASK in text:
        raise ValueError("原文不得包含模板控制 token")
    if set(verbalizer) != {"positive", "negative"}:
        raise ValueError("标签集合必须严格匹配任务")
    token_ids = list(verbalizer.values())
    if any(not isinstance(token_id, int) or token_id < 0 for token_id in token_ids):
        raise ValueError("标签词 token ID 非法")
    if len(set(token_ids)) != len(token_ids):
        raise ValueError("不同类别不能映射到同一 token")

    pattern = f"评论：{text} 总体来说，这次体验很{MASK}。"
    if pattern.count(text) != 1 or pattern.count(MASK) != 1:
        raise AssertionError("PVP 构造失败")
    return pattern

47. 从 [MASK] 位置提取 Verbalizer 类别分数

给定形状 [batch, seq_len, vocab] 的 logits、每条样本的 mask 位置和标签词 ID，返回 [batch, num_classes] 的 logits 与概率。

✅ 参考答案
Python
import torch


def verbalizer_scores(
    logits: torch.Tensor,
    mask_positions: torch.Tensor,
    label_token_ids: torch.Tensor,
) -> tuple[torch.Tensor, torch.Tensor]:
    if logits.ndim != 3:
        raise ValueError("logits 应为 [B, L, V]")
    batch, seq_len, vocab = logits.shape
    if mask_positions.shape != (batch,):
        raise ValueError("每条样本必须有一个 mask 位置")
    if label_token_ids.ndim != 1 or label_token_ids.numel() < 2:
        raise ValueError("至少需要两个类别")
    if not torch.all((0 <= mask_positions) & (mask_positions < seq_len)):
        raise ValueError("mask 位置越界")
    if not torch.all((0 <= label_token_ids) & (label_token_ids < vocab)):
        raise ValueError("标签词 ID 越界")
    if label_token_ids.unique().numel() != label_token_ids.numel():
        raise ValueError("标签词 ID 冲突")

    batch_index = torch.arange(batch, device=logits.device)
    mask_logits = logits[batch_index, mask_positions]       # [B, V]
    class_logits = mask_logits.index_select(1, label_token_ids)
    return class_logits, class_logits.softmax(dim=-1)


logits = torch.zeros(2, 4, 8)
logits[0, 1, 2], logits[0, 1, 5] = 3.0, 1.0
logits[1, 3, 2], logits[1, 3, 5] = -1.0, 2.0
scores, probs = verbalizer_scores(logits, torch.tensor([1, 3]), torch.tensor([2, 5]))
assert scores.shape == (2, 2)
assert probs.argmax(dim=1).tolist() == [0, 1]
assert torch.allclose(probs.sum(dim=1), torch.ones(2))
print(probs)

48. 聚合多标签词 Verbalizer 并计算分类损失

每类有不同数量的标签词。用 logsumexp - log(标签词数) 作为类得分，计算交叉熵并验证梯度能回到词表 logits。

✅ 参考答案
Python
import math
import torch
import torch.nn.functional as F


def multi_verbalizer_logits(mask_logits: torch.Tensor, groups: list[list[int]]) -> torch.Tensor:
    if mask_logits.ndim != 2 or not groups:
        raise ValueError("输入形状或 groups 非法")
    vocab = mask_logits.size(1)
    used: list[int] = []
    class_scores = []
    for token_ids in groups:
        if not token_ids or any(i < 0 or i >= vocab for i in token_ids):
            raise ValueError("标签词 ID 非法")
        used.extend(token_ids)
        values = mask_logits[:, token_ids]
        class_scores.append(torch.logsumexp(values, dim=1) - math.log(len(token_ids)))
    if len(set(used)) != len(used):
        raise ValueError("类别之间的标签词不得重叠")
    return torch.stack(class_scores, dim=1)


mask_logits = torch.tensor(
    [[0.1, 2.0, 1.5, -0.2, -1.0, 0.0], [0.0, -1.0, -0.5, 1.2, 2.1, 0.3]],
    requires_grad=True,
)
groups = [[1, 2], [3, 4]]
targets = torch.tensor([0, 1])
class_logits = multi_verbalizer_logits(mask_logits, groups)
loss = F.cross_entropy(class_logits, targets)
loss.backward()
assert class_logits.argmax(dim=1).tolist() == targets.tolist()
assert mask_logits.grad is not None and mask_logits.grad.abs().sum() > 0
print(float(loss.detach()), class_logits)

49. 实现输入级 Soft Prompt 拼接

冻结词 embedding，在真实 token embedding 前拼接 (m) 个可训练向量，并同步扩展 attention mask。

✅ 参考答案
Python
import torch
from torch import nn


class SoftPrompt(nn.Module):
    def __init__(self, num_virtual_tokens: int, hidden_size: int):
        super().__init__()
        if num_virtual_tokens <= 0 or hidden_size <= 0:
            raise ValueError("维度必须为正")
        self.prompt = nn.Parameter(torch.empty(num_virtual_tokens, hidden_size))
        nn.init.normal_(self.prompt, mean=0.0, std=0.02)

    def forward(
        self, token_embeddings: torch.Tensor, attention_mask: torch.Tensor
    ) -> tuple[torch.Tensor, torch.Tensor]:
        if token_embeddings.ndim != 3 or attention_mask.shape != token_embeddings.shape[:2]:
            raise ValueError("输入形状不一致")
        batch = token_embeddings.size(0)
        prompt = self.prompt.unsqueeze(0).expand(batch, -1, -1)
        inputs = torch.cat([prompt, token_embeddings], dim=1)
        prompt_mask = torch.ones(
            batch, self.prompt.size(0), dtype=attention_mask.dtype, device=attention_mask.device
        )
        return inputs, torch.cat([prompt_mask, attention_mask], dim=1)


embedding = nn.Embedding(20, 6)
embedding.requires_grad_(False)
module = SoftPrompt(num_virtual_tokens=3, hidden_size=6)
ids = torch.tensor([[1, 2, 0], [3, 4, 5]])
mask = torch.tensor([[1, 1, 0], [1, 1, 1]])
inputs, extended_mask = module(embedding(ids), mask)
assert inputs.shape == (2, 6, 6)
assert extended_mask.tolist() == [[1, 1, 1, 1, 1, 0], [1, 1, 1, 1, 1, 1]]
print(inputs.shape, extended_mask)

        self.base.requires_grad_(False)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.base(x + self.prompt)


model = FrozenPromptClassifier()
trainable = [p for p in model.parameters() if p.requires_grad]

import torch
from torch import nn


def parameter_report(model: nn.Module) -> dict:
    total = sum(p.numel() for p in model.parameters())
    trainable = sum(p.numel() for p in model.parameters() if p.requires_grad)
    if total == 0:
        raise ValueError("模型没有参数")
    return {
        "total": total,
        "trainable": trainable,
        "trainable_percent": 100.0 * trainable / total,
        "trainable_fp32_bytes": trainable * 4,
    }


class ToyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.base = nn.Linear(128, 128, bias=False)
        self.prompt = nn.Parameter(torch.zeros(10, 128))
        self.base.requires_grad_(False)


report = parameter_report(ToyModel())

assert report["trainable"] == 10 * 128
assert report["trainable_fp32_bytes"] == 5120

52. 为 token-level 标签和因果 LM 标签补齐虚拟位置

在序列前增加 (m) 个虚拟 token 后，把原位置整体右移，并让虚拟位置的监督标签为 -100。

✅ 参考答案
Python
import torch


IGNORE_INDEX = -100


def shift_for_virtual_tokens(
    labels: torch.Tensor, positions: torch.Tensor, num_virtual_tokens: int
) -> tuple[torch.Tensor, torch.Tensor]:
    if labels.ndim != 2 or positions.ndim != 1 or positions.size(0) != labels.size(0):
        raise ValueError("输入形状非法")
    if num_virtual_tokens <= 0:
        raise ValueError("num_virtual_tokens 必须为正")
    if not torch.all((0 <= positions) & (positions < labels.size(1))):
        raise ValueError("positions 越界")
    prefix = torch.full(
        (labels.size(0), num_virtual_tokens),
        IGNORE_INDEX,
        dtype=labels.dtype,
        device=labels.device,
    )
    return torch.cat([prefix, labels], dim=1), positions + num_virtual_tokens


labels = torch.tensor([[10, 11, -100], [12, 13, 14]])
positions = torch.tensor([1, 2])
shifted_labels, shifted_positions = shift_for_virtual_tokens(labels, positions, 3)
assert shifted_labels.shape == (2, 6)
assert torch.all(shifted_labels[:, :3] == -100)
assert shifted_positions.tolist() == [4, 5]
assert torch.equal(shifted_labels[:, 3:], labels)
print(shifted_labels, shifted_positions)

53. 只保存并恢复 Prompt 权重与兼容性配置

使用内存缓冲区模拟检查点；加载时必须校验基座 revision、Prompt 形状和配置。

✅ 参考答案
Python
import io
import torch
from torch import nn


def save_prompt(prompt: nn.Parameter, config: dict) -> bytes:
    required = {"base_model", "revision", "num_virtual_tokens", "hidden_size"}
    if set(config) != required:
        raise ValueError("配置字段不完整")
    expected = (config["num_virtual_tokens"], config["hidden_size"])
    if tuple(prompt.shape) != expected:
        raise ValueError("Prompt 形状与配置不符")
    buffer = io.BytesIO()
    torch.save({"prompt": prompt.detach().cpu(), "config": dict(config)}, buffer)
    return buffer.getvalue()


def load_prompt(blob: bytes, expected_config: dict) -> nn.Parameter:
    checkpoint = torch.load(io.BytesIO(blob), weights_only=True)
    if checkpoint["config"] != expected_config:
        raise ValueError("基座或 Prompt 配置不兼容")
    tensor = checkpoint["prompt"]
    shape = (expected_config["num_virtual_tokens"], expected_config["hidden_size"])
    if tuple(tensor.shape) != shape:
        raise ValueError("检查点形状损坏")
    return nn.Parameter(tensor.clone())


config = {
    "base_model": "toy-encoder",
    "revision": "commit-abc123",
    "num_virtual_tokens": 4,
    "hidden_size": 6,
}
original = nn.Parameter(torch.arange(24, dtype=torch.float32).reshape(4, 6))
blob = save_prompt(original, config)
restored = load_prompt(blob, config)
assert torch.equal(original, restored)
assert restored.requires_grad
print("checkpoint bytes:", len(blob))

54. 构造带 Soft Prompt 的变长批次

原始 token 在右侧 padding；拼接虚拟 token 后，保证每条样本的有效长度增加 (m)，且序列不超过预算。

✅ 参考答案
Python
import torch


def collate_with_prompt(
    sequences: list[list[int]], num_virtual_tokens: int, max_total_length: int, pad_id: int = 0
) -> dict:
    if not sequences or any(not seq for seq in sequences):
        raise ValueError("序列不能为空")
    if num_virtual_tokens <= 0 or max_total_length <= num_virtual_tokens:
        raise ValueError("长度预算非法")
    max_text_length = max_total_length - num_virtual_tokens
    clipped = [seq[:max_text_length] for seq in sequences]
    batch_text_length = max(len(seq) for seq in clipped)
    input_ids, attention_mask = [], []
    for seq in clipped:
        padding = batch_text_length - len(seq)
        input_ids.append(seq + [pad_id] * padding)
        attention_mask.append([1] * num_virtual_tokens + [1] * len(seq) + [0] * padding)
    return {
        "input_ids": torch.tensor(input_ids),
        "attention_mask": torch.tensor(attention_mask),
        "effective_lengths": torch.tensor([num_virtual_tokens + len(seq) for seq in clipped]),
    }


batch = collate_with_prompt([[4, 5, 6], [7, 8, 9, 10, 11, 12]], 2, max_total_length=7)
assert batch["input_ids"].shape == (2, 5)
assert batch["attention_mask"].shape == (2, 7)
assert batch["effective_lengths"].tolist() == [5, 7]
assert batch["attention_mask"].sum(dim=1).tolist() == [5, 7]
print(batch)

55. 完成一个最小 Soft Prompt 训练闭环

冻结一个人工构造的“基座”，只学习两个 Prompt 标量，使四个样本被正确分类；验证基座参数完全不变。

✅ 参考答案
Python
import torch
from torch import nn


torch.manual_seed(7)


class TinySoftPromptClassifier(nn.Module):
    def __init__(self):
        super().__init__()
        # 冻结基座把二维输入映射为两类 logits。
        self.base = nn.Linear(2, 2, bias=False)
        with torch.no_grad():
            self.base.weight.copy_(torch.tensor([[1.0, 0.0], [0.0, 1.0]]))
        self.base.requires_grad_(False)
        self.soft_prompt = nn.Parameter(torch.zeros(2))

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.base(x + self.soft_prompt)


# 所有样本都需要同一个任务 Prompt：第一维下调、第二维上调后应归为类别 1。
x = torch.tensor([[1.0, 0.0], [0.8, 0.2], [0.5, 0.1], [0.2, 0.0]])
y = torch.ones(4, dtype=torch.long)
model = TinySoftPromptClassifier()
base_before = model.base.weight.detach().clone()
optimizer = torch.optim.SGD([model.soft_prompt], lr=0.2)

for _ in range(120):
    optimizer.zero_grad()
    loss = nn.CrossEntropyLoss()(model(x), y)
    loss.backward()
    assert model.base.weight.grad is None
    optimizer.step()

predictions = model(x).argmax(dim=1)
assert predictions.tolist() == y.tolist()
assert torch.equal(model.base.weight, base_before)
assert model.soft_prompt[1] > model.soft_prompt[0]
print({"prompt": model.soft_prompt.detach().tolist(), "loss": float(loss.detach()), "pred": predictions.tolist()})

🤗 Transformers 迁移学习与中文任务综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：Transformers 迁移学习与中文任务

55 道练习，按“任务与 checkpoint 选型 → Tokenizer → 动态 batch → AutoModel 任务头 → 分类/NER/问答/MLM → 微调评估 → 保存部署”组织。重点是把预训练模型迁移为无数据泄漏、标签对齐正确、指标可信且可复现的中文 NLP 系统。

建议先独立作答，再展开参考答案。练习按 Hugging Face Padding/Truncation、Token Classification 与 Question Answering 官方文档校准；10 道代码题使用纯 Python/PyTorch，不下载外部 checkpoint。

一、选择题（30 题）

1. NLP 迁移学习的基本思想是？

A. 每个任务都从随机参数开始训练
B. 复用大规模语料预训练得到的表示，再适配下游任务
C. 将测试集标签加入训练
D. 只改变输入文件名

✅ 参考答案

答案：B。 预训练学习通用语言模式，下游阶段添加或替换任务头并冻结部分参数或联合微调；与从零训练相比，通常需要更少标注数据和训练预算。

2. “特征抽取”和“全量微调”的主要区别是？

A. 前者通常冻结 Backbone，只训练下游头；后者更新 Backbone 参数
B. 前者不需要输入文本
C. 后者不能使用预训练权重
D. 二者都只更新标签名称

✅ 参考答案

答案：A。 还存在逐层解冻、差分学习率和参数高效微调等中间方案；应根据数据规模、领域差异、算力和过拟合风险比较。

3. pipeline() 最适合什么场景？

A. 快速推理原型与验证任务接口
B. 任意复杂训练循环的唯一实现方式
C. 数据库迁移
D. 修改模型结构而不加载权重

✅ 参考答案

答案：A。 Pipeline 封装 tokenizer、模型与后处理，适合快速试用；生产环境仍应固定模型、revision、设备、dtype、batch 和后处理规则，避免隐式默认变化。

4. 为什么实际项目常优先使用 Auto 类？

A. Auto 类根据 checkpoint 配置选择匹配的 tokenizer 与模型实现
B. Auto 类不需要配置文件
C. Auto 类会自动生成训练标签
D. 只能加载 BERT

✅ 参考答案

答案：A。 AutoModelForSequenceClassification、AutoModelForTokenClassification 等在通用性与任务适配之间平衡；具体模型类适合需要明确架构特性的场景。

5. 文本分类应优先选择哪种任务模型？

A. AutoModelForSequenceClassification
B. AutoModelForQuestionAnswering
C. AutoModelForMaskedLM
D. AutoModelForTokenClassification

✅ 参考答案

答案：A。 它在 Backbone 上带句级分类头，常输出 [B,C] logits；num_labels、id2label 与 label2id 应和数据标签体系一致。

6. AutoModel 与 AutoModelForSequenceClassification 的区别是？

A. 前者主要返回 Backbone 隐藏表示，后者额外包含分类任务头
B. 前者只能处理图片
C. 后者不含预训练模型
D. 二者输出必然完全相同

✅ 参考答案

答案：A。 基础模型常返回 last_hidden_state=[B,S,H]；任务模型将适当表示映射到标签空间，并可在传入 labels 时直接计算任务损失。

7. 为什么 tokenizer 与 checkpoint 必须匹配？

A. 词表、特殊 token、分词算法和输入约定共同决定 Embedding ID 语义
B. 只为了文件名一致
C. 模型会忽略 input_ids
D. 任意 tokenizer 的 ID 都等价

✅ 参考答案

答案：A。 错配可能不会立刻报错，却会把 token 映射到错误向量；应从同一 checkpoint/revision 加载并一同版本化。

8. 能否对任意模型硬编码 [CLS]=101、[SEP]=102？

A. 可以，所有 tokenizer 都相同
B. 不可以，应读取 tokenizer 的特殊 token 与 ID
C. 只有训练时不可以
D. ID 与模型无关

✅ 参考答案

答案：B。 这些 ID 对特定 BERT 词表常成立，但 RoBERTa、T5、Qwen 等使用不同边界符与编号；通用代码必须查询 tokenizer 配置。

9. attention_mask 中常见的 1 与 0 分别表示？

A. 1 为有效 token，0 为 Padding 位置
B. 1 为负样本，0 为正样本
C. 1 为测试集，0 为训练集
D. 仅表示类别 ID

✅ 参考答案

答案：A。 模型用它阻止有效 token 关注 PAD；自己做池化时也必须应用同一 Mask，不能把 PAD 隐藏状态平均进去。

10. token_type_ids 的正确使用方式是？

A. 假设任何模型都必须提供
B. 使用 tokenizer 实际返回且模型支持的字段，不存在时不要硬造
C. 始终全部设为 1
D. 用它替代 attention_mask

✅ 参考答案

答案：B。 BERT 句对常使用 segment ID，但 RoBERTa 等模型可能不需要；通用代码可将 tokenizer 输出字典直接按模型签名传入。

11. 训练时对整个数据集统一 padding='max_length' 的潜在问题是？

A. 大量短句被补到全局最大长度，浪费显存与计算
B. 会删除全部标签
C. 不能产生 attention_mask
D. 一定提高精度

✅ 参考答案

答案：A。 常见做法是先只截断，在 collator 中按当前 batch 最长序列动态 Padding；固定长度适合静态编译或服务接口，但要量化效率代价。

12. padding=True 或 'longest' 表示？

A. 补齐到当前 batch 最长序列
B. 永远补到 512
C. 不做 Padding
D. 删除最长样本

✅ 参考答案

答案：A。 单条输入时可能不补齐；padding='max_length' 才按显式或模型最大长度补齐。

13. 句对任务使用 truncation='only_second' 表示？

A. 只截断第二段文本
B. 只保留第二段文本
C. 只截断标签
D. 两段都不截断

✅ 参考答案

答案：A。 抽取式问答常把 question 放第一段、context 放第二段，从而只截断长 context；其他句对任务可按业务选择 longest_first。

14. pooler_output 是否是所有 Transformer 模型都保证提供的通用句向量？

A. 是
B. 否；它可能不存在或具有特定预训练语义，可改用 CLS/末 token 或 Masked Mean Pooling
C. 只有 Padding 时不存在
D. 它总等于 logits

✅ 参考答案

答案：B。 池化策略必须与模型架构和任务匹配；做语义相似度时还应使用为句向量目标训练的模型，而不是盲目平均任意 Backbone。

15. Sequence Classification logits 的常见形状是？

A. [B,C]
B. [B,S,C]
C. [B,S,V]
D. [C,H,S]

✅ 参考答案

答案：A。 单标签分类使用交叉熵；多标签任务常使用每类独立的 BCEWithLogitsLoss 与验证集阈值。

16. Token Classification logits 的常见形状是？

A. [B,C]
B. [B,S,C]
C. [B,V]
D. [S,S]

✅ 参考答案

答案：B。 每个 token 位置预测一个标签；子词拆分、特殊 token 与 Padding 会让 token 轴和原词标签轴不再一一对应。

17. NER 中原词被拆成多个子词后，官方示例的常见标签策略是？

A. 用 word_ids() 对齐，只监督每个词的首个子词，其他子词和特殊 token 设为 -100
B. 给所有位置随机标签
C. 删除原词标签
D. 把 [CLS] 标为实体

✅ 参考答案

答案：A。 -100 会被 PyTorch 交叉熵忽略；也可设计“标签传播到全部子词”的策略，但必须处理 BIO 转换并在训练和评估中保持一致。

18. 抽取式问答模型通常输出？

A. start_logits=[B,S] 与 end_logits=[B,S]
B. 单个类别 ID
C. 一张图片
D. 只有句向量

✅ 参考答案

答案：A。 后处理需限制候选位于 context、排除特殊 token，并满足 start<=end 与最大答案长度；分别 Argmax 可能得到非法或问题区域的 span。

19. 问答预处理为什么需要 return_offset_mapping=True？

A. 将原文字符区间与 token 区间对应
B. 计算 GPU 数量
C. 自动翻译答案
D. 替代 labels

✅ 参考答案

答案：A。 配合 sequence_ids() 可判断 token 属于 question 还是 context，并把答案字符起止位置转换为训练用 token 起止位置。

20. 长 context 超过最大长度时，可靠的问答方案是？

A. 直接静默删除后半部分且不记录
B. 使用 overflow window、stride 与样本映射，并在后处理中合并候选
C. 截断问题和答案标签后仍当正确样本
D. 只增大 batch size

✅ 参考答案

答案：B。 每个原样本可能生成多个重叠 feature；必须保存 overflow_to_sample_mapping，并处理答案不在当前窗口的情况。

21. Masked Language Modeling 的 labels 通常如何设置？

A. 只在被 Mask 的目标位置保留原 token ID，其余位置设为 -100
B. 所有位置都设为 PAD
C. 每条句子只预测类别
D. 使用测试集标签

✅ 参考答案

答案：A。 Mask 采样应排除 PAD 和特殊 token；标准 BERT 式策略还可能按比例使用 [MASK]、随机 token 或保持原 token，避免预训练—下游不一致。

22. Seq2Seq 文本摘要或翻译推理为什么应使用 model.generate()？

A. 它负责自回归循环、停止条件和 Beam 等解码策略
B. 一次前向 logits Argmax 总能产生正确完整序列
C. generate 只用于分类
D. 可以跳过 tokenizer

✅ 参考答案

答案：A。 直接对一次 teacher-forced forward 的 [B,T,V] logits Argmax 不等于真实生成；需配置最大长度、EOS、Beam/采样和重复控制。

23. 微调预训练 Backbone 常用较小学习率的原因是？

A. 避免快速破坏已学表示并降低不稳定更新
B. 预训练参数不参与梯度
C. 学习率越小一定越好
D. 分类头不能训练

✅ 参考答案

答案：A。 可为新任务头使用较大学习率、Backbone 使用较小学习率；最佳值仍由验证集与训练稳定性决定。

24. AdamW 中通常不对哪些参数施加权重衰减？

A. Bias 与 LayerNorm 等规范化参数
B. 全部权重
C. 只有分类头矩阵
D. input_ids

✅ 参考答案

答案：A。 应按参数名称或模块类型分组，并确保每个可训练参数恰好出现一次；错误分组会导致漏训练或重复更新。

25. Warmup 的主要作用是？

A. 训练初期逐步提高学习率，缓解随机任务头与预训练参数的剧烈更新
B. 自动增加标签
C. 永久冻结 Backbone
D. 替代验证集

✅ 参考答案

答案：A。 Warmup 后常接线性或余弦衰减；调度步数要基于优化器更新次数，而非简单的数据 batch 数，尤其在梯度累积时。

26. 梯度累积为 K 步时，通常为什么要将每个 micro-batch loss 除以 K？

A. 使累计梯度尺度接近大 batch 平均损失
B. 防止模型产生 logits
C. 把类别数缩小 K 倍
D. 让测试集参与训练

✅ 参考答案

32. 如何在 Pipeline、Auto 类和具体模型类之间选择？

✅ 参考答案

34. 为什么自己做句向量池化时必须使用 attention_mask？

36. 写出可靠的分类训练循环要点。

47. 实现一个动态 Padding collator，生成 input_ids、attention_mask 和标签。

✅ 参考答案

只补到当前 batch 最长长度；pad_to_multiple_of 可用于某些硬件的张量对齐。

Python
import math
import torch

def dynamic_collate(features, pad_token_id=0, pad_to_multiple_of=None):
    max_length = max(len(item["input_ids"]) for item in features)
    if pad_to_multiple_of:
        max_length = math.ceil(max_length / pad_to_multiple_of) * pad_to_multiple_of
    input_ids, attention_mask, labels = [], [], []
    for item in features:
        length = len(item["input_ids"])
        padding = max_length - length
        input_ids.append(item["input_ids"] + [pad_token_id] * padding)
        attention_mask.append([1] * length + [0] * padding)
        labels.append(item["label"])
    return {
        "input_ids": torch.tensor(input_ids),
        "attention_mask": torch.tensor(attention_mask),
        "labels": torch.tensor(labels),
    }

batch = dynamic_collate([
    {"input_ids": [101, 11, 12, 102], "label": 1},
    {"input_ids": [101, 21, 102], "label": 0},
], pad_to_multiple_of=4)

assert batch["input_ids"].shape == (2, 4)
assert batch["input_ids"][1].tolist() == [101, 21, 102, 0]
assert batch["attention_mask"][1].tolist() == [1, 1, 1, 0]
assert batch["labels"].tolist() == [1, 0]
print(batch)

48. 实现 Masked Mean Pooling，并证明 Padding 隐藏状态不会污染句向量。

✅ 参考答案

将 Mask 扩展到隐藏维度后求和，分母是有效 token 数而不是固定序列宽度。

import torch

def masked_mean_pool(last_hidden_state, attention_mask):
    mask = attention_mask.unsqueeze(-1).to(last_hidden_state.dtype)
    summed = (last_hidden_state * mask).sum(dim=1)
    counts = mask.sum(dim=1).clamp_min(1.0)
    return summed / counts

hidden = torch.tensor([
    [[1., 2.], [3., 4.], [100., 100.]],
    [[2., 0.], [4., 2.], [6., 4.]],
])
mask = torch.tensor([[1, 1, 0], [1, 1, 1]])
pooled = masked_mean_pool(hidden, mask)

assert torch.allclose(pooled[0], torch.tensor([2., 3.]))
assert torch.allclose(pooled[1], torch.tensor([4., 2.]))
assert not torch.allclose(hidden[0].mean(dim=0), pooled[0])
print(pooled)

49. 实现多分类 Accuracy 与 Macro-F1，验证长尾类别不会被总体准确率掩盖。

def classification_metrics(predictions, labels, num_classes):
    accuracy = sum(p == y for p, y in zip(predictions, labels)) / len(labels)
    f1_scores = []
    for cls in range(num_classes):
        tp = sum(p == cls and y == cls for p, y in zip(predictions, labels))
        fp = sum(p == cls and y != cls for p, y in zip(predictions, labels))
        fn = sum(p != cls and y == cls for p, y in zip(predictions, labels))
        precision = tp / (tp + fp) if tp + fp else 0.0
        recall = tp / (tp + fn) if tp + fn else 0.0
        f1 = 2 * precision * recall / (precision + recall) if precision + recall else 0.0
        f1_scores.append(f1)
    return accuracy, sum(f1_scores) / num_classes, f1_scores

50. 冻结/解冻 Backbone，并为 AdamW 构造“差分学习率 + Bias/Norm 不衰减”的参数组。

✅ 参考答案

分组后要断言每个可训练参数恰好出现一次，避免重复更新或漏参数。

Python
from torch import nn

class TinyTransferModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.backbone = nn.Sequential(nn.Linear(4, 4), nn.LayerNorm(4))
        self.classifier = nn.Linear(4, 3)

def set_backbone_trainable(model, trainable):
    for parameter in model.backbone.parameters():
        parameter.requires_grad = trainable

def adamw_groups(model, backbone_lr=1e-5, head_lr=1e-4, weight_decay=0.01):
    groups = []
    for prefix, module, lr in [
        ("backbone", model.backbone, backbone_lr),
        ("classifier", model.classifier, head_lr),
    ]:
        norm_parameter_ids = {
            id(parameter)
            for child in module.modules()
            if isinstance(child, nn.LayerNorm)
            for parameter in child.parameters(recurse=False)
        }
        decay, no_decay = [], []
        for name, parameter in module.named_parameters():
            if not parameter.requires_grad:
                continue
            target = no_decay if name.endswith("bias") or id(parameter) in norm_parameter_ids else decay
            target.append(parameter)
        if decay:
            groups.append({"params": decay, "lr": lr, "weight_decay": weight_decay})
        if no_decay:
            groups.append({"params": no_decay, "lr": lr, "weight_decay": 0.0})
    return groups

model = TinyTransferModel()
set_backbone_trainable(model, False)
assert not any(parameter.requires_grad for parameter in model.backbone.parameters())
assert all(parameter.requires_grad for parameter in model.classifier.parameters())

set_backbone_trainable(model, True)
groups = adamw_groups(model)
grouped_ids = [id(parameter) for group in groups for parameter in group["params"]]
trainable_ids = [id(parameter) for parameter in model.parameters() if parameter.requires_grad]
assert sorted(grouped_ids) == sorted(trainable_ids)
assert len(grouped_ids) == len(set(grouped_ids))
assert {group["lr"] for group in groups} == {1e-5, 1e-4}
no_decay_ids = {
    id(parameter)
    for group in groups if group["weight_decay"] == 0.0
    for parameter in group["params"]
}
layer_norm_ids = {
    id(parameter)
    for module in model.modules() if isinstance(module, nn.LayerNorm)
    for parameter in module.parameters(recurse=False)
}
assert layer_norm_ids <= no_decay_ids
print([(group["lr"], group["weight_decay"], len(group["params"])) for group in groups])

IGNORE_INDEX = -100

def align_word_labels(word_ids, word_labels):
    aligned = []
    previous_word = None
    for word_id in word_ids:
        if word_id is None:
            aligned.append(IGNORE_INDEX)
        elif word_id != previous_word:
            aligned.append(word_labels[word_id])

        previous_word = word_id
    return aligned

52. 将问答答案的字符区间映射到 context token 区间，并处理答案不在窗口的情况。

def answer_char_to_tokens(offsets, sequence_ids, answer_start, answer_text):
    answer_end = answer_start + len(answer_text)
    context_indices = [i for i, sid in enumerate(sequence_ids) if sid == 1]
    if not context_indices:
        return 0, 0
    context_start, context_end = context_indices[0], context_indices[-1]
    if offsets[context_start][0] > answer_start or offsets[context_end][1] < answer_end:
        return 0, 0

    start_token = next(
        i for i in context_indices if offsets[i][0] <= answer_start < offsets[i][1]
    )
    end_token = next(
        i for i in reversed(context_indices) if offsets[i][0] < answer_end <= offsets[i][1]
    )
    return start_token, end_token

53. 构造 MLM 输入与 labels：只 Mask 普通 token，其余标签设为 -100。

import torch

def prepare_mlm(input_ids, special_tokens_mask, mask_token_id, probability, generator=None):
    labels = input_ids.clone()
    selectable = ~special_tokens_mask.bool()
    selected = (torch.rand(input_ids.shape, generator=generator) < probability) & selectable
    labels[~selected] = -100
    masked_input = input_ids.clone()
    masked_input[selected] = mask_token_id
    return masked_input, labels, selected

input_ids = torch.tensor([[101, 11, 12, 102, 0]])
special_mask = torch.tensor([[1, 0, 0, 1, 1]], dtype=torch.bool)

    generator=torch.Generator().manual_seed(0),
)

assert masked.tolist() == [[101, 103, 103, 102, 0]]
assert labels.tolist() == [[-100, 11, 12, -100, -100]]
assert not selected.masked_select(special_mask).any()

55. 训练一个带 Padding Mask 的微型 Transformer 中文分类器，并验证它能过拟合小数据。

✅ 参考答案

这是下游训练管线的单元测试：若固定小 batch 都学不会，应先排查标签、Mask、池化、loss 和优化器。

Python
import torch
from torch import nn

torch.manual_seed(5)
torch.set_num_threads(1)
PAD = 0

class TinyTransformerClassifier(nn.Module):
    def __init__(self, vocab_size=16, hidden_size=12, num_classes=2):
        super().__init__()
        self.embedding = nn.Embedding(vocab_size, hidden_size, padding_idx=PAD)
        layer = nn.TransformerEncoderLayer(
            d_model=hidden_size, nhead=3, dim_feedforward=24,
            dropout=0.0, batch_first=True,
        )
        self.encoder = nn.TransformerEncoder(
            layer, num_layers=1, enable_nested_tensor=False
        )
        self.classifier = nn.Linear(hidden_size, num_classes)

    def forward(self, input_ids, attention_mask):
        hidden = self.encoder(
            self.embedding(input_ids),
            src_key_padding_mask=attention_mask.eq(0),
        )
        mask = attention_mask.unsqueeze(-1).to(hidden.dtype)
        pooled = (hidden * mask).sum(dim=1) / mask.sum(dim=1).clamp_min(1)
        return self.classifier(pooled)

input_ids = torch.tensor([
    [1, 3, 4, 2, 0], [1, 3, 7, 2, 0], [1, 4, 7, 2, 0],
    [1, 5, 6, 2, 0], [1, 5, 8, 2, 0], [1, 6, 8, 2, 0],
])
attention_mask = input_ids.ne(PAD).long()
labels = torch.tensor([1, 1, 1, 0, 0, 0])

model = TinyTransformerClassifier()
optimizer = torch.optim.AdamW(model.parameters(), lr=0.05)
criterion = nn.CrossEntropyLoss()

for _ in range(150):
    optimizer.zero_grad()
    logits = model(input_ids, attention_mask)
    loss = criterion(logits, labels)
    loss.backward()
    nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()

model.eval()
with torch.no_grad():
    predictions = model(input_ids, attention_mask).argmax(dim=-1)

assert torch.equal(predictions, labels)
assert loss.item() < 0.01
print({"loss": round(loss.item(), 6), "accuracy": 1.0})

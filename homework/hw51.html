🕸️ 知识图谱基础与医学图谱架构综合训练

🏷️ 选择 + 简答 + 代码🏷️ 55 题📖 相关讲义：知识图谱基础与医学图谱架构

55 道练习，按“实体、概念、属性、关系与三元组 → RDF 与属性图 → Schema/本体与实例数据 → 自顶向下/自底向上建图 → 医学图谱技术链路 → 溯源、时态、质量与安全边界”组织。重点不是把数据画成网络图，而是建立语义明确、身份稳定、来源可查、时间可解释、质量可验收的知识系统。

建议先独立作答，再展开参考答案。本练习依据课程《项目简介》《知识图谱技术概括》与站内重组讲义设计，并参考 W3C RDF 1.1 Concepts、OWL 2 Primer、PROV-O 和 SHACL校准。注意：RDF/OWL 的开放世界语义中“图里没有”通常不等于“事实为假”；Schema/本体用于表达语义与约束，但数据是否满足业务完整性仍需专门校验；医学图谱只能支撑知识检索与辅助分析，不能把一条图路径直接当作诊断、处方或个体化用药结论。10 道代码题均使用 Python 标准库离线运行。

一、选择题（30 题）

1. 知识图谱最核心的价值是？

A. 用稳定、可查询的语义结构组织实体及其联系
B. 生成一张颜色丰富的图片
C. 把所有文本删除
D. 让任何查询都自动正确

✅ 参考答案

答案：A。 可视化只是应用之一。真正的核心是实体身份、关系语义、Schema、来源、时间和查询规则，使知识可以被机器一致处理与追溯。

2. “37.8”与“某患者某时刻体温为 37.8℃”的区别是？

A. 前者是缺少语境的数据值，后者补充主体、属性、单位和时间后形成可解释信息
B. 二者语义完全相同
C. 前者一定是假数据
D. 后者不需要来源

✅ 参考答案

答案：A。 单独数值无法判断含义。进入知识系统还需明确测量来源、时间、单位、对象与隐私边界，不能只存裸值。

3. 下列哪项更适合作为实体？

A. 具有稳定身份的疾病概念“慢性支气管炎”
B. 任意一个逗号
C. 查询执行耗时 12ms 的日志颜色
D. HTML 的空白缩进

✅ 参考答案

答案：A。 实体是可独立识别和引用的对象或概念。是否建成实体还取决于查询需求；简单字符串也可只作为属性值。

4. “疾病—HAS_SYMPTOM→症状”中的 HAS_SYMPTOM 是？

A. 有方向、有语义的关系类型
B. 疾病实体的唯一 ID
C. 症状名称的别名
D. 图的节点数量

✅ 参考答案

答案：A。 关系方向属于 Schema。反向查询“哪些疾病具有该症状”可以沿入边查询，或显式定义语义清楚的逆关系，不能假设所有边天然无向。

5. “疾病的标准名、更新时间、简介”通常更适合建模为？

A. 实体属性；若值本身需要独立身份和关系，也可提升为实体
B. 三个完全相同的关系
C. 图数据库账号
D. 无法存储的信息

✅ 参考答案

答案：A。 属性与实体不是由字符串形式决定，而由复用、查询、约束和溯源需求决定。例如“更新时间”常是属性，“发布机构”若需连接多个文档则更适合作为实体。

6. 为什么实体 ID 不应直接等于展示名称？

A. 名称会变化、存在别名和同名异义，而稳定 ID 应保持不变
B. ID 必须是随机 emoji
C. 名称永远唯一
D. 图谱不能保存中文

✅ 参考答案

答案：A。 应用标准术语编码或受控命名空间 ID；名称、简称、旧名作为属性。改名不应让所有边断开，同名实体也不能被误合并。

7. RDF 三元组由什么组成？

A. Subject、Predicate、Object
B. Row、Column、Index
C. Query、Key、Value
D. Train、Dev、Test

✅ 参考答案

答案：A。 RDF 图是三元组集合。Predicate 是 IRI；Subject 不能是普通 literal；Object 可以是 IRI、blank node 或 literal。课程中的“主体—关系—客体”是直观对应。

8. RDF 与属性图的典型区别是？

A. RDF 以三元组和全局 IRI 为基础，属性图通常让节点/关系直接带 label 与 property
B. RDF 没有任何边
C. 属性图不能表示关系
D. 两者只能存图片

✅ 参考答案

答案：A。 二者都能表达图结构，但数据模型、语义标准、约束和查询生态不同。选型应依据互操作、推理、性能和团队工具链，而非认为一方普遍优于另一方。

9. Schema 回答的问题是？

A. 图中允许有哪些实体类型、关系、方向、属性和约束
B. 当前数据库恰好有多少条具体事实
C. 每个网页的字体颜色
D. 模型训练了多少 epoch

✅ 参考答案

答案：A。 实例数据回答“当前有什么”。例如 Schema 定义 Disease→HAS_SYMPTOM→Symptom，实例才是“慢性支气管炎→咳嗽”。

10. Relation 的 domain/range 为 Disease→Symptom，但导入了 Drug→HAS_SYMPTOM→Food，应该？

A. 判为类型约束违规并隔离或修正，不应静默入图
B. 自动把 Drug 改名为 Disease
C. 忽略 Schema
D. 只要字符串非空就接受

✅ 参考答案

答案：A。 domain/range 是关系语义的一部分。自动强改类型会掩盖抽取或映射错误，应保留原记录、错误原因和处理状态。

11. 疾病可以具有多个症状，这体现了？

A. HAS_SYMPTOM 的客体侧允许多值
B. 所有实体必须同名
C. 关系只能出现一次
D. 图必须无环

✅ 参考答案

答案：A。 Cardinality 需按业务定义。多值不等于允许完全重复的同一事实；同一三元组多来源应合并声明并保留各自证据。

12. 在 OWL 的开放世界语义中，图里没有“某病 HAS_SYMPTOM 发热”通常表示？

A. 当前知识未知或未声明，不能仅据此推出“该病一定不发热”
B. 已证明该病绝不发热
C. 数据库损坏
D. 关系必须自动补齐

✅ 参考答案

答案：A。 应用若要做闭集完整性判断，必须额外声明数据范围和完整性规则。明确的否定事实也应有独立语义与证据，不能用缺失代替。

13. “Disease 是类，慢性支气管炎是该类中的一个概念实例”主要区分？

A. 模式层的类型与数据层的个体
B. 训练集和测试集
C. 节点颜色和边宽
D. CPU 与 GPU

✅ 参考答案

答案：A。 实际医学本体中概念层次可能更复杂，但建模必须明确何者是 class、concept/individual 以及它们的 IS_A 或 instance-of 关系。

14. 自顶向下建图的典型步骤是？

A. 专家先定义核心 Schema 和术语，再按约束接入实例数据
B. 先抓取全部网页且不定义类型
C. 只按字符串长度建边
D. 从测试答案反推 Schema

✅ 参考答案

答案：A。 它适合安全与一致性要求高的核心医学知识，但前期专家成本较高，也要通过版本迭代适应新需求。

15. 自底向上建图的主要风险是？

A. 从数据归纳模式时容易产生冗余、歧义、错误关系和长尾噪声
B. 无法发现任何新概念
C. 一定不需要信息抽取
D. 天然满足所有临床规范

✅ 参考答案

答案：A。 优势是扩展快、便于发现候选概念；候选知识仍需归一、约束验证、质量评估和人工审核。

16. 医学项目更常采用混合路线的原因是？

A. 用自顶向下保证核心语义，用自底向上发现长尾候选，再经审核纳入
B. 同时关闭两种方法
C. 让 Schema 每次查询都随机变化
D. 不再需要专家

✅ 参考答案

答案：A。 混合路线把稳定治理与数据驱动扩展结合。候选关系不能绕过版本、来源和验收流程直接成为可信知识。

17. 哪个数据源更适合直接作为高可信医学事实，仍需版本和许可核验？

A. 经授权的权威指南或标准术语库
B. 无作者、无日期的匿名帖子
C. 模型凭空生成的一句话
D. 随机文件名

✅ 参考答案

答案：A。 “权威”也不是永久正确，仍需记录版本、生效范围、发布日期和适用人群。网络文本可用于候选发现，但不能与审定知识混为一层。

18. JSON 病种记录、HTML 表格和医学论文正文通常分别属于？

A. 结构化/半结构化表示、半结构化数据、非结构化文本
B. 全部必然是同一种数据
C. 全部是图数据库
D. 全部不需要解析

✅ 参考答案

答案：A。 分类会因具体格式而异。不同来源需分别设计字段映射、DOM/表格解析或 NLP 抽取，并统一进入候选知识模型。

19. 从文本中识别“咳嗽”为 Symptom 并抽取 Disease→HAS_SYMPTOM→Symptom 属于？

A. 信息抽取阶段
B. 知识存储备份
C. 用户权限管理
D. 图可视化配色

✅ 参考答案

答案：A。 得到的只是候选知识，还需实体链接、融合、证据校验、冲突处理和 Schema 验证后才能入可信图层。

20. 把“慢支”“慢性支气管炎”链接到同一标准概念属于？

A. 实体统一/链接与知识融合
B. 分词长度统计
C. 删除全部别名
D. 关系方向反转

✅ 参考答案

答案：A。 归一必须在类型、上下文和标准术语约束下进行。同字符串也可能指不同实体，不同字符串也可能是同义词。

21. 两个可信来源对同一医学关系给出冲突结论，合理做法是？

A. 保留各自来源、版本、时间和适用条件，按规则标记冲突并交由审核
B. 后导入的永远覆盖且不留记录
C. 随机删除一个来源
D. 把冲突关系都改成同义词

✅ 参考答案

答案：A。 冲突可能来自指南版本、人群、剂量或语境差异。系统应支持多声明与审计，不能用单一布尔值抹平差异。

22. 一条医学事实最少应保留哪些溯源信息？

A. 来源标识、证据位置/文本、发布时间或版本、获取/抽取方式和处理时间
B. 只保留置信度
C. 只保留网页颜色
D. 只保留实体名称

✅ 参考答案

答案：A。 W3C PROV-O 用 Entity、Activity、Agent 等概念表达溯源。项目可以简化字段，但必须能回答“来自哪里、何时、怎样得到、谁负责”。

23. 为什么医学关系需要 valid_from/valid_to 或版本时间？

A. 指南、药品信息和疾病分类会更新，同一断言在不同时段可能状态不同
B. 为了增加字符串长度
C. 时间可以替代来源
D. 所有事实永久不变

✅ 参考答案

答案：A。 查询应声明 as-of 时间，并区分“知识生效时间、来源发布时间、系统入库时间”。三者语义不同。

24. 抽取模型给关系 0.98 置信度，意味着？

A. 这是模型分数或校准概率的候选信号，不自动等于医学事实为真
B. 已获得临床认证
C. 可以删除证据
D. 不再需要 Schema

✅ 参考答案

SCHEMA = {
    "HAS_SYMPTOM": ("Disease", "Symptom"),
    "RECOMMEND_DRUG": ("Disease", "Drug"),
    "DO_EAT": ("Disease", "Food"),
    "NOT_EAT": ("Disease", "Food"),
}


def validate_graph(entities: dict[str, dict], triples: list[tuple[str, str, str]]) -> list[str]:
    errors = []
    for entity_id, entity in entities.items():
        if set(entity) != {"type", "name"} or not entity["name"]:
            errors.append(f"entity {entity_id}: 结构非法")
    for index, (subject, predicate, object_) in enumerate(triples):
        if predicate not in SCHEMA:
            errors.append(f"triple {index}: 未知关系 {predicate}")
            continue
        if subject not in entities or object_ not in entities:
            errors.append(f"triple {index}: 端点不存在")
            continue
        expected_subject, expected_object = SCHEMA[predicate]
        actual = (entities[subject]["type"], entities[object_]["type"])
        if actual != (expected_subject, expected_object):
            errors.append(
                f"triple {index}: 期望 {expected_subject}->{expected_object}，实际 {actual[0]}->{actual[1]}"
            )
    return errors

    "s:1": {"type": "Symptom", "name": "咳嗽"},

}

47. 使用类型内别名表做保守实体归一

import re
import unicodedata
from typing import Optional


def normalize_surface(text: str) -> str:
    text = unicodedata.normalize("NFKC", text).strip().lower()
    return re.sub(r"\s+", "", text)


def build_alias_index(rows: list[dict]) -> dict[tuple[str, str], str]:
    index = {}
    for row in rows:
        for name in [row["standard_name"], *row["aliases"]]:
            key = (row["type"], normalize_surface(name))
            if key in index and index[key] != row["id"]:
                raise ValueError(f"别名冲突: {key}")
            index[key] = row["id"]
    return index


def resolve(
    entity_type: str, mention: str, index: dict[tuple[str, str], str]
) -> Optional[str]:
    return index.get((entity_type, normalize_surface(mention)))

48. 建立稳定 ID 注册表并防止冲突覆盖

ID 来自受控命名空间和标准编码；重复导入同一实体允许补充别名，但同 ID 的类型或标准名冲突必须报错。

✅ 参考答案

import re


ID_PATTERN = re.compile(r"^(disease|symptom|drug|food):[A-Z0-9._-]+$")
TYPE_PREFIX = {"Disease": "disease", "Symptom": "symptom", "Drug": "drug", "Food": "food"}


class EntityRegistry:
    def __init__(self):
        self.entities: dict[str, dict] = {}

    def register(self, entity_id: str, entity_type: str, standard_name: str, aliases=()) -> None:
        prefix = entity_id.split(":", 1)[0] if ":" in entity_id else ""
        if (not ID_PATTERN.fullmatch(entity_id)
                or TYPE_PREFIX.get(entity_type) != prefix
                or not standard_name.strip()):
            raise ValueError("ID 或标准名非法")
        incoming = {"type": entity_type, "standard_name": standard_name.strip()}
        if entity_id in self.entities:
            current = self.entities[entity_id]
            if any(current[key] != incoming[key] for key in incoming):
                raise ValueError("同一 ID 出现身份冲突")
            current["aliases"].update(a.strip() for a in aliases if a.strip())
            return
        self.entities[entity_id] = {**incoming, "aliases": {a.strip() for a in aliases if a.strip()}}

49. 合并相同三元组并保留全部证据

from collections import defaultdict


def merge_claims(claims: list[dict]) -> list[dict]:
    grouped = defaultdict(dict)
    for claim in claims:
        required = {"subject", "predicate", "object", "source", "published_at", "evidence"}
        if set(claim) != required or not all(claim[field] for field in required):
            raise ValueError("claim 字段不完整")
        key = (claim["subject"], claim["predicate"], claim["object"])
        evidence_key = (claim["source"], claim["published_at"], claim["evidence"])
        grouped[key][evidence_key] = {
            "source": claim["source"],
            "published_at": claim["published_at"],
            "evidence": claim["evidence"],
        }
    result = []
    for (subject, predicate, object_), evidence_map in sorted(grouped.items()):
        result.append({
            "subject": subject,
            "predicate": predicate,
            "object": object_,
            "evidence": sorted(evidence_map.values(), key=lambda item: (item["source"], item["published_at"])),
        })
    return result

50. 实现有方向的一跳查询

支持从主体沿出边查询，也支持从客体沿入边反查；返回完整三元组以便追溯。

✅ 参考答案
Python
def one_hop(
    triples: list[tuple[str, str, str]],
    node_id: str,
    predicate: str,
    direction: str = "out",
) -> list[dict]:
    if direction not in {"out", "in"}:
        raise ValueError("direction 只能是 out 或 in")
    results = []
    for subject, relation, object_ in triples:
        if relation != predicate:
            continue
        if direction == "out" and subject == node_id:
            results.append({"neighbor": object_, "triple": (subject, relation, object_)})
        elif direction == "in" and object_ == node_id:
            results.append({"neighbor": subject, "triple": (subject, relation, object_)})
    return sorted(results, key=lambda item: item["neighbor"])


triples = [
    ("d:1", "HAS_SYMPTOM", "s:1"),
    ("d:1", "HAS_SYMPTOM", "s:2"),
    ("d:2", "HAS_SYMPTOM", "s:1"),
]
assert [x["neighbor"] for x in one_hop(triples, "d:1", "HAS_SYMPTOM")] == ["s:1", "s:2"]
assert [x["neighbor"] for x in one_hop(triples, "s:1", "HAS_SYMPTOM", "in")] == ["d:1", "d:2"]
print(one_hop(triples, "s:1", "HAS_SYMPTOM", "in"))

51. 查找有限跳数的可追溯路径

用 BFS 查找起点到终点的简单路径，路径中保留每条边，禁止在同一路径重复节点。

✅ 参考答案
Python
from collections import defaultdict, deque


def find_paths(
    triples: list[tuple[str, str, str]], start: str, target: str, max_hops: int
) -> list[list[tuple[str, str, str]]]:
    if max_hops < 1:
        raise ValueError("max_hops 必须为正")
    adjacency = defaultdict(list)
    for triple in triples:
        adjacency[triple[0]].append(triple)
    queue = deque([(start, [], {start})])
    paths = []
    while queue:
        node, path, visited = queue.popleft()
        if len(path) == max_hops:
            continue
        for edge in adjacency[node]:
            next_node = edge[2]
            if next_node in visited:
                continue
            next_path = path + [edge]
            if next_node == target:
                paths.append(next_path)
            else:
                queue.append((next_node, next_path, visited | {next_node}))
    return paths


triples = [
    ("d:1", "HAS_SYMPTOM", "s:1"),
    ("s:1", "RELATED_CONCEPT", "c:1"),
    ("d:1", "RELATED_CONCEPT", "c:2"),
]
paths = find_paths(triples, "d:1", "c:1", max_hops=2)
assert paths == [[("d:1", "HAS_SYMPTOM", "s:1"), ("s:1", "RELATED_CONCEPT", "c:1")]]
print(paths)

    missing_source = []
    connected = set()
    for index, claim in enumerate(claims):
        subject, predicate, object_ = claim["subject"], claim["predicate"], claim["object"]
        if not claim.get("source"):
            missing_source.append(index)
        if subject not in entities or object_ not in entities:
            dangling.append(index)
            continue
        connected.update({subject, object_})
        if predicate not in SCHEMA:
            type_errors.append(index)
        elif (entities[subject]["type"], entities[object_]["type"]) != SCHEMA[predicate]:
            type_errors.append(index)
    return {
        "duplicate_triples": duplicate_count,
        "dangling_claims": dangling,
        "type_error_claims": type_errors,
        "missing_source_claims": missing_source,
        "isolated_entities": sorted(set(entities) - connected),
    }


entities = {
    "d:1": {"type": "Disease"}, "s:1": {"type": "Symptom"},
    "f:1": {"type": "Food"}, "d:isolated": {"type": "Disease"},
}
claims = [
    {"subject": "d:1", "predicate": "HAS_SYMPTOM", "object": "s:1", "source": "src:1"},
    {"subject": "d:1", "predicate": "HAS_SYMPTOM", "object": "s:1", "source": "src:2"},
    {"subject": "f:1", "predicate": "HAS_SYMPTOM", "object": "s:1", "source": ""},
    {"subject": "d:1", "predicate": "HAS_SYMPTOM", "object": "missing", "source": "src:3"},
]
report = quality_report(entities, claims)
assert report["duplicate_triples"] == 1
assert report["dangling_claims"] == [3]
assert report["type_error_claims"] == [2]
assert report["missing_source_claims"] == [2]
assert report["isolated_entities"] == ["d:isolated"]
print(report)

53. 检测 IS_A 分类层级中的环

输入边方向为 child→parent；无环返回 None，有环返回一条闭合路径。

✅ 参考答案
Python
from collections import defaultdict
from typing import Optional


def find_taxonomy_cycle(edges: list[tuple[str, str]]) -> Optional[list[str]]:
    graph = defaultdict(list)
    nodes = set()
    for child, parent in edges:
        graph[child].append(parent)
        nodes.update({child, parent})
    state = {node: 0 for node in nodes}
    stack = []

    def dfs(node: str):
        state[node] = 1
        stack.append(node)
        for parent in graph[node]:
            if state[parent] == 0:
                cycle = dfs(parent)
                if cycle:
                    return cycle
            elif state[parent] == 1:
                start = stack.index(parent)
                return stack[start:] + [parent]
        stack.pop()
        state[node] = 2
        return None

    for node in sorted(nodes):
        if state[node] == 0:
            cycle = dfs(node)
            if cycle:
                return cycle
    return None


valid = [("慢性支气管炎", "呼吸系统疾病"), ("呼吸系统疾病", "疾病")]
invalid = valid + [("疾病", "慢性支气管炎")]
assert find_taxonomy_cycle(valid) is None
cycle = find_taxonomy_cycle(invalid)
assert cycle is not None and cycle[0] == cycle[-1]
print(cycle)

54. 按 as-of 时间筛选事实并检测互斥关系

使用左闭右开区间 valid_from≤as_of<valid_to；valid_to 为 None 表示仍有效。

✅ 参考答案
Python
from datetime import date


OPPOSITES = {frozenset({"DO_EAT", "NOT_EAT"})}


def active_claims(claims: list[dict], as_of: date) -> list[dict]:
    active = []
    for claim in claims:
        start = date.fromisoformat(claim["valid_from"])
        end = date.fromisoformat(claim["valid_to"]) if claim["valid_to"] else None
        if end is not None and end <= start:
            raise ValueError("无效时间区间")
        if start <= as_of and (end is None or as_of < end):
            active.append(claim)
    return active


def find_opposite_conflicts(claims: list[dict]) -> list[tuple[str, str]]:
    predicates_by_pair = {}
    for claim in claims:
        pair = (claim["subject"], claim["object"])
        predicates_by_pair.setdefault(pair, set()).add(claim["predicate"])
    return sorted(pair for pair, predicates in predicates_by_pair.items()
                  if any(opposite <= predicates for opposite in OPPOSITES))


claims = [
    {"subject": "d:1", "predicate": "DO_EAT", "object": "f:1",
     "valid_from": "2024-01-01", "valid_to": "2025-01-01"},
    {"subject": "d:1", "predicate": "NOT_EAT", "object": "f:1",
     "valid_from": "2025-01-01", "valid_to": None},
    {"subject": "d:2", "predicate": "DO_EAT", "object": "f:2",

55. 完成“归一 → 校验 → 合并 → 可追溯回答”最小闭环

从两条带别名的原始记录构建同一标准三元组，合并证据并回答疾病症状查询。

✅ 参考答案
Python
from collections import defaultdict


ENTITIES = {
    "disease:D001": {"type": "Disease", "name": "慢性支气管炎"},
    "symptom:S001": {"type": "Symptom", "name": "咳嗽"},
}
ALIASES = {
    ("Disease", "慢性支气管炎"): "disease:D001",
    ("Disease", "慢支"): "disease:D001",
    ("Symptom", "咳嗽"): "symptom:S001",
}
SCHEMA = {"HAS_SYMPTOM": ("Disease", "Symptom")}


def build_and_answer(rows: list[dict], disease_mention: str) -> dict:
    grouped = defaultdict(list)
    for row in rows:
        subject = ALIASES.get((row["subject_type"], row["subject"]))
        object_ = ALIASES.get((row["object_type"], row["object"]))
        if subject is None or object_ is None:
            raise ValueError("存在未链接实体，进入人工复核")
        expected = SCHEMA.get(row["predicate"])
        actual = (ENTITIES[subject]["type"], ENTITIES[object_]["type"])
        if expected != actual:
            raise ValueError("Schema 类型或关系方向错误")
        key = (subject, row["predicate"], object_)
        grouped[key].append({"source": row["source"], "evidence": row["evidence"]})

    disease_id = ALIASES.get(("Disease", disease_mention))
    if disease_id is None:
        return {"status": "unknown_entity", "items": []}
    items = []
    for (subject, predicate, object_), evidence in sorted(grouped.items()):
        if subject == disease_id and predicate == "HAS_SYMPTOM":
            items.append({
                "entity_id": object_,
                "name": ENTITIES[object_]["name"],
                "path": [subject, predicate, object_],
                "evidence": sorted(evidence, key=lambda item: item["source"]),
            })
    return {
        "status": "ok" if items else "not_in_graph",
        "items": items,
        "notice": "仅为知识检索结果，不构成诊断。",
    }


rows = [
    {"subject_type": "Disease", "subject": "慢支", "predicate": "HAS_SYMPTOM",
     "object_type": "Symptom", "object": "咳嗽", "source": "guide:v1", "evidence": "常见咳嗽"},
    {"subject_type": "Disease", "subject": "慢性支气管炎", "predicate": "HAS_SYMPTOM",
     "object_type": "Symptom", "object": "咳嗽", "source": "textbook:v2", "evidence": "可见咳嗽"},
]

assert len(answer["items"][0]["evidence"]) == 2
assert answer["items"][0]["path"] == ["disease:D001", "HAS_SYMPTOM", "symptom:S001"]
print(answer)

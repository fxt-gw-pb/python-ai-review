🩺 医疗问诊机器人：GPT2 从零训练对话模型

🏷️ GPT2🏷️ 对话生成🏷️ Flask
项目进度 · 共 3 章

FROM-SCRATCH DIALOGUE MODEL PROJECT

从一份医疗问答语料走到可对话、可交付的问诊机器人

这个项目不调用现成的预训练权重，而是基于 GPT2 架构从零训练一个中文医疗对话模型：先把问答语料 token 化成多轮对话样本，再完成训练与验证闭环，最后用 Top-K/Top-P 采样实现终端人机交互，并通过 Flask 交付网页版问诊界面。

核心任务医疗问答对话生成输入健康问题，自回归生成医学解答

生成质量由哪些参数控制

参数作用调大 / 调小的影响
max_history_len生成时携带的历史对话轮数越大上下文越连贯，但输入越长、越易被无关历史干扰
max_len单句最大长度，超出截断限制生成与输入的长度上限，影响显存与信息完整性
repetition_penalty对已生成 token 的重复惩罚调大可缓解复读，过大会让表达不自然
topk只保留概率最高的 k 个候选调小更保守稳定，调大更多样但更易跑偏
topp保留累积概率达 p 的候选集合与 topk 联用动态控制候选范围，越大越发散
🛡️

医疗场景边界：本项目用于学习对话模型的训练与部署流程。小规模语料上从零训练的生成模型可能编造医学信息，输出不能作为真实医疗建议；真实医疗产品还需要权威知识库检索、医学审核、风险拒答与免责声明等机制。

推荐复习方式

  1. 先跑通数据处理脚本，抽查几条序列化样本，确认多轮对话的拼接格式与 token 边界符合预期。
  2. 训练时先用小批次快速过一遍闭环，确认 loss 正常下降后再放大轮数，并保留验证集最优模型。
  3. 交互阶段固定同一组问题，分别调整 topk、topp 和 repetition_penalty，观察生成质量的变化。
  4. 把终端交互逻辑迁移到 Flask 时，注意会话历史的存储位置与并发访问下的隔离。
  5. 最后对照真实医疗场景，思考生成式方案与检索式方案（如 RAG 项目）的边界与组合方式。

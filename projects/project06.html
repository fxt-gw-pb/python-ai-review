LLM MULTI-TASK FINE-TUNING PROJECT

用 LoRA 让 62 亿参数的 ChatGLM 学会两类任务的统一输出

直接使用大模型完成信息抽取和文本分类时，输出格式往往难以严格约束。这个项目把两类任务统一成指令-输出格式的混合数据集，对 ChatGLM-6B 做 LoRA 微调：冻结主干、只训练低秩适配参数，让模型在小算力上学会按要求的格式稳定作答。

核心任务信息抽取 + 文本分类统一指令格式 · LoRA 高效微调

项目范围

3工程阶段
62 亿基座模型参数
2任务统一成一种格式
1份混合指令数据集

项目覆盖 ChatGLM-6B 的架构改动与硬件要求、云端 GPU 环境与模型准备、jsonl 指令数据的解析与预处理、LoRA 的注入与参数冻结，以及多任务训练、验证与推理预测。

为什么选 LoRA 而不是全量微调

维度全量微调LoRA 微调
可训练参数全部模型权重仅低秩矩阵 A、B，占比极小
显存与算力6B 级别需要多卡或大显存单张消费级 / 入门级云 GPU 即可
产物与切换每个任务保存一份完整权重只保存小体积适配权重，按任务加载
效果上限更高，但小数据易过拟合小样本下通常可逼近全量微调

版本与算力提示：课件使用 ChatGLM-6B + transformers 4.33 + peft 0.3.0 的较早版本组合，如今复现需注意版本兼容；GLM 系列已有更新的开源模型，但“指令化数据 + LoRA 微调”的方法本身依然通用。原课件另附的第三方算力平台（趋动云）注册操作截图指南未收录，按第 1 章的云端环境配置即可，任一云 GPU 平台均可替代。

推荐复习方式

  1. 先看懂两类任务如何统一成指令-输出格式，这一步决定了微调后输出是否可被程序稳定解析。
  2. 数据预处理后抽查若干样本，确认指令、输入与目标输出的拼接和截断符合预期。
  3. 打印可训练参数占比，确认 LoRA 只训练适配参数、主干确实被冻结。
  4. 训练中同时观察两类任务的验证表现，防止混合比例失衡导致某一任务退化。
  5. 推理时用未见过的样本检验输出格式的稳定性，并与讲义中的 PEFT 原理互相印证。

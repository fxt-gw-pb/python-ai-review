设计标注规范与数据合同

本章交付物

  • docs/annotation-guideline.md：标签语义、边界、嵌套、否定与冲突裁决；
  • data/raw/manifest.jsonl：原始文档 ID、来源、许可、时间和校验和；
  • data/annotations/*.jsonl：Doccano 导出后的原始标注；
  • data/processed/{train,valid,test}.jsonl：可训练的统一数据合同；
  • reports/data-quality.json：数量、分布、一致性、错误与泄漏检查。

标注边界示例

文本 标注 理由
“确诊慢性支气管炎” 标“慢性支气管炎” 动作词不属于疾病名
“反复剧烈咳嗽” 按指南固定为“咳嗽”或整体 修饰语是否入边界必须全局一致
“否认咳嗽” 可标实体，但增加否定标记 NER 找 mention，建图前还要判断语境
“父亲患高血压” 标实体 + experiencer=家属 不能误当当前对象的疾病
“考虑某病可能” 标实体 + certainty=不确定 假设不应当成确定事实入图

Doccano 项目配置

  1. 建立序列标注项目，配置四类实体；
  2. 如平台版本支持关系标注，为四类关系设置严格方向；
  3. 上传试标语料，不要一次性分发全部数据；
  4. 创建两名标注员 + 一名裁决员的试标流程；
  5. 上传带正反例的标注指南；
  6. 导出时保存项目配置、标签表与导出日期。

工具版本可能改变界面和导出格式，因此项目不应依赖手工点击顺序，而应对导出 JSONL 定义独立 Schema。

统一数据合同

JSON
{
  "doc_id": "doc:000001",
  "text": "某疾病常伴有某症状。",
  "entities": [
    {"id": "T1", "start": 0, "end": 3, "text": "某疾病", "type": "Disease"},
    {"id": "T2", "start": 6, "end": 10, "text": "某症状", "type": "Symptom"}
  ],
  "relations": [
    {"subject": "T1", "predicate": "HAS_SYMPTOM", "object": "T2"}
  ],
  "source": {"uri": "source:001", "retrieved_at": "2026-07-01"}
}

约定 start 包含、end 不包含。校验时要求：

Python
def validate_span(text: str, entity: dict) -> None:
    start, end = entity["start"], entity["end"]
    assert 0 <= start < end <= len(text)
    assert text[start:end] == entity["text"]

BIOES 转换注意事项

  • 先从原文偏移构造标签，再做 tokenization；
  • 子词 tokenizer 需要保留 offset mapping，不要凭 token 字符串回猜原文位置；
  • 嵌套实体如果使用单层 BIOES 无法同时表达，必须在标注指南中定义保留哪一层；
  • 从 BIOES 解码后必须再校验标签序列合法性。

切分与泄漏检查

先按来源文档、患者或业务对象分组，再切分 train / valid / test。不要对句子级随机打乱后直接切分。

泄漏检查至少包含：

  • 文本完全重复；
  • 规范化后重复；
  • 高 n-gram / 编辑相似度的近重复；
  • 同一原始文档的切片跨集合；
  • 人工构造的黄金问题被用作训练样本。

数据质量报告

JSON
{
  "documents": 0,
  "entity_counts": {"Disease": 0, "Symptom": 0, "Drug": 0, "Food": 0},
  "relation_counts": {"HAS_SYMPTOM": 0},
  "invalid_spans": 0,
  "dangling_relation_refs": 0,
  "duplicate_documents": 0,
  "cross_split_duplicates": 0,
  "annotator_agreement": {"entity_f1": 0.0, "relation_f1": 0.0}
}

对 NER 与 RE，标注员一致性可用一名标注员当预测、另一名当参考，计算实体级与三元组级 F1。单纯比较标签数量不能反映边界和关系方向分歧。

验收清单

  • [ ] 数据源可追溯、有授权且已做敏感信息处理；
  • [ ] 每个实体 span 都能用 text[start:end] 复现；
  • [ ] 每条关系的主客体 ID 都存在于同一样本；
  • [ ] 训练、验证、测试按组切分且无近重复泄漏；
  • [ ] 标注一致性达到团队设定门槛，分歧已裁决并回写指南。

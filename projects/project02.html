ENTERPRISE RAG ENGINEERING PROJECT

从企业私有文档走到可检索、可生成、可评估、可服务的智能问答系统

EduRAG 以教育知识服务为场景，把结构化 FAQ 与非结构化知识库串成一套完整工程：高频标准问题走 MySQL + BM25，开放知识问题走 Milvus 混合检索与大模型生成，最终通过 RAGAS 建立评估闭环，并封装为 API 与 WebUI。

核心任务企业私有知识问答FAQ 快速命中 + RAG 复杂问题生成

项目范围

6工程阶段
26完整专题
2FAQ 与 RAG 查询通道
4检索、生成、评估、服务闭环

项目同时覆盖本地模型管理、LangChain 编排、MySQL/Redis 结构化问答、Milvus 向量检索、多格式文档解析、查询改写、Prompt 路由、RAGAS 评估、会话历史以及 FastAPI/WebSocket 服务。

三条问答路径如何协作

路径主要数据检索方式适用问题关键风险
MySQL FAQ标准问答对BM25 + Softmax高频、边界清楚、答案固定的问题覆盖有限，需要阈值和未命中兜底
Milvus RAG切分后的企业文档稠密 + 稀疏混合检索与重排开放式、需要引用知识上下文的问题切分、召回、上下文和生成均会影响质量
融合路由FAQ + 文档 + 会话历史意图判断后分流并合并结果真实线上连续问答需统一权限、引用、缓存、监控与评估口径
🛡️

生产可靠性：RAG 不是“接入向量数据库即可准确”。上线前应验证数据权限、文档版本、召回覆盖、引用一致性、提示注入、隐私泄漏、延迟与成本，并保留无法可靠回答时的拒答和人工转接。

推荐复习方式

  1. 先明确 FAQ 与 RAG 的边界，并固定一组可重复使用的测试问题。
  2. 先跑通 MySQL FAQ，再完成文档解析、切分、向量化和 Milvus 检索，避免一次联调所有模块。
  3. 每次调整切分、召回、重排或 Prompt 时，记录同一评估集上的指标与失败案例。
  4. 服务化前补齐配置隔离、权限校验、日志脱敏、超时重试、并发控制和异常兜底。
  5. 最后通过 RAGAS 指标与人工抽检共同评估，不能只凭少量演示问题判断系统质量。

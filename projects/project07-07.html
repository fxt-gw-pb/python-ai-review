交付可追溯的医疗 KBQA

本章交付物

  • src/kbqa/nlu.py：意图、槽位与实体链接统一语义帧；
  • src/kbqa/policy.py：执行、澄清、未命中、域外与安全路由；
  • src/kbqa/templates.py：意图到已审核 Cypher 模板的白名单；
  • src/kbqa/service.py：输入校验、查询执行与返回合同；
  • app.py：交互界面，不保存未授权敏感内容；
  • eval/qa-set.jsonl 与 reports/kbqa-eval.md：端到端及安全评估。

统一语义帧

JSON
{
  "intent": {"name": "disease_symptoms", "confidence": 0.94},
  "slots": {
    "disease": {
      "mention": "慢性支气管炎",

  "safety": {"status": "allow", "reason": null}
}

NLU 不直接返回可执行 Cypher。它只返回结构化语义帧，由策略层决定是否执行。

查询模板白名单

Python
QUERY_TEMPLATES = {
    "disease_definition": """
        MATCH (d:Disease {id: $disease_id})
        RETURN d.name AS subject,
               [d.description] AS values,
               [d.source] AS sources,
               d.updated_at AS updated_at
    """,
    "disease_symptoms": """
        MATCH (d:Disease {id: $disease_id})-[r:HAS_SYMPTOM]->(s:Symptom)
        RETURN d.name AS subject,
               collect(DISTINCT s.name) AS values,
               collect(DISTINCT r.source) AS sources,
               max(r.updated_at) AS updated_at
    """,
    "disease_drugs": """
        MATCH (d:Disease {id: $disease_id})-[r:RECOMMEND_DRUG]->(x:Drug)
        RETURN d.name AS subject,
               collect(DISTINCT x.name) AS values,
               collect(DISTINCT r.source) AS sources,
               max(r.updated_at) AS updated_at
    """,
}

补充 disease_foods 的宜吃与忌吃查询。对药品相关结果，展示层必须说明“图谱收录的关联”，不能将其改写为面向个人的用药指令。

策略层

Python
SUPPORTED_INTENTS = set(QUERY_TEMPLATES)

def choose_action(frame):
    if frame["safety"]["status"] != "allow":
        return "unsafe"

    intent = frame["intent"]
    if intent["name"] not in SUPPORTED_INTENTS:
        return "unsupported"

    disease = frame["slots"].get("disease")
    if disease is None:
        return "ask_missing_slot"
    if disease["status"] == "nil":
        return "not_found"
    if disease["status"] == "clarify":
        return "clarify_entity"

    confidence = min(intent["confidence"], disease["confidence"])
    if confidence < 0.45:
        return "unsupported"
    if confidence < 0.80:
        return "clarify_intent"
    return "execute"

阈值在验证集调整，并用可靠性图或校准误差检查置信度是否真能表示正确概率。

服务编排

Python
def answer_question(question: str, session_id: str) -> dict:
    frame = nlu.parse(question)
    action = choose_action(frame)

    if action != "execute":
        return build_non_answer(action, frame, session_id)

    intent = frame["intent"]["name"]
    disease_id = frame["slots"]["disease"]["entity_id"]
    records, summary, keys = graph.execute_query(
        QUERY_TEMPLATES[intent],
        disease_id=disease_id,
        database_=settings.neo4j_database,
    )
    return build_answer(records, frame, summary)

请求需要超时、日志脱敏、trace ID 和统一错误处理。图查询未返回记录不能并入系统异常：

  • 实体不存在→ not_found；
  • 实体存在但相关边为空→ no_evidence；
  • 数据库连接或超时错误→ system_error。

安全路由

问题类型 系统动作
图谱内的定义、症状、关联药品、饮食宜忌 在有证据时检索并展示来源
“我是不是得了……” 不诊断，建议寻求合格医疗人员
“我应该吃多少……” 不给个体化剂量或处方建议
胸痛、呼吸困难、意识障碍等紧急信号 停止普通问答，提示联系当地急救服务 / 立即就医
图谱无证据或资料过期 明确说明暂无可用证据，不使用模型自由补全

端到端评估集

JSON
{
  "case_id": "qa-0001",
  "question": "慢性支气管炎常见症状有哪些？",
  "expected_intent": "disease_symptoms",
  "expected_entity_id": "disease:000123",
  "expected_status": "answered",
  "expected_value_ids": ["symptom:000031"],
  "must_have_evidence": true
}

评估维度：

  • 意图 Macro / Micro F1 和置信度校准；
  • 槽位实体级 F1；
  • 实体链接 Accuracy / MRR / NIL F1；
  • 查询模板选择准确率与执行成功率；
  • 答案集合 F1、追溯覆盖率与资料时效覆盖率；
  • 应澄清、应拒答与紧急路由正确率；
  • p50 / p95 延迟、错误率与各模块耗时。

运行前检查

  • [ ] 所有 Cypher 都来自已审核白名单并使用参数绑定；
  • [ ] 密码、密钥和敏感日志不出现在仓库与前端；
  • [ ] 澄清状态按 session 隔离且会过期；
  • [ ] answered、clarification_needed、not_found、unsupported、unsafe、system_error 均有测试；
  • [ ] 回答带来源、时间、trace ID 和安全声明；
  • [ ] 全部黄金问题与安全负例达到团队设定的发布门槛。

项目收官

完整演示应展示三条链路：

  1. 正常命中：问题→意图 / 实体→参数化 Cypher→答案 + 证据；
  2. 歧义澄清：同名 mention→候选实体→用户确认→恢复查询；
  3. 安全兜底：诊断 / 剂量 / 紧急问题→不执行普通图查询→安全提示。

能跑通第一条链路只说明有 demo；三条链路都通过自动化验收，才说明项目具备可交付的边界。

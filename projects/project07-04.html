统一 SPO 合同

Python
from dataclasses import dataclass

@dataclass(frozen=True)
class Triple:
    subject_id: str
    predicate: str
    object_id: str
    confidence: float
    source: str

模型层可以先使用样本内实体 ID，融合后再替换为标准图谱 ID。不要只用实体文本当三元组主键。

规则基线

每条规则明确实体类型、触发模式、方向和否定处理：

Python
RULES = [
    {
        "id": "has-symptom-v1",
        "subject_type": "Disease",
        "object_type": "Symptom",
        "predicate": "HAS_SYMPTOM",
        "triggers": ["表现为", "常伴有", "症状包括"],
    },
    {
        "id": "not-eat-v1",
        "subject_type": "Disease",
        "object_type": "Food",
        "predicate": "NOT_EAT",
        "triggers": ["忌食", "应避免食用"],
    },
]

规则预测必须记录 rule_id，才能在错误分析中定位哪条规则过宽或冲突。

Pipeline 基线

候选实体对

用 Schema 先过滤不可能的类型组合：

Python
ALLOWED_PAIRS = {
    ("Disease", "Symptom"),
    ("Disease", "Drug"),
    ("Disease", "Food"),
}

def candidate_pairs(entities):
    for subject in entities:
        for obj in entities:
            if subject["id"] == obj["id"]:
                continue
            if (subject["type"], obj["type"]) in ALLOWED_PAIRS:
                yield subject, obj

再根据句子边界、Token 距离、段落和否定范围进一步剪枝。剪枝规则也要在测试集上计算候选召回率，避免在分类前就删掉正确实体对。

关系分类输入

Text
[CLS] 某疾病 [SUBJ] 常伴有 [OBJ] 某症状 [SEP]

可使用特殊标记显式告诉模型当前的主体和客体，输出四类关系 + NO_RELATION。训练时要记录负样本生成和下采样策略。

CasRel 实现骨架

Python
import torch
import torch.nn as nn

class CasRelHeads(nn.Module):
    def __init__(self, hidden_size: int, num_relations: int):
        super().__init__()
        self.sub_head = nn.Linear(hidden_size, 1)
        self.sub_tail = nn.Linear(hidden_size, 1)
        self.obj_head = nn.Linear(hidden_size, num_relations)
        self.obj_tail = nn.Linear(hidden_size, num_relations)

    def subject_scores(self, encoded):
        return torch.sigmoid(self.sub_head(encoded)), torch.sigmoid(self.sub_tail(encoded))

    def object_scores(self, encoded, subject_repr):
        conditioned = encoded + subject_repr
        return torch.sigmoid(self.obj_head(conditioned)), torch.sigmoid(self.obj_tail(conditioned))

需要单独测试的形状：

Text
encoded:        [batch, seq_len, hidden]
subject heads:  [batch, seq_len, 1]
subject tails:  [batch, seq_len, 1]
object heads:   [batch, seq_len, num_relations]
object tails:   [batch, seq_len, num_relations]
mask:           [batch, seq_len]

损失是四个带 mask 的多标签二元交叉熵之和。不要先 sigmoid 再使用数值不稳定的手写对数；优先输出 logits 并使用 BCEWithLogitsLoss(reduction="none")，再应用 mask。

解码与阈值

  • 主体 head / tail 可使用贪心近邻配对或受长度约束的配对；
  • 对每个主体单独预测客体与关系；
  • 各关系类型可有独立阈值，不一定全部使用 0.5；
  • 阈值必须在验证集调整，测试集只做一次最终评估；
  • 解码后根据 (subject_id, predicate, object_id) 去重。

Pipeline + gold NER 表示关系分类上限；Pipeline + predicted NER 才是真实端到端效果。两者差距就是上游 NER 错误对 RE 的影响。

错误归因

Text
ner_miss            主体或客体未识别
wrong_entity_type   类型错导致候选被过滤
wrong_relation      实体对正确但关系错
direction_error     主客体反向
negation_scope      否定或假设范围错
overlap_miss        SEO / EPO 未完整抽取
cross_sentence      关系跨句，当前模型不支持

验收清单

  • [ ] 规则、Pipeline 与 CasRel 输出同一 SPO 合同；
  • [ ] 候选实体对阶段已报告召回率；
  • [ ] CasRel 的四个预测头形状和 mask 有单元测试；
  • [ ] 报告同时包含 gold NER 与 predicted NER 两种条件；
  • [ ] 测试集专项覆盖 SEO、EPO、否定、长距离和无关系样例。

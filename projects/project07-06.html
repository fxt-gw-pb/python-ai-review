幂等导入 Neo4j 并验证图质量

本章交付物

  • src/graph/constraints.cypher：唯一约束与索引；
  • src/graph/import_nodes.py：分类批量导入节点；
  • src/graph/import_edges.py：白名单路由各类关系查询；
  • src/graph/quality.cypher：节点数、关系数、缺失、重复、孤立点和 Schema 违反；
  • artifacts/import-manifest.json：数据、Schema、模型与建图批次；
  • tests/graph/test_idempotency.py：连续导入两次后计数不增长。

运行配置

Shell
export NEO4J_URI="neo4j://localhost:7687"
export NEO4J_USER="neo4j"
export NEO4J_PASSWORD="<数据库密码>"
export NEO4J_DATABASE="neo4j"

使用 Bolt / neo4j:// 与官方 Driver，不在代码中写死 HTTP 7474 地址和弱密码。项目不应提交真实 .env 文件。

批量节点导入

Python
NODE_QUERIES = {
    "Disease": """
        UNWIND $rows AS row
        MERGE (n:Disease {id: row.id})
        ON CREATE SET n.created_at = datetime()
        SET n.name = row.name,
            n.aliases = coalesce(row.aliases, []),
            n.source = row.source,
            n.updated_at = datetime()
    """,
    "Symptom": """
        UNWIND $rows AS row
        MERGE (n:Symptom {id: row.id})
        ON CREATE SET n.created_at = datetime()
        SET n.name = row.name,
            n.source = row.source,
            n.updated_at = datetime()
    """,
}

def import_node_batch(driver, entity_type, rows, database):
    query = NODE_QUERIES[entity_type]  # 只从内部白名单取查询
    driver.execute_query(query, rows=rows, database_=database)

为 Drug 和 Food 添加对应的固定查询。用户或数据文件中的 entity_type 不能不经白名单就拼接成 Label。

批量关系导入

Python
EDGE_QUERIES = {
    "HAS_SYMPTOM": """
        UNWIND $rows AS row
        MATCH (a:Disease {id: row.from_id})
        MATCH (b:Symptom {id: row.to_id})
        MERGE (a)-[r:HAS_SYMPTOM]->(b)
        SET r.source = row.source,
            r.confidence = row.confidence,
            r.extractor_version = row.extractor_version,
            r.updated_at = datetime()
    """,
    "RECOMMEND_DRUG": """
        UNWIND $rows AS row
        MATCH (a:Disease {id: row.from_id})
        MATCH (b:Drug {id: row.to_id})
        MERGE (a)-[r:RECOMMEND_DRUG]->(b)
        SET r.source = row.source,
            r.confidence = row.confidence,
            r.extractor_version = row.extractor_version,
            r.updated_at = datetime()
    """,
}

导入前应在文件层验证主客体 ID 存在。Neo4j 中的 MATCH 未命中时，该行将不产生关系；如果不对账，会出现静默丢边。

对账和失败批

Python
def chunks(rows, size=500):
    for i in range(0, len(rows), size):
        yield rows[i:i + size]

def import_with_audit(driver, query, rows, database):
    imported = 0
    failures = []
    for batch_no, batch in enumerate(chunks(rows)):
        try:
            driver.execute_query(query, rows=batch, database_=database)
            imported += len(batch)
        except Exception as exc:
            failures.append({"batch_no": batch_no, "size": len(batch), "error": type(exc).__name__})
    return {"attempted": len(rows), "imported": imported, "failures": failures}

错误日志不记录密码、完整敏感原文或未脱敏的个人信息。生产系统还应对可重试与不可重试错误分类。

幂等性测试

Python
def test_import_is_idempotent(graph_client, sample_bundle):
    import_bundle(graph_client, sample_bundle)
    first = graph_counts(graph_client)

    import_bundle(graph_client, sample_bundle)
    second = graph_counts(graph_client)

    assert second == first

测试环境使用独立数据库或容器。“运行前清库”不算幂等性；幂等性是同一批数据重复导入不产生额外节点与边。

图质量脚本

CYPHER
// 按节点类型计数
MATCH (n)
UNWIND labels(n) AS label
RETURN label, count(*) AS count ORDER BY label;

// 孤立点
MATCH (n)
WHERE NOT (n)--()
RETURN labels(n) AS labels, count(*) AS count;

// 缺少来源的关系
MATCH ()-[r]->()
WHERE r.source IS NULL OR trim(r.source) = ""
RETURN type(r) AS relation, count(*) AS count;

// 关系类型与主客体类型违反
MATCH (a)-[r]->(b)
WHERE NOT (
  (a:Disease AND type(r) = "HAS_SYMPTOM" AND b:Symptom) OR
  (a:Disease AND type(r) = "RECOMMEND_DRUG" AND b:Drug) OR
  (a:Disease AND type(r) IN ["DO_EAT", "NOT_EAT"] AND b:Food)
)
RETURN labels(a), type(r), labels(b), count(*) AS violations;

验收清单

  • [ ] 凭据来自环境变量，仓库中没有真实密码；
  • [ ] 约束在导入前创建，MERGE 仅使用稳定 ID；
  • [ ] 节点 Label 与关系类型只从内部白名单路由；
  • [ ] 导入同一快照两次后节点数和关系数不变；
  • [ ] attempted / imported / failed 完成对账；
  • [ ] 图质量门禁输出可存档报告，不只在 Neo4j Browser 里人工观察。

END-TO-END NLP PROJECT

从一份短文本数据集走到可演示、可部署、可答辩的分类系统

这个项目不是单独训练某一个模型，而是用同一业务任务串起数据分析、基线实验、预训练模型、LLM 方案、工程部署和模型压缩，最后把技术成果转化为完整的项目表达。

核心任务单标签 · 十分类输入新闻短文本，输出唯一类别

项目任务与数据

200,000总样本
180,000训练集
10,000 + 10,000验证集与测试集
10新闻类别

训练集各类别约 18,000 条，类别分布均衡；文本平均长度约 19.2 个字符、最长约 38 个字符。任务目标是根据新闻标题或短文本判断其所属类别，为后续内容推荐提供可靠的语义标签。

四条建模路线如何选择

路线输入表示优势主要代价 / 适用场景
随机森林分词 + TF-IDF 稀疏向量训练快、流程清楚、适合建立首个可比较基线语序与上下文表达有限，适合快速验证数据和流程
FastText字符或词及其 n-gram速度快、模型小，子词信息能缓解未登录词问题语义建模深度有限，适合高吞吐、低成本部署
BERT 微调上下文相关的 Transformer 表示能利用预训练知识和上下文，适合追求更强语义判别训练与推理资源更高，需要规范的数据管道和调参
LLM 提示词自然语言指令 + 标签约束无需专门训练即可快速试验，便于低样本启动受延迟、费用和输出稳定性影响，需做格式校验与兜底
🔐

工程安全：调用第三方模型服务时，密钥应通过环境变量读取，不要写进代码、截图或 Git 仓库。

推荐复习方式

  1. 先用第 1 章建立统一的数据和业务认识，再固定同一套评估口径。
  2. 依次复现随机森林、FastText 和 BERT；每次只改变一类关键变量并记录结果。
  3. 把 LLM 提示词方案看作另一条产品路线，重点评估稳定性、成本和错误兜底。
  4. 压缩前先保存原模型基线，压缩后同时比较精度、模型体积、延迟和运行设备。
  5. 最后用答辩与模拟面试章节，把“做了什么”整理成“为什么这样做、结果如何、还能怎样改进”。

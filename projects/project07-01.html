定义问题、Schema 与验收指标

本章交付物

  • docs/scope.md：业务范围、非目标与安全边界；
  • schemas/kg-schema.yaml：实体、关系、属性与约束；
  • schemas/qa-intents.yaml：问答意图与必需槽位；
  • docs/acceptance.md：抽取、融合、图质量、KBQA 和安全指标；
  • 20–50 个手工构造的端到端“黄金样例”。

项目问题

将目标写成一句可验收的话：

对经授权且来源可追溯的医学文本，抽取疾病、症状、药品与食物及其指定关系，归一后写入 Neo4j，支持定义、症状、药品和饮食宜忌的可追溯问答；系统不进行诊断或个体化用药推荐。

项目非目标

  • 不根据用户症状推断所患疾病；
  • 不生成药品剂量、处方或停药建议；
  • 不将互联网未审核内容直接当成权威医学事实；
  • 不用图可视化替代数据质量验收。

最小 Schema

YAML
version: 1.0.0
entities:
  Disease:
    key: id
    required: [id, name, source, updated_at]
  Symptom:
    key: id
    required: [id, name, source]
  Drug:
    key: id
    required: [id, name, source]
  Food:
    key: id
    required: [id, name, source]

relations:
  HAS_SYMPTOM:
    from: Disease
    to: Symptom
  RECOMMEND_DRUG:
    from: Disease
    to: Drug
  DO_EAT:
    from: Disease
    to: Food
  NOT_EAT:
    from: Disease
    to: Food

relation_properties:
  required: [source, confidence, extractor_version]

Schema 中的关系名使用稳定、不含空格的英文枚举，展示层再映射为中文。这样可以避免展示文案变化导致查询和模型标签全部迁移。

问答意图合同

YAML
intents:
  disease_definition:
    required_slots: [disease]
    query_template: disease_definition_v1
  disease_symptoms:
    required_slots: [disease]
    query_template: disease_symptoms_v1
  disease_drugs:
    required_slots: [disease]
    query_template: disease_drugs_v1
  disease_foods:
    required_slots: [disease]
    query_template: disease_foods_v1
  unsupported:
    required_slots: []
    query_template: null

必须有 unsupported 或等价的域外类别。否则分类器会把任意问题强制分类到某个可执行意图。

端到端黄金样例

JSON
{
  "question": "慢性支气管炎常见症状有哪些？",
  "intent": "disease_symptoms",
  "entity": {"mention": "慢性支气管炎", "id": "disease:000123"},
  "expected_query": "disease_symptoms_v1",
  "expected_answer_ids": ["symptom:000031", "symptom:000052"],
  "expected_status": "answered"
}

还要构造负例：

  • 缺少疾病名：“有什么症状？”→ clarification_needed；
  • 实体歧义：同名概念→澄清候选；
  • 未收录实体→ not_found；
  • 要求诊断或给药量→ unsafe；
  • 系统不支持的问题→ unsupported。

验收指标

层 最低必报指标 验收方式
NER 严格实体级 P/R/F1 分类型 + micro 汇总
RE 严格三元组 P/R/F1 总体 + 重叠三元组子集
链接 Recall@5、Accuracy / MRR、NIL F1 同名与未收录子集
图质量 重复、孤立点、Schema 违反、属性完整率 自动 Cypher 质量查询
KBQA 意图 F1、端到端答案正确率、追溯覆盖率 黄金问题集
安全 应拒答准确率、紧急路由覆盖率 独立安全负例集

指标阈值要根据数据集和业务风险设定，不应盲目复制其他项目的数字。本章先确定“怎么测”，后续才可比较规则、BiLSTM-CRF 和 CasRel。

验收清单

  • [ ] 四类实体和四类关系都有反例与边界说明；
  • [ ] 每种关系的主体、客体类型和方向唯一；
  • [ ] 黄金样例同时覆盖正常、澄清、未命中、域外与安全场景；
  • [ ] 非目标被写入 README 与 API 声明；
  • [ ] Schema 和意图定义有版本号。

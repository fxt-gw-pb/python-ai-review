实体归一、消歧与来源追踪

本章交付物

  • data/catalog/entities.jsonl：标准实体、别名、类型、描述与来源；
  • src/linking/candidate.py：精确、模糊与向量候选生成；
  • src/linking/ranker.py：名称、上下文、类型与图邻域排序；
  • src/linking/decision.py：链接、澄清和 NIL 决策；
  • reports/linking.md：Recall@K、Accuracy / MRR、NIL F1 与误差分析；
  • data/curation/review-queue.jsonl：低置信度人工复核队列。

标准实体目录

JSON
{
  "entity_id": "disease:000123",
  "canonical_name": "慢性支气管炎",
  "aliases": ["慢支"],
  "type": "Disease",
  "description": "...",
  "sources": [
    {"source_id": "authority:001", "updated_at": "2026-06-01"}
  ],
  "status": "active"
}

别名只用于召回，不能单独证明两个概念必然相同。同一别名可以指向多个候选，需要上下文消歧。

候选生成器

Python
def generate_candidates(mention: str, entity_type: str, top_k: int = 20):
    candidates = []
    candidates += exact_alias_lookup(mention, entity_type)
    candidates += fuzzy_name_search(mention, entity_type, top_k=top_k)
    candidates += vector_search(mention, entity_type, top_k=top_k)
    return deduplicate_by_entity_id(candidates)[:top_k]

首先计算 Recall@K。如果正确实体没有进入候选，之后换更强排序器也无法挽回。

可解释排序基线

Python
def score_candidate(mention, context, mention_type, candidate):
    features = {
        "name": name_similarity(mention, candidate["canonical_name"], candidate["aliases"]),
        "context": context_similarity(context, candidate["description"]),
        "type": float(mention_type == candidate["type"]),
        "prior": candidate.get("prior", 0.0),
    }
    score = 0.25 * features["name"] + 0.55 * features["context"]
    score += 0.15 * features["type"] + 0.05 * features["prior"]
    return score, features

上面的权重只是结构示例，需在验证集调整。系统要保留特征分数，便于解释一次误链接是名称、上下文还是先验导致。

链接 / 澄清 / NIL 三态决策

Python
def decide_link(ranked, accept=0.82, margin=0.12):
    if not ranked:
        return {"status": "nil", "entity_id": None}

    top1 = ranked[0]
    top2_score = ranked[1]["score"] if len(ranked) > 1 else 0.0

    if top1["score"] >= accept and top1["score"] - top2_score >= margin:
        return {"status": "linked", "entity_id": top1["entity_id"]}
    if top1["score"] >= accept * 0.65:
        return {"status": "clarify", "candidates": ranked[:3]}
    return {"status": "nil", "entity_id": None}

阈值需通过验证集的精确率—召回率取舍确定。医学知识入图应优先保证精度，低置信度结果进复核队列，不强制选 top-1。

关系对齐表

YAML
relation_aliases:
  HAS_SYMPTOM:
    - has_symptom
    - has_sign
    - symptom_of
  RECOMMEND_DRUG:
    - recommend_drug
    - recommand_drug

symptom_of 的原始方向与 HAS_SYMPTOM 相反，对齐时不能只改谓词名，还要交换主客体。历史拼写 recommand_drug 可作为输入别名，标准谓词使用正确拼写。

来源追踪与冲突

融合不等于覆盖。对于每条节点属性或关系，保留：

JSON
{
  "statement_id": "stmt:opaque-id",
  "subject_id": "disease:000123",
  "predicate": "HAS_SYMPTOM",
  "object_id": "symptom:000031",
  "source_id": "doc:2026-001#p12",
  "published_at": "2026-05-20",
  "extracted_at": "2026-07-16",
  "extractor_version": "re-1.1.0",
  "confidence": 0.93,
  "review_status": "approved"
}

多来源冲突时，业务策略可选择展示最新权威来源、并列多个声明或进入人工复核。不能按“最后写入”静默覆盖。

评估

阶段 指标 目标
候选召回 Recall@1 / 5 / 20 确认正确实体能进入候选
排序 Accuracy、MRR 确认正确实体位置足够靠前
NIL Precision / Recall / F1 不把未收录概念强行误链接
澄清 候选覆盖率、用户选择成功率 候选集中包含正确实体且文案可区分
来源 追溯覆盖率 入图声明均可定位到来源

验收清单

  • [ ] 每个标准实体有稳定 ID，名称修改不会产生新 ID；
  • [ ] 候选召回与候选排序分开评估；
  • [ ] 系统支持 NIL 和澄清，不强制选择 top-1；
  • [ ] 关系对齐同时处理名称、方向和主客体类型；
  • [ ] 每条入图声明都有来源、时间、抽取器版本与审核状态。

🧬 医学实体抽取：规则、BiLSTM-CRF 与 Viterbi

🏷️ NER🏷️ BiLSTM-CRF🏷️ Viterbi✏️ 配套练习：医学实体抽取：规则、BiLSTM-CRF 与 Viterbi 综合训练🚀 配套项目：医学知识图谱：从信息抽取到 KBQA

医学实体抽取：规则、BiLSTM-CRF 与 Viterbi

本讲对应原课件第三章：实体识别基础、规则 NER、BiLSTM-CRF 和 Viterbi 扩展资料。

学习目标

  • 把医学 NER 定义为序列标注任务；
  • 区分词典 / 正则基线、统计学习与深度学习方法；
  • 理解 BiLSTM 的发射分数、CRF 的转移分数与训练目标；
  • 手工推演 Viterbi 的动态规划过程；
  • 用实体级 P/R/F1 而非 Token Accuracy 验收。

任务定义

输入是文本序列 $x=(x_1,\ldots,x_n)$，输出是与之对齐的标签序列 $y=(y_1,\ldots,y_n)$。在字级 BIOES 标注中：

  • B-X：X 类实体的开始；
  • I-X：X 类实体的内部；
  • E-X：X 类实体的结束；
  • S-X：单字 / 单 Token 实体；
  • O：不属于目标实体。
Text
文本：慢  性  支  气  管  炎  常  见  咳  嗽
标签：B-D I-D I-D I-D I-D E-D O   O   B-S E-S

边界和类型必须同时正确才算一个正确实体。“慢性支气管”和“慢性支气管炎”不是部分得分的同一个结果。

三类方法

方法 代表 优势 局限
规则 词典、正则、词性模式 可解释、冷启动快、精确控制 召回有限，维护成本高，语境判断弱
传统序列模型 HMM、CRF 能显式学习标签依赖 特征工程重，表示能力受限
神经网络 BiLSTM-CRF、Transformer-CRF 能学习上下文表示，上限更高 需要标注数据、训练算力与版本化管理

规则基线应保留为可回归的比较组，而不是在训出模型后删除。在高风险或语料很少的区域，规则还可作为高精度保护层。

规则 NER 的基本流程

Text
领域词典 / 后缀词典 / 正则
  → 候选匹配
  → 边界扩展或裁剪
  → 重叠冲突处理
  → 否定、假设、家族史等语境过滤
  → 统一输出实体 span

一个统一的输出合同：

JSON
{
  "text": "慢性支气管炎",
  "type": "Disease",
  "start": 0,
  "end": 6,
  "confidence": 0.99,
  "method": "dictionary"
}

end 是否为开区间必须在数据合同中统一，否则模型、标注和评估会产生系统性的差一位错误。

BiLSTM-CRF 数据流

Text
Token IDs
  → Embedding
  → BiLSTM 上下文编码
  → Linear 产生每个位置对每个标签的发射分数
  → CRF 结合转移分数计算序列分数
  → Viterbi 找最优合法标签路径

BiLSTM 同时编码左侧与右侧上下文。线性层产生发射分数 $P_{i,y_i}$，CRF 学习标签 $y_{i-1}$ 到 $y_i$ 的转移分数 $A_{y_{i-1},y_i}$。

一条路径的总分：

$$ s(x,y)=\sum_{i=1}^{n} P_{i,y_i}+\sum_{i=1}^{n} A_{y_{i-1},y_i} $$

条件概率用全部可能路径归一：

$$ p(y\mid x)=\frac{\exp s(x,y)}{\sum_{y'\in\mathcal Y(x)}\exp s(x,y')} $$

训练最小化负对数似然：

$$ \mathcal L=-\log p(y^{gold}\mid x) =\log\sum_{y'}\exp s(x,y')-s(x,y^{gold}) $$

分母不能枚举 $K^n$ 条路径，它通过前向动态规划与 log-sum-exp 稳定计算。

Viterbi 解码

Viterbi 与前向算法使用同样的状态转移结构，但聚合操作不同：

  • 前向算法用 logsumexp，汇总所有路径；
  • Viterbi 用 max，并保留最优前驱用于回溯。

递推式：

$$ \delta_i(t)=P_{i,t}+\max_s\left[\delta_{i-1}(s)+A_{s,t}\right] $$

每个位置还要保存使最大值成立的前驱标签 $s$ 。到达终止状态后，沿回指针反向得到整条最优路径。

Mask 与合法转移

批处理中的 padding 位置必须被 mask，不应进入发射、转移或损失统计。还可将不可能转移设为极小分数，例如：

  • O -> I-Disease 不合法；
  • B-Disease -> I-Symptom 类型不一致；
  • START 不应直接转移到 I-* 或 E-*。

评估与错误分析

对每个类型和 micro 总体计算：

$$ P=\frac{TP}{TP+FP},\qquad R=\frac{TP}{TP+FN},\qquad F_1=\frac{2PR}{P+R} $$

Token Accuracy 会被大量 O 标签主导，不能代替实体级指标。验收报告至少要包含：

  • 边界过短 / 过长；
  • 类型混淆；
  • 整个实体漏检；
  • 虚假实体；
  • 否定、假设、家族史或时间语境错误；
  • 长实体、缩写、中英文混合和未登录词子集。

实现检查清单

  1. 分词 / 字级序列与原文偏移能互相转换；
  2. 标签 ID 与标签名的映射被保存在模型产物中；
  3. 分组切分防止同一患者或同一文档泄漏；
  4. 损失与解码都正确应用 mask；
  5. 每个 checkpoint 同时保存词表、标签表、配置和阈值；
  6. 测试包含空句、超长句、无实体句和不合法输入。

本讲小结

  1. BiLSTM 负责上下文表示，CRF 负责整体标签序列的一致性。
  2. 前向算法用于训练时汇总全部路径，Viterbi 用于推理时找最优路径。
  3. NER 的主指标是严格实体级 P/R/F1，规则基线与错误分析应长期保留。

知识融合：实体统一、链接与消歧

本讲对应原课件第五章，将实体消歧、实体统一与关系对齐组织成可实施的“候选生成—候选排序—NIL 判定—来源保留”流程。

学习目标

  • 区分实体归一、实体链接、实体消歧和关系对齐；
  • 设计别名表、候选生成器和候选排序器；
  • 理解 TF-IDF + 余弦相似度的基线与边界；
  • 处理知识库中不存在的 NIL 实体；
  • 用 Recall@K、Accuracy / MRR 和 NIL F1 评估。

四个容易混淆的概念

概念 要解决的问题 示例
实体统一 / 归一 多个写法映射到标准名 “高血压病”、“高血压”→标准概念
实体链接 将文本 mention 映射到知识库 ID “感冒”→disease:...
实体消歧 多个候选同名实体中选正确项 “Apple”是公司还是水果
关系对齐 将近义关系合并成统一谓词 has_sign、has_symptom→HAS_SYMPTOM

同名不等于同一实体，字符串不同也不等于不同实体。这就是“只用名称当主键”会导致图谱污染的原因。

标准实体记录

JSON
{
  "entity_id": "disease:000123",
  "canonical_name": "慢性支气管炎",
  "aliases": ["慢支", "慢性支气管炎症"],
  "type": "Disease",
  "description": "...",
  "source": "authority:example",
  "updated_at": "2026-07-01"
}

标准 ID 不应从名称直接生成，因为标准名可能被更正。ID 应稳定，名称与别名都是可更新属性。

实体链接流程

Text
mention + 上下文
  → 文本规范化
  → 候选生成（别名字典 / 模糊检索 / 向量检索）
  → 候选排序（名称 + 上下文 + 类型 + 图邻域）
  → 置信度校准
  → 链接、澄清或 NIL

文本规范化

可以统一全半角、大小写、空白、部分标点与已知缩写，但不要用过度清洗合并具有医学差异的词。

候选生成

候选生成追求高 Recall@K，宁可多召回少量候选，也不要在第一步排除正确实体。常见策略：

  • 标准名 / 别名精确匹配；
  • 编辑距离、字符 n-gram 或 BM25；
  • 描述或 mention 向量的近邻检索；
  • 先用实体类型过滤不可能候选。

候选排序

一个可解释的基线分数：

$$ score(e\mid m,c)=w_1s_{name}(m,e)+w_2s_{context}(c,e)+w_3s_{type}+w_4s_{graph} $$

其中 $m$ 是 mention，$c$ 是上下文，$e$ 是候选实体。权重不应凭感觉设定，可以在验证集上学习或调优。

TF-IDF + 余弦相似度基线

原项目用实体描述构建 TF-IDF 矩阵，将 mention 周边窗口与候选描述比较。余弦相似度：

$$ \cos(\mathbf q,\mathbf d)=\frac{\mathbf q^\top\mathbf d}{|\mathbf q|_2|\mathbf d|_2} $$

基线实现要点：

Python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

vectorizer = TfidfVectorizer(analyzer="char", ngram_range=(2, 4))
entity_matrix = vectorizer.fit_transform(entity_descriptions)

query = vectorizer.transform([mention_context])
scores = cosine_similarity(query, entity_matrix)[0]
topk = scores.argsort()[-5:][::-1]

中文短文中字符 n-gram 能避免对分词错误过度敏感。但 TF-IDF 不理解语义，描述短、关键词稀疏或需要多跳图上下文时，需要更强的向量 / 交互编码排序器。

NIL 不是“选最像的”

如果知识库中没有正确实体，系统必须输出 NIL，而不是强行选择最高分候选。判定可以结合：

  • top-1 置信度低于阈值；
  • top-1 与 top-2 差距过小；
  • 实体类型冲突；
  • 关键属性与上下文矛盾；
  • 模型不确定性过高。

高风险医学场景中，模糊结果应进入澄清或人工复核，不应直接写入主图。

关系对齐与冲突合并

关系对齐先做 Schema 层映射，再处理实例边。对于多来源冲突，不要简单覆盖，可保留：

  • 每个来源的原始声明；
  • 来源可信级别和发布时间；
  • 抽取模型版本与置信度；
  • 当前综合结论与合并规则版本。

评估指标

模块 推荐指标 回答的问题
候选生成 Recall@K 正确实体是否进入前 K 个候选？
候选排序 Accuracy、MRR 正确实体是否排在前面？
NIL 判定 Precision / Recall / F1 未收录概念是否被正确拒绝？
端到端链接 Micro / Macro Accuracy 按实体类型分布时是否仍稳定？

本讲小结

  1. 融合的核心是“概念 ID 与上下文”，不是名称字符串去重。
  2. 候选生成追求 Recall@K，排序器追求正确候选的前置，NIL 负责避免强制误链接。
  3. 医学知识冲突要保留来源和时间，不能用一次覆盖丢失证据。

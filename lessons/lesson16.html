🗄️ MySQL 基础：数据库、DDL 与数据管理

🏷️ 数据库入门🏷️ DDL / DML🏷️ 约束✏️ 配套练习：SQL 加强练习：北风项目 34 题

学习 SQL 的第一步不是背语法，而是搞清楚三件事：数据存在哪里（库和表）、结构怎么定义（DDL）、数据怎么进出（DML）。本讲从数据库的基本概念和 MySQL 环境入手，依次掌握 SQL 通用语法、数据类型、建库建表、增删改数据，最后用约束给数据加上"安全锁"。

本讲与下一讲《MySQL 查询》配套：本讲负责"把数据管起来"，下一讲负责"把数据查出来"。所有示例均可在 MySQL 8.0 中直接执行。

一、数据库概述

1. 为什么需要数据库

先回顾一下我们已经会的存储数据的方式，各自的问题在哪里：

存储方式 特点 局限
变量、列表、字典 读写快，用法灵活 只能临时存储，程序一结束数据就丢失
文件 可以永久存储 不方便精细化管理：查一条、改一条都要自己解析
数据库 永久存储 + 规律化组织 需要额外学习，但这是实际开发中真正存数据的地方

数据库（DataBase）就是存储和管理数据的系统，本质上也是一个文件系统，但它按照固定格式组织数据，让用户可以方便地对数据做精细化管理，即 CRUD 四类操作：

  • Create：新增
  • Update：修改
  • Read：查询
  • Delete：删除

2. 关系型与非关系型数据库

按照数据存储形式的不同，数据库分为两大类：

类别 存储形式 常见产品
关系型数据库（SQL 数据库，RDBMS） 数据以行、列组成的二维表格存储，表与表之间有关系（一对多、多对多、一对一） MySQL、Oracle、SQL Server、DB2、SQLite
非关系型数据库（NoSQL 数据库，了解即可） 键值存储、文档存储、列式存储、图式存储 Redis（键值）、MongoDB（文档）、HBase（列式）、Neo4j（图）

我们要学的 MySQL 属于关系型数据库：数据保存在数据库（database）→ 数据表（table）→ 行（记录）与列（字段） 这样的层级结构中。

二、认识 MySQL

1. 发展简史与版本

MySQL 是目前最流行的关系型数据库管理系统之一，它的归属经历了三家公司：

  • 1995 年：瑞典 MySQL AB 公司开发并发布 MySQL；
  • 2008 年：Sun 公司收购 MySQL AB；
  • 2010 年：Oracle（甲骨文）公司收购 Sun，MySQL 归入 Oracle 旗下至今。

两个重要版本节点：2015 年 10 月发布 MySQL 5.7 正式版；2018 年 4 月发布 MySQL 8.0 正式版。窗口函数是 MySQL 8.0 才加入的功能，这也是课程选用 8.0 系列（8.0.16）的原因。

在命令行查看已安装的 MySQL 版本：

Shell
mysql --version
# 或简写
mysql -V

2. 核心特点

  • C/S 架构：分为 Client（客户端）和 Server（服务端）两部分。服务端以表格形式存储数据；客户端连接服务端，通过 SQL 对数据进行管理。
  • 免费开源：不需要支付额外费用（MySQL 5 以上的企业版开始收费，提供官方技术支持；社区版依旧免费）。
  • 支持大型数据库：可以处理上千万条数据。
  • 使用标准 SQL：以标准的 SQL 语言形式完成增、删、改、查。
  • 跨平台、多语言接口：支持不同操作系统，并提供 C、C++、Python、Java 等多种编程语言的操作接口。
  • 支持多种存储引擎：常用 InnoDB。

3. 安装方式与存储引擎

安装 MySQL 常见有三种途径，选其一即可：

  1. 使用现成环境：例如课程提供的虚拟机中已装好 MySQL，开箱即用；
  2. 官网安装包：从官网（www.mysql.com）下载社区版安装到本机，历史版本可在 https://downloads.mysql.com/archives/community/ 下载；
  3. 集成环境工具：例如 PhpStudy 这类一键部署工具，安装后在其中启动 MySQL 8.x 即可。注意安装路径要合法，不要出现中文和空格。

存储引擎注意：如果用集成工具安装，默认引擎可能是 MyISAM——它不支持事务和约束。应将引擎改为 InnoDB（支持事务、约束），这也是 MySQL 8.0 官方默认引擎。

三、连接 MySQL：命令行与 DataGrip

1. IP 地址与端口

MySQL 是 C/S 软件：假设你在上海，公司的 MySQL 服务在北京，客户端要靠两个信息找到服务端：

  • IP 地址：标识网络上的唯一一台设备（找到哪台电脑），本机默认 127.0.0.1；
  • PORT 端口：标识一台设备上的一个网络程序（找到电脑上的哪个程序），MySQL 默认 3306。

2. 命令行登录与登出

登录的完整命令格式：

Shell
mysql -h数据库服务端IP -P端口 -u用户名 -p密码
  • -h 省略时默认连接本机 127.0.0.1；-P 省略时默认 3306；
  • -u 后是登录用户名，MySQL 安装后自带 root 用户；
  • -p 后直接跟密码即为明文登录（密码暴露在屏幕和命令历史里，不推荐）。

三种常用登录方式：

Shell
# 方式1：明文登录（不推荐，密码可见）
mysql -uroot -p你的密码

# 方式2：暗文登录（推荐）：-p 后什么都不写，回车后再隐式输入密码
mysql -u root -p

# 方式3：远程访问，用完整参数指明主机
mysql --host=主机名 --user=账号 --password=密码

登出有三种等价方式：exit、quit、快捷键 Ctrl + D。

扩展：如果 MySQL 是用集成工具装的，想在任意路径下直接使用 mysql 命令，需要把它的 bin 目录配置到系统环境变量 PATH 中。

3. DataGrip 的使用

DataGrip 是 JetBrains 公司推出的数据库客户端软件，可以兼容各种数据库；它与 PyCharm 同属一家公司，安装和常用设置的操作方式一致。

连接 MySQL 的步骤：

  1. 新建工程（Project），选择打开方式；
  2. 在 Database 面板新建数据源：Data Source → MySQL；
  3. 填写连接信息：主机（Host）、端口（Port）、用户名（User）、密码（Password）；
  4. 第一次连接时根据提示点击下载 MySQL 连接驱动文件；
  5. 点击 Test Connection 测试，成功后保存即可。

两个重要的日常操作：

  • 附加文件夹到项目：在右侧窗口点击右键选择 "Attach Directory To Project"，选中存放 SQL 文件的文件夹后点击 OK，就能在 DataGrip 中管理自己的 .sql 文件；
  • 为 SQL 文件绑定会话：双击打开附加文件夹下的 SQL 文件后，编写 SQL 之前要先给文件设置一个会话（session），否则语句没有执行目标。

常用快捷键（Windows）：

快捷键 作用
Ctrl + Enter 快速执行选中的 SQL 语句
Ctrl + / 快速注释 / 取消注释
Ctrl + Alt + L SQL 代码格式化

扩展：PyCharm 专业版内置了同款 Database 工具，也可以直接连接本机或远程服务器上的 MySQL。

四、SQL 语句概述

1. 什么是 SQL

SQL 全称 Structured Query Language（结构化查询语言），是所有关系型数据库都要遵循的规范：只要是关系型数据库（MySQL、Oracle 等），就支持 SQL。它的作用是对数据库中的数据做管理操作——数据库、数据表、表记录的增删改查。

注意：虽然所有关系型数据库都支持 SQL，但不同数据库还会有一些自己特有的函数或语法（俗称"方言"）。

2. SQL 的四大分类

按功能可以把 SQL 语句分为 4 类：

分类 全称 作用 核心关键字
DDL Data Definition Language 数据定义语言 定义数据库对象：数据库、表、列等（不涉及表中记录） create、alter、drop、show
DML Data Manipulation Language 数据操作语言 对表记录进行增、删、改 insert、update、delete
DQL Data Query Language 数据查询语言 查询表记录（重点，下一讲专门学） select、from、where、group by
DCL Data Control Language 数据控制语言 定义访问权限和安全级别、创建用户、事务操作 grant（授权）、commit（提交）、rollback（回滚）

3. 通用语法与注释

  1. 一条 SQL 语句可以写成一行，也可以写成多行，以分号 ; 结尾；
  2. 可以使用空格和缩进增强可读性；
  3. MySQL 的 SQL 语句不区分大小写，建议关键字大写（本站示例为贴近随堂代码，多为小写，两种写法均可）；
  4. 注释有三种写法，注意 -- 后面必须跟一个空格：
SQL
/*
 多行
 注释
*/

# 单行注释（MySQL 特有）

-- 单行注释，-- 后边必须加空格
select * from student;

五、数据类型

创建表时要为每个字段指定数据类型，只有符合类型要求的数据才能存进来——数据类型本身就是对数据的第一层限制。

1. 数值类型

分类 类型 说明
整型 tinyint（1 字节）、smallint（2 字节）、mediumint（3 字节）、int（4 字节）、bigint（8 字节） 字节数越大，取值范围越大
浮点型 float（单精度）、double（双精度） 存在精度误差
定点型 decimal(M, N) 共存 M 位数字，小数占 N 位，如 decimal(5,2)；要求没有误差的小数（比如金钱）用它

2. 字符串类型

类型 说明
char(n) 定长字符串，n 表示字符数。如 char(3) 存 'ab' 时会补空格为 'ab '
varchar(n) 不定长（可变长度）字符串，如 varchar(3) 存 'ab' 就存 'ab'，按实际长度存储
text / mediumtext / longtext 文本型，容量依次为 65535、16777215、4294967295 字节，存大段文本

3. 日期时间类型

类型 格式 说明
date yyyy-MM-dd 只存日期
time HH:mm:ss 只存时间
datetime yyyy-MM-dd HH:mm:ss 日期 + 时间
timestamp yyyy-MM-dd HH:mm:ss 日期 + 时间；默认使用服务器时区

4. 类型选择的一般原则

  1. 先选合适的：根据存储值的性质选类型——姓名用字符串、年龄用整数、身高体重用小数、手机号建议用字符串（不参与运算且可能有前导 0）；
  2. 再选省空间的：同类类型中，在满足需求的前提下尽量选取值范围小的，够用就行。比如存年龄，tinyint（0~255 足够）比 int 更合适。

六、DDL：操作数据库

数据库层面的常用操作——查看、创建、修改、切换、删除：

SQL
# 1. 查看当前所有的数据库（注意 databases 有 s）
show databases;

# 2. 创建数据库
# 方式1：数据库已存在时会报错
create database db_demo1 charset 'utf8';
# 方式2：加 if not exists，数据库已存在时什么都不做（不报错）
create database if not exists db_demo1 charset 'utf8';
# 方式3：创建时指定其它字符集
create database if not exists db_demo2 charset 'gbk';

# 3. 查看某个数据库的详细信息（能看到字符集）
show create database db_demo1;   # utf8
show create database db_demo2;   # gbk

# 4. 修改数据库的字符集
alter database db_demo2 charset 'utf8';

# 5. 切换（进入）数据库
use db_demo1;

# 6. 查看当前正在使用哪个数据库
select database();

# 7. 删除数据库【慎用！数据会全部丢失】
drop database db_demo2;

两个细节：

  • 创建数据库时建议显式指定 charset 'utf8'（或 utf8mb4），防止存中文出错或乱码；
  • 不切库也能操作其它库的表，但表名前必须加库名前缀：select * from 库名.表名;。

七、DDL：操作数据表

对表本身（不是表里的数据）做增删改查。操作表之前务必先 use 切换到目标数据库。

SQL
# 0. 切库：之后创建的表都归属这个库
use db_demo1;

# 1. 查看当前数据库中所有的数据表
show tables;

# 2. 创建数据表
/*
格式:
    create table [if not exists] 数据表名(
        字段名 数据类型 [约束],
        字段名 数据类型 [约束],
        ......
        字段名 数据类型 [约束]
    );
细节: 中括号里的内容是可选项，写不写都行。
*/
create table if not exists users(
    id int,                 # id列，整型
    username varchar(20),   # 账号，字符串类型
    password varchar(20)    # 密码，字符串类型
);

# 3. 核心：查看某张表的结构信息（字段名、数据类型、约束等）
desc users;

# 4. 修改表名，两种格式等价
alter table users rename user_tmp;
rename table user_tmp to users;

# 5. 删除数据表
drop table users;

八、DDL：操作表字段

实际开发中，第一版表结构往往只满足第一版需求；随着需求增加、数据维度扩展，就需要对已有表的字段做调整。核心都是 alter table：

SQL
# 1. 新增字段
# 格式: alter table 表名 add 字段名 数据类型 [约束];
alter table users add address varchar(200) not null;
# 新增字段时同样可以带默认值、枚举类型等
alter table users add gender enum('男', '女', '未知') default '男';

# 2. 修改字段的数据类型/约束（不改名） —— modify
# 格式: alter table 表名 modify 字段名 新的数据类型 [新的约束];
alter table users modify address varchar(10) not null;

# 3. 修改字段名（可同时改类型和约束） —— change
# 格式: alter table 表名 change 旧字段名 新字段名 新的数据类型 [新的约束];
alter table users change address addr varchar(20) not null;

# 4. 删除字段
# 格式: alter table 表名 drop 字段名;
alter table users drop addr;

三个容易踩的坑：

  • modify / change 时必须写新的数据类型，只想改约束也要把类型带上，否则报错；
  • modify 时如果不写约束，原来的约束会被抹掉。例如 address 原本非空，执行 alter table users modify address varchar(10); 后非空约束就没了——想保留就要重新写上 not null；
  • change 必须指定新字段名（改名专用），modify 不能改名。

modify 与 change 的选择：只动类型或约束用 modify；要改列名用 change。

九、DML：插入数据（INSERT）

DML 语句操作的是表数据（表中的记录），实际开发中增、删、改被统称为变更语句。先看"增"。

1. 四种插入格式

SQL
# 格式1：指定列名插入（列的个数和类型要与值一一对应）
insert into users(id, username, password) values(1, 'admin01', 'pwd111');
# 只给部分列赋值也可以，其余列为 NULL 或默认值
insert into users(username) values('admin02');

# 格式2：省略列名 = 默认全列名，值必须与所有列一一对应
insert into users values(2, 'admin03', 'pwd333');

# 格式3：如果主键设置了自增，主键位置可以传 null，交给数据库维护
insert into users values(null, 'admin04', 'pwd444');

# 格式4：一条语句同时插入多行（批量操作效率更高）
insert into users values
    (5, 'admin05', 'pwd555'),
    (6, 'admin06', 'pwd666');

细节：要添加的值的个数、类型，必须与列名一一对应；不写列名时默认是全列名。

2. 扩展：MySQL 8.0 的"存在则更新，不存在则插入"

当插入的数据导致主键或唯一索引冲突时，on duplicate key update 会自动改为执行更新，否则正常插入（俗称 upsert）：

SQL
# 传统写法：用 values() 函数引用待插入的值
insert into category (cid, cdesc, cname)
values
    (2, '移动通讯设备', '平板电脑'),  # cid=2 已存在 → 执行更新
    (6, '小米手机', '小米手机')       # cid=6 不存在 → 执行插入
on duplicate key update
    cdesc = values(cdesc),
    cname = values(cname);

# MySQL 8.0.19+ 推荐的别名写法（8.0.20 起官方不再推荐 values() 函数）
insert into category (cid, cdesc, cname)
values
    (2, '移动通讯设备', '平板'),
    (3, '笔记本', '笔记本电脑')
as new_data   -- 为待插入的数据行起个别名
on duplicate key update
    cdesc = new_data.cdesc,
    cname = new_data.cname;

注意：该语法必须有主键或唯一索引才能检测冲突。

十、DML：修改与删除（UPDATE / DELETE）

1. 修改数据

SQL
# 格式: update 表名 set 字段名=值, 字段名=值 where 条件;

# 有 where：只更新满足条件的行
update stu set name='小龙女', tel='777', gender='女' where id = 3;

# 没有 where：一次性更改所有行！！
update stu set name='杨过', gender='男';

一个过来人的含泪忠告：写 update 或 delete 语句时，一定一定一定要先想 where 条件。不加 where，更新/删除的就是全表。

2. 删除数据

SQL
# 格式: delete from 表名 where 条件;

# 有 where：只删除满足条件的行
delete from stu where id = 2;

# 没有 where：一次性删除表中所有数据
delete from stu;

实际业务中还有一个重要习惯：往往不做物理删除，而是逻辑删除——给表加一个状态字段（如 is_deleted），"删除"其实是把状态改为已删除，数据仍保留在表里，方便追溯和恢复。

十一、约束：保证数据完整性

1. 约束概述与分类

约束是在数据类型限制的基础上，对列值追加的额外限定（例如不能重复、不能为空），用来保证数据的完整性和一致性。

分类 约束 关键字 特点
单表约束 主键约束 primary key 非空且唯一；一张表只能有一个主键；一般配合 auto_increment 自增使用
单表约束 非空约束 not null 该列值不能为空，但可以重复
单表约束 唯一约束 unique 该列值不能重复，但可以为 NULL
单表约束 默认约束 default 默认值 不给值就用默认值填充（类似 Python 的缺省参数）；给了值以给的为准
多表约束 外键约束 foreign key 约束表与表之间的关系，下一讲详细展开

生活中的对应场景：注册账户时"用户名必须唯一（unique）、密码和手机号必填（not null）、证件类型有默认选项（default）"。

2. 主键约束（PRIMARY KEY）

主键用来唯一标识表中的每一行记录，同时能加快数据查找速度（背后涉及索引，后续课程展开）。比如学号可以唯一标识一个学生，就适合当学生表的主键。

主键的特点：

  • 必须包含唯一的值，不能重复；
  • 不能为 NULL；
  • 每个表都应该有一个主键，并且只能有一个主键。

MySQL 官方建议：所有表的主键字段都叫 id，类型用 int unsigned。

设置主键的两种方式：

SQL
# 方式1：创建表时直接在字段后声明
create table student(
    sid int primary key,    # 主键
    sname varchar(20),
    age int
);
# 也可以在建表语句末尾统一声明: primary key(sid)

# 方式2：建表后追加主键
alter table student add primary key(sid);

# 删除主键约束
alter table student drop primary key;

注意：给已有数据的表补加主键时，如果该列已存在重复数据，添加会失败——要先清理重复数据再加主键。

3. 主键自增（AUTO_INCREMENT）

如果主键 id 的值靠手动维护，很容易冲突。把主键设置为自增后交给数据库自动维护：

SQL
create table student(
    sid int primary key auto_increment,   # 主键 + 自增
    sname varchar(20),
    age int
);

# 插入时主键传 null（或不写主键字段、或传 0），数据库自动在最大主键值基础上 +1
insert into student values(null, '张三', 18);
insert into student(sname, age) values('李四', 19);

自增的规则：

  • 默认从 1 开始，每插入一条自动加 1；
  • 如果手动指定了主键值，后续会从指定的值继续自增。例如手动插入 sid=1001 后，下一条自动分配 1002；
  • 删除数据（delete）不会重置自增序列：删掉 1004 后再插入，新主键是 1005 而不是复用 1004；
  • 已有表也可以补加自增：alter table student modify sid int primary key auto_increment;。

4. 面试题：DELETE 与 TRUNCATE 的区别

delete from 表名; 和 truncate table 表名; 都能清空表数据，但有本质区别：

对比项 delete from truncate table
所属分类 DML 语句 DDL 语句
事务 可以结合事务使用 一般不能结合事务使用
自增序列 不重置（id 接着涨） 重置清零（相当于把表摧毁后按原样重建一张新表）
适用场景 带条件的删除 单纯清空全表数据
SQL
delete from stu;        # 清空数据，自增 id 不清零
truncate table stu;     # 清空数据，自增 id 清零
truncate stu;           # 效果同上，table 可省略

5. 非空、唯一与默认约束

用一张学生表把三种约束的行为一次演示清楚：

SQL
create table if not exists stu(
    id int primary key auto_increment,   # 主键（非空、唯一）、自增
    name varchar(10) not null,           # 非空约束
    phone varchar(11) unique,            # 唯一约束
    gender varchar(2),                   # 没有额外限定
    address varchar(10) default '北京'    # 默认约束
);

# 正常插入
insert into stu values(null, '杨过', '111', '男', '上海');
insert into stu values(null, '郭靖', '222', '男', '广州');

# 测试默认约束：不给 address，自动填充'北京'
insert into stu(id, name, phone, gender) values(null, '尹志平', '444', '男');

# 测试非空约束：name 传 null → 报错 Column 'name' cannot be null
insert into stu values(null, null, '555', '女', '深圳');

# 测试唯一约束：
insert into stu values(null, '黄蓉', '555', '女', '深圳');   # 成功
insert into stu values(null, '黄蓉', null, '女', '深圳');    # 成功：unique 列可以为 NULL
insert into stu values(null, '黄蓉', '555', '女', '深圳');   # 报错 Duplicate entry '555' for key 'phone'

# 传入值的个数与列数不匹配也会报错
insert into stu values(1, '乔峰', '111');    # 报错

从中能提炼出三条容易考察的细节：

  • 非空约束的列不能为 NULL，但值可以重复（两个'黄蓉'没问题）；
  • 唯一约束的列不能重复，但可以为 NULL（NULL 不参与唯一性比较，可以有多个）；
  • 默认约束只在"不给值"时生效，显式给了值就以给的为准。

6. 综合建表示例

把本讲的数据类型和约束组合起来，创建一张规范的员工表：

SQL
create table if not exists employee(
    eid int primary key auto_increment,     # 员工编号：主键 + 自增
    ename varchar(20) not null,             # 姓名：非空
    age int not null,                       # 年龄：非空
    sex enum('男', '女'),                    # 性别：枚举类型，只能取列出的值
    mobile varchar(11) unique,              # 手机号：唯一
    salary decimal(10, 2) default 0.00      # 薪资：定点数（金钱），默认 0.00
);

insert into employee(ename, age, sex, mobile, salary) values('张三', 18, '男', '13800000000', 3000.00);
insert into employee(ename, age, sex, mobile) values('李四', 19, '女', '13900000000');   # salary 用默认值
# 手机号重复，写入失败：
# insert into employee(ename, age, sex, mobile, salary) values('王五', 20, '男', '13800000000', 4000.00);

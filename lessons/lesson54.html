🔗 关系抽取：规则、Pipeline 与 CasRel

🏷️ SPO🏷️ Pipeline🏷️ CasRel✏️ 配套练习：关系抽取：规则、Pipeline 与 CasRel 综合训练🚀 配套项目：医学知识图谱：从信息抽取到 KBQA

关系抽取：规则、Pipeline 与 CasRel

本讲对应原课件第四章，将关系抽取基础、规则方法、BiLSTM+Attention Pipeline 与 CasRel 联合抽取放到同一评估框架中对比。

学习目标

  • 把关系抽取的输出定义为标准 SPO 三元组；
  • 区分封闭关系分类与开放信息抽取；
  • 说清规则、Pipeline 与 Joint 方法的误差来源；
  • 理解 CasRel 如何用级联指针抽取重叠三元组；
  • 用严格三元组 P/R/F1 和专项子集评估。

任务定义与输出合同

JSON
{
  "subject": {"id": "disease:000123", "text": "慢性支气管炎", "type": "Disease"},
  "predicate": "HAS_SYMPTOM",
  "object": {"id": "symptom:000031", "text": "咳嗽", "type": "Symptom"},
  "source": "doc:2026-001#p12",
  "confidence": 0.93
}

根据关系集是否预先定义，可分为：

  • 封闭关系抽取：预先给定 HAS_SYMPTOM、RECOMMEND_DRUG 等关系集，适合 Schema 驱动建图；
  • 开放信息抽取：谓词也由文本中发现，覆盖广但语义难统一。

医学建图项目通常以封闭抽取为主，开放抽取用于发现候选关系，经人工审核后再扩展 Schema。

评估口径

一个预测只有在主体边界与类型、关系类型、客体边界与类型都正确时才计为 TP。

$$ P=\frac{|T_{pred}\cap T_{gold}|}{|T_{pred}|},\qquad R=\frac{|T_{pred}\cap T_{gold}|}{|T_{gold}|} $$

除了总体 micro F1，还应报告：

  • 各关系类型的 P/R/F1；
  • 只包含重叠三元组的子集指标；
  • 不同实体距离的分桶指标；
  • 使用 gold 实体和使用 predicted 实体的两组结果。

最后一项可以区分“RE 模型错误”和“上游 NER 错误”。

规则方法

规则系统通常由三部分组成：

  1. 实体识别或实体词典；
  2. 关系触发词表，如“表现为”、“常伴有”；
  3. 依存、词序、句式和否定范围规则。
Text
[Disease] 可表现为 [Symptom]
[Drug] 用于治疗 [Disease]
[Disease] 应避免食用 [Food]

优势是可解释且少样本可用；主要问题是句式覆盖有限，规则冲突与否定 / 条件范围难维护。

Pipeline 方法

Pipeline 把任务拆成独立步骤：

Text
文本
  → NER 得到实体集
  → 构造候选实体对
  → 对每个实体对进行关系分类
  → 过滤 no_relation

原课件使用 BiLSTM+Attention 作为关系分类器：BiLSTM 编码双向上下文，Attention 为不同 Token 分配权重，再对候选实体对分类。

重要工程问题：

  • NER 漏掉的实体对不会进入 RE；
  • $m$ 个实体可产生 $O(m^2)$ 候选对，no_relation 会压倒正类；
  • 训练时使用 gold 实体、推理时使用 predicted 实体会造成分布差异；
  • 实体对的方向与类型约束应在候选生成时使用，减少无效组合。

Joint 与 CasRel

Joint 方法在同一模型中协同学习实体与关系。CasRel 把三元组看成主体到客体的级联：

  1. BERT 编码全文；
  2. 预测所有主体的 head / tail；
  3. 对每个主体构造主体表示，与全文表示融合；
  4. 对每种关系预测客体 head / tail；
  5. 组合出一个主体对应的多个关系与客体。

对编码后的序列 $H\in\mathbb R^{n\times d}$，主体检测产生：

$$ p_s^{head}=\sigma(HW_s^{head}+b_s^{head}),\qquad p_s^{tail}=\sigma(HW_s^{tail}+b_s^{tail}) $$

指定主体 $s$ 后，对每种关系 $r$ 预测客体：

$$ p_{o,r}^{head}=\sigma((H+h_s)W_{o,r}^{head}+b_{o,r}^{head}) $$

客体边界预测条件于主体，因此同一句中一个实体可与多个客体、多个关系组合。

重叠三元组

类型 含义 示例
Normal 三元组之间无实体共享 A-r-B，C-r-D
SEO 单实体重叠 A-r1-B，A-r2-C
EPO 实体对重叠 A-r1-B，A-r2-B

CasRel 的级联预测能自然表示 SEO 和 EPO，但仍会受制于主体阈值、子词边界、长文截断和类别不平衡。

方法选型

情况 优先方法 理由
数据很少、句式稳定 规则 可快速建立高精度基线
已有高质量 NER，需快速迭代关系集 Pipeline 模块可独立替换、调试和部署
重叠三元组多，有足够标注 CasRel / 其他 Joint 模型 减少模块间误差传递，能表达重叠
高风险生产场景 混合与人工复核 用规则保护关键类别，低置信度结果不自动入图

本讲小结

  1. Pipeline 便于分步优化，但会继承 NER 错误并面临候选对爆炸。
  2. CasRel 通过“先主体、后按关系找客体”的级联指针处理重叠三元组。
  3. 评估要使用严格三元组 F1，并分别测试 gold 实体与 predicted 实体条件。
